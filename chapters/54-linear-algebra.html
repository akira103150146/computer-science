線性代數

線性代數是「向量與其變換」的數學。對電腦科學家來說,它不是一堆要背的矩陣運算規則,而是機器學習、3D 圖學、資料科學、量子計算共用的一門「空間語言」。一旦你能看見矩陣在做什麼,公式就會變成畫面。

你早就活在線性代數裡了

今天早上到現在,你大概已經用了線性代數幾十次,只是沒察覺。手機的人臉解鎖,把你的臉變成一串數字 (一個向量),再和資料庫裡的向量比對「夠不夠像」;你滑開的 3D 遊戲,每一幀都在用 矩陣把成千上萬個頂點旋轉、縮放、投影到螢幕上;Google 在 1998 年靠一個叫 PageRank 的演算法打敗對手,而 PageRank 的核心,是一個矩陣的特徵向量; 你問 ChatGPT 的每一句話,在模型內部都是一連串龐大的矩陣乘法。

這些看似天差地別的東西,底層說的是同一種語言。微積分(第 12 章)是「變化與累積」的數學; 線性代數,則是「向量,以及對向量的變換」的數學。它是機器學習(第 38 章)、 電腦圖學(第 39 章)、資料科學、乃至量子計算的共同母語。

很多人學線性代數學得很痛苦,是因為被教成「一堆計算規則」:矩陣怎麼相乘、行列式怎麼展開、 消去法怎麼操作……公式背了一卡車,卻始終不知道這些東西到底在幹嘛。 這一章我們走另一條路——數學家、YouTuber 3Blue1Brown 推廣的那條路:幾何直覺優先。 我們會反覆問同一個問題:「這個東西,在空間裡長什麼樣子?」一旦你能在腦中看見矩陣在 把空間揉捏變形,那些公式就會從天書變成你早就理解的畫面的註腳。

本章的一條主線

如果這一章你只記得一件事,請記得這句:「矩陣是線性變換,也就是一個把空間整齊變形的函數。」 向量是空間裡的點/箭頭,矩陣是搬動它們的動作。後面所有東西——矩陣乘法、行列式、反矩陣、特徵向量—— 都只是這個畫面的不同側面。先把這句話貼在心裡,我們開始。

向量:同一個東西的三種身分

線性代數的主角是向量(vector)。但「向量是什麼」這個問題,不同的人會給你三種不同的答案, 而厲害的地方在於——他們講的其實是同一件事。

  • 物理學家說:向量是空間中有「方向」與「長度」的箭頭。把它平移到哪裡都還是同一個向量,只要方向與長度不變。
  • CS 與資料科學的人說:向量是一串數字清單,例如一位顧客是 $(年齡, 月消費, 造訪次數)=(28, 3500, 12)$。幾個數字就是幾維。
  • 數學家說:向量是任何「可以相加、可以乘以一個數,而且運算結果仍守規矩」的抽象元素。在這個觀點下,連函數、多項式都可以是向量。

這一章我們主要在前兩種觀點之間來回穿梭:把向量畫成箭頭來培養直覺,把向量寫成數字來計算。 習慣上,我們把向量寫成直的一行(欄向量),例如:

$$\mathbf{v}=\begin{bmatrix}3\\2\end{bmatrix}\qquad \mathbf{w}=\begin{bmatrix}-1\\2\\0\end{bmatrix}$$

$\mathbf{v}$ 是一個二維向量(住在平面 $\mathbb{R}^2$ 裡),$\mathbf{w}$ 是三維的(住在 $\mathbb{R}^3$)。 座標的每個數字,告訴你「沿著各個軸走多遠」。

向量的兩個基本動作:相加與縮放

向量只有兩個最基本的運算,但整座線性代數的大廈都蓋在它們之上:

向量加法(vector addition):對應座標各自相加。 $\begin{bmatrix}3\\1\end{bmatrix}+\begin{bmatrix}1\\2\end{bmatrix}=\begin{bmatrix}4\\3\end{bmatrix}$。 幾何上,這叫首尾相接(tip-to-tail):先走完第一支箭頭,再從它的箭頭尖端接著走第二支, 終點就是它們的和。它捕捉的是「先做一個位移,再做另一個位移」的總效果。

純量乘法(scalar multiplication):把向量裡每個數字都乘上同一個數(這個數叫純量 scalar)。 $2\begin{bmatrix}2\\1\end{bmatrix}=\begin{bmatrix}4\\2\end{bmatrix}$。 幾何上,這是把箭頭沿原方向伸長或縮短;乘上負數則會掉頭。 「scalar(純量)」這個字的字根就是 scale(縮放)——它的工作就是縮放向量。

在 Python 裡,我們幾乎一律用 numpy 來表示與運算向量。它是整個科學計算與 ML 生態的地基:

import numpy as np

v = np.array([3, 1])          # 用一維陣列表示向量
w = np.array([1, 2])

print(v + w)                  # [4 3]   ← 向量加法(對應座標相加)
print(2 * v)                  # [6 2]   ← 純量乘法(每個分量都乘 2)
print(-v)                     # [-3 -1] ← 掉頭
print(np.linalg.norm(v))      # 3.1622... ← 向量長度 |v| = √(3²+1²)
ℹ️
「向量」這個字的小歷史

vector 源自拉丁文 vehere(「攜帶、運送」),和 vehicle(車輛)同源—— 它本來描述的就是「把一個點載送到另一個點」的位移。這個古老的字根,其實一語道破了向量在 線性代數裡的角色:它既是空間中的一個點,也是一個「搬動」的動作。記住這個雙重身分,後面會很有用。

線性組合、生成空間與基:空間是怎麼被「撐」出來的

既然向量可以相加、可以縮放,我們自然會問:拿幾支向量,反覆地縮放再相加,能湊出哪些向量? 這個「縮放再相加」的式子,有個正式名字:線性組合(linear combination)。給定向量 $\mathbf{u},\mathbf{w}$ 與純量 $a,b$,

$$a\,\mathbf{u}+b\,\mathbf{w}$$

就是它們的一個線性組合。「線性」這個詞,你可以理解成「只准縮放與相加,不准做別的(不准相乘、不准開根號)」。 當你讓 $a,b$ 跑遍所有實數,湊得出來的所有向量,構成的集合,叫做這組向量的 生成空間(span)。

想像一下:在平面上,如果 $\mathbf{u}$ 與 $\mathbf{w}$ 指向不同方向,那麼透過調配 $a,b$, 你可以走到平面上的任何一點——它們的 span 是整個平面。但如果 $\mathbf{w}$ 剛好是 $\mathbf{u}$ 的倍數(兩者共線),那無論怎麼調,你都只能停留在那一條直線上——span 縮水成一條線。

線性獨立:有沒有人在「白吃白喝」?

上面右圖的情況,我們叫 $\mathbf{u},\mathbf{w}$ 線性相依(linearly dependent): 其中一支可以被另一支(或其他支)的線性組合表達出來——它沒有貢獻新的方向,是多餘的。 反過來,如果一組向量裡沒有任何一支能被其他支湊出來,每一支都帶來一個全新的、無可取代的方向, 我們就說它們線性獨立(linearly independent)。

有個更俐落的判準:一組向量線性獨立,當且僅當 $a_1\mathbf{v}_1+a_2\mathbf{v}_2+\dots+a_k\mathbf{v}_k=\mathbf{0}$ 這條式子 只有在所有係數都是 $0$ 時才成立。直覺是:如果你能用「非全零」的係數湊出零向量, 代表其中有人可以被別人表示,就有人多餘。

基與維度:用最精簡的一組向量描述整個空間

現在把兩個概念合起來:一組向量,如果它們(1) 線性獨立(沒有冗餘),而且 (2) 它們的 span 撐滿了整個空間,我們就稱這組向量為這個空間的一組 基(basis)。基,就是「描述整個空間所需要的、剛剛好的一組積木」—— 一個都不能少(否則撐不滿),一個都不嫌多(否則就相依了)。

平面 $\mathbb{R}^2$ 最自然的一組基,是標準基 $\hat{\imath}=\begin{bmatrix}1\\0\end{bmatrix}$ 與 $\hat{\jmath}=\begin{bmatrix}0\\1\end{bmatrix}$(沿 $x$ 軸與 $y$ 軸各走一格的單位向量)。 任何向量 $\begin{bmatrix}3\\2\end{bmatrix}$ 其實都是 $3\hat{\imath}+2\hat{\jmath}$—— 你早就一直在用基了,只是沒意識到。記住 $\hat{\imath},\hat{\jmath}$ 這對「標準基向量」, 它們是下一節「矩陣是變換」的主角。

一個空間的基,可以有很多種選法,但它們含有的向量個數永遠一樣——這個個數, 就是空間的維度(dimension)。平面是 2 維(任何基都恰好 2 支向量), 我們生活的空間是 3 維,而一張 1000 像素的灰階圖片,在數學上是一個住在 1000 維空間裡的向量。 資料科學處理的,常常就是這種「高到無法想像」的維度。

💡
高維別硬用眼睛想,用代數想

你沒辦法「看見」一個 784 維的向量(這是手寫數字辨識 MNIST 每張 28×28 圖片的維度),沒關係—— 幾乎沒有人可以。訣竅是:在 2D / 3D 培養直覺,再相信代數會把同樣的規則一字不差地搬到高維。 「線性獨立」「基」「維度」這些概念,在 2 維和 2000 維是完全同一套定義。低維畫圖建立感覺,高維靠公式放心推廣——這是駕馭線性代數的關鍵心法。

核心洞見:矩陣就是線性變換

終於到了這一章的心臟。請先暫時忘掉你以前看過的「矩陣是一個數字方塊」的印象。我們要建立一個全新的畫面: 矩陣是一個函數——一個吃進一個向量、吐出另一個向量的函數;而且是一種特別「守規矩」的函數,它把整個空間整齊地變形。

什麼叫「整齊地變形」?它有兩個鐵律:(1) 原本的直線,變形後仍是直線(不會彎曲);(2) 原點固定不動。 想像平面是一張畫了方格的彈性橡皮膜,你可以拉、可以轉、可以斜推,但不准把格線拉成曲線、不准把中心移走。 這種變形,就叫線性變換(linear transformation)。

這裡是最關鍵、也最漂亮的一步。一個線性變換,看起來要追蹤平面上「無窮多個點」跑去哪,似乎是天文數字的資訊。 但因為它這麼守規矩,其實——你只要知道兩個點跑去哪,就完全確定了整個變換:那兩個點, 就是標準基向量 $\hat{\imath}$ 與 $\hat{\jmath}$。

為什麼?因為任何向量 $\begin{bmatrix}x\\y\end{bmatrix}=x\hat{\imath}+y\hat{\jmath}$。 而線性變換「守規矩」正意味著它保持線性組合:變換後的向量,就是 $x\cdot(\hat{\imath}\text{變到哪})+y\cdot(\hat{\jmath}\text{變到哪})$。 所以只要記下 $\hat{\imath},\hat{\jmath}$ 的落點,所有點的落點都能算出來。

🔑
矩陣的欄,是基向量變換後的落點

把 $\hat{\imath}$ 變換後的落點放成第一欄,$\hat{\jmath}$ 的落點放成第二欄, 拼起來就是這個變換的矩陣。所以矩陣的每一欄,都不是抽象的數字,而是一個基向量被搬到了哪裡。 這就是「為什麼矩陣長那樣」的答案。讀矩陣,就讀它的欄向量——它們直接告訴你空間被怎麼揉捏了。

舉個具體例子。考慮一個剪切(shear)變換:它讓 $\hat{\imath}$ 待在原地 $\begin{bmatrix}1\\0\end{bmatrix}$, 但把 $\hat{\jmath}$ 推到 $\begin{bmatrix}1\\1\end{bmatrix}$(往右斜)。把這兩個落點拼成欄:

$$M=\begin{bmatrix}1 & 1\\0 & 1\end{bmatrix}\quad\Big(\text{第一欄 }\hat{\imath}\to(1,0),\ \text{第二欄 }\hat{\jmath}\to(1,1)\Big)$$

矩陣 × 向量:就是「拿座標去調配欄向量」

有了這個畫面,「矩陣乘以向量」就再自然不過了。要算 $M\mathbf{v}$,就是問「$\mathbf{v}$ 經過這個變換後,落到哪?」 答案是:用 $\mathbf{v}$ 的座標,去調配矩陣的欄向量:

$$\begin{bmatrix}a & b\\c & d\end{bmatrix}\begin{bmatrix}x\\y\end{bmatrix} =x\begin{bmatrix}a\\c\end{bmatrix}+y\begin{bmatrix}b\\d\end{bmatrix} =\begin{bmatrix}ax+by\\cx+dy\end{bmatrix}$$

看穿了嗎?那條你可能背過的「橫的乘直的」公式 $\begin{bmatrix}ax+by\\cx+dy\end{bmatrix}$, 其實只是「$x$ 倍第一欄 加上 $y$ 倍第二欄」的展開。它不是憑空規定的怪規則,而是 「保持線性組合」這個要求的必然結果。用我們的剪切矩陣驗證: 

import numpy as np

M = np.array([[1, 1],
              [0, 1]])          # 剪切:î→(1,0), ĵ→(1,1)

i_hat = np.array([1, 0])
j_hat = np.array([0, 1])
print(M @ i_hat)               # [1 0]  第一欄,正是 î 的落點
print(M @ j_hat)               # [1 1]  第二欄,正是 ĵ 的落點

v = np.array([2, 3])
print(M @ v)                   # [5 3]  = 2*(1,0) + 3*(1,1) = (2+3, 3)

(在 numpy 裡,@ 是矩陣乘法運算子。注意別用 *,那是逐元素相乘,意思完全不同。)

幾種你天天會遇到的變換

讓我們把幾個最常見的線性變換,連同它們的矩陣與「對基向量做了什麼」一起列出來。這張表你之後會一再回頭查:

變換矩陣î 落點(第一欄)ĵ 落點(第二欄)
不變(單位)$\begin{bmatrix}1&0\\0&1\end{bmatrix}$$(1,0)$$(0,1)$
縮放(×2 寬、×1.5 高)$\begin{bmatrix}2&0\\0&1.5\end{bmatrix}$$(2,0)$$(0,1.5)$
逆時針旋轉 90°$\begin{bmatrix}0&-1\\1&0\end{bmatrix}$$(0,1)$$(-1,0)$

旋轉的矩陣值得記:逆時針轉 $\theta$ 角的矩陣是 $\begin{bmatrix}\cos\theta & -\sin\theta\\ \sin\theta & \cos\theta\end{bmatrix}$。 你可以自己驗證:它把 $\hat{\imath}=(1,0)$ 轉到 $(\cos\theta,\sin\theta)$(單位圓上轉了 $\theta$), 把 $\hat{\jmath}$ 轉到 $(-\sin\theta,\cos\theta)$。這正是第 39 章電腦圖學裡, 旋轉一個 2D 圖形所用的同一個矩陣。

矩陣乘法,其實是「變換的接力」

如果矩陣是動作,那兩個矩陣相乘 $AB$ 是什麼?答案優美得不得了:它是先做 $B$ 這個變換,再做 $A$ 這個變換 的「總效果」——也就是兩個變換的合成(composition)。$AB$ 這個新矩陣, 描述的就是「一口氣完成兩步」會把空間變成什麼樣。

這立刻解釋了兩件常讓初學者困惑的事。第一,為什麼是「由右往左」讀:在 $AB\mathbf{v}$ 裡, $\mathbf{v}$ 先碰到最右邊的 $B$,再碰到 $A$,正如函數合成 $f(g(x))$ 先做 $g$ 再做 $f$。 第二,為什麼矩陣乘法那樣定義:它不是誰隨便規定的,而是「兩個變換接力後,新的基向量落點是什麼」的計算結果。

為什麼矩陣乘法不可交換?

$AB\neq BA$,這在數字世界裡很怪($3\times5=5\times3$ 啊),但在變換世界裡再合理不過: 動作的先後順序會影響結果。先「旋轉 90°」再「往右剪切」,和先「剪切」再「旋轉」, 把同一個方塊揉成的形狀根本不一樣——就像「先穿襪子再穿鞋」跟「先穿鞋再穿襪子」結果天差地別。 用 numpy 親眼看一次: 

import numpy as np

A = np.array([[0, -1],     # 逆時針旋轉 90°
              [1,  0]])
B = np.array([[1, 1],      # 水平剪切
              [0, 1]])

print(A @ B)               # [[ 0 -1]   先剪切,再旋轉
                           #  [ 1  1]]
print(B @ A)               # [[ 1 -1]   先旋轉,再剪切
                           #  [ 1  0]]
print(np.allclose(A @ B, B @ A))   # False ← 順序不同,結果不同!

# 也驗證「合成」確實等於「逐步施作」
v = np.array([2, 1])
print((A @ B) @ v)         # [-1  3]
print(A @ (B @ v))         # [-1  3]  ← 完全一致

要算 $AB$ 的每個元素,規則是「$A$ 的第 $i$ 列,點乘 $B$ 的第 $j$ 欄,得到結果的第 $(i,j)$ 格」。 但與其死記這條,不如記住它為什麼長這樣:$AB$ 的第 $j$ 欄,就是「$B$ 的第 $j$ 欄(也就是某個基向量被 $B$ 送到的地方) 再被 $A$ 變換一次」的落點。一切又回到那句話:欄向量,記錄著基向量的去向。

行列式:空間被放大或壓扁了幾倍?

每個線性變換都會改變面積。剪切把方塊推斜但面積不變;放大鏡把每樣東西變大;有些變換甚至把整個平面壓成一條線。 行列式(determinant)就是衡量這件事的單一數字:它告訴你,經過這個變換, 面積(在 3D 是體積)被縮放了幾倍。記作 $\det(A)$ 或 $|A|$。

具體說:取由 $\hat{\imath},\hat{\jmath}$ 撐起的那個面積為 $1$ 的單位正方形,變換後它變成一個平行四邊形; 這個平行四邊形的面積,就是 $\det(A)$。$\det=3$ 表示「所有面積都放大 3 倍」;$\det=1$ 表示「面積不變」(例如純旋轉與剪切); $\det=0$ 則是個大警訊,我們等一下細談。

對 $2\times 2$ 矩陣,行列式有個好背的公式(對角線相乘相減):

$$\det\begin{bmatrix}a & b\\c & d\end{bmatrix}=ad-bc$$

當 det = 0:空間被壓扁了

最重要的特例是 $\det=0$。它代表變換後的面積是 $0$——意思是,原本飽滿的二維平面,被壓扁成一條線(甚至一個點)。 欄向量這時是共線的(線性相依),它們的 span 從整個平面塌縮成一條直線:維度減少了。

這件事為什麼要命?因為一旦空間被壓扁,變換就無法復原了。想像你把一團三維黏土壓成一張二維薄餅—— 你永遠沒辦法只看薄餅,就知道原本黏土的厚度資訊跑哪去了,那個資訊被毀掉了。 所以 $\det=0$ 等價於「這個矩陣不可逆(non-invertible / singular)」。 這是行列式最實用的用途之一:一眼判斷一個矩陣可不可逆、一個方程組有沒有唯一解。 

import numpy as np

A = np.array([[3, 1],
              [0, 2]])
print(np.linalg.det(A))        # 6.0  ← 面積放大 6 倍,可逆

B = np.array([[2, 1],
              [4, 2]])          # 第二欄 (1,2) = 0.5 × 第一欄 (2,4),共線
print(np.linalg.det(B))        # 0.0  ← 被壓扁成一條線,不可逆!

線性方程組:$A\mathbf{x}=\mathbf{b}$ 到底在問什麼

中學解過的聯立方程,其實是線性代數最古老的應用。看這組:

$$\begin{cases}x+2y=5\\3x-y=1\end{cases}\qquad\Longleftrightarrow\qquad \underbrace{\begin{bmatrix}1 & 2\\3 & -1\end{bmatrix}}_{A}\underbrace{\begin{bmatrix}x\\y\end{bmatrix}}_{\mathbf{x}} =\underbrace{\begin{bmatrix}5\\1\end{bmatrix}}_{\mathbf{b}}$$

把一堆方程式打包成 $A\mathbf{x}=\mathbf{b}$ 之後,它有了一個漂亮的幾何詮釋: 「有沒有一個向量 $\mathbf{x}$,經過 $A$ 這個變換之後,剛好落在 $\mathbf{b}$ 的位置?」 換句話說,我們在尋找變換 $A$ 的「原像」——是哪個點被搬到了 $\mathbf{b}$。 若 $A$ 可逆($\det\neq 0$),答案唯一;若 $\det=0$(空間被壓扁),那可能無解(因為 $\mathbf{b}$ 不在那條被壓扁的線上), 也可能有無窮多解。

另一種同樣有用的幾何看法:每一條方程式,是平面上的一條直線;解,就是這些線的交點。

高斯消去法:有系統地解

解兩三個變數靠湊還行,但變數一多就需要系統化的方法——高斯消去法(Gaussian elimination)。 它的精神就是國中學的「加減消去」,只是規範化了:用三種「不改變解」的列運算 (某列乘一個非零數、兩列對調、某列加上另一列的倍數),把矩陣整理成「階梯狀」的上三角形, 然後由下往上一個個回代。我們手解上面那組: 

原始:   x + 2y = 5      ... (1)
        3x −  y = 1      ... (2)

消去 x:把 (2) 減去 3×(1):
        (3x − y) − 3(x + 2y) = 1 − 15
                      −7y = −14   →   y = 2

回代:  代入 (1):x + 2(2) = 5   →   x = 1
解:    (x, y) = (1, 2)  ✓

實務上,當然交給 numpy。但請千萬別用「先求反矩陣再相乘」的方式(理由下一節說明),而要用專門的解算器:

import numpy as np

A = np.array([[1, 2],
              [3, -1]])
b = np.array([5, 1])

x = np.linalg.solve(A, b)      # 內部就是高斯消去(的數值穩定版本)
print(x)                       # [1. 2.]  ← (x, y) = (1, 2)

秩:這個變換「保住」了幾個維度?

和消去法緊密相關的概念是秩(rank):矩陣的秩,是它的欄向量裡「線性獨立的個數」, 也就是變換後的空間還剩幾維。一個 $2\times2$ 矩陣若秩為 2,代表它把平面完整地映成平面(滿秩、可逆); 若秩為 1,代表它把平面壓成了一條線($\det=0$);若秩為 0,壓成了一個點。 秩,就是用「維度」的語言,精準地說出「這個變換損失了多少資訊」。 

import numpy as np

print(np.linalg.matrix_rank(np.array([[1, 2],[3, -1]])))   # 2  滿秩,可逆
print(np.linalg.matrix_rank(np.array([[2, 1],[4, 2]])))    # 1  壓成一條線

反矩陣:把變換倒帶

如果矩陣 $A$ 是一個動作,那「把這個動作還原」的動作,就是它的反矩陣(inverse), 記作 $A^{-1}$。先做 $A$ 再做 $A^{-1}$,等於什麼都沒做——這個「什麼都沒做」的變換, 就是單位矩陣(identity matrix) $I$(對角線全 1、其餘全 0,它讓每個基向量待在原地):

$$A^{-1}A=AA^{-1}=I=\begin{bmatrix}1&0\\0&1\end{bmatrix}$$

有了反矩陣,$A\mathbf{x}=\mathbf{b}$ 在理論上可以「一行解決」:兩邊左乘 $A^{-1}$,得到 $\mathbf{x}=A^{-1}\mathbf{b}$。但能不能倒帶,取決於 $A$ 有沒有把空間壓扁: 如果 $\det(A)=0$(壓成了線),資訊已經毀掉,沒有任何變換能把薄餅還原成黏土——這時反矩陣不存在。 所以:$A$ 可逆 $\iff \det(A)\neq 0 \iff$ 滿秩。這三句話講的是同一件事。

$2\times2$ 的反矩陣有公式(把對角線對調、副對角線變號,再除以行列式):

$$\begin{bmatrix}a&b\\c&d\end{bmatrix}^{-1}=\frac{1}{ad-bc}\begin{bmatrix}d&-b\\-c&a\end{bmatrix}$$

看得出來:$ad-bc$(也就是 $\det$)若為 $0$,公式裡會出現除以零——這又一次說明「$\det=0$ 就不可逆」。

⚠️
大系統千萬不要「先求逆再相乘」

理論上 $\mathbf{x}=A^{-1}\mathbf{b}$ 很美,但在實際數值計算裡,明確地算出反矩陣是個壞主意。 原因有二:(1) 慢——求逆比直接解方程多做好幾倍的工;(2) 不準——求逆會放大浮點誤差, 對「接近不可逆」的矩陣尤其危險。專業的數值函式庫一律用消去法的變體(如 LU 分解)直接解 $A\mathbf{x}=\mathbf{b}$,也就是 np.linalg.solve(A, b),而非 np.linalg.inv(A) @ b。這個「為什麼不直接求逆」的故事,正是第 55 章數值方法的核心關懷之一: 數學上對的算法,在有限精度的電腦上不一定是好算法。

點積與投影:相似度的幾何

除了相加與縮放,向量還有一個極其有用的運算:點積(dot product,又稱內積)。 它吃進兩個向量,吐出一個純量(一個數字),做法是對應分量相乘再加總:

$$\mathbf{v}\cdot\mathbf{w}=v_1 w_1+v_2 w_2+\dots+v_n w_n$$

例如 $(3,2)\cdot(1,4)=3\times1+2\times4=11$。這個不起眼的數字,藏著豐富的幾何意義。 關鍵公式是:

$$\mathbf{v}\cdot\mathbf{w}=\lVert\mathbf{v}\rVert\,\lVert\mathbf{w}\rVert\cos\theta$$

其中 $\theta$ 是兩向量的夾角。它告訴我們:點積衡量兩個向量「方向有多一致」。 當它們指向相近($\theta$ 小,$\cos\theta\approx1$),點積大且為正;當它們垂直(正交,orthogonal) 時 $\cos 90°=0$,點積恰好為零——這是判斷垂直最方便的測試;當它們指向相反,點積為負。

從這個公式還能讀出投影(projection)的意味:點積本質上是在問「$\mathbf{v}$ 在 $\mathbf{w}$ 方向上的影子有多長」。 把光從正上方打下來,$\mathbf{v}$ 在 $\mathbf{w}$ 所在直線上投下的影子,長度正比於點積。第 39 章圖學裡的光照計算 (一個表面被照得多亮,取決於法線與光線方向的點積),用的就是這個。

餘弦相似度:ML 怎麼判斷「兩段文字像不像」

把上面的公式反過來解出 $\cos\theta$,就得到大名鼎鼎的餘弦相似度(cosine similarity):

$$\cos\theta=\frac{\mathbf{v}\cdot\mathbf{w}}{\lVert\mathbf{v}\rVert\,\lVert\mathbf{w}\rVert}$$

它的值落在 $[-1,1]$:愈接近 $1$ 代表兩向量方向愈一致(愈「相似」),$0$ 代表毫不相干(正交),$-1$ 代表完全相反。 這正是第 38 章機器學習裡無所不在的相似度度量。當搜尋引擎、推薦系統或大型語言模型把一篇文章、一個商品、一個詞 變成一個高維向量(稱為 嵌入 embedding)之後,要問「這兩樣東西像不像」, 算的就是它們向量之間的餘弦相似度。「語意相近」這件抽象的事,就這樣被化約成了「向量夾角小」這件幾何的事。 

import numpy as np

def cosine_similarity(v, w):
    return np.dot(v, w) / (np.linalg.norm(v) * np.linalg.norm(w))

a = np.array([1, 0, 1, 0, 1])      # 想像成三篇文件的詞頻向量
b = np.array([1, 1, 1, 0, 1])      # b 和 a 很像
c = np.array([0, 1, 0, 1, 0])      # c 和 a 幾乎相反

print(round(cosine_similarity(a, b), 3))   # 0.866  ← 很相似
print(round(cosine_similarity(a, c), 3))   # 0.0    ← 正交,毫不相干

特徵值與特徵向量:變換中「不改方向」的特殊軸

我們抵達了全章的重頭戲。回到「矩陣是把空間揉捏變形」的畫面:大多數向量在變換後, 會被推往一個新的方向(像圖 54-4 裡那些斜掉的格線)。但對某些特別的矩陣, 會有那麼幾條「黃金方向」——落在這些方向上的向量,變換後方向完全不變,只是被拉長或縮短(甚至反向)。 它們像是變換的「旋轉軸」,任憑空間怎麼扭,自己穩穩待在原本的直線上。

這種「方向不變」的向量,叫特徵向量(eigenvector);它被拉伸的倍率,叫對應的 特徵值(eigenvalue) $\lambda$。用一條式子表達這個「變換後只是被縮放」的條件:

$$A\mathbf{v}=\lambda\mathbf{v}\qquad(\mathbf{v}\neq\mathbf{0})$$

左邊 $A\mathbf{v}$ 是「對 $\mathbf{v}$ 做變換」,右邊 $\lambda\mathbf{v}$ 是「把 $\mathbf{v}$ 單純放大 $\lambda$ 倍」。 這條式子說:對特徵向量而言,「做完整個矩陣變換」竟然和「只是乘上一個數字」效果一樣——它身上發生的複雜變形,塌縮成了單純的伸縮。 ("eigen" 是德文,意思是「自己的、固有的」,所以 eigenvector 直譯就是「(這個矩陣)固有的向量」。)

怎麼求:特徵方程

怎麼找出這些黃金方向?把 $A\mathbf{v}=\lambda\mathbf{v}$ 移項成 $A\mathbf{v}-\lambda\mathbf{v}=\mathbf{0}$, 再寫成 $(A-\lambda I)\mathbf{v}=\mathbf{0}$。我們要找非零的 $\mathbf{v}$ 滿足它, 這代表矩陣 $(A-\lambda I)$ 必須把某個非零向量壓扁成零——也就是它必須是不可逆的,即

$$\det(A-\lambda I)=0$$

這就是特徵方程(characteristic equation)。對 $2\times2$ 矩陣,它會化成一個關於 $\lambda$ 的二次方程 $\lambda^2-(\text{跡})\lambda+\det=0$(其中「跡 trace」是對角線元素之和)。我們以 $A=\begin{bmatrix}2&1\\1&2\end{bmatrix}$ 為例手算:

$$\det\begin{bmatrix}2-\lambda & 1\\ 1 & 2-\lambda\end{bmatrix}=(2-\lambda)^2-1=0 \ \Rightarrow\ (2-\lambda)^2=1\ \Rightarrow\ \lambda=1\ \text{或}\ \lambda=3$$

把 $\lambda=3$ 代回 $(A-3I)\mathbf{v}=\mathbf{0}$,解得特徵向量方向 $(1,1)$; 把 $\lambda=1$ 代回,得方向 $(1,-1)$。和圖 54-10 完全吻合。當然,實際計算交給 numpy: 

import numpy as np

A = np.array([[2, 1],
              [1, 2]])

eigenvalues, eigenvectors = np.linalg.eig(A)
print("特徵值:", eigenvalues)              # [3. 1.]
print("特徵向量(每一欄是一個):")
print(eigenvectors)
# [[ 0.707 -0.707]      第一欄 ≈ (1,1)/√2  對應 λ=3
#  [ 0.707  0.707]]     第二欄 ≈ (-1,1)/√2 對應 λ=1

# 驗證 A v = λ v(取第一個特徵對)
v0 = eigenvectors[:, 0]
print(A @ v0)                 # ≈ [2.12 2.12]
print(eigenvalues[0] * v0)    # ≈ [2.12 2.12]  ← 兩者相等,定義成立!

(numpy 回傳的特徵向量被正規化成長度 1,所以是 $(1,1)$ 除以 $\sqrt 2\approx0.707$;方向不變。)

為什麼特徵向量這麼重要?三個應用

特徵向量遠不只是課本習題。它是線性代數最有威力的概念之一,直接驅動了幾個你天天在用的系統:

  • PageRank(Google 的起家技術,呼應第 30 章):把整個網路想成一個巨大的矩陣 $M$, $M_{ij}$ 描述「從網頁 $j$ 點到網頁 $i$ 的機率」。一個網頁的重要性,取決於有多少重要的網頁連向它—— 這是個循環定義,但線性代數能優雅地解開它:整個網路的重要性分數,正是矩陣 $M$ 對應特徵值 $1$ 的那個 主特徵向量(dominant eigenvector)。實務上用「冪迭代」(反覆把任意向量乘上 $M$,它會自動收斂到主特徵向量)就能算出來。
  • PCA 主成分分析(降維,呼應第 38 章):當你有一堆高維資料,想找出「資料變化最劇烈的幾個方向」來壓縮維度, 答案是資料共變異數矩陣的特徵向量。最大特徵值對應的特徵向量,就是資料散布得最開的那個方向(第一主成分); 只保留前幾個主成分,就能用少少幾個數字捕捉資料大部分的資訊。這是資料視覺化與壓縮的利器。
  • 震動模態與穩定性:橋樑、樂器、分子的自然振動頻率,對應其物理矩陣的特徵值; 一座橋會不會因共振而垮(著名的塔科馬海峽大橋),問的就是特徵值。同樣的數學也出現在 量子力學(能階是算符的特徵值)與動態系統的穩定性分析。
🔑
特徵向量是「看穿一個矩陣本質」的鑰匙

一個複雜的矩陣,在它自己的特徵向量這組「天然座標軸」底下看,會變得極其簡單——它的作用只是「沿著每個特徵方向,各自伸縮對應的特徵值倍」。 這個「換到對的座標系,複雜變簡單」的想法,是 PCA、PageRank、量子力學、乃至更進階的 奇異值分解(SVD)背後共通的靈魂。當你發現特徵向量一再出現在毫不相干的領域,別驚訝: 它們是在告訴你那個系統最深層的結構。

線性代數在 CS 的應用速覽

我們已經一路看到不少應用了,這裡把線性代數在 CS 各角落的身影,做一次鳥瞰式的整理:

  • 機器學習(第 38 章):資料天生就是向量(每個樣本是特徵向量,整個資料集是矩陣)。 線性迴歸本質上是把目標向量「投影」到特徵張成的空間上(用的就是點積與投影); 神經網路每一層都是「一次矩陣乘法 $W\mathbf{x}+\mathbf{b}$ 再套一個非線性函數」—— 矩陣乘法負責「線性地混合與變換」,非線性函數負責「掰彎」。 少了那個非線性,疊幾層都沒用:因為矩陣乘矩陣還是矩陣(合成!),多層會塌縮成單層,模型學不到複雜的東西。
  • 電腦圖學(第 39 章):3D 物體的每個頂點是向量,平移、旋轉、縮放、透視投影全是矩陣; 一整套變換用矩陣乘法串成一條「管線」,GPU 每秒對數百萬頂點做這些矩陣運算。本章的旋轉矩陣,你在那裡會再見到。
  • 圖論(第 15、30 章):一張圖(graph)可以寫成鄰接矩陣 $A$,$A_{ij}=1$ 表示節點 $i,j$ 相連。 神奇的是,$A^k$ 的第 $(i,j)$ 格,正好是「從 $i$ 走 $k$ 步到 $j$ 的路徑數」——圖的結構問題,就這樣變成了矩陣運算問題。PageRank 也是這個家族的成員。
  • 資料壓縮 / 推薦系統:奇異值分解(SVD)能把任何矩陣拆成「旋轉 × 伸縮 × 旋轉」, 只保留最大的幾個伸縮量,就能用極少的資料近似原矩陣——這是影像壓縮與 Netflix 式推薦系統的數學核心。
  • 量子計算:量子位元的狀態是(複數)向量,量子閘是保持長度的特殊矩陣(么正矩陣 unitary), 一段量子演算法就是一連串矩陣乘法作用在狀態向量上。整個量子計算,可以說就是「在複數向量空間裡做線性代數」。
🔍
一個橫貫全 CS 的模式

注意到了嗎?上面每一條,核心動作都是同兩件事:把現實世界的東西編碼成向量,再用矩陣去變換它們。 文字、圖片、3D 模型、網頁、社交網路、量子態——五花八門的對象,一旦變成向量,就都能用同一套線性代數工具去處理。 這就是為什麼說它是 CS 的「共同語言」:它提供了一個讓無數問題長得一樣、因而能用同一批演算法解決的統一框架。

✏️ 練習 54.1 思考

給定 $\mathbf{v}=(2,3)$、$\mathbf{w}=(4,-1)$。(a) 手算 $\mathbf{v}+\mathbf{w}$,並說明它在「首尾相接」圖上的幾何意義。 (b) 手算點積 $\mathbf{v}\cdot\mathbf{w}$,它是正是負?這代表兩向量的夾角大致是銳角還是鈍角?用 numpy 驗證,並算出實際夾角。

看解答

(a) $\mathbf{v}+\mathbf{w}=(2+4,\ 3+(-1))=(6,2)$。幾何上:從原點先走到 $(2,3)$,再從那裡接著走 $\mathbf{w}=(4,-1)$ 的位移(往右 4、往下 1),終點落在 $(6,2)$——這就是首尾相接。

(b) $\mathbf{v}\cdot\mathbf{w}=2\times4+3\times(-1)=8-3=5>0$。點積為正,代表 $\cos\theta>0$,夾角是銳角(兩向量大致同向)。

import numpy as np
v = np.array([2, 3]); w = np.array([4, -1])
print(v + w)                       # [6 2]
dot = np.dot(v, w); print(dot)     # 5
cos = dot / (np.linalg.norm(v) * np.linalg.norm(w))
print(np.degrees(np.arccos(cos)))  # ≈ 60.26 度(銳角,符合預期)
✏️ 練習 54.2 實作

判斷下列兩組向量各自是否線性獨立:(a) $(1,2)$ 與 $(2,4)$;(b) $(1,2)$ 與 $(2,1)$。 先用「其中一支是不是另一支的倍數」的直覺判斷,再用 numpy 的 matrix_rank 或行列式驗證。

看解答

(a) $(2,4)=2\times(1,2)$,第二支是第一支的兩倍——共線,線性相依。 (b) $(2,1)$ 不是 $(1,2)$ 的任何倍數(若 $(2,1)=k(1,2)$ 則 $k=2$ 又 $k=0.5$,矛盾), 方向不同,線性獨立。

把兩向量當成矩陣的欄:獨立 $\iff$ 滿秩 $\iff \det\neq0$。

import numpy as np
A = np.array([[1, 2],[2, 4]])      # 欄為 (1,2),(2,4)
print(np.linalg.matrix_rank(A))    # 1  -> 相依
print(np.linalg.det(A))            # 0.0

B = np.array([[1, 2],[2, 1]])      # 欄為 (1,2),(2,1)
print(np.linalg.matrix_rank(B))    # 2  -> 獨立
print(round(np.linalg.det(B), 1))  # -3.0 (≠0)
✏️ 練習 54.3 實作

(呼應第 39 章)把點 $P=(1,0)$ 逆時針旋轉 $90^\circ$。先用旋轉矩陣手算,再寫程式驗證; 進階:寫一個函式 rotate(point, degrees),能旋轉任意角度,並驗證旋轉 $90^\circ$ 四次會回到原點。

看解答

逆時針 $90^\circ$ 的旋轉矩陣是 $\begin{bmatrix}0&-1\\1&0\end{bmatrix}$ ($\cos90°=0,\ \sin90°=1$)。手算: $\begin{bmatrix}0&-1\\1&0\end{bmatrix}\begin{bmatrix}1\\0\end{bmatrix} =\begin{bmatrix}0\\1\end{bmatrix}$。$(1,0)$ 轉到 $(0,1)$——本來指右,現在指上,正確。

import numpy as np

def rotate(point, degrees):
    t = np.radians(degrees)
    R = np.array([[np.cos(t), -np.sin(t)],
                  [np.sin(t),  np.cos(t)]])
    return R @ np.array(point)

print(rotate([1, 0], 90))          # [0. 1.]  右 -> 上

p = np.array([1.0, 0.0])
for _ in range(4):                 # 轉 90 度四次 = 360 度
    p = rotate(p, 90)
print(np.round(p, 10))             # [1. 0.]  ← 轉回原點
✏️ 練習 54.4 實作

用高斯消去法手解下面這組三元一次方程組,再用 np.linalg.solve 對答案:

$$\begin{cases}x+y+z=6\\ 2y+5z=-4\\ 2x+5y-z=27\end{cases}$$

看解答

用第 1 式消去第 3 式的 $x$:第 3 式減去 $2\times$第 1 式,得 $(2x+5y-z)-2(x+y+z)=27-12$,即 $3y-3z=15\Rightarrow y-z=5$。

現在有 $\begin{cases}2y+5z=-4\\ y-z=5\end{cases}$。由第二式 $y=z+5$ 代入第一式: $2(z+5)+5z=-4\Rightarrow 7z=-14\Rightarrow z=-2$,於是 $y=3$。 回代第 1 式:$x+3+(-2)=6\Rightarrow x=5$。解為 $(x,y,z)=(5,3,-2)$。

import numpy as np
A = np.array([[1, 1,  1],
              [0, 2,  5],
              [2, 5, -1]])
b = np.array([6, -4, 27])
print(np.linalg.solve(A, b))       # [ 5.  3. -2.]  ← 與手算一致
✏️ 練習 54.5 思考

計算 $\det\begin{bmatrix}3&1\\2&4\end{bmatrix}$,並用「面積縮放」解釋這個數字的意義。 接著計算 $\det\begin{bmatrix}1&2\\2&4\end{bmatrix}$,它為什麼是 0?這對「這個矩陣可不可逆」說明了什麼?

看解答

第一個:$\det=3\times4-1\times2=10$。意義:這個變換把所有面積放大 10 倍—— 單位正方形(面積 1)會被變成面積 10 的平行四邊形。因為 $10\neq0$,矩陣可逆。

第二個:$\det=1\times4-2\times2=0$。它為 0 是因為兩欄 $(1,2)$ 與 $(2,4)$ 共線 ($(2,4)=2\times(1,2)$),變換把整個平面壓扁成一條線,面積縮放倍率自然是 0。 既然空間被壓扁、資訊遺失,這個變換無法還原——也就是不可逆(奇異)。

import numpy as np
print(np.linalg.det(np.array([[3, 1],[2, 4]])))   # 10.0
print(np.linalg.det(np.array([[1, 2],[2, 4]])))   # 0.0
✏️ 練習 54.6 實作

求矩陣 $A=\begin{bmatrix}4&1\\2&3\end{bmatrix}$ 的特徵值。先用特徵方程 $\lambda^2-(\text{跡})\lambda+\det=0$ 手算,再用 numpy 驗證,並對其中一個特徵向量檢查 $A\mathbf{v}=\lambda\mathbf{v}$。

看解答

跡 $=4+3=7$,$\det=4\times3-1\times2=10$。特徵方程: $\lambda^2-7\lambda+10=0\Rightarrow(\lambda-2)(\lambda-5)=0\Rightarrow\lambda=2$ 或 $\lambda=5$。

import numpy as np
A = np.array([[4, 1],[2, 3]])
vals, vecs = np.linalg.eig(A)
print(vals)                        # [5. 2.]

# 驗證第一個特徵對:A v 應該等於 λ v
v = vecs[:, 0]; lam = vals[0]
print(np.round(A @ v, 6))          # 例如 [3.16 3.16]
print(np.round(lam * v, 6))        # 同上 ← A v = λ v 成立

解讀:沿 $\lambda=5$ 的特徵方向,空間被拉伸 5 倍;沿 $\lambda=2$ 的方向拉伸 2 倍。 兩個特徵值都為正,代表這個變換在兩個特徵方向上都只是「同向放大」,沒有翻面也沒有壓扁 (它們的乘積 $2\times5=10$ 恰好等於 $\det$——這不是巧合,行列式永遠等於所有特徵值的乘積!)。

✏️ 練習 54.7 挑戰

(二選一)(A) PageRank 與特徵向量:對下面這個 3 節點的網路連結矩陣 $M$ (每一欄已正規化成機率,加總為 1),用「冪迭代」——反覆把一個向量乘上 $M$——找出穩定的重要性分數, 並說明它和「特徵值為 1 的特徵向量」的關係。 $$M=\begin{bmatrix}0 & 0 & 1\\ 0.5 & 0 & 0\\ 0.5 & 1 & 0\end{bmatrix}$$ (B) 簡單 PCA:對一組沿著 $y=x$ 方向散布的 2D 點雲,用共變異數矩陣的特徵向量找出「資料變化最大的方向」。

看解答

(A) PageRank。重要性向量 $\mathbf{r}$ 必須滿足「我的分數 = 連向我的人的分數依連結分配後的總和」, 也就是 $M\mathbf{r}=\mathbf{r}$。這正是「特徵值為 1 的特徵向量」的定義!冪迭代從任意分布出發, 反覆乘 $M$,就會收斂到這個主特徵向量:

import numpy as np
M = np.array([[0,   0, 1],
              [0.5, 0, 0],
              [0.5, 1, 0]])

r = np.array([1/3, 1/3, 1/3])      # 任意起始分布(總和為 1)
for _ in range(50):
    r = M @ r                       # 反覆施作,自動收斂
print(np.round(r, 4))               # [0.4 0.2 0.4] ← 穩定的 PageRank 分數

# 驗證它就是特徵值 1 的特徵向量
vals, vecs = np.linalg.eig(M)
print(np.round(vals.real, 3))       # 其中一個是 1.0

節點 1 與 3 同樣重要(0.4),節點 2 較不重要(0.2)。把網頁重要性問題,化約成了求一個矩陣的主特徵向量——這就是 Google 起家的核心數學。

(B) PCA。主成分就是共變異數矩陣的特徵向量,最大特徵值對應「資料散得最開」的方向:

import numpy as np
# 沿 y=x 拉長的點雲
np.random.seed(0)
t = np.random.randn(200)
data = np.stack([t + 0.1*np.random.randn(200),
                 t + 0.1*np.random.randn(200)], axis=1)

data = data - data.mean(axis=0)              # 置中
cov = np.cov(data.T)                          # 2x2 共變異數矩陣
vals, vecs = np.linalg.eig(cov)
pc1 = vecs[:, np.argmax(vals)]               # 最大特徵值對應的特徵向量
print(np.round(pc1, 3))                       # ≈ [0.707 0.707] -> 正是 y=x 方向!

第一主成分指向 $(0.707,0.707)$,也就是 $y=x$——恰好是資料散布最開的方向。 只保留這一個方向上的投影,就能把二維資料壓成一維而幾乎不失真,這就是 PCA 降維的精髓。

把這一章串起來

我們從手機解鎖、3D 遊戲與 Google 搜尋出發,最後抵達了 PageRank 與 PCA 的引擎室。回顧這趟旅程的路標:

  • 向量有三種身分(箭頭/數字清單/抽象元素),只靠兩個動作運轉:相加(首尾相接)與純量乘法(伸縮)。
  • 反覆「縮放再相加」就是線性組合;它們湊得出的所有向量是生成空間;沒有冗餘的一組叫線性獨立;剛好撐滿空間的精簡組叫基,其大小就是維度。
  • 本章的心臟:矩陣是線性變換——把空間整齊變形的函數。它的欄向量,就是基向量變換後的落點。
  • 矩陣乘法是變換的合成(由右往左、通常不可交換);行列式是面積縮放倍率,$\det=0$ 代表空間被壓扁、不可逆。
  • $A\mathbf{x}=\mathbf{b}$ 問「誰被變換到了 $\mathbf{b}$」;用高斯消去解,秩告訴你保住了幾維;反矩陣是「倒帶」,但大系統別直接求逆(呼應第 55 章)。
  • 點積衡量方向一致程度,衍生出投影與餘弦相似度(ML 的相似度尺;呼應第 38、39 章)。
  • 特徵向量是變換中「打死不轉向、只被縮放」的黃金方向,特徵值是縮放倍率——它驅動了 PageRank、PCA、震動分析與量子力學。

最重要的收穫,不是任何一條公式,而是一種看的方式:當你看到一個矩陣,不再看到一堆冷冰冰的數字, 而是看到一隻手正在把整個空間旋轉、拉扯、剪切;當你看到 $A\mathbf{v}=\lambda\mathbf{v}$, 腦中浮現的是一支在風暴中紋風不動、只是被拉長的箭頭。一旦你「看見」了這些畫面, 線性代數就從一門要硬背的科目,變成了一種視覺直覺——而這正是讀懂機器學習、圖學與現代 CS 大半江山的鑰匙。
