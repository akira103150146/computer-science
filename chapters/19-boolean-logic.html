一場從 0 與 1 開始的造塔運動

在第 1 章,我們說整個電腦世界是一座抽象的高塔:電晶體被抽象成邏輯閘,邏輯閘被抽象成 CPU, 機器碼被抽象成組合語言、再被抽象成 C、再被抽象成 Python……每一層都站在前一層的肩膀上。 當時那只是一句話帶過的承諾。現在,我們要把這座塔一層一層、親手蓋出來。

這正是第五部「計算機系統」要做的事,它沿著一條被稱為 Nand2Tetris(從 NAND 到俄羅斯方塊)的著名路線前進:我們從一顆 NAND 閘這個最原始的零件出發, 逐層往上組——閘 → ALU(算術邏輯單元)→ CPU → 機器語言與組譯器 → 虛擬機 → 編譯器,最後是作業系統。 蓋到頂層時,你會在自己親手造的機器上,跑自己寫的高階語言程式。途中沒有任何魔法,沒有任何「黑盒子」是你不懂的—— 因為每一塊磚都是你自己疊上去的。

塔底那塊磚——NAND 閘——背後其實是一套漂亮的數學。要懂它,我們得先回答兩個更基本的問題: 為什麼電腦只用 0 和 1? 以及 怎麼用 0 和 1 來「思考」? 走吧,我們從地基開始挖。

為什麼是二進位?

我們日常用十進位(decimal),有 0 到 9 十個數字,大概是因為人有十根手指。那電腦為什麼不也用十進位, 搞十種狀態,不是更省空間嗎?答案出乎意料地實際:因為「兩種狀態」最好認、最不會出錯。

電腦內部靠電壓傳遞資訊。如果你想用十進位,就得把電壓切成十個等級(例如 0V、0.5V、1V……), 然後在嘈雜、發熱、訊號會衰減的真實電路裡,精準分辨「這到底是 0.5V 還是 0.6V」。這幾乎不可能可靠。 但如果你只分兩種——「高電壓」與「低電壓」——那中間就有一大片緩衝地帶: 訊號就算被雜訊推高推低個零點幾伏,只要還沒越過中央那條判讀門檻,就還是被讀成原本的值。

這個「高/低」「開/關」「通/斷」的最小資訊單位,我們叫它一個位元(bit,binary digit 的縮寫): 它只能是 0 或 1。一個位元只能表達兩種可能,但把位元排起來,表達力就爆炸成長:8 個位元能表示 $2^8 = 256$ 種值, 16 個位元能表示 $2^{16} = 65536$ 種。整台電腦裡流動的所有東西——數字、文字、圖片、影片、程式本身—— 在最底層,全都是一長串 0 和 1。

而這正是奇妙的連結所在:「0 與 1」剛好對得上「假與真」。 還記得第 13 章的命題邏輯嗎?那裡的每個命題非真($\text{T}$)即假($\text{F}$),用連接詞 $\wedge$(且)、$\vee$(或)、$\neg$(非)組合起來。 現在把 $\text{T}$ 換成 $1$、$\text{F}$ 換成 $0$,把 $\wedge\vee\neg$ 換成 AND/OR/NOT—— 那套「對與錯的邏輯」,就搖身一變,成了能在電路裡計算的代數。這套代數,就是布林代數。

🔑
一個位元,連結了三個世界

同一個位元,可以被看成三種東西,而它們在數學上完全等價:
物理:一條電線的高電壓 / 低電壓。
邏輯:一個命題的真 / 假(第 13 章)。
數學:布林代數裡的 $1$ / $0$。
整個第五部,就是在這三個世界之間自由換算。你在離散數學學的 De Morgan 定律,等一下會直接拿來省電晶體。

布林代數:把邏輯變成算術

布林代數(Boolean algebra)是一套只在 $\{0, 1\}$ 這兩個值上運作的代數。 它的「變數」只能取 0 或 1,它的「運算」吃進 0/1、吐出 0/1。一個吃 $n$ 個位元、吐出一個位元的函數, 就叫一個布林函數(Boolean function)。整顆 CPU 說穿了,就是一大堆布林函數的集合體。 我們先從三個最基本的運算開始。

三個基本運算:AND、OR、NOT

要徹底定義一個布林運算,最乾淨的方法是把所有輸入組合都列出來,逐一寫下輸出——這張表就是 真值表(truth table)。$n$ 個輸入,就有 $2^n$ 列。三個基本運算如下(在硬體傳統裡, AND 寫成 $\cdot$、OR 寫成 $+$、NOT 寫成 $\neg$ 或在變數上加一槓):

📜
兩個人,隔著八十年合作

1854 年,英國數學家 George Boole(布爾)出版《思維的法則》(An Investigation of the Laws of Thought), 想用代數來描述人類的邏輯推理。當時這是純哲學,跟機器毫無關係。八十多年後的 1937 年, 一個叫 Claude Shannon(夏農)的 MIT 碩士生,在他的碩士論文裡指出:布爾那套代數, 剛好可以描述由開關組成的電路——「真/假」對應「通/斷」。這篇論文被譽為史上最重要的碩士論文之一, 它把抽象的邏輯,接上了實體的電路,直接催生了今天的數位電腦。你接下來看到的一切,都站在這兩個人的肩膀上。

更多布林函數:XOR、NAND、NOR

兩個輸入的布林函數,理論上有 $2^{2^2} = 16$ 種(輸出那一欄有 4 格,每格可填 0 或 1)。除了 AND 和 OR, 還有三個特別常用、特別重要,你一定要認得:

  • XOR(互斥或,exclusive or),記號 $A \oplus B$:「剛好一個是 1」才輸出 1。 兩個一樣(都 0 或都 1)就輸出 0。它其實就是「兩者不同嗎?」的偵測器,做加法時的進位邏輯會用到它。
  • NAND(反及,not-and),記號 $\neg(A \cdot B)$:就是「AND 之後再 NOT」。除了「兩個都 1」會輸出 0,其餘全是 1。
  • NOR(反或,not-or),記號 $\neg(A + B)$:就是「OR 之後再 NOT」。只有「兩個都 0」才輸出 1。

布林恆等式:化簡的本錢

既然布林運算像算術,它也有一整套「運算律」。記住它們,你就能把一條複雜的布林式化簡—— 在硬體上,化簡一條式子就等於少用幾顆閘、省下電晶體、省電、跑更快。這就是晶片設計師天天在做的事。

最該背熟的是最後一條De Morgan 定律(第 13 章已經見過它的邏輯版),口訣: 「否定穿過括號時,『且』與『或』要對調」。它在程式裡天天出現: 

# 想表達「這個數『不是』介於 1 到 10 之間」
if not (1 <= x and x <= 10):
    ...

# 用 De Morgan 改寫,意思一模一樣,卻更直觀:「太小,或太大」
if x < 1 or x > 10:
    ...

在晶片上,同一條 De Morgan 定律意味著:一個「AND 後反相」的電路,可以換成「先各自反相再 OR」—— 哪種比較省零件,就用哪種。你在數學課學的等式,在這裡是真金白銀的成本。我們等一下還會看到它更狠的用法: 用它把所有閘都換成 NAND。

== 0 的否定是 != 0,> 10 的否定是 <= 10。化簡後不但少一層否定,讀起來也更像人話。

邏輯閘:讓布林代數長出實體

到目前為止,布林代數還只是紙上的符號。把它變成一塊會動的硬體,靠的就是邏輯閘(logic gate)—— 一個用電晶體做成的小電路,吃進幾條代表 0/1 的電線,吐出一條 0/1 的電線。

🔑
閘 = 布林函數的硬體化身

一個邏輯閘,就是一個布林函數的實體實作。「AND 函數」是數學概念,「AND 閘」是把這個概念焊進矽晶片的零件。 它們的真值表完全一樣。所以——只要你會寫布林式,你就會設計電路;反過來,任何電路在做什麼, 看它的布林式就懂了。這一層的抽象,正是讓我們可以「用代數推理硬體」的關鍵。

NAND 的魔法:一閘造萬物

這是 Nand2Tetris 整個計畫的核心洞見,也是這一章你最該帶走的一句話:

🔑
NAND 是「萬能閘(universal gate)」

只用 NAND 這一種零件,反覆組合,就能造出 NOT、AND、OR、XOR……以及任何你想得到的布林函數, 進而造出 ALU、記憶體、整顆 CPU。換句話說,整台電腦,理論上可以只由 NAND 閘堆成。 這就是為什麼我們的造塔運動,從「一顆 NAND」開始就夠了——它是宇宙的萬用磚。

聽起來像變魔術,但證明出乎意料地簡單:我們只要示範「怎麼用 NAND 做出 NOT、AND、OR」就好。 因為任何布林函數都能用 NOT/AND/OR 組出來(下一節會說明為什麼),而這三者又都能用 NAND 組出來, 那 NAND 自然就能組出一切。我們一個一個來,每一步都附上真值表當鐵證。

📜
NAND 的萬能性早有人證過

NAND 又叫「Sheffer stroke」,記號 $\mid$。哲學家兼邏輯學家 Charles Sanders Peirce 早在 1880 年左右就注意到 NAND(以及 NOR)單獨一個就足以表達所有邏輯運算,只是他的筆記直到 1933 年才出版;Henry M. Sheffer 則在 1913 年正式發表了這個結果, 因此這個記號掛上了他的名字。這個「一個運算就夠了」的性質,術語叫功能完備(functional completeness)——也是現代晶圓廠特別愛大量製造 NAND/NOR 的實際原因之一。

從真值表到電路:為什麼電腦能算「任何」邏輯

我們一直說「任何布林函數都能用閘做出來」。為什麼這麼有把握?這裡有一個漂亮、機械化、永遠有效的步驟, 能把任何一張真值表直接翻成一條布林式(進而翻成電路)。這個式子的標準形式叫 積之和(sum of products,SOP)。

做法只有三步,簡單到像填表:

  1. 看真值表,圈出所有輸出為 1 的列。
  2. 每一個這樣的列,寫出一個 AND 項(稱為最小項 minterm):該列中是 1 的變數寫原樣,是 0 的變數加上否定。這個 AND 項只在這一列為 1。
  3. 把所有這些 AND 項用 OR 串起來。完成。

為什麼一定對?因為每個最小項只負責「點亮」它對應的那一列,其餘列它都輸出 0;把它們 OR 起來, 就剛好在「該為 1 的列」為 1、其他列為 0——完美重現整張表。既然任何真值表都能這樣翻成「AND + OR + NOT」的式子, 而這三者又都能用 NAND 造,那結論就鎖死了:

🔑
這就是「電腦能算任何邏輯」的根據

任何布林函數 → 都能寫成 SOP 標準形式(AND/OR/NOT)→ 都能只用 NAND 實現。 所以,只要你能把一個問題的需求列成一張真值表,它就保證能被一塊電路算出來。 加法、比較、選擇、判斷……無一例外。這條從「真值表」到「電路」的橋,是整個數位硬體的地基。

四個列輸出 1。照步驟,每列寫一個最小項,再 OR 起來,得到未化簡的 SOP:

$$M = \underbrace{\neg A\, B\, C}_{011} + \underbrace{A\, \neg B\, C}_{101} + \underbrace{A\, B\, \neg C}_{110} + \underbrace{A\, B\, C}_{111}.$$

這個式子保證正確,但有點臃腫(4 個三輸入 AND)。用布林恆等式可以把它化簡成更省的形式 (化簡過程留作練習 19.4):

$$M = A\cdot B + B\cdot C + A\cdot C.$$

意思非常直白:「三組配對裡,只要任何一對同時贊成,就過半」。照這條化簡式,電路只要三個 AND 加一個 OR:

✏️ 練習 19.4 思考

承上,用布林恆等式,把三人多數決的未化簡 SOP $$M = \neg A\,B\,C + A\,\neg B\,C + A\,B\,\neg C + A\,B\,C$$ 一步步化簡成 $M = A\cdot B + B\cdot C + A\cdot C$。(提示:等冪律 $X = X + X + X$ 允許你把 $A\,B\,C$ 複製三份來湊對。)

看解答

先用等冪律,把唯一的「三人全贊成」項 $ABC$ 複製成三份(因為 $X + X + X = X$):

$$M = \neg A\,B\,C + A\,\neg B\,C + A\,B\,\neg C + \underbrace{A\,B\,C + A\,B\,C + A\,B\,C}_{\text{把一份拆成三份}}.$$

再把它們兩兩配對,每對提出公因子,括號內用補餘律 $X + \neg X = 1$ 收掉:

  • $A\,B\,\neg C + A\,B\,C = A\,B(\neg C + C) = A\,B \cdot 1 = A\,B$
  • $A\,\neg B\,C + A\,B\,C = A\,C(\neg B + B) = A\,C$
  • $\neg A\,B\,C + A\,B\,C = B\,C(\neg A + A) = B\,C$

三項相加:$M = A\,B + A\,C + B\,C$。化簡完成,電路從「4 個三輸入 AND + 1 個四輸入 OR」瘦身成「3 個二輸入 AND + 1 個三輸入 OR」——少了好幾顆閘。這就是恆等式的現金價值。$\blacksquare$

用 HDL 把晶片寫下來

畫電路圖很直觀,但真要設計大型晶片,工程師是用「寫程式」的方式來描述硬體的——這種語言叫 硬體描述語言(Hardware Description Language,HDL)。注意它和一般程式語言不同:它描述的不是 「依序執行的步驟」,而是「哪些零件、怎麼接線」——比較像在寫一張接線清單。

Nand2Tetris 用了一套教學用的精簡 HDL。一個晶片定義長這樣:用 IN 宣告輸入腳、OUT 宣告輸出腳, PARTS: 之後列出內部用到的零件,每行 Chip(a=..., b=..., out=...) 就像把一顆零件擺上去並接好線, 其中可以自己取名一些「內部線」把零件串起來。我們把剛剛圖 19-4 的構造,原封不動寫成 HDL—— 從唯一的原始零件 Nand 出發,一層層蓋上去: 

/** NOT:把唯一的輸入接到 Nand 的兩個腳 */
CHIP Not {
    IN  in;
    OUT out;
    PARTS:
    Nand(a=in, b=in, out=out);
}

/** AND:Nand 之後再反相 */
CHIP And {
    IN  a, b;
    OUT out;
    PARTS:
    Nand(a=a, b=b, out=nandOut);   // nandOut 是一條內部線
    Not(in=nandOut, out=out);      // 直接重用上面定義好的 Not!
}

/** OR:把 a、b 各自反相後再 Nand(這就是 De Morgan) */
CHIP Or {
    IN  a, b;
    OUT out;
    PARTS:
    Not(in=a, out=notA);
    Not(in=b, out=notB);
    Nand(a=notA, b=notB, out=out);
}

/** XOR:(a 且 非b) 或 (非a 且 b) */
CHIP Xor {
    IN  a, b;
    OUT out;
    PARTS:
    Not(in=a, out=notA);
    Not(in=b, out=notB);
    And(a=a,    b=notB, out=w1);    // a 且 非b
    And(a=notA, b=b,    out=w2);    // 非a 且 b
    Or(a=w1, b=w2, out=out);
}

看出那股「蓋塔」的味道了嗎?Not 只認得 Nand;And 用了 Nand 和剛蓋好的 Not; Or 用了 Not 和 Nand;Xor 又站在 Not/And/Or 之上。 每一層都只依賴前面已經造好的零件,而追到最底,全部都是 NAND。這正是抽象高塔在硬體上的真實長相—— 而且你以後用 And 時,完全不必再去想它裡面那兩顆 NAND,就像你用 find_max 不必想它的迴圈。

從一條線到一排線:多位元與多工器

到目前為止,我們的閘都只處理「一條線」上的 1 個位元。但實際的電腦一次要處理一整個 16 位元(或 32、64 位元)的數。 好消息是:多位元的閘,只是把單位元的閘並排起來而已。

一束並排傳輸的電線,稱為匯流排(bus);例如 16 條線綁成一束,就承載一個 16 位元的值。 想要一個「16 位元的 AND」(在 nand2tetris 裡叫 And16)?不必發明新東西——拿 16 個一位元 AND 閘, 第 $i$ 個負責處理兩個輸入的第 $i$ 位元,各做各的,16 個結果拼成 16 位元輸出就好。 Not16、Or16 全是同樣的「並排」把戲。位元之間互不干擾,平行得很乾脆。

多工器(Mux)與解多工器(Demux):用一個位元做選擇

還有兩個零件,雖然小,卻是後面蓋 CPU、記憶體時的關鍵——它們讓電路會「選擇」。

多工器(multiplexer,Mux)是一個「二選一開關」:它有兩條資料輸入 $a$、$b$,外加一條選擇線 sel, 輸出 $\text{out}$。當 $sel = 0$,輸出 $a$;當 $sel = 1$,輸出 $b$。它就是硬體版的 out = b if sel else a。 解多工器(demultiplexer,Demux)則是反過來:一條輸入 $\text{in}$,由 $sel$ 決定把它送到 $y_0$ 還是 $y_1$,沒被選中的那條輸出 0。

它們的真值表與布林式如下。注意這兩條式子,還是只用 AND/OR/NOT——所以,終究還是只用 NAND:

Mux(out)Demux
$sel$$\text{out}$$\text{in}$$sel$$(y_0, y_1)$
0$a$$x$0$(x, 0)$
1$b$$x$1$(0, x)$

$$\text{Mux: } \text{out} = (\neg sel \cdot a) + (sel \cdot b) \qquad\quad \text{Demux: } y_0 = \neg sel \cdot \text{in},\ \ y_1 = sel \cdot \text{in}.$$

把 Mux 也「並排」成 16 位元(Mux16),你就有了一個能在兩個 16 位元數之間二選一的開關。 手上有了 And16、Or16、Not16、Mux16 這些「一排線」的零件,我們就湊齊了蓋下一座樓的全部磚塊—— 下一章,我們要用它們組出 ALU(算術邏輯單元):一個能做加法、減法、比較、AND、OR…… 並用幾條控制線(很像這裡的 sel!)選擇「這次要算哪一種」的萬能運算核心。塔,要往上長一層了。

布林邏輯,就住在你的程式裡

最後,把鏡頭拉回你熟悉的世界。你寫了那麼多 Python 和 C,其實天天都在用布林邏輯——只是你可能沒意識到, 它們最終都會落到剛才這些閘上。而且,程式裡有兩套長得很像、卻天差地別的運算子,一定要分清楚。

邏輯運算 vs 位元運算

  • 邏輯運算子:Python 的 and / or / not(C 的 && / || / !)。 它們把整個值當成一個「真/假」來運算,結果是一個布林值。這對應「一個位元」的邏輯。
  • 位元運算子(bitwise):&(AND)、|(OR)、^(XOR)、~(NOT)。 它們把整數看成一排位元,對每一個位元各自做運算——這根本就是上一節的「16 位元閘並排」!
a = 0b1100   # 十進位 12,二進位 1100
b = 0b1010   # 十進位 10,二進位 1010

print(bin(a & b))  # 0b1000 → 逐位元 AND:每一位都「兩個都 1 才 1」
print(bin(a | b))  # 0b1110 → 逐位元 OR
print(bin(a ^ b))  # 0b0110 → 逐位元 XOR:每一位「不同才 1」
print(bin(a & 0b0110))  # 0b0100 → 用「遮罩(mask)」只保留中間兩位

看到 a & b 了嗎?它就是 4 個 AND 閘並排,一次處理 4 個位元——和 nand2tetris 的 And16 是同一回事, 只是位元數不同。位元運算之所以這麼快,正因為硬體上它就是「一排閘同時動」,一個時脈就算完。

⚠️
別把 and 和 & 搞混

它們不一樣!and 看整體真假:12 and 10 在 Python 裡得到 10(因為 12 為真,回傳後者)。 但 12 & 10 是逐位元 AND,得到 8。在 C 裡更要小心:x & y 和 x && y 是兩種完全不同的運算, 手滑寫錯一個是經典 bug 來源。判斷條件用邏輯版(and/&&),操作位元才用位元版(&)。

短路求值:邏輯運算的小聰明

邏輯運算子還有一個位元運算子沒有的特性:短路求值(short-circuit evaluation)。 想想 AND 的真值表——只要第一個運算元是「假」,不管第二個是什麼,整體一定是假。 所以聰明的語言發現左邊是假時,根本不會去算右邊。OR 則相反:左邊已經是真,就懶得算右邊了。 

def noisy():
    print("我被呼叫了!")   # 一旦執行就會印出來,讓我們看得見
    return True

# and:左邊是 False,結果鐵定是 False,右邊直接跳過
if False and noisy():       # noisy() 不會執行,什麼都不印
    pass

# or:左邊是 True,結果鐵定是 True,右邊直接跳過
if True or noisy():         # noisy() 也不會執行
    pass

# 這不只是省時間,更是一種保護。下面這行靠短路避免了除以零:
xs = []
if len(xs) > 0 and xs[0] > 5:   # 左邊為假,就不會去碰 xs[0](否則會出錯)
    print(xs[0])

最後那個例子是短路最實用的用途:用左邊的條件守住右邊,避免右邊在不該執行時出錯(這裡是避免存取空串列)。 位元運算 &、| 不會短路——它們一定會把兩邊都算出來再逐位元合併,因為它們的工作是「處理位元」,不是「做決定」。

(b) 用 Python 只定義一個 NAND 函式當原始零件,再用它一層層造出 NOT、AND、OR、XOR、MUX, 然後寫一段測試,用窮舉真值表的方式驗證 NAND 的萬能性(每個自製閘都和 Python 內建運算一致)。

(a) $sel = 0$ 時輸出跟著 $a$,$sel = 1$ 時跟著 $b$:

上半($sel=0$)的 out 完全照抄 $a$ 那欄,下半($sel=1$)完全照抄 $b$ 那欄——正是「二選一」,也吻合 $\neg sel\cdot a + sel\cdot b$。

(b) 整座塔只立在一顆 NAND 上:

def NAND(a, b):
    return int(not (a and b))      # 唯一的原始零件

def NOT(a):    return NAND(a, a)
def AND(a, b): return NOT(NAND(a, b))
def OR(a, b):  return NAND(NOT(a), NOT(b))           # De Morgan
def XOR(a, b): return OR(AND(a, NOT(b)), AND(NOT(a), b))
def MUX(a, b, sel): return OR(AND(NOT(sel), a), AND(sel, b))

# ---- 驗證萬能性:窮舉所有輸入,和 Python 內建運算逐一比對 ----
bits = (0, 1)
for a in bits:
    assert NOT(a) == 1 - a
for a in bits:
    for b in bits:
        assert AND(a, b) == (a & b)
        assert OR(a, b)  == (a | b)
        assert XOR(a, b) == (a ^ b)
        for sel in bits:
            assert MUX(a, b, sel) == (b if sel else a)

print("全部通過:只靠一顆 NAND,就重現了所有基本閘與 Mux!")

程式跑完印出成功訊息——這就是 NAND 萬能性的「可執行證明」。當然,如同第 13 章的提醒:窮舉這裡等於證明, 因為每個閘的輸入組合是有限的(2 或 3 個位元),全部列完就毫無漏網之魚。NAND 是真的萬能。$\blacksquare$

把這一章串起來

我們站在抽象高塔的塔基,鋪好了第一塊、也是最關鍵的一塊磚。回顧這趟旅程:

  • 電腦用二進位,因為「高/低」兩種狀態最能抵抗雜訊、最可靠。一個位元(bit),同時是電壓、是真假、是 0/1。
  • 布林代數把邏輯變成算術:AND、OR、NOT、XOR、NAND、NOR,各有真值表;一整套恆等式(尤其 De Morgan)讓我們能化簡式子、省下硬體。
  • 邏輯閘是布林函數的硬體化身。把閘接起來就是電路;會寫布林式,就會設計電路。
  • 本章的震撼彈:NAND 是萬能閘。只用它就能造出 NOT/AND/OR/XOR/……以及整台電腦。我們用真值表親手驗證了。
  • 任何布林函數都能從真值表寫成 SOP 標準形式(AND/OR/NOT),所以都能用 NAND 實現——這是「電腦能算任何邏輯」的根據。
  • 用 HDL 像寫接線清單一樣描述晶片;Mux/Demux 讓電路會做選擇;多位元閘只是把一位元閘並排。而這一切,就活在你程式裡的 and、&、^ 與短路求值之中。

從兩個字——「真」與「假」——我們一路推到了能算任何邏輯的電路,而且發現整座塔只需要一種磚。 下一章,我們要拿這些磚,砌出電腦的第一個「會算數」的器官:算術邏輯單元(ALU)。 加法器、比較器、會聽指令切換功能的運算核心,全都在等著我們。塔基已穩,往上爬吧。
