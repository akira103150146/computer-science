程式,其實是一連串「狀態轉換」

上一章我們下了一個定義:計算,就是「依照規則,把資訊從一種狀態一步步轉換成另一種狀態」。 現在把這句話放大來看一支真正在跑的程式,你會發現它驚人地貼切。

想像電腦的記憶體是一塊巨大的白板,上面寫著此刻所有變數的值——這就是程式的狀態(state)。 你的程式每執行一行,就可能擦掉白板上的某個數字、再寫上新的。把「餘額 100」改成「餘額 80」, 把「目前最大值 3」改成「目前最大值 9」。一支程式,本質上就是一張「如何把白板從初始狀態, 一步步改寫成最終狀態」的指令清單。

🔑
核心觀念

命令式程式設計(imperative programming)的世界觀很單純:程式 = 狀態 + 一連串改變狀態的指令。 你不是在描述「答案是什麼」,而是在下命令:「先做這個,再做那個,然後根據情況決定下一步。」 這一章學的每樣東西——變數、運算、迴圈——都是在為你提供「改變狀態」與「控制改變順序」的工具。

兩種跑程式的方式:對話 vs 寫信

在開始之前,先認識兩種和 Python 互動的方式,你整章都會用到。

第一種是互動式直譯器(interactive interpreter),又叫 REPL (Read–Eval–Print Loop:讀取、求值、印出、再循環)。在終端機輸入 python 直接按 Enter, 你會看到提示符號 >>>。這時它就像一位隨叫隨到的助手,你打一行,它立刻回你一行: 

>>> 3 + 4
7
>>> "Hello" * 3
'HelloHelloHello'
>>> 2 ** 10
1024

這像是「對話」:適合快速做實驗、驗證一個想法、查一個函式怎麼用。本章很多小片段, 我建議你打開 REPL 親手敲一遍——你問,它答,即時又過癮。

第二種是把指令寫進一個原始碼檔(source file),副檔名 .py, 再整批交給 Python 執行。這像是「寫信」:你把完整的步驟一次寫好,存成檔案,然後執行它。 

# square.py —— 存成檔案,再用 python square.py 執行
x = 9
print(x * x)   # 81
python square.py
💡
什麼時候用哪個?

探索、試水溫、查語法 → 用 REPL,即時回饋最爽快。
要保存、要重複執行、程式有好幾行 → 寫成 .py 檔。
兩者背後是同一個 Python,差別只在「一句一句問」還是「整封信寄出」。本章的長範例請存成檔案執行。

變數:綁在物件上的便利貼

程式要改寫狀態,得先有地方「記住」東西。這個工具叫變數(variable)。 我們用指派(assignment)這個動作,把一個值記到變數裡: 

price = 100        # 把 100 這個值,綁到名字 price 上
print(price)       # 100
price = 80         # 改變它的值(打折了!)
print(price)       # 80

很多教科書會告訴你「變數是一個盒子,你把值放進盒子裡」。這個比喻很常見,但會誤導你, 而且日後談到串列、函式傳參時會害你出錯。請換成一個更精準的比喻:

在 Python 裡,值(物件)才是真正存在的盒子。變數不是盒子,而是貼在盒子上的一張便利貼(名字)。

當你寫 price = 100,Python 做的是:在記憶體裡建立一個內容為 100 的整數物件, 然後撕一張寫著 price 的便利貼,貼到那個物件上。所以 = 唸作「綁定(bind)」最貼切—— 它把名字綁到物件上,而不是「把東西塞進名為 price 的盒子」。

這個比喻立刻能解釋一件令初學者困惑的事:同一個物件可以被貼上好幾張便利貼。 

a = 100
b = a        # 不是「複製一份」,而是讓 b 這張便利貼,貼到 a 指的同一個物件上
print(a, b)  # 100 100

關於命名,Python 有幾條規矩與慣例:名字由英文字母、數字、底線組成,但不能以數字開頭; 習慣上用小寫加底線(例如 total_price、max_score);而且名字要會說話—— x 不如 price,n 不如 student_count。好的命名,是寫給「三個月後的你」看的。

資料的型別:int、float、str、bool

記憶體裡的物件不是都長一樣。一個整數和一段文字,能做的事天差地別——你可以對整數做減法, 卻不能對名字做減法。每個物件都屬於某個型別(type / data type), 型別決定了「這個值能做什麼、不能做什麼」。Python 最基本的四種型別是:

型別中文用途例子
int整數沒有小數點的數0、42、-7
float浮點數帶小數點的數3.14、-0.5、2.0
str字串一串文字(用引號包起來)"hello"、'你好'
bool布林值只有真或假兩種True、False

想知道某個值是什麼型別,用內建函式 type() 問它: 

print(type(42))        # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>

# 小心:2 和 2.0 是不同型別!
print(type(2))         # <class 'int'>
print(type(2.0))       # <class 'float'>,只要有小數點就是 float

Python 的整數,大到天荒地老

這裡藏著一個 Python 非常迷人、也非常重要的特性。試試看算 2 的 100 次方: 

x = 2 ** 100
print(x)          # 1267650600228229401496703205376
print(type(x))    # <class 'int'>,還是整數,沒有任何問題

三十位數,Python 連眉頭都不皺一下。Python 的整數是任意精度(arbitrary precision)的: 只要記憶體裝得下,它能精確表示要多大有多大的整數,永遠不會「爆掉」。

但這其實是 Python 替你做了很多工作。在更貼近硬體的語言裡,真相沒這麼美好。 C 語言的 int 通常只有固定的 32 個位元,能表示的範圍就那麼大 (約正負二十一億)。一旦超過,會發生一種叫溢位(overflow)的現象—— 數字像時鐘從 12 點繞回 1 點那樣,從最大值「繞」回最小值: 

#include <stdio.h>
#include <limits.h>

int main(void) {
    int x = INT_MAX;            /* 32 位元 int 的最大值:2147483647 */
    printf("%d\n", x);          /* 2147483647 */
    printf("%d\n", x + 1);      /* 溢位!竟然印出 -2147483648 */
    return 0;
}

x + 1 不增反減,變成一個巨大的負數!這不是 bug,而是固定位元數的必然後果—— 最大值再加一,就「繞回」最小值。這類溢位曾造成真實世界的災難 (例如 1996 年亞利安五號火箭升空後解體,根源之一就是數值轉換溢位)。

🔍
為什麼要看 C 這一段?

因為它揭示了 Python 幫你藏起來的「底層真相」:電腦的記憶體是有限的位元,數字並非天生無限。 Python 用軟體模擬出無限精度,代價是慢一些;C 直接用硬體的固定位元,代價是你得自己提防溢位。 這種「方便 vs 透明」的取捨,會貫穿整本書。位元如何表示數字、溢位的完整原理,我們在第五部 「計算機系統」會從頭講透。

運算子:讓資料動起來

有了資料,我們得能對它做運算。運算子(operator)就是那些 +、-、 * 之類的符號。先看算術運算子:

運算子名稱範例結果
+加3 + 47
-減10 - 64
*乘3 * 412
/真除法7 / 23.5
//整數除法(地板除)7 // 23
%取餘數(模)7 % 21
**次方2 ** 101024

分清 / 與 //,以及 % 的妙用

這是初學者第一個容易混淆的地方,值得停下來看清楚: 

print(7 / 2)    # 3.5  —— / 是「真除法」,結果一律是 float,連 6 / 2 都會給 3.0
print(7 // 2)   # 3    —— // 是「整數除法/地板除」,丟掉小數部分,只留整數
print(7 % 2)    # 1    —— % 是「取餘數」:7 除以 2,商 3 餘 1,所以是 1
print(8 % 2)    # 0    —— 整除時餘數為 0

%(模數,modulo)看起來不起眼,卻是程式裡的瑞士刀。判斷奇偶(n % 2 == 0 就是偶數)、 判斷整除(n % 3 == 0 是 3 的倍數)、把數字「捲」在固定範圍內(時鐘的 hour % 12)、 取一個數的個位數(n % 10)……你會一輩子用到它。

比較運算子:答案是 True 或 False

比較運算子拿兩個值來比,結果是一個布林值。最容易出事的,是 == 和 = 的混淆: 

x = 5           # 一個等號 = 是「指派」:把 5 綁到 x
print(x == 5)   # 兩個等號 == 是「比較」:x 等於 5 嗎?-> True
print(x == 6)   # False
運算子意義範例結果
==等於3 == 3True
!=不等於3 != 4True
>大於5 > 3True
<小於5 < 3False
>=大於等於3 >= 3True
<=小於等於2 <= 1False
⚠️
最經典的初學者陷阱:= 不是 ==

= 是指派(把值綁到名字),== 是比較(問兩者相不相等)。 在 C 裡寫 if (x = 5) 是合法的(它會先把 5 指派給 x,再判斷,永遠成立), 是無數 bug 的源頭;Python 比較貼心,在 if 條件裡誤用 = 會直接報語法錯誤, 逼你改正。但養成「比較用兩個等號」的肌肉記憶,仍是第一要務。

邏輯運算子:and、or、not

當你需要把多個條件組合起來,就用邏輯運算子。它們吃布林值,也吐布林值: 

age = 20
print(age >= 18 and age < 65)   # True:「成年」且「未滿 65」,兩個都成立才為 True
print(age < 13 or age > 60)     # False:「兒童」或「長者」,有一個成立就 True,這裡都不成立
print(not age == 20)            # False:not 把 True 翻成 False

記法很直覺:and 要「兩個都真」才真;or 只要「至少一個真」就真; not 則把真假對調。注意 Python 用的是英文單字 and / or / not, 不像 C 用符號 && / || / !——這也是 Python 讀起來像英文句子的原因之一。

字串也能運算:串接與重複

有趣的是,+ 和 * 用在字串上有特別的意義: 

greeting = "Hello" + ", " + "world"   # + 對字串是「串接(concatenation)」
print(greeting)                        # Hello, world

line = "=" * 30                        # * 對字串是「重複」
print(line)                            # 印出 30 個等號,很適合畫分隔線
print("哈" * 3)                        # 哈哈哈

同一個 +,對數字是相加,對字串是接起來——這叫運算子多載(operator overloading): 同一個符號,依運算對象的型別而有不同行為。但這也埋了一個坑:"3" + 4 會怎樣? 我們馬上在型別轉換那節揭曉。

運算式 vs 陳述式

這兩個詞你會一直聽到,現在花一分鐘把它們分清楚,之後讀任何程式都會更清明。

運算式(expression)是「一段會算出一個值的程式碼」。它的重點是「求值(evaluate)」—— 算完會留下一個結果。下面每一行單獨拿出來,都會得到一個值: 

3 + 4           # 算出 7
price * 0.8     # 算出打八折後的價格
age >= 18        # 算出 True 或 False
"Hi" + name     # 算出一個新字串

陳述式(statement)則是「一個完整的指令動作」,它做某件事,但本身不一定產生值。 指派就是最典型的陳述式——它執行「把值綁到名字」這個動作: 

price = 100         # 這是陳述式:它「做」了綁定這件事
print(price)        # 這也是陳述式:它「做」了印出這件事
if price > 50:      # if 整段是陳述式,它控制程式流向
    print("好貴")

一句話記住差別:運算式被「算出」成一個值,陳述式被「執行」成一個動作。 兩者常常套在一起——指派陳述式 price = 3 + 4 的右邊 3 + 4 就是一個運算式, Python 先把運算式求值成 7,再執行「把 7 綁到 price」這個陳述式動作。

型別轉換:在型別之間搬家

常常,你手上的資料型別不對。使用者輸入的 "18" 是字串,但你想拿它做算術; 你算出一個數字,但想把它接到一段文字裡。這時就要型別轉換(type casting)。 Python 用「同名函式」做轉換:int()、float()、str(): 

n = int("42")          # 字串 "42" -> 整數 42
pi = float("3.14")     # 字串 "3.14" -> 浮點數 3.14
s = str(100)           # 整數 100 -> 字串 "100"

print(n + 8)                       # 50,現在能算術了
print("價格:" + str(100) + " 元")  # 價格:100 元,先轉成字串才能串接
print(int(3.9))                    # 3,float 轉 int 會「無條件捨去」小數,不是四捨五入

隱式轉換 vs 顯式轉換

上面那種你親手呼叫 int() / str() 的,叫顯式轉換(explicit conversion)—— 你明白地下令。但有些時候 Python 會「自己」幫你轉,這叫隱式轉換(implicit conversion): 

print(3 + 4.0)        # 7.0:int 3 被「隱式」升級成 float,結果是 float
print(True + 1)       # 2:布林值 True 在算術裡被當成 1(False 當 0)

# 但 Python 對「字串和數字」絕不自作主張:
# print("3" + 4)      # TypeError!它不會猜你想要 "34" 還是 7
print("3" + str(4))   # "34":你得自己決定並顯式轉換
print(int("3") + 4)   # 7:或者轉成數字相加

Python 的哲學是「面對模稜兩可,拒絕猜測」。它願意做安全、無損的隱式轉換(int 變 float 不會失真), 但「字串 + 數字」意圖不明,它寧可報錯也不亂猜。這逼你把意圖講清楚,長遠來看是好事。

浮點數,沒你想的那麼精確

現在來看程式設計裡最反直覺、卻最該早點知道的一件事。在 REPL 裡親手敲下這行: 

print(0.1 + 0.2)          # 0.30000000000000004 (!!)
print(0.1 + 0.2 == 0.3)   # False (!!)

沒看錯,在電腦眼中 0.1 + 0.2 不等於 0.3。 這不是 Python 的 bug,而是幾乎所有現代語言(C、Java、JavaScript……)共有的現象。

⚠️
浮點數誤差:為什麼 0.1 + 0.2 ≠ 0.3

電腦用二進位儲存小數,而 0.1 在二進位裡是個無限循環小數 (就像 1/3 在十進位是 0.333… 寫不完)。電腦只能存有限位元,於是存進去的其實是 0.1 的近似值,微小的誤差累積後就露了餡。這套規則叫 IEEE 754 浮點數標準。
實務上的鐵則:絕對不要用 == 直接比較兩個浮點數。要判斷「幾乎相等」, 改判斷差的絕對值夠不夠小,例如 abs(a - b) < 1e-9。這套機制的完整原理, 我們會在第五部「計算機系統」深入剖析。現在,先把這個「常識」刻進腦袋就好。

與世界對話:輸入與輸出

程式要有用,得能和外界溝通。最基本的兩個工具是 print()(輸出)和 input()(輸入)。

print():不只會印,還會排版

print() 可以一次印好幾個東西,並用兩個常用參數調整格式:sep(分隔符)和 end(結尾符): 

print("a", "b", "c")              # a b c    —— 多個值預設用「空格」分隔
print("a", "b", "c", sep="-")     # a-b-c    —— 改用減號分隔
print(2024, 6, 29, sep="/")       # 2024/6/29 —— 拿來組日期很方便

print("沒有換行", end="")          # end 預設是換行符 "\n",改成空字串就不換行
print(",接在同一行")              # 沒有換行,接在同一行

input():拿到的永遠是字串

input() 會暫停程式,等使用者打字按 Enter,然後把輸入交給你。這裡有一個 每個初學者都會中招的關鍵事實:

🔑
input() 永遠回傳字串

不管使用者打的是 18 還是 hello,input() 拿到的一律是 字串。想拿它做算術?你必須先轉型(int() 或 float())。 忘了轉型,要嘛報 TypeError,要嘛得到詭異結果。

name = input("你叫什麼名字?")      # 拿到字串,直接用沒問題
print("嗨," + name + "!")

age_str = input("你幾歲?")         # 即使打 18,拿到的是字串 "18"
age = int(age_str)                 # 必須轉成整數才能算術
print("明年你就", age + 1, "歲了")

# 常見錯誤示範:
# age = input("你幾歲?")
# print(age + 1)   # TypeError:字串不能加整數!忘了轉型的後果

熟練後,很多人會把轉型和輸入寫在同一行,簡潔俐落: 

age = int(input("你幾歲?"))   # 一氣呵成:讀進來、轉成 int、綁到 age

流程控制:選擇、重複、跳出

到目前為止,我們的程式都像直直一條路,從上跑到下。但真正有用的程式得會做選擇 (這種情況做這個、那種情況做那個)和重複(同一件事做很多次)。 這就是流程控制(control flow)。

Python 用「縮排」定義區塊

在學第一個 if 之前,先講 Python 最有辨識度的特色。大多數語言(像 C、Java)用大括號 { } 把「屬於同一個區塊的程式」框起來;Python 不用大括號,而是用縮排 (indentation)——也就是行首的空白——來表達「這幾行同屬一個區塊」。

🔑
縮排在 Python 裡是「語法」,不是「美觀」

在別的語言,縮排只是為了好看,改不改都能跑;在 Python,縮排決定了程式的結構與意義。 縮排錯了,程式的邏輯就跟著錯,甚至直接報錯。慣例是每層縮排用 4 個空格, 且不要混用 tab 和空格(這會引發難以察覺的錯誤)。好消息是,這逼著每個人都把程式排整齊。

if / elif / else:讓程式做選擇

if 後面接一個條件(會算出 True/False 的運算式),條件成立就執行它底下縮排的區塊: 

score = 85

if score >= 90:
    print("優等")
elif score >= 60:       # elif = else if:前面不成立,再試這個條件
    print("及格")
else:                   # 以上都不成立,走這條
    print("不及格")
# 輸出:及格

注意條件是由上往下逐一檢查,一旦有一個成立,就執行那塊、然後跳過其餘所有 elif / else。所以順序很重要——score = 95 雖然也 >= 60, 但因為先撞上 >= 90 就停了,正確印出「優等」。

while 迴圈:條件成立就一直做

while 迴圈會反覆執行底下的區塊,直到條件不再成立為止。它適合 「不知道要做幾次,但知道什麼時候該停」的情況: 

n = 5
total = 0
i = 1
while i <= n:           # 只要 i 還 <= n,就繼續
    total = total + i   # 把 i 累加進 total
    i = i + 1           # 關鍵!每圈都要讓 i 變大,否則條件永遠成立
print(total)            # 15 (= 1+2+3+4+5)
⚠️
小心無限迴圈(infinite loop)

如果上面忘了寫 i = i + 1,i 永遠是 1,i <= n 永遠成立, 程式就會卡在迴圈裡跑不完,畫面瘋狂輸出或乾脆當住。寫 while 時請務必確認: 條件總有一天會變成 False。萬一真的卡死,在終端機按 Ctrl + C 強制中止。

for 迴圈與 range():做固定次數

當你「知道要做幾次」或「要走遍一串東西」,for 迴圈更順手。搭配 range() 產生一串數字最常見: 

for i in range(5):         # range(5) 產生 0, 1, 2, 3, 4(從 0 開始,「不含」5)
    print(i)

for i in range(1, 6):      # 兩個參數:從 1 到「6 之前」,即 1, 2, 3, 4, 5
    print(i)

for i in range(0, 11, 2):  # 三個參數:從 0 到 11 之前,「步長」2 -> 0,2,4,6,8,10
    print(i)
ℹ️
range 的「左閉右開」

range(1, 6) 包含 1 但不包含 6——這叫「左閉右開」區間。一開始覺得彆扭, 但它有個好處:range(0, n) 剛好產生 n 個數字(0 到 n-1),和「長度 n」完美對應。 這也是著名的「差一錯誤(off-by-one error)」最愛出沒的地方,務必數清楚邊界。

break 與 continue:跳出與跳過

在迴圈裡,有兩個特殊指令能改變流向。break 直接跳出整個迴圈; continue 跳過本圈剩下的部分,直接進入下一圈: 

for i in range(1, 100):
    if i == 5:
        break          # 一碰到 5 就跳出,後面的 6, 7, ... 通通不跑
    print(i)           # 只印出 1 2 3 4

for i in range(1, 11):
    if i % 2 == 1:     # 如果是奇數
        continue       # 跳過這一圈剩下的(不執行下面的 print),直接進下一圈
    print(i)           # 只印出偶數 2 4 6 8 10

Python vs C:同一段邏輯,兩種腔調

把這一章學到的東西兜在一起,來寫一個小任務:印出 1 到 10 之間的偶數。 先用 Python: 

for i in range(1, 11):     # 走過 1 到 10
    if i % 2 == 0:         # 餘數為 0 就是偶數
        print(i)           # 印出來

一模一樣的演算法,用 C 寫一次。注意:想法完全相同,差別全在「腔調」: 

#include <stdio.h>

int main(void) {
    for (int i = 1; i <= 10; i++) {   /* 用大括號和分號,迴圈條件寫法也不同 */
        if (i % 2 == 0) {
            printf("%d\n", i);        /* printf 要用 "%d" 指定印整數 */
        }
    }
    return 0;
}

把兩段並排,差異一目了然。這些差異不是誰好誰壞,而是兩種語言不同的設計取捨:

面向PythonC
變數要先宣告型別嗎?不用,i = 1 即可要,int i = 1
用什麼框出區塊?縮排(空格)大括號 { }
每行結尾要分號嗎?不用要,每個陳述式以 ; 結尾
邏輯運算and / or / not&& / || / !
整數會溢位嗎?不會(任意精度)會(固定位元)
輸出print(i)printf("%d\n", i); 要指定格式

看出規律了嗎?C 要求你把一切都講清楚——型別、邊界、格式;Python 則替你打理掉許多細節, 讓你專注在「想法」上。這正呼應上一章的主題:同一個演算法,可以有無數種程式實作。 當你能無視腔調、看穿底下的同一個靈魂,你就真的開始像個程式設計師那樣思考了。

✏️ 練習 2.1 思考

不要執行,先用腦袋預測下面每一行會印出什麼,再實際跑跑看驗證:

print(9 / 2)
print(9 // 2)
print(9 % 2)
print(2 ** 3 ** 2)        # 提示:** 是「由右往左」結合的
print(5 + 3 > 2 * 4)
print("ab" * 3 + "!")
看解答 
4.5          # / 是真除法,給 float
4            # // 地板除,丟掉小數
1            # 9 除以 2 餘 1
512          # 2 ** (3 ** 2) = 2 ** 9 = 512,不是 (2**3)**2=64!
False        # 左邊 8,右邊 8,8 > 8 為 False
ababab!      # "ab" 重複三次再串上 "!"

第四題是最容易答錯的——** 和大多數運算子相反,是「右結合」的。 不確定運算順序時,加上括號讓意圖清楚,永遠是好習慣。

初學者最常踩的雷

在進入練習之前,把本章散落的陷阱集合成一張「避雷地圖」。這幾個錯誤,幾乎每個人都會犯至少一次—— 先認得它們,你就能在出錯時迅速反應「啊,是這個」。

⚠️
六大常見陷阱

1. = 與 == 不分:指派用一個等號,比較用兩個。
2. 整數除法搞錯:要小數結果用 /,要整數結果用 //; 5 / 2 是 2.5 不是 2。
3. 浮點數直接比較:別寫 0.1 + 0.2 == 0.3,改判斷 abs(a - b) < 1e-9。
4. 忘記轉型 input():input() 回傳字串,要算術記得包 int() / float()。
5. 縮排錯誤:同一區塊要對齊,別混用 tab 和空格;縮排錯了邏輯就錯了。
6. 差一錯誤(off-by-one):range(1, n) 不含 n;迴圈邊界、陣列索引最容易差一格, 數不清楚時拿小例子(例如 n = 3)手動跑一遍。

這些陷阱有個共通點:它們都不會「轟」一聲地大爆炸,而是安靜地給你一個錯誤的結果。 程式照跑,只是答案不對——這正是最難抓的 bug。對抗它的最好武器,是上一章提過的習慣: 拿一個小到能用手算的例子,讓程式跑一遍,核對輸出是否如你預期。

動手練習

看懂和會寫是兩回事。下面五題難度遞增,請先自己寫,真的卡住再展開解答。 每一題都建議存成 .py 檔實際執行,並故意輸入幾個刁鑽的值試試看。

✏️ 練習 2.2 實作

寫一支程式,請使用者輸入一個攝氏溫度,換算成華氏並印出。公式是 $F = C \times \frac{9}{5} + 32$。例如輸入 100,應印出 212.0。

看解答

重點:input() 回傳字串,要先用 float() 轉成數字(溫度可能有小數)。

c = float(input("請輸入攝氏溫度:"))
f = c * 9 / 5 + 32
print(c, "°C =", f, "°F")

注意這裡用 /(真除法)而不是 //,否則 9 // 5 會變成 1, 整條公式就錯了——這正是「整數除法搞錯」的陷阱。

✏️ 練習 2.3 實作

輸入一個西元年份,判斷它是不是閏年(leap year),印出「是」或「不是」。 規則是:能被 4 整除的是閏年,但能被 100 整除的不是,除非它又能被 400 整除。 (所以 2000 是閏年,1900 不是,2024 是。)

看解答

這題在練「用 and / or 組合條件」。把規則翻成邏輯式: 「被 4 整除」且(「不被 100 整除」或「被 400 整除」)。 括號很重要,它決定了運算順序。

year = int(input("請輸入年份:"))

if year % 4 == 0 and (year % 100 != 0 or year % 400 == 0):
    print(year, "是閏年")
else:
    print(year, "不是閏年")

拿 1900(不是)、2000(是)、2024(是)、2023(不是)四個值各測一遍,確認都正確。

✏️ 練習 2.4 實作

計算複利(compound interest)。請使用者輸入本金、年利率(例如 0.05 代表 5%)、年數, 算出到期的本利和。公式是 $A = P \times (1 + r)^n$。輸出時請四捨五入到小數點後 2 位 (提示:用內建函式 round(x, 2))。

看解答

這題用到次方運算子 ** 和 round():

principal = float(input("本金:"))
rate = float(input("年利率(例如 0.05):"))
years = int(input("年數:"))

amount = principal * (1 + rate) ** years
print("到期本利和:", round(amount, 2))

進階:你也能不用次方,改用迴圈一年一年滾,順便印出每年餘額—— 這其實更接近銀行「逐年計息」的真實過程:

amount = principal
for year in range(1, years + 1):
    amount = amount * (1 + rate)
    print("第", year, "年底:", round(amount, 2))

兩種寫法結果一致,但後者展示了「迴圈也能取代數學公式」——很多時候,一步步累積比直接套公式更直觀。

✏️ 練習 2.5 挑戰

做一個猜數字遊戲。程式先隨機選一個 1 到 100 的整數,然後反覆讓玩家猜: 猜太小提示「太小了」,太大提示「太大了」,猜中就恭喜並結束。 (提示:這是「不知道要幾次、但知道何時停」的典型,用 while 迴圈。 產生亂數用 import random 後的 random.randint(1, 100)。)

看解答

關鍵是迴圈條件「還沒猜中就繼續」,以及每圈把玩家輸入的字串轉成 int:

import random

answer = random.randint(1, 100)   # 隨機選一個 1~100 的整數當答案
guess = -1                        # 先給一個不可能猜中的初始值

while guess != answer:
    guess = int(input("猜一個 1~100 的數字:"))   # 別忘了轉型!
    if guess < answer:
        print("太小了,再大一點")
    elif guess > answer:
        print("太大了,再小一點")
    else:
        print("答對了!")

再挑戰:加一個計數器,在猜中時告訴玩家「你猜了幾次」。 這只需在迴圈外設 count = 0,每圈 count = count + 1 即可。

✏️ 練習 2.6 挑戰

用迴圈把 1 累加到 n(即 $1 + 2 + \cdots + n$),再用高斯的公式 $\dfrac{n(n+1)}{2}$ 算一次,印出兩個結果並比對它們是否相等。 請使用者輸入 n,試幾個不同的值(例如 10、100、1000)看看是否總是吻合。

看解答

這題把「迴圈累加」和「數學公式」對照——這正是本書反覆出現的主題: 一個結果,常有「一步步算」和「一招算完」兩種路。注意公式那行要用 // 而非 /, 因為 $n(n+1)$ 必為偶數,整數除法能得到乾淨的整數結果,也避開浮點誤差。

n = int(input("請輸入 n:"))

total = 0
for i in range(1, n + 1):     # 1, 2, ..., n(注意右邊是 n+1 才會含到 n)
    total = total + i

formula = n * (n + 1) // 2    # 高斯公式,用 // 保持整數且精確

print("迴圈累加結果:", total)
print("公式 n(n+1)/2:", formula)
print("兩者相等嗎?", total == formula)

傳說高斯小學時被罰算 1 加到 100,他瞬間看出可以把數字頭尾配對 (1+100, 2+99, …, 共 50 對,每對都是 101),於是答案是 $50 \times 101 = 5050$, 這就是公式 $\frac{n(n+1)}{2}$ 的由來。你的迴圈花了 n 步,他只花了一步—— 這種「找到更聰明的算法」的追求,正是演算法這門學問的精神,我們在第六部會深入。

把這一章串起來

我們把「命令式知識」一條條教給了電腦。回顧這趟旅程的路標:

  • 程式是狀態 + 一連串改變狀態的指令。你可以用 REPL 即時對話,或寫成 .py 檔整批執行。
  • 變數不是盒子,是貼在物件上的便利貼(名字);= 是「綁定」,把名字綁到物件。
  • 四種基本型別 int / float / str / bool,用 type() 觀察。Python 整數任意精度永不溢位,C 的固定位元 int 則會溢位。
  • 運算子:分清 /(真除法)與 //(整數除法)、%(取餘數),以及 =(指派)與 ==(比較)。
  • 運算式被「算出」一個值,陳述式被「執行」一個動作。
  • 型別轉換分顯式(你呼叫 int())與隱式(Python 自動把 int 升 float);浮點數有 IEEE 754 誤差,別用 == 比較。
  • 輸入輸出:print() 有 sep / end 可調;input() 永遠回傳字串,要算術記得轉型。
  • 流程控制:if/elif/else 做選擇,while / for + range() 做重複,break / continue 跳出與跳過。Python 用縮排定義區塊。

你現在握有命令式程式設計的全套基本零件了。下一章,我們會把這些零件組裝成更大的東西—— 函式(function)與更豐富的資料結構,讓你能把「找最大值」這種過程封裝起來、 重複使用,真正爬上抽象的下一層階梯。
