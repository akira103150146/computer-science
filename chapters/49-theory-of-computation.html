計算理論

回到那個欠著的問題

還記得第 1 章結尾那個讓人睡不著的句子嗎?我們說:1936 年,圖靈(Alan Turing)用一台極簡的抽象機器 精確定義了「什麼叫可計算」,然後證明了——有些問題,沒有任何演算法能解決, 最著名的就是停機問題。當時我們只丟下這個震撼結論,就匆匆把你帶回疆界之內,去學駕馭計算的力量。

現在,你已經走過了演算法、資料結構、複雜度(第 26 章)、NP 完備性(第 33 章), 也在第 47 章親手用有限狀態機(FSM)讓硬體「依狀態行動」,在第 42 章看著正規表示式與 DFA 如何把原始碼切成 token。你手上的拼圖,終於夠了。這一章,我們要把它們拼成一幅完整的地圖。

核心的策略出奇地簡單。要問「什麼能被計算」,我們先造一台最弱的機器,看它能算什麼、不能算什麼; 然後給它加一點能力,看疆界往外推到哪;再加一點,再看……一路加到一台「要多強有多強」的終極機器為止。 每一台機器,恰好對應一類語言。把這些類別由小到大套疊起來,就是語言學家 喬姆斯基(Noam Chomsky)在 1950 年代提出的喬姆斯基層級(Chomsky hierarchy)。

🔑
這一章在回答三個問題

計算理論(theory of computation)其實在追問三件事,層層遞進:
① 能不能算?(可計算性 computability)——有些問題根本沒有演算法。
② 要多少資源?(複雜度 complexity)——能算的之中,有些快、有些慢到沒意義(第 26、33 章)。
③ 要用什麼機器?(自動機 automata)——不同強度的機器,對應不同類別的問題。
本章主攻 ① 與 ③,並在結尾把線接回你已經學過的 ②。

形式語言:把所有問題講成同一種話

要比較不同機器的能力,我們得先讓所有問題「說同一種話」。這種共通語言,叫做形式語言(formal language)。 它只需要三個樸素到不行的概念。

  • 字母表(alphabet),記作 $\Sigma$:一個有限的符號集合。例如 $\Sigma = \{0, 1\}$(二進位), 或 $\Sigma = \{a, b\}$,或英文 26 個字母,或一個程式語言所有合法字元的集合。
  • 字串(string):把字母表裡的符號排成一列(有限長)。例如在 $\Sigma = \{0,1\}$ 上, 0110、1、000 都是字串。特別地,長度為 0 的空字串記作 $\varepsilon$。
  • 語言(language):就是一堆字串的集合——任意一群你想圈在一起的字串。例如「所有偶數長度的字串」、 「所有合法的 Python 程式」、「所有是質數的二進位表示」,都各自是一個語言。

我們用 $\Sigma^*$ 代表「在 $\Sigma$ 上能拼出的所有字串」(包含 $\varepsilon$)。那麼,一個語言 $L$ 就是 $\Sigma^*$ 的一個子集:$L \subseteq \Sigma^*$。這呼應了你在第 13 章學的集合論—— 語言不過是「字串的集合」而已。

🔑
關鍵翻譯:任何問題 = 「這個字串屬於某語言嗎?」

這是整章最重要的一次轉念。看似五花八門的計算問題,幾乎都能改寫成一個成員判定問題(membership): 給定一個字串 $w$,問「$w \in L$?」。
・「這個數是質數嗎?」→ 把數寫成字串,問它是否屬於語言 $L_{\text{prime}} = \{\,$質數的二進位表示$\,\}$。
・「這段原始碼語法正確嗎?」→ 問它是否屬於語言 $L_{\text{C}} = \{\,$合法的 C 程式$\,\}$。
・「這張圖連通嗎?」→ 把圖編碼成字串,問它是否屬於語言 $L_{\text{conn}}$。
於是「解一個問題」就等於「替一個語言做成員判定」,而「一台機器」就是「一個判定某語言成員資格的裝置」。 所有機器都收同樣的輸入(字串)、回答同樣的問題(屬不屬於),我們才能公平地比較它們誰強誰弱。

這個視角還呼應了第 33 章那句「複雜度理論只研究決策問題(答案非是即否)」。 成員判定正是最純粹的是非題:屬於,或不屬於。接下來,我們就從最弱的那台機器開始,看它能判定哪些語言。

有限自動機:一台沒有記憶體的機器

你在第 47 章已經見過它的真身——有限狀態機。那裡它是硬體裡的紅綠燈控制器、序列偵測器; 這裡我們換上理論的眼鏡,把它當成一台判定語言成員資格的機器,正式名稱叫 有限自動機(finite automaton),確定性的版本縮寫為 DFA(deterministic finite automaton)。

它的工作方式是:從一個起始狀態出發,一次讀入一個字元,每讀一個就依規則跳到下一個狀態。 字串讀完後,如果停在「接受狀態」,就接受(回答「屬於」);否則拒絕。整個過程裡,它唯一的記憶 就是「我現在在哪個狀態」——而狀態只有有限個。這就是它最致命的特性:

🔑
DFA 的記憶,小到只有「有限個格子」

一台 DFA 只有固定、有限個狀態。它不能「記住」一個能任意變大的數字(例如「目前讀了幾個 a」), 因為那需要無限多個狀態。它能記的,只有「事先就規劃好的那幾種情況」。記住這個限制——它正是稍後 「DFA 數不出 $a^n b^n$」的根本原因。

正式定義:五元組

剝掉所有比喻,一台 DFA 就是五樣東西,我們把它打包成一個五元組(5-tuple):

$$M = (Q,\ \Sigma,\ \delta,\ q_0,\ F)$$

  • $Q$:有限的狀態集合。
  • $\Sigma$:輸入的字母表。
  • $\delta$:轉移函數(transition function),型別是 $\delta : Q \times \Sigma \rightarrow Q$。 「在狀態 $q$ 讀到字元 $a$,就跳到狀態 $\delta(q, a)$」。確定性的精髓就在這裡:每個(狀態,字元)組合,下一步唯一確定。
  • $q_0 \in Q$:起始狀態。
  • $F \subseteq Q$:接受狀態(accepting states)的集合,在狀態圖裡畫成雙圈。

DFA 接受的所有字串,合起來就是它辨認(recognize)的語言,記作 $L(M)$。 能被某台 DFA 辨認的語言,就叫正規語言(regular language)。

例子一:含有偶數個 a 的字串

設 $\Sigma = \{a, b\}$,語言 $L = \{\,w : w$ 裡的 $a$ 出現偶數次$\,\}$(像 bb、aa、baab 都算, a、aba 不算)。要記住「目前 a 的個數是奇是偶」,其實只需要兩個狀態—— 因為我們不在乎確切數字,只在乎奇偶這個「一位元」的資訊。讀到 $b$ 不改變奇偶,讀到 $a$ 就翻面。

例子二:以 01 結尾的字串

設 $\Sigma = \{0, 1\}$,語言 $L = \{\,w : w$ 以 01 結尾$\,\}$。我們需要記住「最近讀到的尾巴,離 01 還差多遠」: 用三個狀態——還沒看到有用前綴、剛看到 0、剛看到 01(接受)。這和第 47 章那台偵測 "101" 的序列偵測器 是同一個套路。

用 Python 模擬一台 DFA

DFA 的定義就是五元組,所以寫成程式幾乎是逐字翻譯:把 $\delta$ 存成一個字典,然後寫一個迴圈 「一次吃一個字元、查表換狀態」。整個模擬器不到二十行。 

class DFA:
    def __init__(self, states, alphabet, delta, start, accept):
        self.states = states        # Q:狀態集合
        self.alphabet = alphabet    # Σ:字母表
        self.delta = delta          # δ:(狀態, 字元) -> 下一個狀態
        self.start = start          # q0:起始狀態
        self.accept = accept        # F:接受狀態集合

    def run(self, s):
        state = self.start              # 從起始狀態出發
        for ch in s:                    # 一次讀入一個字元
            state = self.delta[(state, ch)]   # 查轉移函數,換到下一個狀態
        return state in self.accept     # 讀完後,是否停在接受狀態?

# 圖 49-2 那台「含偶數個 a」的 DFA
even_a = DFA(
    states={"even", "odd"},
    alphabet={"a", "b"},
    delta={
        ("even", "a"): "odd",  ("even", "b"): "even",
        ("odd",  "a"): "even", ("odd",  "b"): "odd",
    },
    start="even",
    accept={"even"},
)

for s in ["", "b", "a", "aa", "aba", "baab", "aaab"]:
    print(repr(s), even_a.run(s))
# '' True   'b' True   'a' False   'aa' True
# 'aba' False   'baab' True   'aaab' False

注意這支模擬器有多「笨」:它沒有任何計數器、沒有堆疊、沒有額外陣列, 全部家當就是一個叫 state 的變數。這正是 DFA 的物理真相——它的記憶體是常數大小的。 這個事實,等一下會變成一把鋒利的證明武器。

NFA:讓機器「同時走進所有岔路」

現在放寬一個限制。確定性 DFA 要求「每個(狀態,字元)只有一條出路」。如果我們允許一個字元有好幾條出路 (甚至允許不讀字元就跳的 $\varepsilon$ 轉移),就得到非確定性有限自動機(nondeterministic finite automaton,NFA)。

「非確定性」這個詞,你在第 33 章談 NP 時碰過。直覺一樣:想像機器面臨岔路時, 能同時分身走進所有路;只要其中任何一條路最後停在接受狀態,整台機器就接受。 這讓 NFA 寫起來常常簡單得多。例如「以 01 結尾」,可以這樣設計:在起點不斷「賭」這個 0 還不是結尾(自迴圈), 某一刻「猜」它就是結尾的開頭,於是衝向終點。

這時你一定會問:NFA 能「分身」,豈不是比 DFA 強?令人驚訝的是——並沒有。 每一台 NFA,都能轉換成一台辨認完全相同語言的 DFA。這個轉換叫子集建構法(subset construction)。

🔍
子集建構法的直覺:用「狀態的集合」當新狀態

既然 NFA 會分身到「好幾個狀態同時存在」,那我們就乾脆把「目前可能身處的那一群狀態」整包當成 DFA 的一個新狀態。 上面那台 NFA 從 $\{q_0\}$ 出發,讀到 0 後可能在 $\{q_0, q_1\}$,在這個新狀態下再讀 1 就可能到 $\{q_0, q_2\}$(含接受狀態 $q_2$,所以這個合成狀態是接受的)。 $n$ 個 NFA 狀態,最多生出 $2^n$ 個「子集」狀態——數量爆炸,但仍然有限。有限,就還是 DFA。 所以 NFA 與 DFA 辨認的語言恰好是同一類:正規語言。非確定性只是「方便」,沒有讓它「更強」。

看解答

關鍵洞見:我們不需要記住整個數字(那要無限個狀態),只需要記住「目前數值除以 3 的餘數」——只有 0、1、2 三種可能。三個狀態就夠了,分別代表餘數 0(接受)、餘數 1、餘數 2。

讀到一個位元 $b$ 後,餘數從 $r$ 變成 $(2r + b) \bmod 3$。逐一算出轉移:

現態(餘數)讀到 0 → 次態讀到 1 → 次態
r0(接受)(2·0+0)%3 = 0 → r0(2·0+1)%3 = 1 → r1
r1(2·1+0)%3 = 2 → r2(2·1+1)%3 = 0 → r0
r2(2·2+0)%3 = 1 → r1(2·2+1)%3 = 2 → r2

起始狀態是 r0(空字串代表 0)。狀態圖就是三個圓 r0、r1、r2,r0 畫雙圈(接受),依上表連箭頭。可以直接拿前面的 DFA 類別驗證:

mod3 = DFA(
    states={"r0", "r1", "r2"},
    alphabet={"0", "1"},
    delta={
        ("r0","0"):"r0", ("r0","1"):"r1",
        ("r1","0"):"r2", ("r1","1"):"r0",
        ("r2","0"):"r1", ("r2","1"):"r2",
    },
    start="r0", accept={"r0"},
)
for n in range(10):
    b = format(n, "b")           # n 的二進位字串
    print(n, b, mod3.run(b), "←應為", n % 3 == 0)
# 0,3,6,9 → True;其餘 → False

這題漂亮地示範了 DFA 的核心思想:不存原始資料,只存「足以決定未來」的最小摘要(這裡是餘數)。一個能變到無限大的數字,被壓進了三個狀態。

正規語言 = DFA = 正規表示式

我們剛把「正規語言」定義成「某台 DFA 能辨認的語言」。但你其實早就認得正規語言的另一張臉了—— 第 16 章用 grep 搜尋文字、第 42 章用來切 token 的那個正規表示式(regular expression)。

正規表示式是用三個基本操作,從單一字元「長」出複雜模式的記法:

  • 聯集 / 選擇:a|b 表示「a 或 b」。
  • 串接:ab 表示「a 後面接 b」。
  • Kleene 星號:a* 表示「a 重複零次或多次」。

例如 (0|1)*01 描述的,正是圖 49-3 那台 DFA 辨認的語言:「任意一串 0/1,以 01 結尾」。 這不是巧合。1950 年代,Stephen Kleene 證明了一個美得令人吃驚的定理:

🔑
Kleene 定理:三者其實是同一件事

下面三句話描述的是同一類語言(正規語言),彼此可以互相轉換:
① 存在一台 DFA 辨認它;
② 存在一台 NFA 辨認它;
③ 存在一個正規表示式描述它。
所以你在第 42 章讀到的那句「正規表示式能描述的語言,恰好等於 DFA 能辨認的語言」, 就是 Kleene 定理。lex/flex 這類工具能「讀你的正規表示式、自動吐出 DFA 程式碼」, 底層靠的正是這個定理給的轉換演算法。理論與工程,在此完美閉環。

於是「正規」這個類別,有了三種完全等價的刻畫:一種機器(DFA)、一種更方便的機器(NFA)、一種記法(regex)。 當一個概念能從這麼多不同角度被描述、卻都指向同一個東西時,通常代表你摸到了某種本質。 那麼問題來了——這個本質,它的盡頭在哪?有沒有什麼語言,是正規表示式再怎麼寫、DFA 再怎麼加狀態,都辦不到的?

正規語言的天花板:為什麼 DFA 數不了 aⁿbⁿ

考慮一個看起來人畜無害的語言:

$$L = \{\, a^n b^n : n \ge 0 \,\} = \{\, \varepsilon,\ ab,\ aabb,\ aaabbb,\ \dots \,\}$$

也就是「若干個 a,後面接同樣數量的 b」。人類一眼就會判斷:數 a、數 b、比一比。 但 DFA 辦不到——而且我們能嚴格證明它辦不到。直覺先行:DFA 的記憶只有有限個狀態, 可是要檢查 a 和 b 「一樣多」,它得記住「到底讀了幾個 a」,而 a 的數量可以是任意大的整數, 無限多種可能,塞不進有限個狀態。這個直覺,被一個漂亮的工具精確化,叫泵引理(pumping lemma)。

📝
泵引理(正規語言版)

如果 $L$ 是正規語言,那麼存在一個正整數 $p$(叫泵長度),使得 任何 長度 $\ge p$ 的字串 $s \in L$, 都能被切成三段 $s = xyz$,滿足:
① $|y| \ge 1$(中間段非空);
② $|xy| \le p$(前兩段加起來不超過 $p$);
③ 對所有 $i \ge 0$,$xy^i z$ 仍然屬於 $L$(把中間段 $y$ 重複任意次,還是在語言裡)。

為什麼會這樣?因為一台只有 $p$ 個狀態的 DFA,在讀一個長度 $\ge p$ 的字串時, 前 $p$ 步至少會經過 $p+1$ 個狀態——由鴿籠原理(第 13 章!),其中必有某個狀態被重複造訪。 這兩次造訪之間讀掉的那一段,就是 $y$:它形成一個迴圈。既然是迴圈,你繞它一次、兩次、十次、零次, 機器最後落在的狀態都一樣,所以接受結果也一樣——這就是「$y$ 可以被任意重複」的由來。

泵引理最常見的用法,是反證法(又是第 13 章的老朋友):假設某語言是正規的,那它就必須滿足泵引理; 我們找一個它「泵不下去」的字串,撞出矛盾,於是它不可能是正規的。來證 $a^n b^n$:

🔑
定理:$L = \{a^n b^n : n \ge 0\}$ 不是正規語言。

證明(反證法 + 泵引理)。 假設相反,$L$ 是正規的。那它有一個泵長度 $p$。

挑一個「夠長」的字串:取 $s = a^p b^p$(剛好 $p$ 個 a 接 $p$ 個 b)。它屬於 $L$,長度 $2p \ge p$,所以泵引理保證它能切成 $s = xyz$,且 $|xy| \le p$、$|y| \ge 1$。

關鍵在條件 $|xy| \le p$:因為前 $p$ 個字元全都是 a,所以 $x$ 和 $y$ 都只落在 a 區裡。也就是說 $y$ 完全由 a 組成,$y = a^k$,其中 $k \ge 1$。

現在「泵一次」,取 $i = 2$:$xy^2z = a^{p+k} b^p$。這個字串有 $p+k$ 個 a、卻只有 $p$ 個 b,$k \ge 1$ 代表 a 比 b 多,所以它不屬於 $L$!

但泵引理明明保證 $xy^2z$ 該屬於 $L$——矛盾。所以「$L$ 是正規的」這個假設不成立。$L$ 不是正規語言。$\blacksquare$

品味一下這個證明在說什麼:DFA 的失敗,不是「不夠聰明」,而是結構上注定。 它沒有能隨輸入增長的記憶體,所以一旦字串夠長,它一定會陷入某個迴圈、開始「重複自己」, 而這種重複正好破壞了「a 和 b 必須精確相等」的要求。要突破這道天花板,我們需要給機器一樣新東西: 一塊能存放「我數到幾」的記憶體。

加一個堆疊:上下文無關文法與下推自動機

要記住「目前讀了幾個 a」,最自然的工具,你在第 23 章就熟得很——堆疊(stack)。 把有限自動機加上一個堆疊,就得到下推自動機(pushdown automaton,PDA)。

它的策略對 $a^n b^n$ 來說簡單到優雅:讀到每個 a,就往堆疊推入一個記號; 進入 b 區後,每讀一個 b,就彈出一個記號。最後若堆疊剛好清空、字串也剛好讀完,就接受。 堆疊就像一個「計數器」,而且它的高度不受限——這正是 DFA 缺的那塊記憶體。

用文法描述語言:上下文無關文法(CFG)

PDA 是「機器」的角度;同一類語言還有一個「生成」的角度,而且實務上更常用, 那就是你在第 42 章談語法分析時見過的上下文無關文法(context-free grammar,CFG)。 DFA/PDA 是「辨認」字串(給字串、判斷屬不屬於);CFG 則是一套「生產」字串的規則。

一個 CFG 由一組產生規則(production rules)組成。$a^n b^n$ 的文法美得只有兩條:

S → a S b
S → ε

這裡 S 是非終端符號(nonterminal)(可以繼續展開的變數), a、b 是終端符號(terminal)(最終出現在字串裡的字元),S 是起始符號。 規則的意思是:「一個 S,可以換成『a、再一個 S、再 b』,或者直接消失(換成空字串 ε)」。 反覆套用、直到沒有非終端符號為止,這個過程叫推導(derivation)。推導 aabb: 

S ⇒ a S b          (用 S → aSb)
  ⇒ a a S b b      (再用 S → aSb)
  ⇒ a a ε b b      (用 S → ε)
  = a a b b

每次推導都能畫成一棵剖析樹(parse tree)——根是起始符號,每次展開長出子樹,葉子由左到右讀出來就是字串。 這正是第 42 章編譯器把程式碼變成抽象語法樹(AST)的理論底座。

為什麼程式語言的語法是 CFG

你寫程式時天天倚賴 CFG,只是沒意識到。括號要成對巢狀、if 裡可以再包 if、 函式呼叫的引數裡可以再放函式呼叫、運算式 (a + b) * (c + d) 有任意深的結構—— 這些「巢狀、遞迴、要配對」的特性,正規語言全都做不到(它連括號配不配對都檢查不了,理由和 $a^n b^n$ 一模一樣), 但 CFG 駕輕就熟。一個典型的算術運算式文法長這樣: 

Expr   → Expr + Term  |  Term
Term   → Term * Factor  |  Factor
Factor → ( Expr )  |  數字

這幾條規則,就把「先乘除後加減」「括號可無限巢狀」全部編碼進去了。第 42 章的遞迴下降剖析器, 本質上就是把這份文法的每一條規則,翻譯成一個對應的遞迴函式。

ℹ️
CFG = PDA,但 PDA 的堆疊有它的極限

和 Kleene 定理對應,這裡也有一個等價定理:一個語言是上下文無關的,若且唯若有一台 PDA 辨認它—— 文法(生成)與堆疊機(辨認)是同一類語言的兩面。但堆疊只能「後進先出」地存取,這個限制讓 PDA 仍有死角。 經典反例是 $a^n b^n c^n$(a、b、c 三者數量都相等):堆疊數完 a 對掉 b 之後就空了, 沒辦法再拿來比對 c。$a^n b^n c^n$ 不是上下文無關語言(可以用 CFG 版的泵引理證明)。 要處理它,我們需要一塊不只能「後進先出」、而是能自由讀寫、來回移動的記憶體。 那就是下一台、也是最後一台機器。

✏️ 練習 49.2 思考

用泵引理的直覺(不必寫完整形式證明),解釋為什麼語言 $L = \{\, ww : w \in \{a,b\}^* \,\}$ (一個字串連續重複兩次,例如 abab、aa、abbabb)不是正規語言。 再想一想:如果只是 $\{\, w w^{R} : w \in \{a,b\}^* \,\}$($w^R$ 是 $w$ 的反轉,即迴文),它是上下文無關語言嗎?

看解答

$ww$ 不是正規:正規語言的記憶是有限的,但要驗證「後半段和前半段逐字相同」,機器必須把整個前半段一字不漏地記下來——而前半段可以任意長,需要無限的記憶,DFA 的有限狀態裝不下。

形式上仿照課文:取 $s = a^p b a^p b$(屬於 $L$,前後半都是 $a^p b$)。由 $|xy| \le p$,$y$ 落在最前面那段 a 裡,$y = a^k$($k \ge 1$)。泵成 $xy^2z = a^{p+k} b a^p b$,前半變成 $a^{p+k}b$、後半仍是 $a^p b$,兩半不再相等,$\notin L$,矛盾。故非正規。

迴文 $ww^R$:它是上下文無關語言。直覺:迴文的對稱是「由外往內」配對(第一個字元配最後一個、第二個配倒數第二……),這種「巢狀對稱」正合堆疊脾胃——把前半推進堆疊,後半逐字彈出來比對即可。對應的 CFG 也很短:$S \to aSa \mid bSb \mid \varepsilon$(偶長迴文)。

關鍵對比:$ww$(重複)需要「同序」比對,堆疊的後進先出剛好幫倒忙,它連 CFL 都不是;$ww^R$(迴文)需要「逆序」比對,正中堆疊下懷。同樣是「比對兩段」,順序差一個方向,難度就跨了一整級——這是喬姆斯基層級最迷人的細節之一。

圖靈機:計算的終極形狀

現在登場的,是本章的主角,也是第 1 章欠你的那台機器:圖靈機(Turing machine,TM)。 1936 年,23 歲的圖靈為了回答「什麼是『機械可計算』」這個哲學問題,設計了一台極簡到近乎簡陋、 卻被證明要多強有多強的抽象機器。

它只比有限自動機多了一樣東西,但這樣東西改變了一切:一條無限長、可任意讀寫、讀寫頭能來回移動的紙帶。

  • 一條紙帶(tape),切成一格格無限延伸的方格,每格放一個符號(或空白)。它同時是輸入、輸出,也是無限大的草稿紙。
  • 一個讀寫頭(head),指著某一格,能讀該格符號、覆寫它,然後左移或右移一格。
  • 一個有限的控制器(狀態),和 DFA 一樣只有有限個狀態,決定下一步怎麼動。

每一步,圖靈機看「目前狀態 + 讀寫頭所讀到的符號」,然後做三件事:寫一個符號回該格、 把讀寫頭左移或右移一格、切換到新狀態。如此反覆,直到進入特殊的「接受」或「拒絕」狀態為止 ——也可能永遠停不下來(這個「可能不停」的特性,稍後是整章的高潮)。

正式定義:七元組

一台(確定性)圖靈機是一個七元組:

$$M = (Q,\ \Sigma,\ \Gamma,\ \delta,\ q_0,\ q_{\text{accept}},\ q_{\text{reject}})$$

  • $Q$:有限狀態集合;$q_0$ 起始狀態;$q_{\text{accept}}$、$q_{\text{reject}}$ 兩個特殊的停機狀態。
  • $\Sigma$:輸入字母表;$\Gamma \supseteq \Sigma$:紙帶字母表(比輸入多了空白符號 ␣ 與運算用的暫存符號)。
  • $\delta$:轉移函數 $\delta : Q \times \Gamma \rightarrow Q \times \Gamma \times \{L, R\}$。 讀「(狀態,讀到的符號)」,輸出「(新狀態,要寫的符號,左移或右移)」——一行就把「寫、移、換」三件事全包了。

一台真的能跑的圖靈機:辨認 aⁿbⁿcⁿ

我們挑那個 PDA 辦不到的 $a^n b^n c^n$ 來示範,正好證明圖靈機嚴格更強。策略很像小學生劃記: 每一輪,從左到右劃掉一個 a、一個 b、一個 c(把它們改寫成記號),然後回到最左邊重來; 如果某一輪三種劃不齊(例如 a 還有、b 卻沒了),就拒絕;如果三種同時劃完,就接受。 靠的就是讀寫頭能來回掃描整條紙帶——這正是堆疊做不到的事。把它寫成轉移表: 

def run_tm(transitions, tape, start="q0", accept="q_accept",
           reject="q_reject", blank="_", max_steps=10000):
    """通用圖靈機模擬器。transitions:(狀態,符號) -> (新狀態,寫入,移動 L/R/S)"""
    tape = list(tape)            # 把輸入字串攤成可讀寫的紙帶
    head, state, steps = 0, start, 0
    while state not in (accept, reject):
        steps += 1
        if steps > max_steps:    # 安全閥:有些圖靈機真的會永遠跑下去
            return "loop?"
        if head < 0:                       # 讀寫頭掉出左端 → 紙帶向左長一格
            tape.insert(0, blank); head = 0
        if head == len(tape):             # 掉出右端 → 紙帶向右長一格(無限延伸)
            tape.append(blank)
        key = (state, tape[head])
        if key not in transitions:        # 沒定義的情況 = 拒絕
            return "reject"
        state, write, move = transitions[key]
        tape[head] = write                # 寫
        head += {"L": -1, "R": 1, "S": 0}[move]   # 移
    return "accept" if state == accept else "reject"

# 辨認 aⁿbⁿcⁿ:每輪把一個 a→X、一個 b→Y、一個 c→Z,回到最左再來一輪
delta = {
    # q0:找下一個未劃記的 a(或確認 a 全劃完了)
    ("q0", "X"): ("q0", "X", "R"),
    ("q0", "a"): ("q1", "X", "R"),
    ("q0", "Y"): ("q4", "Y", "R"),     # 沒有 a 了 → 進入最終檢查
    ("q0", "_"): ("q_accept", "_", "S"),   # 空字串(n=0)→ 接受
    # q1:向右找第一個未劃記的 b
    ("q1", "a"): ("q1", "a", "R"),
    ("q1", "Y"): ("q1", "Y", "R"),
    ("q1", "b"): ("q2", "Y", "R"),
    # q2:向右找第一個未劃記的 c
    ("q2", "b"): ("q2", "b", "R"),
    ("q2", "Z"): ("q2", "Z", "R"),
    ("q2", "c"): ("q3", "Z", "L"),
    # q3:一路向左,回到紙帶最左端
    ("q3", "a"): ("q3", "a", "L"), ("q3", "b"): ("q3", "b", "L"),
    ("q3", "X"): ("q3", "X", "L"), ("q3", "Y"): ("q3", "Y", "L"),
    ("q3", "Z"): ("q3", "Z", "L"),
    ("q3", "_"): ("q0", "_", "R"),     # 碰到左邊空白,回到 q0 開新一輪
    # q4:確認剩下只有 Y、Z(代表三者數量都對得上)
    ("q4", "Y"): ("q4", "Y", "R"), ("q4", "Z"): ("q4", "Z", "R"),
    ("q4", "_"): ("q_accept", "_", "S"),
}

for w in ["", "abc", "aabbcc", "aaabbbccc", "aabbc", "abcc", "acb", "aabbcccc"]:
    print(f"{w!r:12} -> {run_tm(delta, w)}")
# ''  abc  aabbcc  aaabbbccc → accept;  其餘(數量不齊或順序錯)→ reject

花點時間追一遍 aabbcc 的執行,你會看到讀寫頭像梭子一樣在紙帶上來回穿梭, 每趟劃掉一組 abc。這台機器用了那塊可讀寫、可重訪的記憶體, 做到了 DFA(沒記憶)和 PDA(只能後進先出)都做不到的事。喬姆斯基層級的最外圈,就是它的疆域。

🔑
能力階梯,一句話總結

有限自動機:沒有額外記憶體 → 只能辨認正規語言(連 $a^n b^n$ 都不會)。
下推自動機:加一個後進先出的堆疊 → 能辨認上下文無關語言($a^n b^n$、巢狀括號、程式語法),但敗給 $a^n b^n c^n$。
圖靈機:加一條無限、可讀寫、可來回移動的紙帶 → 目前所知一切能被機械計算的東西,它都能算。
記憶體的「形狀」,決定了機器的「智商」。

邱奇–圖靈論題:「可計算」到底是什麼意思

圖靈機看起來這麼陽春,憑什麼說它「要多強有多強」?這就是 1936 年那場思想風暴的核心。 當時好幾位數學家,各自從完全不同的方向,去定義「什麼叫做能被機械、按部就班地算出來」:

  • 圖靈用他的圖靈機(模仿一個人拿著紙筆、按固定規則一步步算)。
  • 阿隆佐·邱奇(Alonzo Church)用 λ 演算(lambda calculus) ——一套只有「函數」與「代入」的極簡形式系統,正是你在第 6 章碰到的函數式程式設計的祖先。
  • 哥德爾(Gödel)與克林尼(Kleene)用遞迴函數(recursive functions),從基本算術運算遞迴地堆出一切。

震撼的結果是:這三套(以及後來無數套——register machine、cellular automaton、你電腦的 CPU、Python、C……) 看似毫不相干的定義,被一一證明計算能力完全相同:凡是其中一套能算的,其他每一套都能算。 當這麼多獨立的路徑全部匯流到同一個點,科學家相信:我們捕捉到的不是某個人造模型的怪癖,而是「可計算」這個概念本身。

🔑
邱奇–圖靈論題(Church–Turing thesis)

「凡是能被任何機械程序計算出來的東西,都能被圖靈機計算出來。」
注意它叫「論題(thesis)」而不是「定理(theorem)」——因為「機械程序」是個直覺概念, 無法被數學證明,只能被一次次的證據支持。但八十多年來,從沒有人造出任何「能算、圖靈機卻算不出」的合理計算模型。 這個論題給了我們一份大禮:從此「可計算」有了精確的定義——就是「圖靈機可計算」。 於是「某問題沒有演算法能解」這種話,終於變成可以被嚴格證明的數學命題,而不只是「我們還沒想到」。

📜
一切的起點:希爾伯特的判定問題

圖靈和邱奇為什麼忽然要去定義「可計算」?導火線是 1928 年大數學家希爾伯特(David Hilbert)提出的 判定問題(Entscheidungsproblem):有沒有一套機械程序,輸入任何一個數學命題,就能判定它是真是假? 希爾伯特樂觀地相信「有」。結果 1936 年,圖靈和邱奇各自獨立地證明了:沒有。 而圖靈為了證明這個「不可能」,必須先精確定義「機械程序」是什麼——於是,圖靈機誕生了。 換句話說,現代電腦的理論藍圖,是作為一個「不可能性證明」的副產品而被發明出來的。

能算 vs 算不出:可判定性

有了「可計算 = 圖靈機可計算」這把尺,我們終於能精確地談論「界線」了。圖靈機處理一個語言,有兩種「成功」的程度, 差別非常微妙、卻至關重要。

類別圖靈機的行為舊名(來自遞迴論)
可判定(decidable)對每個輸入都保證會停:屬於就停在接受、不屬於就停在拒絕。一個永遠會給答案的機器。遞迴(recursive)
可識別(recognizable)若輸入屬於語言,機器一定會停下來說「接受」;但若不屬於,機器可能拒絕、也可能永遠跑下去不停。遞迴可枚舉(recursively enumerable, RE)

差別就在那個惱人的「不屬於的時候會不會停」。可判定的機器像個誠實的法官,有罪無罪當庭宣判; 可識別的機器像個只會說「有罪」的證人——它指認得出有罪的,但面對無辜者時,它可能永遠在思考, 害你乾等到天荒地老,永遠等不到「無罪」這句話。可判定一定可識別(會停的當然更好);反過來不一定。 而牆外還有更黑暗的一層:連可識別都辦不到的語言(連「屬於時保證會停」都做不到)—— 它們是計算徹底的盲區。

先數一數:為什麼「算不出」的問題必然存在

在動手抓出那個著名的不可判定問題之前,先看一個四兩撥千斤的計數論證, 它直接用第 13 章的可數性(countability)就能證明「不可計算的問題一定存在」。

🔑
程式只有可數多個,問題卻有不可數多個

程式(圖靈機)的數量是可數的。每個程式都是一串有限長的文字(原始碼), 而所有有限字串可以排成一列、一一編號——所以程式至多可數無限多個(第 13 章:能和正整數配對)。
但「問題」(語言)的數量是不可數的。一個語言是 $\Sigma^*$ 的子集,而一個無限集合的所有子集 (它的冪集)是不可數的——這正是康托爾用對角論證證明的(第 13 章預告過、馬上要兌現)。
結論:可數的程式,配不完不可數的問題。就像座位比客人少,必然有客人沒位子坐—— 必然存在某些語言,沒有任何程式能判定它,而且這種「算不出」的問題, 數量上還遠遠多於「算得出」的!可計算的世界,只是無窮汪洋裡的一座小島。

這個論證很美,但它「非建構性」(第 13 章的詞):它證明了「壞問題存在」,卻沒指出哪一個是壞問題。 圖靈做了更狠的事——他直接指名道姓抓出一個,而且是你第 1 章就見過的那個。

停機問題:那個漂亮到不可思議的證明

停機問題(halting problem)問的是:能不能寫一個萬能程式 H, 輸入「任意一個程式 P」和「一筆輸入 x」,就準確判斷 「P 跑在 x 上,究竟會停下來,還是會永遠跑下去」?

這需求合情合理到不行——誰不想有個工具能事先警告「你這段程式會無窮迴圈」?但圖靈證明了: 這樣的 H 不可能存在。不是「很難寫」,是邏輯上不可能。 證明用的是反證法,而它的靈魂,正是康托爾的對角論證——這次,我們把它對準「程式」自己。

🔑
定理(圖靈,1936):停機問題不可判定。

證明(反證法 / 自我指涉)。

第一步:假設它可判定。也就是假設存在一個總是會停、永遠答對的函式 halts(P, x): 當程式 P 跑在輸入 x 上會停機時,它回傳 True;會永遠跑下去時,回傳 False。

第二步:用它造一個「唱反調」的程式。注意一件事:程式本身也是一串文字,所以一個程式可以被當成輸入,餵給另一個程式,甚至餵給它自己。我們造一個搗蛋鬼 D,它吃一個程式 P 當輸入,然後故意跟 halts 的預測作對:

def D(P):
    if halts(P, P):      # 問:「P 餵自己當輸入,會停嗎?」
        while True:       # 若預測「會停」→ D 偏要永遠跑下去
            pass
    else:
        return           # 若預測「不會停」→ D 偏要馬上停下來

第三步:把 D 餵給它自己,問一個致命的問題——D(D) 會停嗎?只有兩種可能,而兩種都爆炸:

  • 假設 D(D) 會停。那麼 halts(D, D) 必須回傳 True。但翻開 D 的程式碼:halts(D,D) 為真時,它走進 while True 永遠跑下去——也就是 D(D) 不會停。矛盾!
  • 假設 D(D) 不會停。那麼 halts(D, D) 必須回傳 False。但翻開程式碼:halts(D,D) 為假時,它走 else 立刻 return——也就是 D(D) 會停。又矛盾!

結論:無論 D(D) 停或不停,都導出矛盾。唯一出錯的地方,是我們最初的假設——那個萬能的 halts 根本不存在。停機問題不可判定。$\blacksquare$

為什麼說這是「對角論證」?把它和康托爾的證明並排,結構一模一樣。想像一張無限大的表: 每一列是一個程式 $P_i$,每一欄是一筆輸入(也用程式編碼當輸入),格子記錄「$P_i$ 跑在那筆輸入上會不會停」。 D 這個程式,被刻意設計成沿著對角線跟每個程式唱反調:它在第 $i$ 格的行為, 故意和 $P_i$ 在第 $i$ 格相反。於是 D 不可能等於表上任何一個 $P_i$ (它至少在對角線那一格和每個人都不同)——可是 D 明明也是個程式,它應該在表上才對。矛盾。 這正是康托爾證明「實數比整數多」時,造出一個「和清單上每個數都不同」的新數的同一招。

🔍
停機問題「可識別、但不可判定」

停機問題不是徹底的盲區——它落在圖 49-1 那個「可識別卻不可判定」的環帶裡。 為什麼可識別?因為要確認「P 在 x 上會停」,你只要真的去跑它: 它若會停,你終究會等到那一刻,然後回答「會停」。麻煩出在另一邊——如果它不會停, 你跑下去就只是陪它一起永遠跑,永遠無法在有限時間內斷定「它不會停」。 這就是「可識別」與「可判定」之間那道精準的裂縫:能抓到所有的「是」,卻無法為所有的「否」蓋棺論定。

不可判定性的蔓延:Rice 定理

你可能想:好吧,停機問題是個極端例子,日常程式分析應該還好吧?壞消息是—— 不可判定性會傳染,而且傳染得極廣。一旦有了停機問題這個「種子」, 我們可以用第 33 章學的歸約(reduction),把它的「不可解」傳遞給一大票其他問題。

🔁
用歸約證明「不可判定」(方向與第 33 章相反的味道)

套路是:假設問題 X 可判定,然後說明「如果我能解 X,我就能拿它來解停機問題」。 但停機問題不可解,矛盾——所以 X 也不可解。歸約方向是「已知不可解的(停機問題)→ 待證的 X」: 把停機問題的實例,翻譯成 X 的實例。例如要證「判斷一個程式會不會印出 hello」不可判定, 就把任意 (P, x) 改寫成一個新程式「先模擬 P 跑 x,停了之後才印 hello」—— 這個新程式會印 hello,若且唯若 P 在 x 上會停。於是「會不會印 hello」要是可判定,停機問題就可判定了。矛盾。

這種傳染最終被一個驚人的定理一網打盡——萊斯定理(Rice's theorem)。它說的事情,殘酷得令人佩服:

🔑
Rice 定理(直覺版)

「任何關於『程式行為』的非平凡性質,都是不可判定的。」
這裡「行為」指的是程式算出什麼函數(輸入對應到什麼輸出),而不是它的程式碼長相; 「非平凡」指的是「有些程式有這性質、有些沒有」(不是對全部程式都成立、也不是對全部都不成立)。 在這個範圍內,幾乎你想得到的問題——「這程式會停嗎?」「它會印出 hello 嗎?」「它算的是不是排序?」 「它和那個程式功能等價嗎?」「它有沒有 bug(會不會在某輸入崩潰)?」——通通不可判定。

Rice 定理是給所有夢想「完美自動分析工具」的人的一盆冷水,但也是一份解脫。它解釋了為什麼 不存在能 100% 抓出所有 bug 的工具、為什麼編譯器的最佳化永遠只能「盡力」、 為什麼防毒軟體無法完美判定「這支程式是不是惡意的」。這些不是業界不夠努力,而是數學的鐵律。 於是實務工具(型別檢查器、靜態分析、linter)都聰明地退一步:它們保守地近似—— 寧可偶爾誤報(說沒問題的有問題),也不在「完美但不可能」的目標上撞牆。這正是第 33 章那句 「證明一個問題很難,是好消息」在可計算性層級的回響:知道哪裡是死路,才能把力氣花在活路上。

串回複雜度:能算之後,還要問「要多快」

到這裡,我們已經畫好了第一道、也是最外面的界線:可計算 vs 不可計算。 但故事還沒完。回到可計算的島嶼之內,還有一條更貼近日常工程的界線——同樣能算, 有的眨眼就好,有的卻要跑到宇宙終結。這就是計算複雜度(computational complexity)的領域, 你在第 26 章與第 33 章已經深入探訪過。計算理論的視角,只是把它們安放回同一張地圖上。

關鍵的轉念是:當我們不再問「圖靈機能不能停」,而問「圖靈機要走幾步才停」「要用紙帶上幾格」, 可計算性就細分成了一個個複雜度類別(complexity class):

類別用圖靈機的話說一句話
P確定性圖靈機在多項式時間內能判定「能快速解出」的問題(第 33 章)
NP非確定性圖靈機在多項式時間內能判定(等價於:解能被多項式時間驗證)「能快速驗證」的問題;NP 的「N」就是這台非確定性圖靈機
PSPACE用多項式大小的紙帶空間就能判定(時間不限)「省空間」的問題,含 P 與 NP,被相信更大
EXPTIME指數時間內能判定已被證明嚴格大於 P(有些問題真的非指數不可)

已知的包含關係是 $\mathrm{P} \subseteq \mathrm{NP} \subseteq \mathrm{PSPACE} \subseteq \mathrm{EXPTIME}$, 而第 33 章那句「NP 是非確定性多項式時間」的承諾,在這裡終於用圖靈機兌現了——「N」指的就是那台會分身的非確定性圖靈機。 其中 P vs NP 是否相等,仍是第 33 章那個懸著一百萬美元的世紀難題。

ℹ️
階層定理:更多時間/空間,真的能算更多

有件事是確定的(不像 P vs NP 還是謎):時間階層定理(time hierarchy theorem)與 空間階層定理證明了——給圖靈機明顯更多的時間或空間,它就真的能解出原本解不了的問題。 所以 $\mathrm{P} \subsetneq \mathrm{EXPTIME}$ 是被證明過的嚴格包含:存在一些問題,可判定,但無論如何都需要指數時間, 快不起來。換句話說,即使在「能算」的島上,難度也是一層一層、真實存在的山脈,不是錯覺。 有趣的是,這些階層定理的證明,骨子裡還是對角論證——又一次,那把鑰匙。

把這一章串起來

我們從第 1 章欠下的那個問題出發——「什麼能被計算?」——然後用三台機器,一路把答案問到了盡頭。 計算理論最終替我們畫出了三道界線:

  • 用什麼機器(自動機):記憶體的形狀決定能力。沒有記憶的有限自動機 → 正規語言(= DFA = NFA = 正規表示式,Kleene 定理);加一個堆疊的下推自動機 → 上下文無關語言(= CFG,程式語法的家);加一條無限可讀寫紙帶的圖靈機 → 可計算的一切。每一級的天花板,都靠泵引理這類論證劃定($a^n b^n$ 出局正規、$a^n b^n c^n$ 出局上下文無關)。
  • 能不能算(可計算性):邱奇–圖靈論題把「可計算」定義成「圖靈機可計算」。而對角論證(第 13 章的康托爾,被圖靈轉向程式自身)證明了停機問題不可判定;Rice 定理更指出,幾乎任何關於程式行為的非平凡問題都不可判定。可計算的世界,只是不可數汪洋裡的一座小島。
  • 要多快(複雜度):在能算的島內,用「要花多少時間/空間」把問題分成 P、NP、PSPACE、EXPTIME……階層定理保證這些層級真實存在,而 P vs NP 仍是世紀懸案(第 26、33 章)。

這三道界線——什麼能算、用什麼機器算、要多快算——合起來,是人類對「計算」這件事最深刻的理解之一。 它的驚人之處在於:這一切,早在第一台真正的電子電腦問世之前,就被圖靈、邱奇、哥德爾這些人, 單憑紙、筆和純粹的邏輯推導出來了。他們研究的從來不是某台機器,而是「計算」本身—— 這正好完美呼應了第 1 章開篇那句話:「電腦科學不是關於電腦的科學,正如天文學不是關於望遠鏡的科學。」 你手上這本書,就此繞了一個漂亮的大圈,回到了它最初的命題。

✏️ 練習 49.3 實作

設 $\Sigma = \{a, b\}$。請替語言 $L = \{\, w : w$ 同時含有子字串 ab 與 ba $\,\}$ 設計一台 DFA,畫出狀態圖,並用課文那個 DFA 類別把它寫成程式驗證。

看解答

思路:我們要記住「到目前為止,ab 看到了沒、ba 看到了沒」。但不能各記一個獨立旗標就完事——因為「看到 ab」這件事,本身要先看到一個 a。比較乾淨的設計,是追蹤「最近一個字元是什麼,以及我們已經湊齊了哪些目標」。一個可行的 5 狀態 DFA:

  • S(起始):還沒讀到任何字元,或只讀過開頭一連串相同字元前的空狀態。
  • A:目前結尾是 a,但還沒湊出 ab 或 ba 任一個(代表整串到目前為止都是 a)。
  • B:目前結尾是 b,且還沒湊出任何目標(整串到目前為止都是 b)。
  • P:已湊出「其中一個」目標,目前結尾 a。
  • Q:已湊出「其中一個」目標,目前結尾 b。
  • F(接受):ab 與 ba 都湊齊了。

與其手刻容易出錯的轉移表,更可靠的做法是:狀態直接用 (最後一個字元, 是否見過ab, 是否見過ba) 這個三元組來編碼,讓程式自動算出轉移——這也示範了「狀態 = 足以決定未來的最小摘要」:

# 狀態 = (last, seen_ab, seen_ba);last ∈ {"", "a", "b"}
def step(state, ch):
    last, ab, ba = state
    ab = ab or (last == "a" and ch == "b")   # 出現 a 後接 b → 湊出 ab
    ba = ba or (last == "b" and ch == "a")   # 出現 b 後接 a → 湊出 ba
    return (ch, ab, ba)

def accepts(w):
    state = ("", False, False)
    for ch in w:
        state = step(state, ch)
    return state[1] and state[2]             # ab 與 ba 都見過才接受

for w in ["", "ab", "ba", "aba", "bab", "abba", "aabb", "abab", "baab"]:
    print(f"{w!r:8} {accepts(w)}")
# 含 ab 也含 ba 的(aba, bab, abba, abab, baab)→ True;其餘 → False

觀察:「ab 與 ba 都出現」其實等價於「字串裡同時有 a 也有 b,且不是『全 a 接全 b』也不是『全 b 接全 a』」——更簡單地說,只要字串裡出現了至少一次「相鄰兩字元不同」兩種方向都有,就成立。把上面的有限狀態畫成圓圈與箭頭(把 (last, ab, ba) 會實際走到的組合列出來),就是你的狀態圖。因為狀態數有限,它確實是一台 DFA,故 $L$ 是正規語言。

✏️ 練習 49.4 思考

用泵引理證明 $L = \{\, a^{n^2} : n \ge 0 \,\}$(a 的個數是完全平方數:0、1、4、9、16……)不是正規語言。 提示:相鄰兩個平方數之間的「間隔」會越來越大。

看解答

證明(反證法 + 泵引理)。假設 $L$ 正規,設泵長度為 $p$。取 $s = a^{p^2} \in L$(長度 $p^2 \ge p)。泵引理給出 $s = xyz$,其中 $1 \le |y| \le |xy| \le p$。設 $|y| = k$,所以 $1 \le k \le p$。

泵一次($i = 2$):$xy^2z$ 的長度是 $p^2 + k$。我們證明它不是完全平方數,方法是把它夾在兩個連續平方數之間。因為 $1 \le k \le p$:

$$p^2 \;<\; p^2 + k \;\le\; p^2 + p \;<\; p^2 + 2p + 1 = (p+1)^2.$$

也就是說 $p^2 + k$ 嚴格落在 $p^2$ 與 $(p+1)^2$ 之間,而這兩者是相鄰的完全平方數,中間沒有別的平方數。所以 $p^2 + k$ 不是完全平方數,$xy^2z \notin L$。

但泵引理保證 $xy^2z \in L$——矛盾。故 $L$ 不是正規語言。$\blacksquare$

直覺:平方數越來越稀疏(間隔 $1, 3, 5, 7, \dots$ 不斷變大),但 DFA 的「迴圈」只會以固定週期 $k$ 反覆加長字串,固定步伐永遠追不上越拉越開的平方數間隔——這就是它注定失敗的原因。

✏️ 練習 49.6 思考

不看課文,用你自己的話,把「停機問題為什麼不可判定」的對角論證從頭複述一遍。 重點要講清楚三件事:(1) 我們一開始假設了什麼?(2) 那個「唱反調」的程式 D 是怎麼造的? (3) 為什麼問「D(D) 會停嗎」會兩面都撞牆?

看解答

(1) 假設:假設停機問題可判定,也就是存在一個永遠會給答案、而且永遠答對的函式 halts(P, x)——它能正確回答「程式 P 跑在輸入 x 上會不會停」。我們的目標是證明這個假設會自相矛盾。

(2) 造反調程式:利用「程式本身也是一串文字、可以被當輸入」這件事,造一個程式 D,它吃一個程式 P,先問 halts(P, P)(「P 餵自己會停嗎?」),然後故意做相反的事:預測「會停」它就無窮迴圈;預測「不會停」它就立刻結束。

(3) 自食其果:把 D 餵給它自己,問「D(D) 會停嗎?」

  • 若 D(D) 會停 → 表示 halts(D,D) 回答「會停」→ 但這樣 D 的程式碼會讓它進無窮迴圈、不停。矛盾。
  • 若 D(D) 不停 → 表示 halts(D,D) 回答「不停」→ 但這樣 D 的程式碼會讓它立刻結束、會停。矛盾。

兩條路都導致矛盾,所以唯一的錯誤源頭,就是最初「halts 存在」的假設。因此停機問題不可判定。$\blacksquare$

一句話抓住靈魂:我們造出一個「行為和每個程式都對著幹」的程式,它因此不可能等於任何程式——包括它自己,於是把『萬能判定器』的存在逼成了悖論。這就是把康托爾的對角線,對準了「程式」這張無限大的表。

✏️ 練習 49.7 挑戰

用歸約論證下面這個問題不可判定:
EQ:「給定兩個程式 P 與 Q,判斷它們是否對所有輸入都給出相同的輸出(功能等價)?」
提示:從某個你已知不可判定的問題出發,把它「翻譯」成 EQ。你可以用「空集問題」EMPTY:「給定程式 P,它是否對所有輸入都不接受(接受的語言是空集)?」——這已知不可判定。

看解答

策略:證明「EQ 可判定 ⟹ EMPTY 可判定」。由於 EMPTY 已知不可判定,反推 EQ 也不可判定。歸約方向是已知難的 EMPTY → 待證的 EQ(把 EMPTY 的實例翻譯成 EQ 的實例)。

歸約構造:假設我們有一台能判定 EQ 的機器 decideEQ(P, Q)。現在給一個 EMPTY 的實例(一個程式 P),我想判斷「P 的語言是不是空集」。我這樣做:

  • 準備一個固定的「什麼都不接受」的程式 P_reject:它對任何輸入都直接拒絕(語言是空集)。這個程式我們隨手就能寫出來。
  • 呼叫 decideEQ(P, P_reject)。

正確性:P 和「永遠拒絕」的 P_reject 功能等價,若且唯若 P 自己也是「永遠拒絕」,也就是 P 的語言是空集。所以:

def decideEMPTY(P):              # 假想:用 EQ 來解 EMPTY
    def P_reject(x):             # 一個「對任何輸入都拒絕」的程式
        return False
    return decideEQ(P, P_reject) # P ≡ P_reject  ⟺  P 的語言是空集

翻譯本身(寫出 P_reject、組裝呼叫)只花常數時間,而且一定會停。於是如果 decideEQ 存在且永遠會停,decideEMPTY 也會——但 EMPTY 不可判定,矛盾。所以 EQ 不可判定。$\blacksquare$

方向再叮嚀一次:歸約一定是「已知不可解的 → 待證的」。我們是用「假想中能解 EQ 的能力」去解掉「已知不能解的 EMPTY」,逼出矛盾。若不嫌麻煩,EMPTY 自己也能再往回歸約到停機問題,整條「不可解」的鏈,源頭都是那個對角論證。順帶一提,這個結果其實也是 Rice 定理的一個特例:「功能等價」是關於程式行為的非平凡性質,所以注定不可判定。
