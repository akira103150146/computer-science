一切的起點:遞迴窮舉

當一個問題的規模很小($n$ 大概在 20 上下,甚至更小),而你又「想不到聰明解」時,最樸實也最可靠的武器就是 窮舉(brute force):把所有可能性一個不漏地試一遍,挑出符合條件或最好的那個。而窮舉最優雅的寫法, 正是第 3 章教過的遞迴。AP325 第 1 章把遞迴的用途分成兩種,這個分類非常實用,值得你記在心裡:

  • 根據定義實作:題目本身就是用「答案和更小答案的關係」定義的(例如合成函數、把棍子從中點切開),你只要把定義翻譯成遞迴。
  • 以遞迴窮舉暴搜:題目沒有現成的遞迴定義,但你可以用遞迴枚舉所有選擇(子集合、排列、回溯),在過程中找答案。

我們先攻第二種——它是回溯與後面所有「枚舉型」題目的地基。

子集合枚舉:每個元素「選或不選」

最基本的窮舉,是枚舉一個集合的所有子集合(subset)。$n$ 個元素,每個都有「選」與「不選」兩種命運, 所以共有 $2^n$ 個子集合。用遞迴想:站在第 i 個元素面前,做一個二選一的決定,然後遞迴去處理第 i+1 個。 當所有元素都決定完(i == n),手上的 chosen 就是一個完整的子集合。 

#include <bits/stdc++.h>
using namespace std;

int a[20], n;

// 對「第 i 個元素」做決定:選它,或不選它
void dfs(int i, vector<int> &chosen) {
    if (i == n) {                       // 邊界:每個元素都決定完了
        for (int x : chosen) cout << x << ' ';
        cout << '\n';                    // 印出這個子集合
        return;
    }
    dfs(i + 1, chosen);                 // 分支一:不選 a[i]
    chosen.push_back(a[i]);             // 分支二:選 a[i]
    dfs(i + 1, chosen);
    chosen.pop_back();                  // 還原現場(這就是「回溯」!)
}

int main() {
    n = 3; a[0] = 1; a[1] = 2; a[2] = 3;
    vector<int> chosen;
    dfs(0, chosen);                     // 印出 {1,2,3} 的全部 8 個子集合
}

把這支遞迴的呼叫過程畫成一棵樹,你會看到一棵漂亮的二元決策樹:每一層對應一個元素,每往下一層就分岔成「不選 / 選」兩條路, 走到底的每一片葉子,就是一個子集合。

如果你不愛遞迴,子集合枚舉也有一個用位元(bit)表示的迭代寫法:把 0 到 2^n - 1 的每個整數 當成一張「點名表」,第 i 個位元是 1 就代表選了第 i 個元素。這個技巧在後面的狀態壓縮 DP會大放異彩: 

// 用位元枚舉所有子集合(mask 的第 i 位為 1 ⇒ 選 a[i])
for (int mask = 0; mask < (1 << n); mask++) {
    for (int i = 0; i < n; i++)
        if (mask & (1 << i))            // 檢查第 i 位是不是 1
            cout << a[i] << ' ';
    cout << '\n';
}
🥋
AP325 例題:子集合乘積 / 子集合的和

AP325【P-1-7 子集合乘積】是這個套路的典型應用:給一堆數,問能不能挑出一個子集合,讓乘積(或和)等於某個目標。 做法就是上面的遞迴 rec(i, prod)——考慮第 i 個元素「選或不選」,prod 帶著「目前選到的乘積」往下傳, 走到底時檢查 prod 是否命中目標。$n$ 小($\le 20$ 左右)時直接窮舉 $2^n$ 就夠; $n$ 稍大時還有「折半枚舉(meet in the middle)」把 $2^n$ 降成 $2^{n/2}$ 的進階招(AP325 第 2 章),這裡先知道有這條路。

排列生成:用 used[] 記住「誰還沒上場」

子集合是「每個元素選或不選」;排列(permutation)則是「把所有元素排成一列,問有幾種順序」。$n$ 個相異元素有 $n!$ 種排列。 遞迴的想法:一個位置一個位置地填,每個位置從「還沒用過的元素」裡挑一個放進去。用一個 used[] 陣列記錄誰已經上場了: 

int a[20], perm[20], n;
bool used[20];

void gen(int depth) {               // depth = 目前要填第幾個位置
    if (depth == n) {               // 邊界:全部位置都填滿了
        for (int i = 0; i < n; i++) cout << perm[i] << ' ';
        cout << '\n';
        return;
    }
    for (int v = 0; v < n; v++) {
        if (used[v]) continue;      // a[v] 已經在前面用過,跳過
        used[v] = true;             // 選擇
        perm[depth] = a[v];
        gen(depth + 1);             // 遞迴:填下一個位置
        used[v] = false;            // 還原(回溯!)
    }
}

看到那三拍了嗎?選擇(used[v]=true)→ 遞迴 → 還原(used[v]=false)。 這個節奏會在這一章一再出現,它就是回溯的指紋。

💡
競賽小抄:next_permutation

如果你只是要「依字典序走過所有排列」,C++ STL 直接幫你做好了:把陣列排序後,反覆呼叫 do { ... } while (next_permutation(a, a + n)); 就能不重複地枚舉全部 $n!$ 種排列,連遞迴都省了。 但手寫遞迴版仍然值得會——因為很多題目需要在「填到一半」時就剪枝或記錄狀態,那是 next_permutation 做不到的。

組合生成:強迫「由小到大」就不會重複

組合(combination)是「從 $n$ 個裡挑 $k$ 個,不管順序」。和排列的差別在於 {1,2} 和 {2,1} 算同一個。 要避免重複,有個簡單漂亮的招:規定選出來的元素索引必須由小到大。所以遞迴時,下一個元素一定從「上一個的後面」開始挑: 

int n, k, comb[20];

void choose(int start, int depth) {     // 從 start 開始挑,已挑了 depth 個
    if (depth == k) {                   // 邊界:挑滿 k 個
        for (int i = 0; i < k; i++) cout << comb[i] << ' ';
        cout << '\n';
        return;
    }
    // 剪枝:剩下的元素不夠湊滿 k 個就別試了
    for (int v = start; v <= n - (k - depth); v++) {
        comb[depth] = v;
        choose(v + 1, depth + 1);       // 下一個從 v+1 開始 ⇒ 永遠遞增、不重複
    }
}

注意那行迴圈條件 v <= n - (k - depth)——這是一個剪枝(pruning):還要挑 k - depth 個, 若 v 太大,後面根本湊不齊,試了也是白試,乾脆不進這個分支。剪枝是回溯的靈魂,我們馬上在 N 皇后看到它的威力。

回溯與剪枝:在死路出現的那一刻就掉頭

前面的子集合、排列、組合,其實都已經是回溯(backtracking)了——回溯就是「有系統地嘗試所有選擇, 在每一步試一個選項,行不通就退回上一步換下一個」。它和純窮舉的差別,在於回溯會在「確定此路不通」的瞬間 就放棄整片子樹(剪枝),而不是傻傻把那片子樹全走完。一次好的剪枝,常常能把指數級的搜尋砍掉大半。

🔑
回溯的三拍:選擇 → 遞迴 → 還原

每一個回溯函式都長這樣:(1) 做一個選擇並標記狀態 → (2) 遞迴處理剩下的子問題 → (3) 撤銷剛才的選擇(把狀態還原),好讓迴圈能乾淨地試下一個選項。 再加上剪枝(在進入分支前先判斷「這條路有沒有救」),你就握住了所有搜尋題的萬用骨架。 看到「列出所有解 / 數有幾種 / 在小範圍內找最佳」,先想回溯就對了。

經典:N 皇后

題目:在 $n \times n$ 的西洋棋盤上擺 $n$ 個皇后,使得任兩個皇后都不會互相攻擊 (不同列、不同行、不在同一條斜線)。問有幾種擺法?(AP325【P-1-9】)

關鍵觀察:既然每一列恰好要放一個皇后,我們就一列一列地放,問題瞬間變成「第 r 列的皇后放在哪一行?」。 用一維陣列 col[r] 記「第 r 列的皇后在第幾行」。放第 r 列時,只要檢查它和前面 0..r-1 列的皇后會不會打架: 同行(col[i] == c)或同斜線(行差等於列差,abs(col[i]-c) == abs(i-r))就不行。 

int n, cnt = 0;
int col[20];                        // col[r] = 第 r 列皇后所在的「行」

bool ok(int r, int c) {             // 第 r 列想放在第 c 行,可行嗎?
    for (int i = 0; i < r; i++) {
        if (col[i] == c) return false;                   // 同一行
        if (abs(col[i] - c) == abs(i - r)) return false; // 同一斜線
    }
    return true;
}

void solve(int r) {                 // 正在決定第 r 列
    if (r == n) { cnt++; return; }  // 全部 n 列都放好了 ⇒ 找到一組解
    for (int c = 0; c < n; c++) {
        if (ok(r, c)) {             // 剪枝:不可行的行,連遞迴都不進去
            col[r] = c;
            solve(r + 1);           // 不必手動還原:下一輪 col[r] 會被覆蓋
        }
    }
}

int main() {
    n = 8;
    solve(0);
    cout << cnt << '\n';            // 8 皇后共有 92 組解
}

那個 if (ok(r, c)) 就是剪枝:一旦發現第 c 行會和前面打架,我們連那一整片子樹都不展開。 沒有它,你得盲目枚舉 $n^n$ 種擺法;有了它,8 皇后只需試非常少的分支就能數出全部 92 組解。

💡
加速:用位元標記攻擊範圍

上面的 ok() 每次都要回頭掃前面所有皇后,是 $O(n)$。高手會改用三個位元遮罩(bitmask)分別記「哪些行、哪兩種斜線」已被占據, 放皇后時直接 $O(1)$ 查表、$O(1)$ 更新。這正是 AP325 提到的「對之前的皇后標記其可攻擊位子,遇到直接跳過」。 位元運算讓 N 皇后快到能算到 14、15 皇后,是很值得練的技巧——位元的思維,也是後面狀態壓縮 DP 的暖身。

另一種遞迴:把「定義」直接翻成程式

AP325 第 1 章的另一半是「根據定義實作」的遞迴。這類題不需要你窮舉,而是題目本身就帶著遞迴結構, 你的工作是把它老實翻譯出來。最經典的是合成函數(P-1-1):輸入是像 f g 3 5 f 2 這種前序式, 其中 f 吃一個參數、g 吃兩個參數、數字就是數字。要算出整串的值,只要寫一個「讀一個 token 就遞迴」的 eval(): 

// 假設 f(x)、g(x,y) 是題目給定的函數
int eval() {
    string tok;
    cin >> tok;
    if (tok == "f") return f(eval());                 // f 要 1 個參數 ⇒ 遞迴 1 次
    if (tok == "g") { int x = eval(), y = eval();     // g 要 2 個參數 ⇒ 遞迴 2 次
                      return g(x, y); }
    return stoi(tok);                                 // 是數字,直接回傳
}

看出來了嗎?遞迴的形狀,直接照著資料的形狀長。「函數的參數本身可能又是一個函數」這句話, 翻成程式就是「eval() 裡面呼叫 eval()」。同一個系列的棍子中點切割(P-1-3)也是如此: 「把一段 [left, right] 從最接近中點的切點 m 切開,成本是這段長度加上左右兩段各自的切割成本」—— 這句話幾乎是逐字翻成遞迴式: 

long long cut(int left, int right) {
    if (right - left <= 1) return 0;          // 邊界:相鄰兩點,不必再切
    int m = 找出最接近中點的切點;              // (座標遞增可用二分搜)
    return (p[right] - p[left])               // 切這一刀的成本 = 這段長度
         + cut(left, m) + cut(m, right);      // 加上左右兩段的成本
}
🔑
兩種遞迴,一眼分辨

遞迴定義題:題目給你「答案 = 某種更小答案的組合」,你照抄就好(合成函數、棍子切割)。 遞迴窮舉題:題目沒給遞迴式,但解空間能用「一步一步做選擇」展開,你用遞迴枚舉(子集合、排列、N 皇后)。 前者是「翻譯」,後者是「搜尋」。看清楚自己在做哪一種,你就不會把窮舉硬塞進定義題、或對著窮舉題乾等一個不存在的公式。

分治實戰:把問題對半劈開

第 27 章告訴你分治的三步驟:分割(divide)、克服(conquer)、合併(merge)——把問題切成兩半, 遞迴解決,再把兩半的答案拼起來。合併排序、快速排序都是它的代表作。在競賽裡,分治真正的價值往往不在「切」,而在「合併」那一步能不能順便算出跨越中線的答案。 AP325 第 5 章一句點破:「碰到沒見過的問題時,分治往往是第一個思考方向。」

典型架構長這樣(這就是你在第 27 章看過的骨架):

f(S):
    若 S 小到可直接算,回傳答案;          // 遞迴終端
    把 S 切成 S1、S2;                      // 分割
    L = f(S1);  R = f(S2);                 // 克服(遞迴)
    回傳 合併(L, R, 跨越中線的部分);       // 合併 ← 競賽的勝負手

反序數量:在合併排序裡「順便」數出來

題目:給一個陣列,問有幾組反序對(inversion)——也就是 i < j 但 a[i] > a[j] 的配對數。 (反序對的數量,直觀上衡量一個陣列「有多亂」。)(AP325【P-5-4】,APCS201806)

暴力是 $O(n^2)$ 兩兩比較。分治的洞見是:把反序對分成三類——兩個都在左半、兩個都在右半、 一個在左一個在右(跨越中線)。前兩類由遞迴算出;第三類,我們在合併兩個已排序的半邊時順手就能數出來: 當我們從右半取出一個元素 a[j] 插到答案前面時,左半還沒被取走的那些元素(它們都比 a[j] 大、且位置在前) 全都和它構成反序對——一次加上 m - i 個: 

// 對 a[l..r) 做合併排序,同時回傳這段裡的反序對數
long long merge_count(vector<int> &a, int l, int r) {
    if (r - l <= 1) return 0;            // 0 或 1 個元素,沒有反序對
    int m = (l + r) / 2;
    long long inv = merge_count(a, l, m) + merge_count(a, m, r);  // 左、右半

    vector<int> tmp;                     // 合併,並數「跨越中線」的反序對
    int i = l, j = m;
    while (i < m && j < r) {
        if (a[i] <= a[j]) tmp.push_back(a[i++]);
        else { tmp.push_back(a[j++]); inv += m - i; }  // a[i..m-1] 都比 a[j] 大
    }
    while (i < m) tmp.push_back(a[i++]);
    while (j < r) tmp.push_back(a[j++]);
    for (int k = l; k < r; k++) a[k] = tmp[k - l];     // 寫回
    return inv;
}

因為反序對是「邊合併邊數」,沒有增加額外的數量級,總複雜度還是合併排序的 $T(n) = 2T(n/2) + O(n) = O(n\log n)$, 把暴力的 $O(n^2)$ 一舉打下來。

最大連續子陣列:分治版 vs 掃描版

題目:給一個含正負數的陣列,找一段連續區間,使其總和最大(AP325【P-4-13 / P-5-2】, 這題同時是貪心/掃描章與分治章的例題,正好讓我們對照兩種範式)。

分治的切法:最大區間要嘛整段在左半、要嘛整段在右半、要嘛跨越中點。 前兩種遞迴解決;跨中點的那種,從中點分別往左、往右貪心地延伸,各取一段最大的前綴和相加: 

int best(vector<int> &a, int l, int r) {     // 處理 a[l..r)
    if (r - l == 1) return a[l];            // 只剩一個元素
    int m = (l + r) / 2;
    int L = best(a, l, m), R = best(a, m, r);   // 全在左 / 全在右

    int s = 0, leftMax = INT_MIN;           // 跨中點:從 m-1 往左延伸的最大和
    for (int i = m - 1; i >= l; i--) { s += a[i]; leftMax = max(leftMax, s); }
    s = 0; int rightMax = INT_MIN;          // 從 m 往右延伸的最大和
    for (int i = m; i < r; i++)   { s += a[i]; rightMax = max(rightMax, s); }

    return max({L, R, leftMax + rightMax});
}

這是 $O(n\log n)$。但同一題用第 31 章的掃描思維(Kadane 演算法)只要 $O(n)$: 令 f(i) 為「以 i 結尾的最大區間和」,則 $f(i) = \max(f(i-1) + a[i],\ a[i])$—— 前面累積是正資產就繼承、是負資產就拋棄重來。一個迴圈掃過去就好: 

long long kadane(vector<int> &a) {
    long long cur = a[0], best = a[0];
    for (int i = 1; i < (int)a.size(); i++) {
        cur = max((long long)a[i], cur + a[i]);   // 繼承 or 拋棄重來
        best = max(best, cur);
    }
    return best;
}

為什麼把這兩個版本擺在一起?因為它示範了一個重要的競賽認知:同一題常常有多個範式都能解,複雜度卻不同。 分治給你 $O(n\log n)$ 的通用思路,但若你看出「以 i 結尾」的遞推關係(那其實是個一維 DP / 掃描),就能更快。 AP325 也特別點出:分治的「合併(跨左右的解)」常常可以改寫成掃描線——逐步把右邊的資料加進來。 分治與掃描,是同一枚硬幣的兩面。

ℹ️
分治還能避開「超綱資料結構」

AP325 第 5 章有個務實的觀察:像「最近點對(closest pair)」「反序數量」這類題,用掃描線解常常需要 multiset / 線段樹這種較進階的資料結構;但改用分治,往往只需要單純的陣列就能搞定,在還沒學線段樹時特別好用。 另一個漂亮的分治例題是「大樓外牆廣告 / 最大矩形」:把區間從中點切開、由高到低外擴合併,或在合併階段用一個遞增堆疊(stack)$O(n)$ 處理。 競賽中,「能不能換一個範式繞過我還不會的工具」本身就是一種解題能力。

貪心:賭一條路,但要賭對

第 31 章說過貪心的精神:每一步都選「當下看起來最好」的,選了就不回頭。它快、它短、它優雅。 問題只有一個:它常常是錯的。第 32 章那個 coins = [1, 3, 4] 湊 6 的反例還記得嗎? 貪心先拿 4,結果用了 3 枚;最佳解是兩個 3,只要 2 枚。AP325【P-4-1 少林寺的代幣】用的正是同一組反例。

所以競賽裡用貪心,最重要的不是「想到貪心策略」(那通常是直覺),而是說服自己這個貪心是對的。 最常用的工具叫交換論證(exchange argument):

🔑
交換論證:證明貪心正確的萬用招

假設存在一個「最佳解」,它在某一步和你的貪心選擇不一樣。 證明你總能把最佳解裡的那一步換成貪心的選擇,而答案不會變差(甚至更好)。 既然換了不吃虧,那「按貪心做」就和最佳解一樣好——於是貪心也是最佳解。 這招的精髓是:不必從頭構造最佳解,只要證明「貪心的選擇可以無痛塞進任何最佳解裡」。

區間排程:挑「最早結束」的那個

題目(活動選擇 activity selection):有 $n$ 個活動,各有開始與結束時間。你一次只能參加一個(時間不能重疊), 問最多能參加幾個?(AP325【P-4-4 幾場華山論劍】)

直覺也許是「挑最短的」或「挑最早開始的」,但兩者都會出錯。正確的貪心:每次都挑「結束時間最早」且不和已選衝突的活動。 交換論證的直覺:結束得越早,留給後面的時間就越多,後面能塞進的活動只會更多、不會更少。所以先把活動依結束時間排序, 再從頭貪心地挑: 

struct Act { int s, e; };           // 開始 s、結束 e
int n;
Act act[100005];

int main() {
    cin >> n;
    for (int i = 0; i < n; i++) cin >> act[i].s >> act[i].e;

    sort(act, act + n, [](const Act &a, const Act &b){
        return a.e < b.e;           // 依「結束時間」由早到晚
    });

    int cnt = 0, lastEnd = -1;       // lastEnd = 已選活動裡最晚的結束時間
    for (int i = 0; i < n; i++) {
        if (act[i].s >= lastEnd) {   // 開始時間 ≥ 上一個的結束 ⇒ 不衝突
            cnt++;
            lastEnd = act[i].e;
        }
    }
    cout << cnt << '\n';
}

更多排序型貪心:交換論證的家族

AP325 第 4 章一連串例題,都是「找對排序的鑰匙」的貪心,證明清一色靠相鄰交換論證:

題型貪心策略(排序鑰匙)交換論證的直覺
最少完成時間總和
(P-4-3 十年磨一劍)		工作由短到長排若短的排在長的後面,把它們交換,總完成時間只會減少
加權最小完成時間
(P-4-5)		依 $t/w$ 由小到大;比較式用整數 a.t*b.w < b.t*a.w相鄰兩件交換,比較兩種順序的加權成本誰小
強弱對戰
(P-4-2 三戰)		我方最弱 a vs 對方最弱 b:$a\le b$ 則棄 a;否則派 a 戰 b「一定存在最佳解派最弱戰最弱」
Two-way merge / Huffman
(P-4-7)		每次取最小的兩個合併、把和放回(用 priority_queue)最小的兩個應該在合併樹最深處

其中加權排序那個「用整數比較式 a.t*b.w < b.t*a.w 取代浮點 a.t/a.w < b.t/b.w」是競賽必備的防雷技巧: 浮點數比較會有精度誤差,化成兩邊同乘的整數比較就乾淨無誤(前提是不溢位,必要時用 long long)。

⚠️
貪心最大的陷阱:沒證明就用

貪心策略「看起來很對」是最危險的時候。比賽中無數的 WA(Wrong Answer)都來自一個沒驗證的貪心。 養成習慣:想到貪心後,先試著用交換論證說服自己,或至少手造幾個小反例去打它。 打不倒,再動手寫。若你連反例都想不出、又證不出來,而 $n$ 很小,那就退一步用 DP 或暴搜保命—— 穩穩拿分,比賭一個沒把握的貪心強。

💡
外掛二分搜:對「答案」二分

有一類最佳化題,本身不好貪心,但「答案具有單調性」——例如 AP325【P-4-9 基地台】: 用 K 條等長線段蓋住所有點,求最短長度 R。直接求 R 很難,但反過來問「給定 R,最少需要幾條線段?」卻能用貪心快速回答, 而且這個數字隨 R 增加只會單調下降。於是我們對 R 二分搜,找最小的 R 使所需線段數 $\le K$。 這個「把最佳化問題轉成可行性判定 + 二分搜」的套路(第 60 章二分搜會細談),是貪心與二分搜最常見的合體技。

掃描線:讓一條線替你掃過所有事件

很多題目的資料是「一堆區間」或「平面上一堆點」。掃描線(sweep line)的想法是: 想像一條線沿著某個方向(通常是 x 軸,或時間軸)緩緩掃過去,在掃描的過程中維護一些資訊。 AP325 一句話抓住精髓:「掃描線的第一步往往是排序。」把所有「會發生的事」變成事件(event), 依座標排好,然後讓線一個一個事件地推進。

最多重疊區間:把端點變成 +1 / −1 事件

題目:給 $n$ 個區間 $[s_i, e_i]$,問在某個瞬間,最多有幾個區間同時覆蓋同一個點? (這就是「最多同時進行的活動數」「最少需要幾間教室」這類問題的核心。)

關鍵轉換:我們不關心區間長什麼樣,只關心「重疊數什麼時候變化」。一個區間開始時,重疊數 +1; 結束時,重疊數 -1。把每個區間拆成兩個事件 (s, +1) 和 (e, -1), 全部依座標排序,再讓掃描線從左到右走,維護一個「目前重疊數」cur,過程中的最大值就是答案: 

int main() {
    int n; cin >> n;
    vector<pair<int,int>> ev;            // (座標, +1 或 -1)
    for (int i = 0; i < n; i++) {
        int s, e; cin >> s >> e;
        ev.push_back({s, +1});          // 區間開始:重疊數 +1
        ev.push_back({e, -1});          // 區間結束:重疊數 -1
    }
    sort(ev.begin(), ev.end());         // 依座標排序(同座標時 -1 排在 +1 前)

    int cur = 0, best = 0;
    for (auto [x, d] : ev) {            // 掃描線一個事件一個事件推進
        cur += d;
        best = max(best, cur);
    }
    cout << best << '\n';
}
⚠️
同座標時,先 +1 還是先 −1?

當一個區間在 x 結束、另一個剛好在 x 開始,它們算不算重疊?這取決於題目用閉區間還是半開區間。 若「碰到端點不算重疊」(半開),要讓 -1 在 +1 之前處理——上面的 pair 排序剛好做到(因為 -1 < +1)。 若「碰到也算重疊」(閉區間),則要反過來讓 +1 先處理。這個一行之差常常是 AC 與 WA 的分水嶺,務必照題意決定。

線段聯集:依左端排序,邊掃邊併

題目:給 $n$ 條線段,求它們聯集(union)覆蓋的總長度(AP325【P-4-11 線段聯集】,APCS201603)。 把線段依左端排序,掃描線維護「目前這一團」連續覆蓋的右界 curR;遇到下一條線段: 若它的左端超過 curR,代表斷開了,把前一團的長度結算掉、另起新的一團;否則延長 curR: 

sort(seg, seg + n);                  // pair 預設先比左端,正好依左端排序
long long total = 0;
int curL = seg[0].first, curR = seg[0].second;
for (int i = 1; i < n; i++) {
    if (seg[i].first > curR) {        // 和目前這團斷開了
        total += curR - curL;        // 結算前一團
        curL = seg[i].first;
        curR = seg[i].second;
    } else {
        curR = max(curR, seg[i].second);  // 重疊,延長右界
    }
}
total += curR - curL;                // 別忘了最後一團
cout << total << '\n';

其他掃描線經典還有:一次買賣(P-4-12)——掃描時維護「到目前為止的最小買價(prefix-min)」, 用今天賣價減去它就是今天賣出的最大獲利;控制點 / 2D-max(P-4-14)——由右往左掃,維護「右方出現過的最大 Y」。 它們的共同骨架都是:排序 → 一條線掃過去 → 沿途維護一兩個關鍵量。一旦你習慣把問題翻譯成「事件流」, 很多看似複雜的區間題會瞬間變簡單。

動態規劃實戰:狀態 + 轉移的競賽兵器庫

終於來到本章的重頭戲。第 32 章已經把 DP 的原理講透了——記憶化、表格法、最佳子結構、重疊子問題, 還有那套「定義狀態 → 寫轉移方程 → 確定 base case → 決定計算順序 → 回推」的五步框架。 這一節我們不重講原理,而是把競賽最常考的幾種 DP「題型」打包成可辨認、可套用的兵器。AP325 第 6 章把 DP 用一個簡潔的維度系統分類,很值得借用:

ℹ️
DP 的三性質與 xDyD 分類

一個問題能用 DP,需要三個性質:最佳子結構(大解由小解拼出)、重疊子問題(同樣的子問題一再出現)、 以及無後效性(no aftereffect)——「未來只取決於目前的狀態,與『怎麼走到這個狀態』無關」。第三點是競賽 DP 的隱形前提: 狀態一旦定好,過去的細節就該被完全濃縮在裡面。
分類上,AP325 用 xDyD 速記:狀態有 $O(n^x)$ 個、每個狀態的轉移要看 $O(n^y)$ 個更小狀態,未優化的總複雜度就是 $O(n^{x+y})$。 例如 LIS 樸素版是 1D1D($O(n^2)$),0/1 背包是 2D0D($O(nW)$),區間 DP 多是 2D1D($O(n^3)$)。 這個記法能幫你在動手前先估好複雜度,判斷會不會超時。

背包:0/1 與完全,只差一個迴圈方向

背包(knapsack)是 DP 的鎮店之寶,第 32 章用二維表講過 0/1 背包。競賽裡更常用一維滾動陣列版本, 而且 0/1 背包與完全背包的程式碼幾乎一模一樣,只差內層迴圈的方向——這個對比是背包題的精華,務必記牢。

0/1 背包(P-6-9 大賣場免費大搬家):每件物品最多拿一次。狀態 dp[j] = 容量 j 時的最大價值, 內層容量迴圈由大到小跑,確保每件物品在這一輪只被用到「上一輪(還沒拿這件)」的結果: 

int dp[MAXW + 1] = {0};
for (int i = 0; i < n; i++)
    for (int j = W; j >= w[i]; j--)              // 0/1:由大到小!
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
cout << dp[W] << '\n';

完全背包(換零錢、無限拿同一物):每件物品可以拿無限多次。只要把內層迴圈改成由小到大, dp[j - w[i]] 就會是「這一輪已經可能拿過這件」的結果,於是同一件物品被允許重複拿: 

int dp[MAXW + 1] = {0};
for (int i = 0; i < n; i++)
    for (int j = w[i]; j <= W; j++)              // 完全:由小到大!
        dp[j] = max(dp[j], dp[j - w[i]] + v[i]);
cout << dp[W] << '\n';
💡
背包是個「變形大師」

很多題目根本沒提「背包」,骨子裡卻是背包。AP325 與 LeetCode 的常見偽裝:分割等和子集 (能否選出一個子集和恰為總和一半 ⇒ 布林版 0/1 背包)、湊零錢的最少枚數(完全背包把 max 換成 min、價值換成「枚數 1」)、 湊出某金額的方法數(把 max 換成「累加」)。認出「這題其實是背包」,是 DP 練到中期最有價值的能力。

最長遞增子序列 LIS:$O(n\log n)$ 的 DP × 二分搜

第 32 章給過 LIS 的 $O(n^2)$ 版(dp[i] = 以 a[i] 結尾的最長遞增長度)。競賽裡資料量一大,$O(n^2)$ 會超時, 必須會 $O(n\log n)$ 版(AP325【P-6-15 一覽衆山小】)。它的想法非常巧,值得細品:

維護一個陣列 tails,讓 tails[k] 代表「所有長度為 k+1 的遞增子序列裡,結尾值最小的那個」。 為什麼要記「最小的結尾」?因為結尾越小,後面越容易接更多數上去——這是個藏在 DP 裡的貪心。 tails 必定是嚴格遞增的,所以每來一個新數 x,就用二分搜(第 60 章)找到它該落腳的位置:

  • 若 x 比所有 tails 都大 → 它能讓最長序列再長一格,push_back 到尾端。
  • 否則 → 找到第一個 $\ge x$ 的位置,用 x 把它替換掉(同長度但結尾更小,後續更有利)。
int lis(vector<int> &a) {
    vector<int> tails;                     // tails[k]:長度 k+1 的 LIS 的最小結尾
    for (int x : a) {
        auto it = lower_bound(tails.begin(), tails.end(), x);  // 第一個 ≥ x
        if (it == tails.end()) tails.push_back(x);  // x 最大 ⇒ 接長
        else *it = x;                               // 否則替換,壓低結尾
    }
    return (int)tails.size();             // tails 的長度 = LIS 的長度
}

每個元素做一次 $O(\log n)$ 的二分搜,總共 $O(n\log n)$。

區間 DP:狀態是「一段區間」,轉移是「枚舉切點」

前面的 DP 狀態都靠「索引 / 容量」描述。還有一大類 DP,它的狀態是「一段連續區間 [i, j]」, 轉移時枚舉「在哪裡把區間切成兩半」。這叫區間 DP(interval DP), 經典代表是石子合併 / 切棍子(AP325 P-6-17、矩陣鏈乘 Q-6-18、合併成本)。

題目(石子合併):一排 $n$ 堆石子,每次只能合併相鄰兩堆,成本是這兩堆的重量和。 把所有石子合成一堆,求最小總成本。

套五步框架:

  1. 狀態:dp[i][j] = 把第 i..j 堆合併成一堆的最小總成本。
  2. 轉移:最後一次合併,一定是把 [i,k] 合成的一堆與 [k+1,j] 合成的一堆併起來, 這最後一併的成本是整段重量和 sum(i..j)。枚舉切點 k 取最小: $$dp[i][j] = \min_{i \le k \lt j}\big(dp[i][k] + dp[k+1][j]\big) + \big(\text{sum}(i..j)\big)$$
  3. base case:dp[i][i] = 0(單獨一堆,不必合併)。
  4. 順序:依區間長度由短到長填表(算長區間前,它依賴的所有短區間都已備妥)。
  5. 回推:若要知道合併順序,額外記每格的最佳切點 k。
int dp[305][305], pre[305];          // pre[i] = a[0]+...+a[i-1] 前綴和
// 讀入 a[0..n-1],建好前綴和,dp[i][i] 預設為 0

for (int len = 2; len <= n; len++)               // 區間長度由短到長
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;
        dp[i][j] = INT_MAX;
        int sum = pre[j + 1] - pre[i];           // 這段石子的總重
        for (int k = i; k < j; k++)              // 枚舉切點
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum);
    }
cout << dp[0][n - 1] << '\n';

狀態有 $O(n^2)$ 個、每個轉移枚舉 $O(n)$ 個切點,所以是 2D1D 的 $O(n^3)$——這正是區間 DP 的招牌複雜度。

樹形 DP:一句話先打個招呼

還有一整類 DP 活在樹上,叫樹形 DP(tree DP):狀態定義在「以某個節點為根的子樹」上, 轉移時把各個子節點的結果由下而上(從葉子往根)彙整起來。最經典的是樹的最大權重獨立集 (公司派對:不能同時邀請一個員工和他的直屬主管,求邀請到的最大歡樂值):每個節點 v 維護兩個狀態—— $f_0(v)$(不選 v)與 $f_1(v)$(選 v),轉移是 $f_1(v) = r(v) + \sum_u f_0(u)$、$f_0(v) = \sum_u \max(f_0(u), f_1(u))$。 這一支牽涉到樹的走訪與更多技巧,我們留到第 63 章專門招待——你現在只要記得:看到「樹」加上「最佳化 / 計數」,DP 往往就藏在 DFS 的回傳值裡。

狀態壓縮 DP:用一個整數記住「走過哪些」

當「狀態」需要記錄「一個集合的成員有沒有被選 / 被造訪」,而集合大小 $n$ 又很小(通常 $\le 20$)時, 我們可以用一個整數的位元來表示這個集合——第 i 位是 1 代表「第 i 個元素在集合裡」。 這叫狀態壓縮 / 位元 DP(bitmask DP)。它把「子集合」這個原本要用容器表示的東西,壓成一個可以當陣列索引的整數,優雅至極。

最經典的入門題是旅行商問題(travelling salesman problem, TSP)(AP325【Q-6-25 貨郎問題】): $n$ 個城市兩兩有距離,從城市 0 出發,每個城市恰好造訪一次再回到 0,求最短總路程。 暴力枚舉所有排列是 $O(n!)$,$n=15$ 就爆了;狀態壓縮 DP 能做到 $O(2^n \cdot n^2)$,$n=18$ 仍可行。

套五步框架:

  1. 狀態:dp[mask][i] = 「已造訪的城市集合是 mask、目前人停在城市 i」時的最短路程。
  2. 轉移:從 (mask, i) 走到一個還沒造訪的城市 j: $$dp[\,mask \,|\, (1\!\ll\! j)\,][j] = \min\big(\cdots,\ dp[mask][i] + d[i][j]\big)$$
  3. base case:dp[1][0] = 0(只造訪了城市 0、人在城市 0,路程為 0)。
  4. 順序:mask 由小到大枚舉(加入新城市只會讓 mask 變大,天然滿足依賴順序)。
  5. 回推 / 收尾:所有城市都造訪後(mask = (1<<n)-1),再從停留點 i 走回城市 0。
const int INF = 1e9;
int d[18][18];                          // 城市間距離
int dp[1 << 18][18];

int tsp(int n) {
    for (int m = 0; m < (1 << n); m++)
        for (int i = 0; i < n; i++) dp[m][i] = INF;
    dp[1][0] = 0;                       // 從城市 0 出發(集合 {0} = 二進位 ...0001)

    for (int mask = 1; mask < (1 << n); mask++)
        for (int i = 0; i < n; i++) {
            if (!(mask & (1 << i))) continue;     // i 必須已在 mask 裡
            if (dp[mask][i] == INF) continue;
            for (int j = 0; j < n; j++) {
                if (mask & (1 << j)) continue;     // j 還沒造訪才能去
                int nm = mask | (1 << j);
                dp[nm][j] = min(dp[nm][j], dp[mask][i] + d[i][j]);
            }
        }

    int ans = INF, full = (1 << n) - 1;
    for (int i = 1; i < n; i++)         // 全造訪後,從 i 走回起點 0
        ans = min(ans, dp[full][i] + d[i][0]);
    return ans;
}

看到題目,該用哪個範式?

練了這麼多套路,最後要回答那個讓人對著螢幕發呆的問題:拿到一題,我到底該用哪一招? 沒有萬無一失的公式,但有一條很實用的「提問順序」。下面這張決策地圖,把競賽選手腦中那套快速判斷攤了開來—— 遇到新題目,就照著一條一條問自己。

🔑
最重要的判斷:先看 $n$

競賽老手讀題的第一件事,常常是瞄一眼資料範圍,因為 $n$ 直接暗示了預期複雜度與範式: $n \le 20$ → 大概是指數級(回溯 / 狀態壓縮 DP);$n \le 500$ → 容得下 $O(n^3)$(區間 DP); $n \le 5000$ → $O(n^2)$(樸素 DP);$n \le 10^5 \sim 10^6$ → 只能 $O(n\log n)$ 或 $O(n)$(排序、貪心、掃描線、二分搜、$O(n\log n)$ 的 LIS)。 範圍會說話——它常常在你還沒想清楚演算法前,就先替你縮小了範式的選項。

動手練習

競賽能力是純粹「刷出來」的。以下七題覆蓋本章每個範式,難度遞增。請先自己寫,再展開對照解答; 每一題都先逼自己回答:這是哪個範式?狀態 / 貪心策略 / 切法是什麼?複雜度夠不夠?所有解答以 C++ 給出。

✏️ 練習 62.1 實作

回溯生成排列與組合:給整數 n,(a) 印出 1..n 的所有排列;(b) 印出從 1..n 中選 k 個的所有組合(每個組合內由小到大)。用回溯實作,並在組合版加上一個剪枝。

看解答

兩者都是「選擇 → 遞迴 → 還原」的回溯。排列用 used[] 避免重複用同一個數;組合靠「下一個從 v+1 開始」保證遞增不重複,並用 v <= n - (k - depth) + 1 剪掉湊不滿的分支。

#include <bits/stdc++.h>
using namespace std;
int n, k, perm[20], comb[20];
bool used[20];

void permute(int depth) {
    if (depth == n) {
        for (int i = 0; i < n; i++) cout << perm[i] << " \n"[i == n - 1];
        return;
    }
    for (int v = 1; v <= n; v++) if (!used[v]) {
        used[v] = true;  perm[depth] = v;
        permute(depth + 1);
        used[v] = false;                 // 還原
    }
}

void choose(int start, int depth) {
    if (depth == k) {
        for (int i = 0; i < k; i++) cout << comb[i] << " \n"[i == k - 1];
        return;
    }
    // 剪枝:剩下的數不夠湊滿 k 個就停
    for (int v = start; v <= n - (k - depth) + 1; v++) {
        comb[depth] = v;
        choose(v + 1, depth + 1);
    }
}

int main() {
    n = 3; permute(0);          // 3! = 6 個排列
    cout << "---\n";
    n = 4; k = 2; choose(1, 0); // C(4,2) = 6 個組合
}

小技巧:" \n"[i == last] 用一個兩字元字串當「空格 / 換行」的選擇器,是競賽常見的省事寫法。

✏️ 練習 62.2 實作

N 皇后計數:輸入 n($1 \le n \le 12$),輸出在 $n \times n$ 棋盤上擺 $n$ 個互不攻擊皇后的方案數。(進階:用位元遮罩加速到能算 n = 14。)

看解答

基本版就是本章的 solve。這裡直接給位元加速版:用三個遮罩 cols(已占的行)、d1(↘ 斜線)、d2(↙ 斜線),放第 r 列時,可放的位置是這三者聯集的補集。

#include <bits/stdc++.h>
using namespace std;
int n; long long cnt = 0;

void solve(int row, int cols, int d1, int d2) {
    if (row == n) { cnt++; return; }
    int full = (1 << n) - 1;
    int avail = full & ~(cols | d1 | d2);     // 還能放皇后的行(位元為 1)
    while (avail) {
        int p = avail & (-avail);             // 取最低位的 1(一個候選位置)
        avail -= p;                           // 把它從候選中移除
        // 下一列:斜線整體左右各移一位
        solve(row + 1, cols | p, (d1 | p) << 1, (d2 | p) >> 1);
    }
}

int main() {
    cin >> n;
    solve(0, 0, 0, 0);
    cout << cnt << '\n';        // 例:n=8 → 92
}

avail & (-avail) 是擷取最低位 1 的經典位元技巧。斜線之所以「左移 / 右移一位」,是因為走到下一列時,同一條對角線的攻擊欄會平移一格。

✏️ 練習 62.3 實作

掃描線:最多重疊區間:給 n 個區間 $[s_i, e_i)$(半開,碰到端點不算重疊),求任一瞬間最多有幾個區間同時覆蓋。$n \le 2\times10^5$。

看解答

把每個區間拆成 (s, +1)、(e, -1) 兩個事件,排序後累加。半開區間 ⇒ 同座標時讓 -1 先於 +1(pair 排序天然如此,因為 -1 < +1)。

#include <bits/stdc++.h>
using namespace std;
int main() {
    int n; cin >> n;
    vector<pair<int,int>> ev;
    for (int i = 0; i < n; i++) {
        int s, e; cin >> s >> e;
        ev.push_back({s, +1});
        ev.push_back({e, -1});
    }
    sort(ev.begin(), ev.end());          // 同座標 -1 先處理(半開區間)
    int cur = 0, best = 0;
    for (auto [x, d] : ev) { cur += d; best = max(best, cur); }
    cout << best << '\n';
}

若題目改成閉區間(碰到端點也算重疊),把結束事件的座標改用 (e, +1) 的相反排序,或更穩妥地令結束事件座標為 e 但讓 +1 先處理(例如把事件值反過來定義)。一行之差,務必照題意。

✏️ 練習 62.4 實作

完全背包(湊零錢方法數):有面額 coins[](每種無限多),問湊出金額 amount 共有幾種組合方法(只看用了哪些硬幣、不計順序)。例如 coins=[1,2,5], amount=5 有 4 種。

看解答

這是完全背包的「計數」變形:把 max 換成「累加」。關鍵在迴圈順序——外層跑硬幣、內層跑金額(由小到大), 這樣每種硬幣只在它「那一輪」被加入,自然避免把 (1 後 2) 和 (2 後 1) 算成兩種。

#include <bits/stdc++.h>
using namespace std;
int main() {
    vector<int> coins = {1, 2, 5};
    int amount = 5;
    vector<long long> dp(amount + 1, 0);
    dp[0] = 1;                            // 湊 0 元:一種(什麼都不拿)
    for (int c : coins)                   // 外層:硬幣
        for (int j = c; j <= amount; j++) // 內層:金額由小到大(完全背包)
            dp[j] += dp[j - c];
    cout << dp[amount] << '\n';           // 4
}

對照:若把兩層迴圈對調(外層金額、內層硬幣),算的就是「排列數」(順序不同算不同),那是另一個問題。迴圈順序在背包計數題裡決定了題意,要想清楚。

✏️ 練習 62.5 挑戰

LIS $O(n\log n)$ + 還原序列:給陣列 a,輸出最長嚴格遞增子序列的長度,並還原出一個這樣的子序列。

看解答

$O(n\log n)$ 版只給長度;要還原,額外記錄兩件事:pos[i] = 元素 a[i] 被放進 tails 的位置(即「以它結尾時的 LIS 長度 − 1」),以及 prev[i] = 它接在誰後面。最後從「位置最大」的元素往回串。

#include <bits/stdc++.h>
using namespace std;
int main() {
    vector<int> a = {2, 5, 3, 7, 1, 6};
    int n = a.size();
    vector<int> tails, idx, prev(n, -1), pos(n);
    for (int i = 0; i < n; i++) {
        int p = lower_bound(tails.begin(), tails.end(), a[i]) - tails.begin();
        if (p == (int)tails.size()) { tails.push_back(a[i]); idx.push_back(i); }
        else { tails[p] = a[i]; idx[p] = i; }
        pos[i] = p;
        prev[i] = (p > 0) ? idx[p - 1] : -1;   // 接在上一段的結尾後面
    }
    int len = tails.size();
    cout << "LIS 長度 = " << len << '\n';
    // 還原:從最後一段的結尾往回走
    vector<int> seq;
    for (int cur = idx[len - 1]; cur != -1; cur = prev[cur])
        seq.push_back(a[cur]);
    reverse(seq.begin(), seq.end());
    for (int x : seq) cout << x << ' ';        // 例:2 3 6
    cout << '\n';
}

注意 prev[i] 在「放進去的當下」就要記 idx[p-1](那時它還是正確的前一段結尾)。最長非遞減則把 lower_bound 換成 upper_bound。

✏️ 練習 62.6 挑戰

區間 DP:石子合併:一排 n 堆石子,重量 a[i]。每次只能合併相鄰兩堆,成本為兩堆重量和。求合成一堆的最小總成本。$n \le 300$。

看解答

狀態 dp[i][j] = 合併 i..j 的最小成本;轉移枚舉切點 k,最後一併的成本是整段和(用前綴和 $O(1)$ 取得);依區間長度由短到長填。$O(n^3)$。

#include <bits/stdc++.h>
using namespace std;
int main() {
    int n; cin >> n;
    vector<int> a(n), pre(n + 1, 0);
    for (int i = 0; i < n; i++) { cin >> a[i]; pre[i + 1] = pre[i] + a[i]; }

    vector<vector<int>> dp(n, vector<int>(n, 0));   // dp[i][i] = 0
    for (int len = 2; len <= n; len++)
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;
            dp[i][j] = INT_MAX;
            int sum = pre[j + 1] - pre[i];
            for (int k = i; k < j; k++)
                dp[i][j] = min(dp[i][j], dp[i][k] + dp[k + 1][j] + sum);
        }
    cout << dp[0][n - 1] << '\n';
}

若石子排成環(首尾相鄰),標準技巧是把陣列複製成兩倍長 a + a,對所有長度 n 的窗口取 dp 最小值。另外,本題可用「四邊形不等式」優化到 $O(n^2)$,屬進階。

✏️ 練習 62.7 挑戰

狀態壓縮 DP:旅行商(小 n):n 個城市($n \le 16$),d[i][j] 為城市間距離。從城市 0 出發,每城恰造訪一次後回到 0,求最短總路程。

看解答

就是本章的 dp[mask][i]:mask 記造訪集合、i 記目前位置。$O(2^n \cdot n^2)$。注意陣列大小 1<<n 在 n=16 時是 65536,乘上 16 完全可行。

#include <bits/stdc++.h>
using namespace std;
const int INF = 1e9;
int d[16][16], dp[1 << 16][16];

int main() {
    int n; cin >> n;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++) cin >> d[i][j];

    for (int m = 0; m < (1 << n); m++)
        for (int i = 0; i < n; i++) dp[m][i] = INF;
    dp[1][0] = 0;                                  // 起點:集合 {0}、在城 0

    for (int mask = 1; mask < (1 << n); mask++)
        for (int i = 0; i < n; i++) {
            if (!(mask & (1 << i)) || dp[mask][i] == INF) continue;
            for (int j = 0; j < n; j++) {
                if (mask & (1 << j)) continue;       // j 已造訪
                int nm = mask | (1 << j);
                dp[nm][j] = min(dp[nm][j], dp[mask][i] + d[i][j]);
            }
        }

    int ans = INF, full = (1 << n) - 1;
    for (int i = 1; i < n; i++)
        ans = min(ans, dp[full][i] + d[i][0]);     // 收尾:走回城 0
    cout << ans << '\n';
}

若只要求「走完所有城市的最短路徑(不必回起點)」,就把收尾那段改成直接取 dp[full][i] 的最小值。狀態壓縮 DP 的變形很多,但「mask 當集合、最外層枚舉 mask」這個骨架是共通的。

把這一章串起來

這一章,我們把第六部那些「演算法理論」拿到競賽場上重新淬鍊了一遍。不是再學一次它們是什麼,而是學會認出題目該用哪一招、並把每一招寫成可複用的套路。回顧幾個關鍵路標:

  • 遞迴窮舉 / 回溯:子集合(選/不選)、排列(used[])、組合(強迫遞增)的範本,以及回溯的三拍「選擇 → 遞迴 → 還原」加剪枝。N 皇后是它的代表作。也分清「遞迴定義題」(翻譯)與「遞迴窮舉題」(搜尋)。
  • 分治:勝負手在「合併」——反序數量在合併排序裡順手數出、最大子陣列靠跨中點延伸。同題的掃描 / DP 版常常更快;分治也能繞過超綱資料結構。
  • 貪心:核心不是想到策略,而是用交換論證證明它對。區間排程挑「最早結束」、加權題找對排序鑰匙、用整數比較式防浮點誤差。沒證明就用,是 WA 的溫床。
  • 掃描線:排序 → 一條線掃過去 → 沿途維護關鍵量。最多重疊區間靠 +1/−1 事件、線段聯集靠依左端邊掃邊併。
  • 動態規劃(重頭戲):背包的 0/1 與完全只差迴圈方向、LIS 的 $O(n\log n)$ 是 DP × 二分搜、區間 DP 枚舉切點且依長度填表、樹形 DP 藏在 DFS 回傳值、狀態壓縮 DP 用位元當集合解小 $n$ 指數題。
  • 決策地圖:從資料範圍 $n$ 出發一路往下問。$n$ 會說話——它常在你想清楚演算法前就先替你縮小了範式選項。

如果這一章只帶走一句話,讓它是這句:解題的第一步不是「想演算法」,而是「認題型」。 當你能一眼從題敘和資料範圍裡讀出骨架——「這是區間排程」「這是偽裝的背包」「$n=16$,八成是狀態壓縮」—— 剩下的,不過是把你早已練熟的套路填進去。而練熟套路的唯一辦法,就是打開評測系統,一題一題地刷下去。 下一章,我們會把這些範式裡最後一塊重要拼圖——樹上的演算法與樹形 DP——好好補齊。
