怎麼用這份模擬卷自我檢測

題庫不是拿來「看」的,是拿來「考」的。把這份附錄當成正式的一場小考,你才測得出真實水準。建議照下面這條循環走一遍,效果遠勝於邊看題邊看答案:

  1. 計時、先遮答案:挑一卷,照標示的建議作答時間設好鬧鐘(基礎 60 分、進階 70 分、挑戰 80 分)。先不要展開任何詳解,把全部 30 題當成一張完整考卷,在紙上寫下你選的選項。觀念題的功力是「在腦中執行程式」算出來的,不是看出來的。

  3. 回對應章複習:每一題詳解都標了「對應觀念」。把答錯的題目的對應章節抄下來,回到本書相應章節(基礎觀念在前面幾部、遞迴在第 3 章、複雜度在第 5 章、資料結構在後續各章)重讀,再回頭把那題親手重算一次,直到不看詳解也能算對。
  4. 循序加難:基礎卷穩了(答對 24 題以上)再進攻進階卷,進階卷穩了再挑戰挑戰卷。三卷由淺入深,範圍逐步擴大到指標、作用域、進位、複雜度與除錯。

考前檢查清單(上場前掃一遍)

觀念題最常見的失分,不是不會,而是「算到一半踩到陷阱」。每做一題前,先問自己這幾件事:

  • 整數除法:兩邊都是整數時 / 會無條件捨去(7/2=3);型別轉換發生在「遇到浮點數那一刻」,且運算由左而右。
  • 運算子優先序:* / % 先於 + -;同級由左而右。不確定就在心裡補上括號。
  • switch 有沒有 break:漏寫就會「落下(fall-through)」繼續執行下一個 case。
  • 迴圈變數被偷改:迴圈內若又動了控制變數,實際步進量會疊加(for 的 +1 再加上body裡的 +1)。
  • 遞迴從基底往上算:先找 base case,再用一張「參數 → 回傳值」的表一層層往上填,別在腦中硬背。
  • 傳值 vs 傳址:傳值(int x)改不到外面;要透過指標(int *x)解參考才會真的改寫。
  • 陣列邊界與索引:迴圈上界是 < 還是 <=?用「值當索引」「索引位移」時,務必在紙上畫格子慢慢填。
  • 作用域:同名的區域變數會「遮蔽」全域變數;函式內改的是哪一個?

✏️ 挑戰卷 第 26 題 選擇

int f1(int a[], int value) {        // 線性搜尋
    int r_value = -1; int i = 0;
    while (i < 100) {
        if (a[i] == value) { r_value = i; break; }
        i = i + 1;
    }
    return r_value;
}
int f2(int a[], int value) {        // 二分搜尋
    int r_value = -1; int low = 0, high = 99; int mid;
    while (low <= high) {
        mid = (low + high)/2;
        if (a[mid] == value) { r_value = mid; break; }
        else if (a[mid] < value) { low = mid + 1; }
        else { high = mid - 1; }
    }
    return r_value;
}

答案:(D)

3k+1=100→k=33,線性搜尋的 if 執行 34 次;二分搜尋 mid 依序 49,24,36,30,33,共 5 次。
