Vim 與編輯器

你這輩子在電腦前敲的字,有一大半不是寫文章,而是「編輯」既有的文字。當編輯的速度能跟上思考的速度,寫程式會變成一件流暢得近乎暢快的事——這,就是 Vim 想給你的禮物。

一個你每天都在做、卻沒注意到的動作

打開你最近寫的任何一段程式,回想一下:你真的是「一個字一個字從頭打到尾」的嗎?幾乎不是。 你大部分的時間,是在修改既有的文字——把這個變數改名、把那一行刪掉、 把這個函式搬到上面、在每一行結尾補個分號、把雙引號換成單引號…… 寫程式這件事,與其說是「打字」,不如說是「編輯」。

既然編輯佔了這麼大的比重,那麼「編輯的效率」就直接決定了你工作的順暢度。 想像一下:你腦中已經清楚知道「我要把游標跳到那個括號裡、刪掉裡面所有東西、重打」, 但你的手卻得先去摸滑鼠、瞄準、拖曳反白、再回鍵盤——這中間的每一次「手離開鍵盤」, 都是一次思緒被打斷的微小停頓。Vim 的整套哲學,就是要消滅這些停頓, 讓「編輯的速度」對齊「思考的速度」。

🔑
這一章在教的不是一個工具,是一種「手感」

Vim 的指令初看像天書,但它們其實組成了一套有文法、可組合的「編輯語言」。 一旦你學會這套語言,它會內化成肌肉記憶,讓你用想的就能改文字。 這種手感,就算你日後不用 Vim 本體,也會跟著你一輩子——因為幾乎每個現代編輯器都提供「Vim 模式」。

2020 年代了,為什麼還要學 Vim?

我們得先誠實面對房間裡的大象:現在是 VS Code、JetBrains、Cursor 的時代。 這些現代編輯器有滑鼠、有圖形介面、有自動補全、有 AI 助手,好用得不得了。 那為什麼還有這麼多資深工程師,死守著一個 1991 年的文字編輯器?有三個很實際的理由。

一、Vim 無所不在

當你 SSH 連線到一台遠端伺服器要改個設定檔,那裡沒有 VS Code,沒有圖形介面, 只有一個黑漆漆的終端機。這時候你能用的,幾乎一定是 vi 或 vim—— 它是 POSIX 標準的一部分,幾乎每一台 Unix/Linux 機器、每一個 Docker 容器裡都預裝著。 當你打 git commit 卻忘了加 -m,Git 也會直接把你丟進 Vim 寫提交訊息。 會基本的 Vim,等於拿到一張「在任何機器上都能編輯文字」的通行證。 你可以不把它當主力,但你逃不掉它。

二、雙手永遠不離開鍵盤的家

Vim 的設計圍繞著一個信念:移動滑鼠是浪費生命。它讓你用鍵盤完成一切—— 移動、選取、刪除、複製、跳轉、搜尋。你的雙手停在鍵盤中央, 靠的不是去夠那些遙遠的方向鍵或滑鼠,而是用 h j k l 這四個就在你右手食指到小指底下的鍵來移動。每一個微小的「不用移開手」, 累積起來就是整天的流暢。

三、可組合的「編輯語言」

這才是 Vim 真正的殺手鐧,我們整章的重頭戲。Vim 的指令不是一堆要死記的快捷鍵, 而是像語言一樣可以自由組合的「動詞 + 名詞」。學會少數幾個動詞和名詞, 你就能拼出成千上萬種精準的編輯動作——這一點,我們稍後會講透。

⚖️
誠實的話:你不是非用 Vim 不可

別誤會,這一章不是要你拋棄 VS Code。對大型專案的開發、除錯、重構, 現代 IDE 的整合體驗通常更好。比較務實的現實是:很多人最後選擇「兩全其美」—— 在 VS Code 或 JetBrains 裡裝一個 Vim 模式外掛,享受 Vim 的編輯手感, 又保留 IDE 的強大功能。所以學 Vim 的鍵位,從來不是白學的。這個話題我們留到本章最後再談。

編輯器的小小演化史

要理解 Vim 為什麼長這樣,得稍微回到它出生的年代。今天的 Vim,是一條長達半世紀的演化鏈的最新一環。

  • ed(1969):最早的編輯器之一,出自 Unix 之父 Ken Thompson 之手。 它是「行編輯器(line editor)」——那個年代沒有螢幕,只有電傳打字機(像打字機般把字印在紙上), 所以你看不到整個檔案,只能用指令說「印出第 5 行」「把第 5 行換成這句」。聽起來很痛苦,但它是一切的起點。 Vim 那些以冒號開頭的指令(:w、:s),正是 ed 的血脈。
  • vi(1976):Bill Joy 寫出了 vi(visual editor,「視覺化」編輯器), 終於能在螢幕上看到整個檔案、用游標四處移動。它奠定了「模式編輯」的設計,影響至今。
  • Vim(1991):Bram Moolenaar 打造了 Vim——名字是「Vi IMproved」 (改良版的 vi)。它在 vi 的基礎上加了多層復原、語法高亮、外掛系統、巨集等等,成為今天事實上的標準。
  • Neovim(2014):一群人覺得 Vim 的程式碼老舊難以維護, 於是分支(fork)出 Neovim,現代化它的內部架構、內建 Lua 腳本與 LSP(語言伺服器)支援。 今天許多人講「Vim」其實用的是 Neovim,兩者鍵位幾乎完全相容。
📜
編輯器聖戰:Vim vs Emacs

在 Vim 的平行宇宙裡,還有另一個傳奇編輯器 Emacs(由 Richard Stallman 於 1976 年起打造)。 如果說 Vim 的哲學是「小巧、模式化、用組合鍵說一句句編輯指令」,那 Emacs 的哲學就是 「一個能用 Lisp 程式無限擴充的作業系統,順便能編輯文字」(這是個玩笑,但也半真)。 兩派信徒纏鬥數十年,史稱「編輯器聖戰(editor war)」。 結論是:它們都好得不得了,選哪個都不會錯——真正重要的是「挑一個,然後把它練到精」。 本章選 Vim,純粹因為它最普及、最容易在任何地方派上用場。

核心觀念:Vim 是一個「模式編輯器」

如果這一章你只能記住一件事,就是這個:Vim 是「模式編輯器(modal editor)」。 這四個字,是理解 Vim 的鑰匙,也是初學者最大的撞牆點。

在一般的編輯器(像記事本或 VS Code)裡,鍵盤永遠在做同一件事:你按 j, 螢幕上就出現一個 j。它是「無模式(modeless)」的。 但 Vim 不一樣:同一個按鍵,在不同的「模式」下,意義完全不同。 在 Vim 的「一般模式」下按 j,游標會往下移動一行,而不是打出一個 j。 為什麼要這麼「反人類」?因為這正是 Vim 速度的來源——

既然你大部分時間都在「編輯」而非「輸入」,Vim 乾脆讓鍵盤預設處於「下指令」的狀態。 於是整個鍵盤的每一個鍵,都不再只是「印一個字母」,而是化身為一個個強大的編輯指令。 只有當你真的要「打字」時,才切換到「輸入模式」。這個取捨,就是模式編輯的全部精神。

初學 Vim 時你一定會搞不清楚自己在哪個模式、按鍵怎麼亂跳。 這時候的萬靈丹是:用力按 Esc(甚至連按兩下),你就會回到安全的一般模式, 重新開始。把 Esc 想成電玩裡的「回到存檔點」。許多人還會把 Ctrl+[ 當成 Esc 的替身(兩者等效),因為它離手指更近。

第一課:在 Vim 裡「活下來」

網路上有個流傳已久的笑話:「怎麼產生一串隨機字串?讓新手試著離開 Vim 就好。」 這玩笑點出了初學者的頭號恐懼——進得去,出不來。所以我們的第一課, 不談花俏技巧,只談兩件保命的事:怎麼打字,以及怎麼全身而退。

進入輸入模式:開始打字

打開 Vim 後你在「一般模式」,這時亂打鍵盤只會觸發一堆奇怪的指令。 要開始打字,先按下面任一個鍵進入輸入模式(你會看到畫面左下角出現 -- INSERT --):

按鍵意思從哪裡開始打字
iinsert在游標前插入(最常用)
aappend在游標後追加
oopen在下面開一個新行並進入輸入
O大寫 O在上面開一個新行
AAppend跳到行尾開始打字
IInsert跳到行首(第一個非空白字元)開始打字

打完字後,按 Esc 回到一般模式。記住這個節奏:「i 打字 Esc」是你跟 Vim 的基本對話。

存檔與離開:四個保命指令

存檔和離開,要先回到一般模式(按 Esc),再按 : 進入命令列模式, 最後打指令並按 Enter:

指令意思
:wwrite——存檔(但不離開)
:qquit——離開(若有未存的更動會拒絕離開)
:wq存檔並離開(也可寫成 :x,或一般模式下按 ZZ)
:q!不存檔、強制離開——把所有更動丟掉
🆘
救命指令:Esc 然後 :q! 然後 Enter

當你不小心把檔案改得亂七八糟、想假裝什麼都沒發生過時,這就是你的逃生門: 先 Esc 確保回到一般模式,輸入 :q! 再按 Enter。 那個 驚嘆號 ! 是 Vim 的「我意已決,別攔我」——它會放棄所有未存的更動直接離開。 把這串按鍵背起來,你就再也不會被困在 Vim 裡了。

移動的藝術:motion

在一般模式,整個鍵盤都是你的移動工具。新手的第一個震撼,就是不要用方向鍵, 而是用右手指尖下的 h j k l:

但逐字移動太慢了。Vim 真正的威力,是「跳躍式」的移動——以「詞」「行」「段落」為單位:

按鍵移動到記憶法
w / b下一個詞首 / 上一個詞首word / back
e當前詞的詞尾end
0 / $行首(第 0 欄) / 行尾$ 在正規表達式裡也代表「結尾」
^行內第一個非空白字元—
gg / G檔案開頭 / 檔案結尾Go
{ / }上一段 / 下一段(以空行分段)—
Ctrl-d / Ctrl-u向下半頁 / 向上半頁down / up
fx / tx跳到本行下一個字元 x 上 / x 前一格find / till
%跳到配對的括號 () [] {}—
42G 或 :42跳到第 42 行數字 + G

注意最後一列那個小細節:許多 motion 前面可以加數字,代表「重複幾次」。 3w 是「往前跳 3 個詞」,5j 是「往下 5 行」。 這個「數字 + 動作」的模式,馬上要在下一節大放異彩。

Vim 的精髓:一套「編輯語言」

現在我們來到本章的核心,也是 Vim 之所以是 Vim 的理由。前面學的指令如果只是「一堆要背的快捷鍵」, 那 Vim 不過是個比較難用的記事本。但它的真正設計,是一套有文法的語言:

操作子(動詞) + 移動或文字物件(名詞) = 一個編輯動作

就像自然語言裡「動詞 + 受詞」可以組出無限的句子,Vim 讓你用少數幾個操作子(operator) 搭配各種移動(motion),拼出成千上萬種精準的編輯。先認識三個最重要的操作子:

操作子動詞意思
ddelete——刪除(剪下)
cchange——刪除後直接進入輸入模式(改寫)
yyank——複製(yank 是 Vim 對「複製」的稱呼)

關鍵在於:這三個動詞,可以接上任何你學過的 motion。「d(刪除) + w(一個詞)」 就是 dw「刪除一個詞」;「y(複製) + $(到行尾)」 就是 y$「複製到行尾」。你不需要為「刪一個詞」「刪到行尾」「刪兩行」 各記一個快捷鍵——你只要學會動詞和名詞,剩下的是排列組合:

組合讀作效果
dwdelete word從游標刪到下一個詞首
d$(= D)delete to end of line從游標刪到行尾
dddelete line刪掉整行(動詞疊兩次 = 作用於整行)
2dd2 delete line刪掉兩行
cwchange word刪掉一個詞並開始打字(改字超常用)
yyyank line複製整行
dfxdelete find x刪到(含)本行下一個 x
d}delete paragraph刪到下一個空行(整段)
🔑
為什麼這套「可組合」的設計這麼強?

因為它的學習效益是相乘而非相加的。假設你會 5 個操作子和 10 個 motion, 你不是會 15 個指令,而是會 5 × 10 = 50 種編輯動作。 每多學一個新 motion,你立刻就「免費」獲得它跟所有操作子的組合。 這就是為什麼 Vim 老手能用幾下按鍵,做到別人要反白拖曳老半天的事—— 他們不是在「記快捷鍵」,他們是在「講一種語言」。

文字物件:更聰明的「名詞」

motion 描述的是「從這裡到那裡」。但很多時候你想操作的是一個結構性的整塊: 「括號裡的全部東西」「這對引號之間的字串」「這個 HTML 標籤的內容」—— 而且你不在乎游標現在站在這塊的哪個位置。這就是文字物件(text object)的舞台。

文字物件由兩個字母組成,開頭是 i(inner,裡面)或 a(around,連邊界一起):

文字物件選取範圍
iw / aw一個詞 / 一個詞連同旁邊的空白
i( / a(小括號內的內容 / 連括號本身(i)、ib 等同)
i{ / a{大括號內 / 連大括號(程式區塊超好用)
i" / a"雙引號內的字串 / 連引號
it / atHTML/XML 標tag 內的內容 / 連標籤
ip / ap一個段落 / 連後面的空行

把文字物件接到操作子後面,魔法就發生了。最經典的組合,是改寫括號或引號裡的內容, 而且游標在哪都行——你不用先跑到開頭:

組合讀作效果
ciwchange inner word不管游標在詞的哪裡,整個詞刪掉重打
di(delete inner ()刪掉括號裡的全部內容(括號留著)
ci"change inner "清空引號內的字串並開始重打
yi{yank inner {}複製整個大括號區塊的內容
dapdelete a paragraph刪掉整段(連同後面空行)

體會一下 ci" 的暢快:游標只要落在 "任意位置" 這對引號中間的任何地方, 按下這三鍵,引號內的字就瞬間清空、游標就位、等你重打。這在別的編輯器裡, 是「雙擊選字→確認範圍→刪除→點進去」好幾個動作。一旦你習慣了文字物件,就再也回不去了。

複製、貼上、復原、重做、重複

有了編輯能力,還需要這些日常的配套動作:

複製貼上(yank & put)

Vim 把複製叫 yank(y),貼上叫 put(p)。 很重要的一點:你刪掉(d)的東西其實是「剪下」, 會跟複製一樣存進暫存區,所以刪掉後一樣能貼回來。

按鍵效果
yy複製整行
yiw複製游標所在的詞
p / P貼在游標後 / 游標前(整行的話是下方 / 上方)

復原與重做(undo & redo)

在一般模式按 u 復原(undo),按 Ctrl-r 重做(redo)。 Vim 的復原是多層的,可以一路退回去——不像某些老編輯器只能退一步。

「重複」這把瑞士刀:.

一般模式下的句點 .,意思是「把我剛剛做的那個編輯,再做一次」。 這看似不起眼,卻是 Vim 效率的另一根支柱。假設你剛用 cw新詞Esc 把一個詞換掉了,接下來你只要把游標移到下一個要改的詞,按一下 .,就重播同樣的修改。 「找下一個 + 按點」的節奏,能讓你飛快地做完一連串相似的小修改。

搜尋與取代

搜尋用 /(往下找)或 ?(往上找):

按鍵效果
/關鍵字 Enter向下搜尋「關鍵字」
n / N跳到下一個 / 上一個符合處(next)
*搜尋游標目前所在的那個詞(超方便)

取代則用命令列模式的 :s(substitute),它繼承自老祖宗 ed,語法值得記熟: 

:s/old/new/        把「本行第一個」old 換成 new
:s/old/new/g       把「本行所有」old 換成 new(g = global,全部)
:%s/old/new/g      把「整個檔案所有」old 換成 new(% = 全部行)
:%s/old/new/gc     同上,但每次取代前都問你 yes/no(c = confirm)

:%s/old/new/g 是你會用上千次的指令:全檔把某個詞換成另一個。 記憶法:% 是「整份」、s 是「換」、結尾的 g 是「一行裡也全部換,不要只換第一個」。

視覺模式與巨集

視覺模式(Visual mode):先選,再動手

有時候你就是想「先把範圍反白出來看清楚,再決定要做什麼」。這時用視覺模式: 按 v 進入後,用 motion 移動來擴大選取範圍,選好了再按一個操作子(d 刪、y 複製、c 改)。

按鍵哪一種視覺模式
v逐字元選取
V逐整行選取(改多行縮排、刪多行的好幫手)
Ctrl-v區塊(欄)選取——能選出一個矩形範圍,做多行「直行」編輯

舉個多行操作的例子:想把連續 5 行往右縮排,按 V 再按 4j 選滿 5 行, 然後按 > 就整塊縮排。區塊模式 Ctrl-v 更神:它能選出一個直的矩形, 例如在好幾行的同一欄位置,一次插入相同文字。

巨集(macro):把一串操作錄起來重播

當「. 重複」不夠用、你想重播的是「一整套組合動作」時,就用巨集: 按 q 再接一個字母(例如 qa)開始錄製到暫存器 a,做你要重複的動作, 再按一次 q 停止;之後用 @a 重播一次,10@a 重播十次。 巨集是 Vim 用來處理「大量重複性編輯」的終極武器——下面的練習裡你會親手錄一個。

讓 Vim 變成你的:.vimrc

Vim 的行為,由你家目錄下的設定檔 ~/.vimrc(Neovim 則是 ~/.config/nvim/init.vim 或 init.lua)決定。 每次 Vim 啟動都會讀它。一份對新手友善的起手式長這樣(" 開頭的是註解): 

" ---------- 顯示 ----------
set number              " 顯示行號
set relativenumber      " 相對行號:讓 5j、3k 這種跳轉一眼就能算出數字
syntax on               " 語法高亮
set cursorline          " 高亮游標所在行

" ---------- 縮排 ----------
set expandtab           " 把 Tab 變成空白
set shiftwidth=4        " 自動縮排寬度 4 格
set tabstop=4           " 一個 Tab 顯示 4 格
set autoindent          " 換行時沿用上一行的縮排

" ---------- 搜尋 ----------
set incsearch           " 邊打邊即時跳到符合處
set hlsearch            " 高亮所有符合的字
set ignorecase          " 搜尋忽略大小寫……
set smartcase           " ……但你打了大寫字母時就變回區分大小寫

把上面這段存進 ~/.vimrc,重開 Vim 就生效了。 關於外掛(plugin):Vim 有龐大的外掛生態(用 vim-plug、lazy.nvim 等管理器安裝), 能加上模糊檔案搜尋、檔案樹、Git 整合、甚至透過 LSP 做到媲美 IDE 的自動補全—— 但這些是後話,初學階段請先把內建功能練熟,別急著掉進「無止盡調整設定檔」的兔子洞。

⚠️
小心「設定檔成癮」

Vim 社群有個著名的陷阱:新手裝了 Vim,結果花了整個週末在調整 .vimrc、 安裝五十個外掛、追求「完美的設定」,卻一行正事都沒做。記住:工具是用來完成工作的,不是用來把玩的。 先用最陽春的設定把編輯練熟,等你真的感覺到某個痛點,再去找對應的設定或外掛來解決它。

該去哪裡練?一份 30 天的務實計畫

Vim 不是「讀懂」就會的,它是「練出來」的肌肉記憶。好消息是,有兩個絕佳的練習場:

  • vimtutor:Vim 內建的官方互動教學。打開終端機,直接輸入 vimtutor 就會啟動, 約 30 分鐘,手把手帶你練過所有基礎。這是每個 Vim 學習者該做的第一件事,而且它就在你電腦裡,免安裝。
  • Vim Adventures:一個把 Vim 鍵位變成闖關解謎的網頁小遊戲,用玩的把 hjkl、 wbe、f 等練成反射。對覺得 vimtutor 太枯燥的人很有效。

但工具再好,沒有循序漸進的計畫也容易半途而廢。給你一條務實的 30 天路線:

週次目標練什麼
第 1 週活下來跑完一次 vimtutor;練熟 i/Esc、hjkl、:w :q :q!。目標:能用 Vim 寫完一個小檔案而不抓狂。
第 2 週跑得快跳躍式移動 w b e 0 $ gg G;基本編輯 dd yy p u。逼自己不用方向鍵。
第 3 週說「語言」「操作子 + motion」的組合;文字物件 ciw di( ci";搜尋 / 與取代 :%s;善用 .。
第 4 週進階與安家視覺模式多行操作、錄一個巨集;寫好自己的 .vimrc;嘗試 1–2 個外掛。把 Vim 設成日常編輯器之一。
💡
學習 Vim 的唯一秘訣:每次只加一招

不要試圖一天記住五十個指令——你會全部忘光。正確的方法是:每隔幾天,只刻意練一個新指令, 直到它變成不用想的反射,再加下一個。前兩週你的速度甚至會比用滑鼠還慢,這完全正常, 幾乎每個人都會經歷這段「先變慢才變快」的低谷。撐過去,曲線就會往上翹。

成熟的觀點:工具是手段,不是身分

最後,讓我們把視野拉高。網路上「Vim vs VS Code」的爭論永遠吵不完,但一個成熟的工程師, 心裡很清楚一件事:編輯器只是手段,把事情做好才是目的。沒有人因為「用 Vim」 而自動變成更好的工程師,正如沒有人因為用了名牌鋼筆就自動寫出好文章。

所以,與其在門派之間二選一,絕大多數人選擇了最聰明的折衷方案: 在現代 IDE 裡啟用「Vim 模式」。VS Code 有 VSCodeVim 外掛, JetBrains 系列有 IdeaVim,它們讓你一邊享受 Vim 那套飛快的鍵盤編輯語言, 一邊保有 IDE 的除錯器、重構、Git 整合與 AI 補全。這就是為什麼本章一開始說: 學 Vim 的鍵位,永遠不會白學——你學的不是某個程式,而是一種跟著你走到哪用到哪的「編輯母語」。

🔑
一句話總結這一章的態度

挑一個編輯器,把它練到「閉著眼睛都能用」的程度,然後就別再糾結了。 你省下來的精力,該拿去解決真正的問題。Vim、Emacs、VS Code—— 能讓「你的手」跟上「你的腦」的,就是好工具。

動手練:這一章是「操作題」

這一章沒辦法用讀的學會,只能用練的。下面的練習都是操作題:解答會寫出具體的按鍵序列。 強烈建議你真的打開一個 Vim(vim練習.txt),把每一題敲一遍。

✏️ 練習 17.1 操作

游標停在這一行的某個地方,行的內容是 name = "Alice"。 請用最少的按鍵,把名字改成 Bob(只動引號裡的內容,游標一開始不一定在引號內)。

看解答

如果游標已經在引號之間,直接:ci" 然後打 Bob 再按 Esc。 三個按鍵就清空了引號內容並讓你重打——這就是文字物件的威力,而且游標在引號內哪裡都行。

如果游標還在行首(引號外),最穩的做法是先跳到引號上再用文字物件: f"(跳到本行第一個引號)接著 ci"、打 Bob、Esc。 關鍵心法:想改「某個成對符號裡的內容」,反射動作就是 ci( / ci" / ci{。

✏️ 練習 17.2 操作

有一行程式 foo(a, b, c, d),你想把括號裡的所有參數都刪掉,變成 foo()。 游標可能停在括號內的任何位置。怎麼一下完成?

看解答

di((delete inner parens)。它會刪掉括號裡面的全部內容, 但保留括號本身,瞬間得到 foo()。同理,di{ 清空大括號區塊、 di" 清空字串。

對照組:如果用 da((delete around), 連括號一起刪掉,結果會變成 foo。差別就在 i(inner,只裡面)和 a(around,連邊界)。

✏️ 練習 17.3 操作

檔案某處有個單字 config,你想把它複製一份,貼到下面好幾行的某個位置。 寫出「複製這個詞 → 移動到目標 → 貼上」的按鍵序列。

看解答

第一步,游標移到 config 上(在詞的任何位置都行),按 yiw (yank inner word)複製整個詞。

第二步,移動到你要貼的目標位置(例如用 j 往下幾行、用 f 或 w 移到定點)。

第三步,按 p 貼在游標後,或 P 貼在游標前。

小提醒:yiw 複製的是「乾淨的詞」;若用 yaw 則會連旁邊一個空白也複製進去,貼上時要留意。

✏️ 練習 17.4 操作

你接手一個檔案,裡面把變數叫 tmp,你想把整個檔案裡所有的 tmp 都改成 buffer。一條指令搞定。

看解答

先確定在一般模式(按 Esc),然後輸入:

:%s/tmp/buffer/g

再按 Enter。拆解一下:% = 對整個檔案的每一行、s = substitute(取代)、 /tmp/buffer/ = 把 tmp 換成 buffer、結尾的 g = 一行內所有出現都換(不只第一個)。

進階:如果你想「換之前每個都先問我一下」,在最後再加一個 c: :%s/tmp/buffer/gc,它會在每個符合處停下來問 y/n。 另外要小心:這樣會把 tmpfile 裡的 tmp 也換掉;若只想換「完整的詞」, 用單字邊界::%s/\<tmp\>/buffer/g。

✏️ 練習 17.5 挑戰

你有一段 C 程式,連續好幾行都忘了加結尾的分號。請寫出「在每一行結尾補上一個分號」的兩種做法: (a) 用一條取代指令;(b) 用一個巨集。

看解答

(a) 取代指令(最快):利用正規表達式裡 $ 代表「行尾」這個特性, 把「行尾」換成「分號」:

:%s/$/;/

它會在每一行的結尾各插入一個 ;。如果只想對選取的某幾行做,先用 V 選好範圍, 再打 :s/$/;/(Vim 會自動把範圍填成 :'<,'>)。

(b) 巨集:把游標放在第一行,然後:

  • qa —— 開始錄製到暫存器 a
  • A;Esc —— 跳到行尾、補上分號、回一般模式
  • j —— 移到下一行(這樣重播時才會逐行往下)
  • q —— 停止錄製

現在暫存器 a 裡存著「補分號 + 下一行」這套動作。要對接下來 9 行重播,按 9@a 即可。這就是巨集處理重複性編輯的威力—— 錄一次,重播無數次。

✏️ 練習 17.6 挑戰

不准用滑鼠、也不准用方向鍵:游標現在在檔案最上方,你想跳到第 100 行的行尾, 並在那裡開始打字。最短按鍵序列是什麼?

看解答

100G 跳到第 100 行,接著 A 跳到行尾並進入輸入模式。 合起來是 100G 然後 A。

A(大寫的 append)= 「移到行尾 + 進入輸入」一鍵完成,比先按 $ 再按 a 更省。 這題想讓你體會:Vim 裡常常有「把兩三個動作合成一鍵」的捷徑,多認識它們,效率就再進一階。

把這一章串起來

我們從「你每天都在編輯文字」這個觀察出發,認識了 Vim 這套讓手跟上腦的編輯哲學。回顧路標:

  • 為什麼學 Vim:它無所不在(伺服器、Git、終端)、讓雙手不離鍵盤、而且提供一套可組合的編輯語言。即使主力用 VS Code,Vim 鍵位也透過「Vim 模式」處處可用。
  • 模式編輯是核心:一般 / 輸入 / 視覺 / 命令列。同一個鍵在不同模式意義不同,迷路就按 Esc。
  • 活命三件事:i 打字、Esc 回家、:wq 存檔離開(救命的是 :q!)。
  • 編輯語言:操作子(d c y)+ motion 或文字物件(iw i( i"),能相乘出無數精準編輯。這是 Vim 真正的靈魂。
  • 配套:p 貼上、u/Ctrl-r 復原重做、. 重複、/ 搜尋、:%s 取代、視覺模式選取、巨集重播。
  • 態度:工具是手段。挑一個、練到精、別糾結;省下的精力拿去解決真正的問題。

這一章是第四部「CS 工具」的一站。你已經學會在任何機器上自在地編輯文字了。 接下來這套工具鏈裡,最重要的夥伴就是「版本控制」——它記住你每一次的修改、 讓你能安心地實驗與回退、也讓一群人能協作同一份程式。下一章,我們就來認識 Git。
