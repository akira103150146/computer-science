程式的另一半:為什麼需要資料結構

前三章,我們一直在談「動作」:函式、迴圈、條件、遞迴。但程式從來不是只有動作—— 動作總得作用在某些資料上。一支真實的程式,一半是「怎麼做」(演算法), 另一半是「資料長什麼樣、被裝在什麼容器裡」。這後半邊,就是資料結構(data structure)。

想像你是個木工。工具箱裡有各式各樣的收納:一條長長的釘條,釘子整齊排成一排, 要第幾根伸手就抽;一個貼著標籤的抽屜櫃,寫著「螺絲」「墊片」, 要哪類零件直接拉開對應抽屜;還有一個篩網,把重複的、雜的東西濾掉,只留下不重複的一份。 同樣是「裝東西」,但裝的方式不同,拿取的效率天差地遠。

資料結構就是程式設計裡的這些收納容器。釘條像串列, 標籤抽屜像字典,篩網像集合。 把資料丟進「對的」容器,很多原本要寫一大段迴圈的事,會瞬間變成一行; 丟進「錯的」容器,你會寫出又長又慢又容易出錯的程式。

🔑
核心觀念

有一句在工程師之間流傳的話,出自 Linux 之父 Linus Torvalds: 「爛程式設計師擔心程式碼;好程式設計師擔心資料結構與它們的關係。」 先想清楚「資料該怎麼擺」,程式碼往往就自己浮現了。

Python 內建了四個威力強大、你每天都會用到的容器: 串列(list)、元組(tuple)、 字典(dict)、集合(set), 外加一個其實也是序列的字串(string)。這一章,我們一個一個認識它們的個性。

串列(list):會伸縮的釘條

串列(list)是 Python 裡最常用的容器:一排有順序的元素, 而且它會自動伸縮——你可以隨時往裡面塞東西、拿東西出來。用一對中括號 [] 建立: 

fruits = ["蘋果", "香蕉", "芭樂", "西瓜"]
empty = []                       # 空串列
mixed = [1, "嗨", 3.14, True]    # 串列可以混裝不同型別(但通常不建議)

print(len(fruits))   # 4 —— len() 回傳長度

索引(index):為什麼從 0 開始?

要拿出串列裡的某一個元素,用索引(index)——也就是它的「位置編號」。 關鍵來了:Python(以及幾乎所有主流語言)的索引從 0 開始, 不是從 1。第一個元素是 fruits[0],第二個是 fruits[1]。 

fruits = ["蘋果", "香蕉", "芭樂", "西瓜"]
print(fruits[0])   # 蘋果  ← 第一個
print(fruits[1])   # 香蕉
print(fruits[3])   # 西瓜  ← 最後一個(長度 4,最大索引是 3)

為什麼是 0?這不是電腦在刁難你。把索引想成「從串列開頭往後走幾步」: 第一個元素就在開頭,走 0 步就到,所以是 0;第二個要往後走 1 步,所以是 1。 這種「偏移量(offset)」的觀點,在我們講到 C 的記憶體時會變得無比自然—— 因為索引其實就是「離起點多遠」。

⚠️
差一錯誤(off-by-one error)

長度為 n 的串列,合法索引是 0 到 n-1。 很多人習慣性地寫 fruits[len(fruits)] 想拿最後一個,結果拿到 IndexError—— 因為最後一個是 fruits[len(fruits) - 1]。這種「差一」的錯誤,是初學者最常見的 bug 之一。

負索引(negative index):從尾巴數回來

Python 有個貼心的設計:用負索引可以從尾端往回數。 -1 是最後一個,-2 是倒數第二個,以此類推。 再也不用寫醜醜的 fruits[len(fruits) - 1] 了: 

fruits = ["蘋果", "香蕉", "芭樂", "西瓜"]
print(fruits[-1])   # 西瓜  ← 最後一個
print(fruits[-2])   # 芭樂  ← 倒數第二個

切片(slicing):一次取一段

索引一次只能取一個元素;切片(slicing)讓你一次取出一整段。 語法是 a[start:stop:step],三個數字都可省略,分別代表「從哪開始、到哪結束、每幾個取一個」。 最重要的一條規則:包含 start,但不包含 stop(數學上叫「左閉右開區間」)。 

a = [10, 20, 30, 40, 50]

print(a[1:4])    # [20, 30, 40]   ← 從索引 1 到 3(不含 4!)
print(a[:3])     # [10, 20, 30]   ← 省略 start = 從頭開始
print(a[2:])     # [30, 40, 50]   ← 省略 stop  = 一直到尾
print(a[:])      # [10,20,30,40,50] ← 整個複製一份(很常用!)
print(a[::2])    # [10, 30, 50]   ← step=2,每隔一個取
print(a[::-1])   # [50,40,30,20,10] ← step=-1,整串反轉!

為什麼「不含 stop」?回頭看圖 4-1 那排灰色的切片邊界。切片其實不是在數「元素」, 而是在數元素之間的「縫」。a[1:4] 的意思是「從第 1 道縫切到第 4 道縫」, 中間夾住的正好是索引 1、2、3 三個元素。這個設計有個漂亮的好處: a[:k] 和 a[k:] 剛好把串列切成不重不漏的兩半, 而且 a[1:4] 取出的元素個數正好是 4 - 1 = 3 個,心算很方便。

💡
兩個超好用的切片慣用法

a[::-1] 是反轉序列最簡潔的寫法(對字串也通!); a[:] 則是「淺拷貝」一份新串列的經典手法——等一下你就會明白為什麼有時候非複製不可。

串列的常用方法

串列是「可變的」(下一節詳談),所以它有一票方法可以「就地」改動自己:

方法 / 語法作用範例
a.append(x)在尾端加一個元素[1,2].append(3) → [1,2,3]
a.insert(i, x)在索引 i 處插入[1,3].insert(1,2) → [1,2,3]
a.pop()取出並移除最後一個(可給索引)[1,2,3].pop() → 回傳 3
a.remove(x)移除第一個值為 x 的元素[1,2,2].remove(2) → [1,2]
a.sort()就地排序(由小到大)[3,1,2].sort() → [1,2,3]
len(a)元素個數len([1,2,3]) → 3
x in a檢查 x 在不在裡面(回傳布林)2 in [1,2,3] → True
 
tasks = ["買菜", "寫程式"]
tasks.append("運動")          # ["買菜", "寫程式", "運動"]
tasks.insert(0, "刷牙")       # ["刷牙", "買菜", "寫程式", "運動"]
done = tasks.pop()            # done = "運動";tasks 少了最後一項
tasks.remove("買菜")          # 把「買菜」拿掉
print("寫程式" in tasks)      # True
tasks.sort()                  # 依字串順序就地排序
⚠️
sort() 回傳 None!

初學者常寫 tasks = tasks.sort(),結果 tasks 變成 None。 因為 sort() 是「就地」改動原串列,它不回傳新串列(回傳 None)。 如果你要的是「排好的新串列、保留原本的」,請用內建函式 sorted(tasks)。 「就地改動 vs 回傳新物件」是 Python 一個反覆出現的主題,要養成分辨的習慣。

可變 vs 不可變:初學者的最大坑

接下來這一節,可能是整章最重要、也最容易被忽略的一節。它解釋了一種會讓你抓破頭的 bug: 「我明明只動了 b,為什麼 a 也跟著變了?」

Python 的物件分兩種。可變(mutable)的物件,內容可以原地修改, 例如 list、dict、set。不可變(immutable)的物件,一旦建立就永遠不能改, 例如 tuple、str、以及數字。

變數是「名字」,不是「盒子」

要搞懂這件事,你得先丟掉一個常見的錯誤心像。很多教材說「變數是裝資料的盒子」—— 對 Python 來說,這是錯的。在 Python 裡, 變數比較像貼在物件上的「便利貼標籤」。當你寫 b = a, 你不是把 a 的內容「複製」一份到 b,而是把另一張標籤 b 貼到了同一個物件上。這就叫別名(aliasing)——同一個東西有了兩個名字。 

a = [1, 2, 3]
b = a            # b 不是複製,而是「同一個物件的第二個名字」
b.append(4)      # 透過 b 改動那個 list

print(b)         # [1, 2, 3, 4]
print(a)         # [1, 2, 3, 4]  ← a 也變了!因為 a 和 b 是同一個東西

這就是惡名昭彰的淺拷貝陷阱(aliasing trap)。如果你真的想要一份獨立的副本, 改了不會牽連原本的,要明確地「複製」: 

a = [1, 2, 3]
b = a[:]          # 切片複製一份新 list(也可用 a.copy() 或 list(a))
b.append(4)

print(b)          # [1, 2, 3, 4]
print(a)          # [1, 2, 3]    ← a 沒被動到,因為 b 現在是另一個物件

is 與 == 不是同一回事

既然「同一個物件」這麼關鍵,Python 給了你一個專門的問法。 == 問的是「內容一不一樣?」; is 問的是「是不是同一個物件?」(背後比的是身分,可用 id() 看)。 

a = [1, 2, 3]
b = a[:]          # 內容相同,但是另一個物件
c = a             # 同一個物件的別名

print(a == b)     # True  ← 內容一樣
print(a is b)     # False ← 但不是同一個物件
print(a is c)     # True  ← c 和 a 是同一個物件
ℹ️
什麼時候該用 is?

絕大多數時候你想比的是「內容」,用 == 就對了。is 幾乎只在一個地方用: 檢查某個東西是不是 None,慣例寫法是 if x is None:。 除此之外,別拿 is 去比數字或字串,那會踩到難解釋的小陷阱。

那不可變物件呢?因為它本來就不能改,所以別名與否根本無所謂—— 沒有人能「偷偷改動」它而牽連到別人。這正是不可變型別讓人安心的地方,也是下一位主角的賣點。

元組(tuple):上了鎖的串列

元組(tuple)幾乎就是「不能改的串列」。用小括號 () 建立(其實逗號才是關鍵)。 它有順序、可索引、可切片,但一旦建立就不能新增、刪除或修改裡面的元素。 

point = (3, 5)            # 一個座標
print(point[0])          # 3
print(point[1])          # 5

point[0] = 9             # ❌ TypeError:tuple 不支援修改

既然不能改,那它有什麼用?恰恰因為「不能改」,它有兩個串列辦不到的本事。

本事一:可以當字典的鍵

等一下會講的字典,它的「鍵(key)」必須是不可變的。所以 list 不能當鍵,但 tuple 可以—— 這讓你能用「一對座標」當作查表的鍵: 

board = {}
board[(0, 0)] = "X"      # 用座標 (0,0) 當鍵,沒問題
board[(1, 2)] = "O"
print(board[(0, 0)])     # X

本事二:多重指派與一行交換

元組支援漂亮的多重指派(multiple assignment):一行同時拆出好幾個變數。 最經典的應用,是不需要暫存變數就能交換(swap)兩個值: 

a, b = 10, 20            # 同時指派:a=10, b=20
a, b = b, a              # 一行交換!現在 a=20, b=10

# 其他語言通常得這樣寫:
# temp = a;  a = b;  b = temp

name, age, city = ("小明", 18, "台北")   # 拆解一個元組
print(age)               # 18

a, b = b, a 為什麼可行?因為等號右邊 b, a 會先被打包成一個元組 (20, 10), 然後一次解開分配給左邊。這個小技巧你之後寫排序、走訪會用到無數次。

🔑
list vs tuple,怎麼選?

一個好用的直覺:如果這是「一堆同類、數量會變」的東西(一串待辦事項、一批分數),用 list; 如果這是「一筆固定欄位的紀錄」(一個座標、一筆 (姓名, 年齡)),用 tuple。 tuple 的不可變,本身就是一種「我保證這筆資料不會被偷改」的承諾。

字串(string):其實是一串字元

你早就在用字串了,但現在你有了新視角:字串(string)就是一個由字元組成的序列, 而且它和 tuple 一樣是不可變的。這代表——你對 list 學的索引與切片, 對字串原封不動全部適用: 

s = "Python"
print(s[0])      # P    ← 索引,和 list 一樣 0-based
print(s[-1])     # n    ← 負索引
print(s[0:3])    # Pyt  ← 切片
print(s[::-1])   # nohtyP  ← 反轉字串!
print(len(s))    # 6

因為不可變,你不能寫 s[0] = "J"(會報錯)。 想「改」字串,實際上是「產生一個新字串」。所有字串方法都是這個原則:它們回傳新字串,原字串不動。

最常用的字串方法

方法作用範例(回傳值)
.split(sep)依分隔符切成串列"a,b,c".split(",") → ["a","b","c"]
sep.join(list)把字串串列黏成一個字串"-".join(["a","b"]) → "a-b"
.strip()去掉頭尾的空白" hi ".strip() → "hi"
.replace(a, b)把所有 a 換成 b"aaa".replace("a","b") → "bbb"
.find(x)回傳 x 首次出現的索引(找不到回 -1)"hello".find("l") → 2
.upper() / .lower()轉大寫 / 小寫"Hi".lower() → "hi"

split 與 join 是一對搭檔,處理文字時形影不離:一個把字串「拆」成串列, 一個把串列「合」回字串。記住它們,你就能應付大半的文字處理。 

line = "  小明,18,台北  "
parts = line.strip().split(",")   # ["小明", "18", "台北"]
print(parts[0])                   # 小明
print(" / ".join(parts))          # 小明 / 18 / 台北

f-string:把變數塞進字串

要把變數的值「填」進一段文字,最現代、最好讀的方法是 f-string: 在字串前面加一個 f,然後在大括號 {} 裡直接放變數或運算式。 

name, score = "小美", 92
print(f"{name} 考了 {score} 分")          # 小美 考了 92 分
print(f"{name} 的平均是 {score / 2:.1f}")  # 小美 的平均是 46.0  (:.1f 取小數一位)

# 老式寫法(仍會在舊程式碼看到),效果類似:
print("{} 考了 {} 分".format(name, score))

走訪字串

字串既然是序列,就能直接用 for 一個字元一個字元走訪:

vowels = "aeiou"
count = 0
for ch in "programming":     # 逐字元走訪,不需要索引
    if ch in vowels:         # in 用在字串上 = 「是不是其中一個字元」
        count += 1
print(count)                 # 3  (o, a, i)
💡
字串拼接的小陷阱

用 + 把很多字串一塊一塊接起來(尤其在迴圈裡)效率很差,因為每次 + 都會產生一個全新的字串(別忘了字串不可變)。要組合大量片段, 請先把它們收進一個 list,最後再用 "".join(list) 一次黏好。

字典(dict):貼著標籤的抽屜櫃

前面所有容器,你都得用「位置編號」去找東西。但生活中我們更常用「名字」找: 查電話簿是用「人名」查號碼,不是「第 37 個號碼」。這就是字典(dict)的天職—— 它儲存一組組鍵值對(key-value pair):給它一個鍵(key), 它立刻吐出對應的值(value)。 

ages = {"小明": 18, "小美": 17, "阿強": 19}   # 用大括號,key: value

print(ages["小美"])      # 17   ← 用鍵查值,不是用位置
ages["小華"] = 20        # 新增一對
ages["小明"] = 21        # 鍵已存在 → 更新它的值
print(len(ages))         # 4

為什麼字典查找「快得不可思議」

在串列裡找一個值,電腦得從頭一個一個比對,平均要看一半的元素——資料越多越慢($O(n)$)。 字典卻幾乎不管裡面有多少資料,查找都一樣快(平均 $O(1)$,常數時間)。這怎麼做到的?

字典背後是一種叫雜湊表(hash table)的精巧結構。當你存入一個鍵, Python 會先用一個雜湊函式(hash function)把這個鍵換算成一個數字, 再用這個數字直接算出「該把值放在記憶體的哪一格」。下次要查,同一個鍵算出同一個數字、 指向同一格——一步到位,不必逐一比對。

安全查值:get 與 in

直接用 ages["不存在的人"] 查一個不存在的鍵,會直接報 KeyError 當掉。 兩個更安全的做法:用 in 先問「有沒有這個鍵」,或用 .get() 給一個「查不到時的預設值」。 

ages = {"小明": 18, "小美": 17}

print("小明" in ages)        # True   ← in 檢查的是「鍵」存不存在
print(ages.get("阿強"))      # None   ← 查不到不報錯,回傳 None
print(ages.get("阿強", 0))   # 0      ← 查不到就回傳指定的預設值 0

走訪字典

ages = {"小明": 18, "小美": 17, "阿強": 19}

for name in ages:               # 直接走訪 = 走訪「鍵」
    print(name)

for name, age in ages.items():  # .items() 同時拿到鍵和值(最常用)
    print(f"{name} 今年 {age} 歲")

print(list(ages.keys()))        # ['小明', '小美', '阿強']  所有鍵
print(list(ages.values()))      # [18, 17, 19]            所有值

殺手級應用:計數(統計詞頻)

字典最經典的用途之一,是計數:統計每樣東西出現幾次。 搭配 .get(key, 0) 的慣用法特別優雅——「拿目前的次數(沒有就當 0),加一,存回去」: 

text = "蘋果 香蕉 蘋果 西瓜 香蕉 蘋果"
counts = {}
for word in text.split():
    counts[word] = counts.get(word, 0) + 1

print(counts)   # {'蘋果': 3, '香蕉': 2, '西瓜': 1}

短短四行,就完成了詞頻統計。試著想像如果只用 list 要怎麼寫——你會發現難寫又慢得多。 這就是「選對容器,程式簡單一半」最好的範例。

集合(set):只認得「不重複」的篩網

集合(set)是數學集合的直接化身:一堆不重複、無順序的元素。 它的兩個招牌能力是去重與快速成員測試(和字典一樣是 $O(1)$,因為背後也是雜湊)。 用大括號 {} 建立,或用 set() 把其他容器轉成集合。 

nums = [1, 2, 2, 3, 3, 3]
unique = set(nums)       # {1, 2, 3}  ← 重複的自動消失!
print(3 in unique)       # True       ← 成員測試極快

unique.add(4)            # 加一個
unique.add(2)            # 2 已存在 → 沒有任何效果(集合不重複)
print(len(unique))       # 4

集合運算:交集、聯集、差集

數學課本上的集合符號,在 Python 裡都有對應的運算子:

運算數學符號Python意思
聯集(union)$A \cup B$a | b在 A 或 B 裡的(全部合起來)
交集(intersection)$A \cap B$a & b同時在 A 和 B 裡的(共同)
差集(difference)$A \setminus B$a - b在 A 但不在 B 裡的
 
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

print(a | b)   # {1, 2, 3, 4, 5, 6}  聯集
print(a & b)   # {3, 4}              交集(兩邊共同有的)
print(a - b)   # {1, 2}              差集(只在 a 裡的)

「找兩群人的共同好友」「比對兩份名單的差異」「過濾掉已經看過的網址」—— 這些任務用集合運算寫,往往就是一行。

串列生成式:更 Pythonic 的寫法

你常常需要「從一個序列,造出一個新序列」。傳統寫法是開一個空 list,用迴圈一個個 append: 

# 求 1~5 的平方
squares = []
for n in range(1, 6):
    squares.append(n * n)
print(squares)   # [1, 4, 9, 16, 25]

Python 提供一種更精煉的寫法,叫串列生成式(list comprehension), 把「迴圈 + append」濃縮成一行,讀起來幾乎像在念數學:「對每個 n,給我 n*n」: 

squares = [n * n for n in range(1, 6)]
print(squares)   # [1, 4, 9, 16, 25]

還能在後面加一個 if 做過濾,只留下符合條件的元素:

# 只要偶數的平方
evens = [n * n for n in range(1, 11) if n % 2 == 0]
print(evens)     # [4, 16, 36, 64, 100]

# 對字串也通:抽出一句話裡的所有母音
text = "computer science"
vowels = [ch for ch in text if ch in "aeiou"]
print(vowels)    # ['o', 'u', 'e', 'i', 'e', 'e']
⚠️
別為了炫技而濫用

生成式很美,但塞太多邏輯(巢狀迴圈、複雜條件)會變成沒人讀得懂的天書。 原則:一行裝得下、一眼看得懂就用生成式;否則乖乖寫成多行迴圈,可讀性永遠優先。

順帶一提:字典與集合生成式

同樣的語法,把中括號換成大括號,就能生出字典或集合:

# 字典生成式:數字 → 它的平方
sq_map = {n: n * n for n in range(1, 6)}
print(sq_map)    # {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}

# 集合生成式:一段文字裡有哪些不重複的字元
chars = {ch for ch in "banana"}
print(chars)     # {'b', 'a', 'n'}

該用哪一個?一張決策表

五個容器各有個性。下次卡住不知道該用哪個時,問自己三個問題: 「順序重要嗎?」「之後要改它嗎?」「需要用名字查、還是只想去重?」 這張表幫你做決定:

容器有序?可變?可重複?查找速度典型用途
list [] 是是是 取索引 $O(1)$
找值 $O(n)$		 一串會增減、有順序的東西(待辦、分數)
tuple () 是否是 取索引 $O(1)$ 固定欄位的一筆紀錄、可當 dict 的鍵
str "" 是否是 $O(n)$ 文字(本質是字元序列)
dict {k:v} 保留插入序*是鍵不可重複 平均 $O(1)$ 用「名字」查「值」、計數、查表
set {} 否是否(自動去重) 平均 $O(1)$ 去重、成員測試、交集/聯集/差集

* 從 Python 3.7 起,dict 會「記住」鍵被插入的順序;但它本質上仍是靠雜湊查找,別把它當成「有序容器」來依賴順序排序。

🔑
一句話記住它們的個性

list 是會伸縮的隊伍,tuple 是上了鎖的隊伍, str 是一排字元,dict 是貼標籤的抽屜,set 是只認得不重複的篩網。

與 C 對照:Python 的 list 底下是什麼?

Python 的 list 用起來像魔法:能混裝型別、能無限長大、能 append。 但天下沒有白吃的午餐。掀開抽象的蓋子,在 C 這個更貼近機器的語言裡, 最接近 list 的東西是陣列(array)——而陣列「死板」得多:

  • 固定大小:宣告時就要決定長度,事後不能說長就長。
  • 同一型別:一個 int 陣列裡,每一格都必須是 int,不能混。
  • 連續記憶體:所有元素緊挨著、一個接一個排在記憶體裡。
  • 沒有內建字典:C 標準庫裡沒有現成的 dict,要嘛自己用陣列實作雜湊表,要嘛用第三方函式庫。
#include <stdio.h>

int main(void) {
    /* 宣告一個長度固定為 5、且每格都是 int 的陣列 */
    int scores[5] = {90, 85, 70, 60, 100};
    int n = 5;
    int sum = 0;

    /* 用索引走訪 —— 注意索引一樣從 0 到 n-1,和 Python 完全一致 */
    for (int i = 0; i < n; i++) {
        sum += scores[i];
    }

    printf("平均 = %.1f\n", (double)sum / n);   /* 平均 = 81.0 */
    /* scores[5] = 50;  ← 越界!C 不會幫你檢查,可能默默寫壞別人的記憶體 */
    return 0;
}

看出 C 為什麼把索引從 0 開始了嗎?因為 scores[i] 在底層的意思,字面上就是 「從陣列起點往後移 i 格」。scores[0] 就是起點本身(移 0 格)。 Python 沿用了這個從 C、甚至更早的語言傳承下來的慣例。

🔑
Python 的 list 其實是「動態陣列」

那 Python 的 list 為什麼能一直長大?因為它底層用的是動態陣列(dynamic array): 先偷偷配置一塊比目前需要還大的連續空間,空間用完了就另開一塊更大的、把舊資料整批搬過去。 這也是為什麼 append 通常很快、但偶爾會「卡一下」(正在搬家)。 至於 dict 與 set 為什麼能 $O(1)$ 查找、動態陣列搬家的成本怎麼算(攤銷分析), 這些迷人的底層真相,我們留到第六部「演算法與資料結構」好好拆解。

動手練習

讀十遍不如寫一遍。下面六題難度遞增,請務必先自己寫,卡住再看解答。

✏️ 練習 4.1 實作

用切片寫一個函式 reverse(s),回傳字串 s 反轉後的結果。 例如 reverse("hello") 應回傳 "olleh"。請不要用迴圈。

看解答

關鍵是切片的第三個參數 step。把它設成 -1,就會從尾巴往前每次走一格,自然得到反轉:

def reverse(s):
    return s[::-1]

print(reverse("hello"))   # olleh
print(reverse("資料結構")) # 構結料資

同樣的招式對 list 也通:[1,2,3][::-1] 會得到 [3,2,1]。一個切片,通吃所有序列。

✏️ 練習 4.2 實作

寫一個函式 word_count(text),輸入一段以空白分隔的文字, 回傳一個字典,記錄每個單字出現的次數。例如 word_count("a b a c a b") 應回傳 {'a': 3, 'b': 2, 'c': 1}。

看解答

這就是本章「計數」慣用法的直接應用。用 split() 切詞,用 get(word, 0) + 1 累加:

def word_count(text):
    counts = {}
    for word in text.split():
        counts[word] = counts.get(word, 0) + 1
    return counts

print(word_count("a b a c a b"))   # {'a': 3, 'b': 2, 'c': 1}

進階:標準庫其實有現成工具 collections.Counter, Counter(text.split()) 一行就搞定。但先親手寫過一次,你才會真正懂它在做什麼。

✏️ 練習 4.3 實作

給你兩個 list,寫一個函式 common(a, b),回傳同時出現在兩者中的元素(不重複)。 例如 common([1,2,2,3,4], [2,4,4,6]) 應得到 {2, 4}。提示:想想哪個容器天生擅長這件事。

看解答

把兩個 list 都轉成 set,然後取交集 &。重複的元素會在轉 set 時自動消失:

def common(a, b):
    return set(a) & set(b)

print(common([1, 2, 2, 3, 4], [2, 4, 4, 6]))   # {2, 4}

如果硬用兩層巢狀迴圈逐一比對,程式碼又長、又是 $O(n \times m)$ 的慢速度。 選對容器,一行解決——這正是本章想讓你體會的事。

✏️ 練習 4.4 思考

寫一個函式 dedup(items),移除 list 裡的重複元素, 但必須保持原本的出現順序。例如 dedup([3, 1, 3, 2, 1, 4]) 應回傳 [3, 1, 2, 4]。

(陷阱:直接 list(set(items)) 雖然能去重,但 set 沒有順序,結果次序會亂掉。)

看解答

訣竅是同時用兩個容器:用一個 set 來「快速記住看過誰」,用一個 list 來「保留順序」。 這完美展示了不同容器各司其職:

def dedup(items):
    seen = set()          # 負責「這個看過了嗎?」的 O(1) 查詢
    result = []           # 負責保留順序
    for x in items:
        if x not in seen:
            seen.add(x)
            result.append(x)
    return result

print(dedup([3, 1, 3, 2, 1, 4]))   # [3, 1, 2, 4]

小知識:其實 list(dict.fromkeys(items)) 也能達成同樣效果—— 因為現代 dict 會記住插入順序、且鍵不重複。兩種寫法都值得認識。

✏️ 練習 4.5 思考

把下面這段用迴圈寫的程式,改寫成一行串列生成式。它的功能是: 取出 1 到 20 裡所有「3 的倍數」,並把每個都乘以 10。

result = []
for n in range(1, 21):
    if n % 3 == 0:
        result.append(n * 10)
看解答

生成式的結構是 [運算式 for 變數 in 序列 if 條件]。把迴圈的三個部位對號入座:

result = [n * 10 for n in range(1, 21) if n % 3 == 0]
print(result)   # [30, 60, 90, 120, 150, 180]

對照著看:n * 10 是 append 進去的東西、for n in range(1, 21) 是迴圈本體、 if n % 3 == 0 是過濾條件。學會這個對應關係,你就能自在地在兩種寫法間轉換。

✏️ 練習 4.6 挑戰

用一個 dict 實作一個簡單的電話簿,支援四種基本操作 (這四種操作合稱 CRUD:Create 新增、Read 查詢、Update 更新、Delete 刪除)。 請寫出這四個函式,並妥善處理「查詢/刪除不存在的人」這種邊界情況。

看解答

字典天生就是電話簿:鍵是人名,值是號碼。重點在於用 in 或 .get() 優雅地處理「找不到」的情況,而不是讓程式直接 KeyError 當掉:

phonebook = {}

def create(name, number):           # C:新增(或覆蓋)
    phonebook[name] = number
    print(f"已儲存 {name}: {number}")

def read(name):                     # R:查詢
    return phonebook.get(name, "查無此人")

def update(name, number):           # U:更新(僅在存在時)
    if name in phonebook:
        phonebook[name] = number
        print(f"已更新 {name}: {number}")
    else:
        print(f"找不到 {name},無法更新")

def delete(name):                   # D:刪除
    if name in phonebook:
        del phonebook[name]
        print(f"已刪除 {name}")
    else:
        print(f"找不到 {name},無法刪除")

# 試跑一輪 CRUD
create("小明", "0912-345-678")
print(read("小明"))     # 0912-345-678
update("小明", "0900-000-000")
print(read("小明"))     # 0900-000-000
delete("小明")
print(read("小明"))     # 查無此人

注意刪除用的是 del phonebook[name]。 你也可以改用 phonebook.pop(name, None),它在鍵不存在時不會報錯—— 又一次,「安全地處理找不到」是寫 dict 程式時永遠要記得的事。

把這一章串起來

我們打開了 Python 的工具箱,認識了五個你天天都會用到的容器。把這趟巡禮收進腦海:

  • 資料結構是程式的另一半。選對容器,常常比想出聰明演算法還重要。
  • list:有序、可變、會伸縮。掌握索引(0-based)、負索引、切片 a[start:stop:step](左閉右開),以及 append/pop/sort 等方法。
  • 可變 vs 不可變是頭號大坑。變數是「標籤」不是「盒子」,b = a 只是別名;要獨立副本得明確複製。== 比內容,is 比身分。
  • tuple:不可變的 list,能當 dict 的鍵,還支援 a, b = b, a 這種漂亮的交換。
  • str:本質是不可變的字元序列,索引切片全適用;split/join/strip/replace 與 f-string 是文字處理的主力。
  • dict:鍵值對,雜湊表撐腰,平均 $O(1)$ 查找;get、in、items() 與「計數」慣用法要熟。
  • set:去重與快速成員測試,交集 &、聯集 |、差集 - 直通數學集合。
  • 串列生成式讓「迴圈+append」濃縮成優雅的一行,但別為了炫技犧牲可讀性。
  • 對照 C:陣列固定大小、同型別、連續記憶體、沒有內建 dict;Python 的 list 其實是底層會自動搬家的「動態陣列」。

現在你既會「動作」(前三章),也會「資料」(這一章)。兩者合起來, 你已經能寫出真正解決問題的程式了。下一章,我們要談一件讓你的程式從「能跑」邁向「可靠」的事: 演算法思維、除錯與測試——當程式不照你想的跑時,該怎麼系統化地把它修好。
