專修軌道的起點:從 Python 到 C++

歡迎來到本書的「專修軌道」。前面五十幾章,我們一路用 Python 建立觀念—— 從第 2 章的變數、迴圈、型別,到第 4 章的 list/dict/set,再到第 29 章親手打造堆疊、佇列、雜湊表與堆積。 Python 語法乾淨、回饋即時,是「把想法說清楚」的最佳語言,讓你能專注在演算法本身,而不被語法綁手綁腳。

但當你走進程式競賽(competitive programming)的世界——ZeroJudge、Codeforces、AtCoder、 以及台灣的 APCS 與資訊學科能力競賽——你會發現一件事:幾乎所有人都在寫 C++。為什麼?

🔑
為什麼競賽選 C++

一是快。C++ 直接編譯成機器碼,跑同一個 $O(n)$ 迴圈,常比 Python 快上數十倍。 評測機(Online Judge)有嚴格的時間限制(常是 1 秒),很多題目用 Python 寫對了邏輯也會 TLE(Time Limit Exceeded), 換成 C++ 卻輕鬆通過。
二是 STL。C++ 標準函式庫(Standard Template Library)內建了一整櫃調校到極致的容器與演算法—— 動態陣列、雜湊表、平衡樹、堆積、排序……拿來就用,而且複雜度有保證。這正是第 29 章 那些你親手寫過的結構的「工業級成品」。

好消息是:你已經懂的那些觀念——變數、迴圈、條件、函式、list、dict、set、堆積——一個都不會浪費。 這一章不教你「重新學程式」,而是教你「換一種腔調說同一件事」。我們會時時把 C++ 的寫法和你熟悉的 Python 並排, 讓你看見:底下其實是同一個靈魂。準備好,我們發動引擎。

第一支 C++ 程式:Hello, Judge

Python 裡印出一行字只要 print("Hello World!")。C++ 則需要一點「儀式」: 

#include <bits/stdc++.h>   // 萬用標頭檔:一行引入競賽會用到的所有功能
using namespace std;       // 使用 std 命名空間,之後 cout、sort 等不用加 std::

int main() {              // 程式從 main 函式開始執行
    cout << "Hello World!\n";   // cout 配 << 輸出;\n 是換行
    return 0;            // 回傳 0 表示正常結束(可省略)
}

逐行拆解,每一行都對應一個 C++ 的核心概念:

  • #include <bits/stdc++.h>:C++ 的功能分散在許多標頭檔(header)裡—— cout 在 <iostream>、sort 在 <algorithm>。一個個 include 很煩, 所以競賽圈用這個「萬用標頭檔」一次把它們全部拉進來。它不是 C++ 標準的一部分(正式比賽外的環境可能沒有), 但在 ZeroJudge、Codeforces 等都能用,直接記起來就對了。
  • using namespace std:標準庫的東西都住在 std 這個命名空間(namespace)裡, 全名是 std::cout、std::sort。加了這一行,就像跟程式說「我接下來叫的人都在 std 這班」, 於是可以直接寫 cout,省去到處打 std::。
  • int main() { ... }:C++ 程式的進入點。和 Python 把程式碼直接攤在檔案裡不同, C++ 所有可執行的指令都得待在某個函式裡,而 main 就是那個「被第一個呼叫」的函式。 大括號 { } 框出它的範圍。
  • cout << "...":輸出。cout(character output)後面接 << (想成箭頭把東西「送進」輸出),再接要印的東西。字串用雙引號。
  • \n:換行字元。在 Online Judge 上,輸出是逐字元嚴格比對的—— 少一個換行,整題就算錯。請養成「輸出結尾一定換行」的肌肉記憶。
  • 分號 ;:C++ 每一個陳述句結尾都要分號。漏掉會編譯錯誤(Compile Error)。
ℹ️
C++ 不靠縮排,靠大括號與分號

還記得第 2 章說 Python 用縮排定義區塊嗎?C++ 不一樣: 區塊用大括號 { } 框起來,陳述句用分號 ; 結尾, 換行與空白對它幾乎沒有意義(你甚至能把整支程式擠成一行)。縮排在 C++ 裡純粹是給人看的禮貌, 不影響程式邏輯——但請務必還是好好縮排,不然三個月後的你會恨現在的你。

型別:這次,大小得你自己管

Python 的整數是「任意精度」的,你算 2 ** 100 它連眉頭都不皺。C++ 不一樣: 每個變數都得先宣告型別,而且型別決定了它佔多少位元、能裝多大的數。這是 C++ 快的代價, 也是初學者最容易栽跟頭的地方。先看怎麼宣告: 

int a = 1;            // 整數,最常用
long long b = 2;      // 更大範圍的整數
double d = 2.564;     // 浮點數(帶小數)
char c = 'a';         // 單一字元,用單引號
bool flag = true;     // 布林值:true 或 false
string s = "ABC";     // 字串,用雙引號

int 會溢位,long long 是你的救生圈

int 用 32 個位元儲存,範圍約是 $-2^{31}$ 到 $2^{31}-1$,也就是大約正負 21 億。 一旦運算結果超出這個範圍,就會溢位(overflow)——數字像時鐘繞圈一樣,從最大值繞回最小值, 給你一個莫名其妙的負數。這正是第 2 章那段 C 溢位範例的真實重演。 

int a = 2000000000;       // 20 億,還在 int 範圍內
cout << a + a << "\n";    // 40 億超出 int!印出負數(溢位)

long long b = 2000000000;
cout << b + b << "\n";    // 4000000000,正確,因為 long long 有 64 位元
⚠️
你「絕對」會因為忘了用 long long 而 WA

這是競賽新手最常見的致命傷之一。題目說「答案可能很大」、要算「總和」、「乘積」、「階乘」、 「前綴和」時,先問自己:會不會超過 21 億?只要有疑慮,就用 long long。
更陰險的是:運算過程中只要某一刻超出 int,即使最終答案在 int 範圍內,也會錯。 例如 (a + b) * c / 2,中間的 (a+b)*c 可能就爆了。常見補救:讓其中一個運算元變成 long long(例如寫 1LL * a * b),整條運算式就會以 64 位元進行。

double 有誤差,別用 == 比較

double 存小數,但和第 2 章講的 IEEE 754 一樣,它不精確—— 100.0 * 99.0 可能算成 9899.9999999。所以兩個浮點數要判斷「相等」, 絕不能用 ==,而要看它們的差的絕對值夠不夠小(這個小門檻習慣叫 eps,常取 1e-9): 

double a, b;
cin >> a >> b;
const double eps = 1e-9;
if (abs(a - b) < eps) cout << "相等\n";   // 正確:看差距,不用 ==
// 競賽中能用整數就用整數,double 只在真的需要小數時才登場(別碰 float,誤差太大)

char 其實是一個小整數

char 存一個字元,但它的真身是一個整數——存的是該字元的 ASCII 編碼。 'a' 是 97、'A' 是 65、'0' 是 48。因為這樣,字元可以做算術,這在處理字串時超好用: 

char c = 'a';
cout << (int)c << "\n";       // 97,把字元轉成它的 ASCII 值
cout << char(99) << "\n";     // 'c',把數字轉回字元

// 因為 'a'~'z'、'A'~'Z'、'0'~'9' 在 ASCII 表都是連續的:
char up = 'D';
char low = up + ('a' - 'A');  // 大寫轉小寫:加上兩段編碼的差
cout << low << "\n";          // 'd'
int digit = '7' - '0';        // 字元 '7' 轉成數字 7(超常用!)
cout << digit << "\n";        // 7
💡
型別速查:對照你熟悉的 Python

Python 的 int 永不溢位 → C++ 用 int(快但會溢位)/long long(範圍大)。 Python 的 float → C++ 的 double。Python 的 str → C++ 的 string。 Python 沒有獨立的「字元」型別(單字元也是字串)→ C++ 的 char 是個獨立、且能當數字用的型別。 C++ 還多了一件你以前不用操心的事:變數使用前必須宣告型別。

輸入輸出:cin/cout、加速,與多筆測資

競賽程式幾乎都是「讀進輸入 → 算 → 印出答案」。C++ 的標準作法是 cin(輸入)配 cout(輸出): 

int a;
double c;
string d;
cin >> a >> c >> d;            // 用 >>(箭頭朝變數)依序讀入,空白/換行都會自動跳過
cout << a << " " << c << " " << d << "\n";   // 用 << 依序輸出

注意方向:輸入用 cin >>(把資料送進變數),輸出用 cout <<(把資料送出去)。 箭頭方向相反,別搞混。cin 讀數字時會自動略過前面的空白與換行,所以輸入排成一行或分成好幾行都讀得到。

要讀「整行含空白的字串」,cin >> 會在空白處停下來,這時改用 getline: 

string line;
getline(cin, line);   // 讀整行,直到換行為止(含空白)

// 陷阱:先 cin >> n 再 getline,會讀到殘留的換行而拿到空字串
int n;
cin >> n;
cin.ignore();         // 用 cin.ignore() 吃掉那個殘留的換行
getline(cin, line);   // 這樣才讀得到真正的下一行

競賽 I/O 加速:一行讓 cin/cout 飛起來

cin/cout 為了和 C 的 scanf/printf 保持同步,預設其實偏慢。 當輸入量達到數十萬、上百萬筆,這個慢會直接讓你 TLE。解法是在 main 開頭加上這一行「咒語」: 

int main() {
    ios_base::sync_with_stdio(false);   // 關掉與 C stdio 的同步,大幅加速
    cin.tie(nullptr);                   // 解除 cin/cout 的綁定,再快一點
    // ... 你的程式 ...
}
⚠️
加速的兩個搭配:別用 endl

加速要兩件事一起做才完整。除了上面那行,還要避免用 endl 換行。 endl 等於 '\n' 再加一次強制清空緩衝區(flush),那個 flush 很慢。 所以本章自始至終都用 "\n" 或 '\n' 換行,不用 endl。
另外提一下 scanf/printf:這是從 C 繼承來的另一套 I/O,速度也很快, 格式用 %d(int)、%lld(long long)、%s(字串)指定。 兩套擇一用熟即可——本章統一用加速過的 cin/cout。

多筆測資與 EOF:while 的兩個經典寫法

很多題目不是只有一組輸入,而是「一直讀到檔案結尾(EOF)」或「先給測資數 t,做 t 次」。 這兩種模式各有一個慣用寫法,記起來受用無窮: 

// 模式一:讀到 EOF 為止。cin >> n 讀成功回傳 true,讀到結尾回傳 false
int n;
long long sum = 0;
while (cin >> n) {        // 不斷讀整數,直到沒有輸入
    sum += n;
}
cout << sum << "\n";

// 模式二:第一行給測資數 t,接著做 t 次
int t;
cin >> t;
while (t--) {            // t-- 先判斷 t 是否非 0,再把 t 減 1,剛好做 t 次
    int a, b;
    cin >> a >> b;
    cout << a + b << "\n";
}

while (cin >> n) 利用了「讀取成功與否」這個布林結果;while (t--) 則利用了 「整數非 0 即為真」與後置遞減的時機。兩者都是競賽裡天天見到的句型。

運算子:四則、位元與三元

基本的四則運算 + - * / 和 Python 幾乎一樣,也遵守先乘除後加減。但有兩個差異要記牢: 

int a = 5, b = 3;
cout << a / b << "\n";       // 1!兩個整數相除,結果只取整數部分(不是 1.666)
cout << a % b << "\n";       // 2,取餘數(模)
cout << 1.0 * a / b << "\n"; // 1.66667,先乘 1.0 變浮點數,才會得到小數

a += 5;   // 等同 a = a + 5;  也有 -= *= /= %=
a++;      // 等同 a += 1
a--;      // 等同 a -= 1
⚠️
整數除法與「鏈式比較」兩個雷

整數 / 整數 = 整數:在 Python 裡 5 / 3 是 1.666,但 C++ 的 5 / 3 是 1 (小數被丟掉)。要小數,讓運算元至少一個是浮點數(乘 1.0)。
不能寫 a < b < c:這在 Python 是合法且符合直覺的,但 C++ 會把它解讀成 (a < b) < c——先算出 a < b 的 true(1)或 false(0),再拿這個 0/1 跟 c 比, 結果完全不是你想的。要判斷 $a < b < c$,得寫成 a < b && b < c。而且這不會編譯錯誤,是個安靜的 bug!

邏輯運算用符號而非單字:Python 的 and/or/not 在 C++ 是 &&/||/!。

位元運算:直接操作 0 與 1

電腦裡的整數本就是一串二進位位元。位元運算(bitwise operation)讓你逐位操作這些 0 與 1, 速度極快,是許多技巧(狀態壓縮、集合運算、快速冪)的基石。四個基本運算加上位移:

運算子名稱規則(逐位)範例(6=110, 5=101)
&AND(且)兩位都 1 才得 16 & 5 = 100 = 4
|OR(或)任一位是 1 就得 16 | 5 = 111 = 7
^XOR(互斥或)兩位不同才得 16 ^ 5 = 011 = 3
~NOT(取反)每位 0↔1 對調~5 = -6
<<左移所有位往左移,等同乘 21 << 3 = 8
>>右移所有位往右移,等同整數除以 220 >> 1 = 10
⛔
^ 不是次方!

從 Python(或數學)來的人最大的陷阱:^ 在 C++ 是 XOR,不是次方! C++ 根本沒有次方運算子。2 ^ 10 不是 1024,而是 2 XOR 10 = 8。整數次方要自己用迴圈乘, 或用(有誤差的)pow 函式處理小數次方。

XOR 有兩個迷人的性質,讓它在演算法(尤其加密、找落單的數)裡無所不在: 

// 性質一:任何數 XOR 自己 = 0
cout << (7 ^ 7) << "\n";        // 0
// 性質二:XOR 可逆。a ^ x ^ x = a(XOR 同一個數兩次會還原)
int a = 42, x = 17;
cout << ((a ^ x) ^ x) << "\n";  // 42,被 x 加密又解密回來了
// 經典應用:一陣列裡每個數都出現兩次,只有一個出現一次,全部 XOR 起來就剩它

三元運算子:一行版的 if-else

當你只是想「依條件二選一」,三元運算子(ternary operator) 條件 ? A : B 比寫整個 if-else 簡潔。它的意思是:條件成立就取 A,否則取 B。對應 Python 的 A if 條件 else B: 

int a = 5, b = 9;
int bigger = (a > b) ? a : b;   // a 比 b 大就取 a,否則取 b → 9
cout << (a % 2 == 0 ? "偶數" : "奇數") << "\n";   // 直接印出判斷結果

流程控制與陣列

if / else if / else 和迴圈,概念和 Python 完全一樣,只是換成大括號與分號: 

int a, b;
cin >> a >> b;
if (a > b) {
    cout << a << " is bigger\n";
} else if (a == b) {           // else if 串接,等同 Python 的 elif
    cout << "equal\n";
} else {
    cout << b << " is bigger\n";
}

// for 迴圈:三段式(初始; 條件; 每圈結尾動作)
for (int i = 0; i < 10; i++) {   // i 從 0 跑到 9,共 10 次
    cout << i << " ";
}

// while 迴圈
int n = 5;
while (n > 0) {
    n--;
}
// break 跳出整個迴圈、continue 跳過本圈剩下的部分,和 Python 一樣

C++ 的 for 是「三段式」:for (初始化; 繼續條件; 每圈結尾)。 for (int i = 0; i < n; i++) 就是最標準的「跑 n 次」,對應 Python 的 for i in range(n)。

陣列:固定大小的連續格子

C++ 最基本的容器是陣列(array):一排連續的、同型別的格子。宣告時要指定大小,而且大小固定、不能改: 

int arr[100];              // 100 個 int 的陣列,索引 0 ~ 99
int b[4] = {1, 4, 2, 3};   // 宣告時直接給值
cout << b[0] << " " << b[3] << "\n";   // 1 3,用 [索引] 取值,從 0 開始
int grid[100][100];        // 二維陣列(矩陣)
💡
競賽慣例:大陣列開在全域

競賽裡通常把大陣列宣告在 main 外面(全域),並照題目上限直接開夠大(例如 int arr[200005];)。 原因有二:一,全域變數自動初始化為 0(區域陣列不初始化的話裡面是垃圾值); 二,全域記憶體區比區域(堆疊)大得多,能開到數千萬而不會崩潰。
另一個鐵則:不要存取超出範圍的索引(如大小 5 的陣列碰 arr[5] 或 arr[-1])。 C++ 不會幫你檢查,輕則拿到垃圾值,重則 RE(Runtime Error)——而且有時本機跑起來「碰巧」正常, 上傳卻錯,極難抓。

函式與傳參:傳值 vs 傳參考 &

函式的寫法是:回傳型別 + 名稱 +(參數列)+ 大括號主體。每個參數都要寫型別。 不回傳東西的函式,回傳型別寫 void。函式要在被呼叫之前定義(通常寫在 main 上方)。 

int biggest(int a, int b, int c) {   // 回傳 int,吃三個 int
    if (a >= b && a >= c) return a;
    if (b >= a && b >= c) return b;
    return c;
}

void greet() {                       // void:不回傳東西
    cout << "hello!\n";
}

int main() {
    cout << biggest(5, 8, 9) << "\n";  // 9
    greet();
}

這裡藏著 C++ 一個關鍵、也常讓人困惑的觀念。預設情況下,參數是傳值(pass by value): 函式拿到的是引數的一份複製品,在函式裡怎麼改都不會影響外面的原變數。 

void try_change(int x) {   // x 是複製品
    x = x * 100;           // 改的是複製品
}

void really_change(int &x) {   // 注意 &:這是「傳參考」
    x = x * 100;               // 改的就是外面那個變數本人
}

int main() {
    int a = 3;
    try_change(a);
    cout << a << "\n";      // 3,沒被改到(改的是複製品)
    really_change(a);
    cout << a << "\n";      // 300,被改到了!
}
🔑
一個 & 的天差地別:傳參考(pass by reference)

在參數型別後加一個 &,就從「傳複製品」變成「傳本人」——函式內對它的修改會反映到外面。 這叫傳參考(pass by reference)。它有兩個用途:一是讓函式能改動外部變數 (例如 swap 交換兩數);二是避免複製大型物件——如果你把一個有一百萬元素的 vector 傳值進函式,它會整個複製一遍,又慢又耗記憶體;改成傳參考(常搭配 const 防止誤改) 就只是傳一個「別名」,瞬間完成。這個 & 在 C++ 競賽裡無處不在,務必分清楚。

struct:把幾個值打包成一個型別

當你需要把幾個相關的值綁在一起(例如平面上的一個點有 x、y),struct 讓你自訂一個新型別: 

struct Point {
    int x, y;
    // 可以在 struct 裡定義「怎麼比大小」,排序時就用得到
    bool operator<(const Point &b) const {
        if (x != b.x) return x < b.x;   // 先比 x
        return y < b.y;                 // x 相同再比 y
    }
};

int main() {
    Point a = {3, 5};        // 用大括號給值
    cout << a.x << " " << a.y << "\n";   // 用 . 取成員 → 3 5
}

那個 operator< 是「運算子多載」:你親自定義了「兩個 Point 怎麼比小於」。 定義好之後,一整排 Point 就能直接拿去 sort 了——這正是下一節的主角。

排序與常用工具

排序是競賽裡最常用的操作。你大可自己寫泡沫排序,但那是 $O(n^2)$,資料一大就 TLE。 C++ 內建的 sort 是 $O(n \log n)$ 的(混合了快速排序等),又快又可靠——直接用它。 

int arr[5] = {3, 1, 4, 1, 5};
sort(arr, arr + 5);   // 排序 [arr, arr+5) 這段,由小到大
// arr 變成 {1, 1, 3, 4, 5}

sort 吃「開始位置」和「結束位置」兩個指標,採左閉右開: 排 arr 的前 n 個就寫 sort(arr, arr + n)。預設由小到大。

自訂比較:cmp 函式與 lambda

想由大到小、或依自訂規則排序,給 sort 第三個參數——一個比較函式。 規則:當 cmp(a, b) 回傳 true,代表「a 應該排在 b 前面」。 

// 寫法一:具名的 cmp 函式(由大到小)
bool cmp(int a, int b) {
    return a > b;   // a 比 b 大時,a 排前面 → 由大到小
}
sort(arr, arr + n, cmp);

// 寫法二:就地寫一個 lambda(匿名函式),不必另外命名
sort(arr, arr + n, [](int a, int b) {
    return a > b;
});
⚠️
比較函式:相等時必須回傳 false

比較函式定義的是「嚴格小於」。當 a 和 b 等價時必須回傳 false (例如由大到小要寫 a > b 而非 a >= b)。若相等時回傳 true,sort 會陷入矛盾, 可能直接 RE。多關鍵字排序時,先比第一關鍵字,相等才比下一個——就像上面 Point 的 operator<。

口袋裡的瑞士刀:常用內建函式

這些小工具能少打很多 code、少出很多 bug,記熟它們:

函式作用範例
min(a,b) / max(a,b)取較小 / 較大者max(3, 7) → 7;三個以上用 max({a,b,c})
swap(a,b)交換兩變數的值swap(x, y)
abs(x)絕對值abs(-5) → 5
__gcd(a,b)最大公因數,$O(\log)$__gcd(12, 18) → 6

int arr[] = {1, 2, 3, 3, 5};   // 必須已排序!
// lower_bound:第一個 >= x 的位置;upper_bound:第一個 > x 的位置
cout << *lower_bound(arr, arr + 5, 3) << "\n";   // 3(第一個 >=3)
// 用相減算出 3 出現幾次(這是個好用招式)
cout << upper_bound(arr, arr + 5, 3) - lower_bound(arr, arr + 5, 3) << "\n"; // 2

int dp[1005];

ℹ️
memset 的陷阱

memset 是一個位元組一個位元組地填,所以只有填 0(全 0)和 -1(全 1) 會得到你想要的整數結果。想把陣列填成「很大的數」當作無限大,別用 memset(dp, 1e9, ...)—— 那會得到怪數字。要填任意值,用 fill(dp, dp+n, value)。 這呼應第 26 章的提醒:工具要懂它「實際怎麼運作」,才不會踩坑。

STL 登場:容器的世界

到此為止,你已經把 C++ 的「語言本身」走過一遍了。現在進入這一章真正的重頭戲—— STL(Standard Template Library,標準模板庫)。 還記得第 29 章你親手打造的那些資料結構嗎?動態陣列、堆疊、佇列、雜湊表、平衡樹、堆積—— STL 把它們全都做成了調校到極致的成品,你只要 #include 就能用,而且複雜度有保證。

STL 容器大致分三家族,理解這張地圖,你就知道面對問題該去哪個架子上拿工具:

三個共同的詞彙:迭代器、範圍 for、成員函式

在逐一拆解容器前,先建立三個貫穿所有容器的共同語言。

1. 成員函式(member function):容器自帶一堆「附在自己身上的函式」,用 容器.函式() 呼叫。 最常見的有 .size()(目前幾個元素)、.empty()(是不是空的)、 .push_back(x)(尾端加入)、.begin() / .end()(起點/終點)。

2. 迭代器(iterator):陣列可以用 arr + i 算位址,但 STL 容器(像 set)的元素不一定連續存放, 於是用「迭代器」來代表「指向某個元素的位置」。v.begin() 指向第一個,v.end() 指向 最後一個的再下一個(尾後位置)。取值用 *it。它就是第 29 章 節點指標的抽象化。

3. 範圍 for(range-based for):走訪整個容器的簡潔寫法,對應 Python 的 for x in v: 

vector<int> v = {3, 1, 4};
for (int x : v) cout << x << " ";       // 3 1 4(複製每個元素到 x)
for (int &x : v) x *= 2;               // 加 & 取參考,就能就地修改
for (auto x : v) cout << x << " ";      // auto 讓編譯器自動推型別,省事

vector:會長大的陣列

用途:vector 是「可以隨時改變長度的陣列」, 是第 29 章「動態陣列」的成品,也是你 99% 時候會用的容器, 直接對應 Python 的 list。固定大小的原生陣列能做的它都能做,還能自由增長。 

vector<int> v;                 // 空的
vector<int> a(n);              // n 個元素,初值為 0
vector<int> b(n, 7);           // n 個元素,初值全為 7
vector<int> c = {1, 2, 5};     // 直接給初值
vector<vector<int>> grid(n);   // 巢狀:二維(每列長度可不同)

v.push_back(10);    // 尾端加入(對應 Python list.append),攤銷 O(1)
v.pop_back();       // 移除尾端,O(1)
cout << v[0];       // 用 [索引] 取值,O(1)
cout << v.size();   // 目前元素個數
sort(v.begin(), v.end());     // 排序整個 vector

操作與複雜度:用索引取值 $O(1)$、尾端加入/刪除攤銷 $O(1)$、 但在開頭或中間插刪要搬動後面全部,$O(n)$。 何時用:幾乎隨時。需要一串會增長的同型別資料、需要傳二維陣列進函式、 或需要許多長度不一的陣列(用 vector 才不會浪費記憶體)時,它都是首選。

💡
Python list ↔ C++ vector

lst.append(x) → v.push_back(x);lst[-1] → v.back(); len(lst) → v.size();lst.sort() → sort(v.begin(), v.end()); for x in lst → for (int x : v)。觀念完全一樣,只是換腔調。

pair:把兩個值綁在一起

用途:pair 把兩個值打包成一個——例如平面上的一個點 (x, y)、 或「值 + 它的索引」。它像一個迷你版的、現成的兩欄 struct,對應 Python 的 tuple(雙元素)。 

pair<int, int> p = {3, 5};       // 或 make_pair(3, 5)
cout << p.first << " " << p.second << "\n";   // 用 .first / .second 取值 → 3 5

vector<pair<int, int>> vp;
vp.push_back({1, 2});
vp.emplace_back(3, 4);           // emplace_back 不必包大括號,稍快

最香的性質:pair 自帶比較規則——先比 first,相同再比 second。 所以一堆 pair 直接 sort 就會「先照第一個、再照第二個」排好,完全不必自訂比較函式。 這讓「依 x 為第一關鍵字、y 為第二關鍵字排序」這種需求變成一行: 

vector<pair<int, int>> pts = {{3, 1}, {1, 9}, {1, 2}};
sort(pts.begin(), pts.end());    // 結果:{1,2}, {1,9}, {3,1}

何時用:需要把兩個值「綁著一起搬動」(尤其排序時)而又懶得寫 struct,就用 pair。 要綁三個以上,可以巢狀(pair<int, pair<int,int>>)或改用 tuple,但通常 struct 更清楚。

stack 與 queue:LIFO 與 FIFO 兩兄弟

用途:這兩個是第 29 章的招牌—— stack(堆疊)是「後進先出(LIFO)」,像一疊盤子,只能從頂端放和拿; queue(佇列)是「先進先出(FIFO)」,像排隊,從尾端進、從前端出。 它們其實都能用 vector/deque 模擬,但用專門的名字能讓讀程式的人一眼看出你在用的概念。 

stack<int> st;
st.push(1); st.push(2); st.push(3);
cout << st.top() << "\n";   // 3,看頂端
st.pop();                  // 移除頂端(注意:pop 不回傳值)
cout << st.size() << "\n";  // 2

queue<int> q;
q.push(1); q.push(2); q.push(3);   // 從尾端加入(enqueue)
cout << q.front() << "\n";  // 1,看最前面
q.pop();                   // 從前端移除(dequeue)
⚠️
pop() 只移除、不回傳;空容器別碰 top/front

C++ 的 pop() 和 Python 的 list.pop() 不同:它只移除、不回傳值。 想拿到那個值,要先 top()(stack)或 front()(queue),再 pop()。 還有,對空的容器呼叫 top() / front() / pop() 是未定義行為(常導致 RE), 所以用之前記得先 if (!st.empty())。

操作與複雜度:三種操作(push/pop/top 或 front)都是 $O(1)$。 何時用:stack 用於括號匹配、運算式求值、深度優先(DFS)、單調堆疊; queue 用於廣度優先搜尋(BFS,見第 30 章)、模擬排隊與緩衝。

priority_queue:永遠先給你最大的(堆)

用途:如果你只需要「隨時取出目前最大的元素」,有個比平衡樹更快、常數更小的結構: 堆積(heap)。STL 的 priority_queue(優先佇列)底層就是二元堆積, 正是第 29 章那個堆積的成品,對應 Python 的 heapq。

堆積是一棵「父親一定不小於兒子」的完全二元樹,但巧妙地用一個陣列就能表示:第 i 個節點的兩個兒子 在 2i 和 2i+1,父親在 i/2。取最大值就是看樹根($O(1)$);插入與刪除最大值則沿著樹「上浮/下沉」調整,各 $O(\log n)$。 

priority_queue<int> pq;          // 預設:最大堆,top() 給最大
pq.push(3); pq.push(9); pq.push(5);
cout << pq.top() << "\n";         // 9,看最大,O(1)
pq.pop();                        // 移除最大,O(log n)

// 想要「最小堆」(top 給最小)?多寫兩個參數:
priority_queue<int, vector<int>, greater<int>> minpq;
minpq.push(3); minpq.push(9); minpq.push(5);
cout << minpq.top() << "\n";      // 3,最小
ℹ️
預設是最大堆,反直覺要小心

C++ 的 priority_queue 預設 top() 給最大值——這和 Python 的 heapq(預設最小堆)相反!想要最小堆,要把宣告寫成 priority_queue<int, vector<int>, greater<int>>。 (在 Python 用 heapq 模擬最大堆則是把值取負放進去。)

操作與複雜度:取最大 $O(1)$、插入 $O(\log n)$、刪除最大 $O(\log n)$。它只能存取「最大(或最小)那一個」, 不能查中間的元素。何時用:每次都要拿極值的場合——Dijkstra 最短路、合併果子(Huffman)、 即時找前 k 大、模擬事件排程。功能上是 set 的子集,但常數更小,被卡時間時換它。

set / multiset:自動有序的集合

用途:set 是一個自動保持有序、且不含重複的集合, 底層是第 29 章的平衡二元搜尋樹(紅黑樹)。 它支援插入、刪除、查找,全都是 $O(\log n)$,而且任何時刻元素都是排好序的。對應 Python 的 set, 但 C++ 的 set 額外保證「有序」這件強大的事。 

set<int> s;
s.insert(5); s.insert(2); s.insert(5);   // 第二個 5 被忽略(不重複)
cout << s.size() << "\n";                 // 2
if (s.count(2)) cout << "2 在裡面\n";     // count 回傳 0 或 1
s.erase(2);                              // 刪除元素 2
cout << *s.begin() << "\n";              // 最小元素
cout << *s.rbegin() << "\n";             // 最大元素
for (int x : s) cout << x << " ";        // 由小到大走訪

set 真正的殺手鐧是內建的 lower_bound / upper_bound:在 $O(\log n)$ 內找到 「第一個 ≥ x」或「第一個 > x」的元素。這讓「插入一個數、查詢比 x 大的最小數」這種需求變得輕鬆: 

set<int> s = {1, 3, 5, 7};
auto it = s.lower_bound(4);   // 第一個 >= 4 的元素 → 指向 5
if (it != s.end()) cout << *it << "\n";   // 5
auto it2 = s.upper_bound(5);  // 第一個 > 5 的元素 → 指向 7
cout << *it2 << "\n";         // 7
⚠️
用 s.lower_bound(x),別用 lower_bound(s.begin(), s.end(), x)

set 一定要用它的成員函式版 s.lower_bound(x),才是 $O(\log n)$。 如果誤用第 28 章那個吃迭代器的全域版 lower_bound(s.begin(), s.end(), x),因為 set 的迭代器不能隨機跳,複雜度會退化成糟糕的 $O(n)$。 這是高手也會偶爾踩的坑。

需要允許重複的有序集合?用 multiset,語法幾乎相同,但要注意: ms.erase(x) 會刪掉所有等於 x 的元素;只想刪一個要寫 ms.erase(ms.find(x))。 若不需要有序、只想快速判斷在不在,用 unordered_set(雜湊實作,平均 $O(1)$), 但它沒有 lower_bound。

map:鍵到值的對應

用途:還記得陣列只能用「0 到 n-1 的整數」當索引嗎?map 打破這個限制—— 它讓你用任意型別當鍵(key)去對應一個值(value)。想用字串、用很大的數、用 pair 當索引?map 都行。 它就是第 29 章雜湊表/平衡樹的成品,直接對應 Python 的 dict。 

map<string, int> age;
age["Alice"] = 20;            // 用 [鍵] 指派
age["Bob"] = 25;
cout << age["Alice"] << "\n"; // 20,用 [鍵] 取值

// 超常見:統計次數。不存在的鍵會自動建立、初值 0
map<string, int> cnt;
cnt["apple"]++;              // apple 從 0 變 1
cnt["apple"]++;             // 變 2

for (auto &p : cnt) ...      // 走訪,p.first 是鍵、p.second 是值,自動照鍵排序
for (auto &[k, v] : cnt) cout << k << ": " << v << "\n";  // C++17 結構化綁定
⚠️
用 [] 查不存在的鍵,會「偷偷」建立它

對 map 寫 mp[k],如果 k 不存在,map 會自動新增一個 k、值為預設值(0 或空), 再回傳。所以單純「想檢查 k 在不在」別用 if (mp[k])(那會把 k 建出來、還可能讓 size 變大), 要用 if (mp.count(k)) 或 if (mp.find(k) != mp.end())。

操作與複雜度:插入/查詢/刪除都是 $O(\log n)$,走訪時自動照鍵由小到大。 不需要鍵有序、只想快?用 unordered_map(雜湊,平均 $O(1)$)。 何時用:統計次數、把名字/字串對應到編號、稀疏的「索引→值」對應。

deque 與 bitset:兩個特化工具

deque(雙端佇列,double-ended queue):同時具備 stack 與 queue 的能力—— 兩端都能 $O(1)$ 進出,而且還支援用索引隨機存取。可以把它想成「前端也能插刪的 vector」。 

deque<int> dq;
dq.push_back(1);    // 尾端加入
dq.push_front(5);   // 前端加入(這是 vector 做不到的)
dq.pop_front();     // 前端移除
cout << dq[0] << "\n";   // 也能用索引

deque 比 vector 慢約 3 倍(常數較大),沒有特別需要前端操作時,優先用 vector。 它的招牌應用是「單調隊列」——在滑動視窗中維護區間最大/最小值。

bitset:壓縮的 bool 陣列。bitset<N> 是一個固定大小 N 的位元陣列(每格只有 0/1), 但它用每個位元存一格,既省空間,又能對整個陣列做位元運算——而位元運算一次處理 64 個位元, 速度約是普通迴圈的 1/64(常說的「除以 32」是保守估計)。 

bitset<1000> b;       // <> 裡放的是大小,不是型別
b[5] = 1;             // 像陣列一樣設值
cout << b.count();    // 1 的個數
b.set();              // 全設 1;b.reset() 全設 0
// 殺手級用法:整段平移與聯集
b = (b << 3) | (b >> 2);   // 一口氣把可達狀態往兩邊擴散
💡
bitset 的經典場景:背包/可達性的常數優化

當你在做「哪些分數/重量是可達的」這類布林 DP(第 32 章的背包問題), 狀態轉移常常是「把目前可達的集合往左/往右平移後取聯集」。用 bool 陣列要 $O(n)$ 跑一輪; 用 bitset 的 (b << w) | b 一行搞定,速度快 64 倍。 bitset 用到的機會不多,但在那少數時刻無可替代。

動手練習

看懂和會寫是兩回事。下面七題由淺入深,每題都附可展開的 C++ 解答,但請先自己寫,真的卡住再看。 強烈建議到 ZeroJudge 找對應題目把它們 AC(Accepted)掉—— 競賽程式是練出來的,不是讀出來的。

✏️ 練習 59.1 實作

多筆測資的平方和。輸入有不定組數的整數,以檔案結尾(EOF)結束。 請讀入每一個整數 x,輸出它的平方 x*x,每個一行。 (這是練「讀到 EOF」與「注意溢位」——x 可能到 10 萬,平方就破 int 了嗎?算算看。)

看解答

用 while (cin >> x) 讀到結尾。x 到 10 萬時平方是 100 億,遠超 int, 所以結果要用 long long,而且要 1LL * x * x 讓乘法以 64 位元進行。

#include <bits/stdc++.h>
using namespace std;
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    long long x;
    while (cin >> x) {        // 讀到 EOF 為止
        cout << x * x << "\n"; // x 已是 long long,乘法不會溢位
    }
    return 0;
}

若 x 宣告成 int,要寫成 cout << 1LL * x * x 才安全——這是新手最常見的溢位陷阱。

✏️ 練習 59.2 實作

成績排行榜(sort + 自訂 cmp)。輸入 n,接著 n 行,每行是一個學生的姓名與分數。 請依分數由高到低輸出姓名;分數相同時,依姓名字典序由小到大。

看解答

用 struct 打包姓名與分數,再給 sort 一個 lambda 比較函式:先比分數(大的在前), 相等再比姓名(小的在前)。注意「相等時回傳 false」。

#include <bits/stdc++.h>
using namespace std;
struct Student { string name; int score; };
int main() {
    int n; cin >> n;
    vector<Student> v(n);
    for (auto &s : v) cin >> s.name >> s.score;
    sort(v.begin(), v.end(), [](const Student &a, const Student &b) {
        if (a.score != b.score) return a.score > b.score;  // 分數高的在前
        return a.name < b.name;                            // 同分:姓名字典序
    });
    for (auto &s : v) cout << s.name << "\n";
    return 0;
}

lambda 的參數用 const Student &(傳參考避免複製、const 防誤改)是好習慣, 尤其物件大時能省下大量時間。

✏️ 練習 59.3 實作

詞頻統計(map)。輸入一段文字(以空白分隔的單字,讀到 EOF)。 請統計每個單字出現幾次,並依字典序輸出 單字 次數,每個一行。

看解答

這是 map<string, int> 的招牌應用:cnt[word]++ 一行完成統計 (不存在的鍵會自動建立、初值 0)。map 走訪時自動照鍵字典序,所以直接印就排好了。

#include <bits/stdc++.h>
using namespace std;
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    map<string, int> cnt;
    string word;
    while (cin >> word) cnt[word]++;     // 統計
    for (auto &[w, c] : cnt)             // map 自動照鍵排序
        cout << w << " " << c << "\n";
    return 0;
}

若不在意輸出順序、只求最快,可把 map 換成 unordered_map(均攤 $O(1)$), 但就不會自動排序了。

✏️ 練習 59.4 實作

找後繼(set + upper_bound)。有一個一開始裝著 1 到 n 的集合。處理 m 筆詢問, 每筆給一個數 x:請輸出集合中比 x 大的最小數,然後把那個數從集合中刪除; 若不存在這樣的數,輸出 -1。(這正是 set 的 upper_bound + erase 的舞台。)

看解答

用 set 維護集合。s.upper_bound(x) 在 $O(\log n)$ 找到「第一個 > x」的迭代器; 若它不是 end(),輸出並 erase 掉。整體 $O((n+m)\log n)$。

#include <bits/stdc++.h>
using namespace std;
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int n, m; cin >> n >> m;
    set<int> s;
    for (int i = 1; i <= n; i++) s.insert(i);
    while (m--) {
        int x; cin >> x;
        auto it = s.upper_bound(x);   // 第一個 > x 的元素
        if (it == s.end()) cout << "-1\n";
        else { cout << *it << "\n"; s.erase(it); }
    }
    return 0;
}

注意一定要用成員函式 s.upper_bound;用全域版會退化成 $O(n)$ 而 TLE。 erase(it) 傳迭代器刪除是 $O(\log n)$(攤銷甚至 $O(1)$)。

✏️ 練習 59.5 挑戰

合併果子(priority_queue)。有 n 堆果子,第 i 堆重 a[i]。每次你可以把任意兩堆合併成一堆, 花費等於兩堆重量之和。把所有果子合成一堆,求最小總花費。 (提示:每次都該合併當前最輕的兩堆——這需要一個能反覆取出最小值的結構。)

看解答

這是經典的 Huffman 貪婪(見第 31 章):每次取最小的兩堆合併,花費累加, 合併結果再放回去。要「反覆取最小值」,用最小堆 priority_queue<..., greater>。 重複 n-1 次。總花費可能很大,用 long long。

#include <bits/stdc++.h>
using namespace std;
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int n; cin >> n;
    priority_queue<long long, vector<long long>, greater<long long>> pq; // 最小堆
    for (int i = 0; i < n; i++) { long long x; cin >> x; pq.push(x); }
    long long total = 0;
    while (pq.size() > 1) {        // 直到只剩一堆
        long long a = pq.top(); pq.pop();   // 最小
        long long b = pq.top(); pq.pop();   // 次小
        total += a + b;            // 這次合併的花費
        pq.push(a + b);            // 合併後放回
    }
    cout << total << "\n";
    return 0;
}

每次 push/pop 是 $O(\log n)$,共做 n-1 次,總複雜度 $O(n \log n)$。 為什麼「每次合併最小兩堆」會最優?因為越早被合併的堆,會被重複累加越多次,所以要讓輕的先合、晚點才碰到重的。

✏️ 練習 59.6 挑戰

活動選擇(pair 排序 + 貪婪)。有 n 個活動,第 i 個的開始與結束時間是 (s, e)。 同一時間只能參加一個活動(前一個的結束時間 ≤ 後一個的開始時間)。求最多能參加幾個活動。

看解答

經典貪婪:依結束時間由早到晚排序,然後貪婪地能選就選。 用 pair<int,int> 存 {結束時間, 開始時間}——把結束時間放 first, 這樣直接 sort 就會照結束時間排序(pair 自帶字典序),省去寫 cmp。

#include <bits/stdc++.h>
using namespace std;
int main() {
    ios_base::sync_with_stdio(false);
    cin.tie(nullptr);
    int n; cin >> n;
    vector<pair<int,int>> a(n);   // {結束時間, 開始時間}
    for (auto &p : a) cin >> p.second >> p.first;  // 讀 開始、結束
    sort(a.begin(), a.end());    // 自動依結束時間排序
    int cnt = 0, lastEnd = -1;
    for (auto &[e, s] : a) {
        if (s >= lastEnd) {      // 開始時間不早於上一個的結束 → 可以選
            cnt++;
            lastEnd = e;
        }
    }
    cout << cnt << "\n";
    return 0;
}

排序 $O(n \log n)$,掃一遍 $O(n)$。把要排序的關鍵字放進 pair 的 first, 是「借 pair 的內建比較省掉 cmp」的常用技巧。

✏️ 練習 59.7 挑戰

可能的總分(bitset)。一份考卷有 n 題,第 i 題分數為 w[i],每題只能「全對(+w)」或「全錯(0)」。 問:總分有哪些「可能值」?輸出所有可能的總分(由小到大)。當 n 很大、總分上限也大時,請用 bitset 把布林 DP 加速。

看解答

令 bitset 的第 i 位代表「總分 i 是否可達」。一開始只有 0 分可達(b[0] = 1)。 處理一題分數 w 時,新可達集合 = 原集合 ∪(原集合每個值都 +w)= b | (b << w)—— 一行就把全部狀態平移聯集,比逐格更新快數十倍。最後掃一遍輸出所有為 1 的位。

#include <bits/stdc++.h>
using namespace std;
const int MAXSUM = 100000;       // 總分上限
bitset<MAXSUM + 1> b;
int main() {
    int n; cin >> n;
    b[0] = 1;                    // 0 分一定可達
    for (int i = 0; i < n; i++) {
        int w; cin >> w;
        b |= (b << w);           // 加上「這題答對」的所有可能,取聯集
    }
    for (int s = 0; s <= MAXSUM; s++)
        if (b[s]) cout << s << " ";   // 所有可達的總分
    cout << "\n";
    return 0;
}

每題的轉移從 $O(\text{總分})$ 變成 $O(\text{總分}/64)$,整體快上約 64 倍。 這是 0/1 背包(第 32 章)的布林版,也是 bitset 最經典的用武之地。

我們把你在 Python 世界練就的直覺,平移到了競賽世界的母語 C++。回顧這趟轉軌的路標:

  • 骨架與 I/O:#include <bits/stdc++.h> + using namespace std + main; cin >> / cout << 配上 sync_with_stdio(false) 加速與 "\n"; 多筆測資用 while (cin >> n) 與 while (t--)。
  • 型別的代價:int 會溢位,大數用 long long; double 有誤差,別用 ==;char 是能算術的小整數。
  • 運算子:整數除法會丟小數;位元 & | ^ ~ << >>(^ 是 XOR 不是次方!);三元 ?:。
  • 函式傳參:預設傳值(複製);加 & 變傳參考(本人),用來修改外部變數或避免複製大物件。
  • 排序與工具:sort + 自訂 cmp/lambda(相等回 false);min/max/swap/__gcd/memset/lower_bound。
  • STL 容器:vector(動態陣列)、pair(打包)、stack/queue(LIFO/FIFO)、 priority_queue(堆,預設最大)、set/multiset(有序集合 + lower/upper_bound)、 map(鍵值對應)、deque(雙端)、bitset(位元加速)。每個都對應你在第 29 章親手打造過的結構。

你現在握有一整套競賽程式的零件了。但工具只是手段——真正分出高下的,是選對工具與 估對複雜度。面對一題,先問:資料量多大?電腦一秒約跑 $10^8$ 次基本操作(呼應第 26 章), 所以 $n = 10^6$ 時 $O(n^2)$ 必死、要找 $O(n \log n)$ 的解。再問:我需要哪種操作? 要極值用 priority_queue、要有序查找用 set、要計數用 map……這張「需求→容器」的對應,就是第 29 章 決策表的 C++ 實戰版。接下來的章節,你會帶著這些武器,真正走進競賽演算法的戰場。
