形式邏輯

兩千年來,「正確的推理」是哲學家爭辯的對象。直到有人問:能不能把推理本身,變成一套像算術一樣、可以交給機器一步步檢查的符號遊戲?這個念頭點燃了現代邏輯——也讓我們第一次看清「可證明」的疆界長什麼樣子。

把「推理」搬上手術台

你已經在第 13 章學會用邏輯與證明替程式背書:命題、連接詞、真值表、∀ 與 ∃、五種證明武器、歸納法。 那一章教你怎麼用邏輯去證明別的東西(數列公式、$\sqrt 2$ 無理、質數無限多)。這一章,我們要做一件更奇怪、也更深刻的事: 把邏輯與證明本身當成研究對象,放上手術台,問一些「後設(meta)」的問題——

  • 「一個證明」到底是什麼?能不能精確到讓一台機器逐行檢查它對不對?
  • 「為真」和「可被證明」,是同一件事嗎?如果不是,差在哪?
  • 有沒有一套規則,強大到能證出所有為真的數學敘述?

最後一個問題,聽起來像痴人說夢——但二十世紀初,全世界最頂尖的數學家(以希爾伯特為首)真的相信答案是「有」, 並把畢生精力押在上面。結果一個二十四歲的年輕人,用一頁紙的構想,證明了答案是「永遠沒有」。 這趟旅程的終點,就是 CS 與數學史上最震撼的結論之一:哥德爾不完備定理。

但先別急。要讀懂那個高潮,我們得先把工具磨利。而這一切的起點,是一個樸素到近乎挑釁的想法——

🔑
什麼是「形式」邏輯?

形式化(formalization)就是:把推理變成純粹的符號操作,規則明確到「不需要理解意義,光靠對照規則就能機械地執行與檢查」。 就像下西洋棋——你不必懂「為什麼」要這樣走,只要看棋子擺在哪、規則允許怎麼動,就能判斷一步棋合不合法。 形式邏輯把「推論」也變成這樣一場棋:證明就是一連串合法的「棋步」,而對不對,一台不懂任何數學的機器也能驗證。

這個「把推理機械化」的夢想,正是電腦科學的胎記。它是底下這一切的共同根基:

  • 數學證明——把「無懈可擊的論證」講到精確;
  • 電腦電路(第 19 章)——每個邏輯閘都是一條命題邏輯規則的物理化身;
  • 程式驗證——用邏輯證明程式正確,而不只是測試它;
  • 資料庫(第 37 章)——SQL 查詢底層就是一階邏輯;
  • Prolog(第 45 章)——直接「用邏輯來寫程式」,執行就是證明;
  • AI——知識表示、自動推理、SAT/SMT 求解器。

學會這一章,你會發現上面這些看似無關的東西,骨子裡是同一套語言。我們從最熟悉的命題邏輯開始,但這次玩真的。

語法 vs 語意:同一個式子的兩種身分

第 13 章我們把 $P \wedge Q$、$P \rightarrow Q$ 當成理所當然的東西在用。現在要拆開它,你會發現一個邏輯式其實同時擁有兩種身分, 而現代邏輯的一切,都建立在分清這兩者之上:

  • 語法(syntax):它是一串符號,長得對不對、合不合文法。跟「意義」「真假」完全無關。
  • 語意(semantics):它表達什麼、在什麼情況下為真。這才碰到「真假值」。

這就像自然語言裡,「綠色的想法憤怒地沉睡」是一個文法正確(語法 OK)、但沒有意義(語意空)的句子。 邏輯把這兩件事徹底切開,而且各自講到一絲不苟。

語法:什麼叫「合式公式」

並不是隨便把符號排在一起就算數。「$P \wedge \rightarrow Q\,)$」是一串垃圾,不是合法的邏輯式。 合法的那些,叫合式公式(well-formed formula,WFF)。WFF 用遞迴定義——這個結構你在第 7 章(遞迴)、第 13 章(結構歸納)都見過:

  1. 基底:每個命題變數($P, Q, R, \dots$)本身,是一個 WFF(稱為原子公式 atom)。
  2. 遞迴:若 $\varphi$ 與 $\psi$ 都是 WFF,則 $\neg\varphi$、$(\varphi \wedge \psi)$、$(\varphi \vee \psi)$、$(\varphi \rightarrow \psi)$、$(\varphi \leftrightarrow \psi)$ 也都是 WFF。
  3. 封閉:除此之外,沒有別的東西是 WFF。

注意這個定義完全沒提到「真假」。它只管「形狀」。就像程式語言的文法(grammar)只管你的程式碼能不能通過剖析(parse),不管它跑起來對不對。 每個 WFF 都對應一棵剖析樹(parse tree):

語意:賦予符號真假

現在輪到語意上場。要讓一個 WFF 有真假,我們得先決定每個原子是真是假——這叫一個真值賦值(truth assignment / valuation), 記成 $v$。$n$ 個原子就有 $2^n$ 種可能的賦值。給定一個 $v$,連接詞的規則(第 13 章那五張真值表)會像沿著剖析樹「由下往上」計算, 最終算出整個式子在這個 $v$ 下的真假。

這裡藏著本章第一個關鍵切換:語法是「一個」式子;語意是它在「所有 $2^n$ 種世界」裡的命運。 把一個式子放進所有可能的賦值去看,我們就能替它分類——

分類定義(在所有賦值下……)例子
恆真式 / 有效式(tautology / valid)永遠為真$P \vee \neg P$(排中律)
矛盾式(contradiction)永遠為假$P \wedge \neg P$
可滿足(satisfiable)至少一個賦值為真$P \rightarrow Q$、$P \wedge Q$
適然式 / 偶真式(contingent)有時真、有時假$P \rightarrow Q$

這幾個概念之間有兩條漂亮的「對偶」關係,務必記住——它們是後面 SAT 與證明的關節:

  • $\varphi$ 是恆真式 $\iff$ $\neg\varphi$ 是矛盾式(不可滿足)。
  • $\varphi$ 可滿足 $\iff$ $\neg\varphi$ 不是恆真式。

換句話說:「證明 $\varphi$ 恆真」和「證明 $\neg\varphi$ 找不到滿足解」是同一件事。 這個小小的觀察,後面會讓「驗證一個論證有效」直接變成一個 SAT 問題。我們先讓電腦幫我們做分類——把第 13 章的真值表升級成一個分類器: 

import itertools

def classify(expr, variables):
    """把一個布林函式分類:恆真式 / 矛盾式 / 適然式(可滿足但非恆真)"""
    results = [expr(**dict(zip(variables, vals)))
               for vals in itertools.product([False, True], repeat=len(variables))]
    if all(results):
        return "恆真式 (tautology)"
    if not any(results):
        return "矛盾式 (contradiction)"
    return "適然式 (contingent,可滿足但非恆真)"

print(classify(lambda P: P or (not P), ["P"]))            # 排中律 → 恆真式
print(classify(lambda P: P and (not P), ["P"]))           # P ∧ ¬P → 矛盾式
print(classify(lambda P, Q: (not P) or Q, ["P", "Q"]))    # P → Q  → 適然式

# 一個經典恆真式:皮爾士定律 ((P → Q) → P) → P
print(classify(lambda P, Q: ((not ((not P) or Q)) or P) or P, ["P", "Q"]))  # 恆真式

最後那個皮爾士定律(Peirce's law)$((P \rightarrow Q) \rightarrow P) \rightarrow P$ 很有意思:它恆真,卻怎麼看都不直覺。 這正是真值表的價值——它是一台完全機械、絕不出錯的判定機器。事實上,因為命題邏輯只有有限種賦值, 「窮舉真值表」就是一個判定恆真/可滿足的演算法。記住這件事,第八節談可判定性時它會回來。

邏輯蘊涵 $\models$:論證「有效」的真正意思

我們平常說一個論證「站得住腳」,到底是什麼意思?形式邏輯給了精確答案。設一堆前提 $\Gamma = \{\psi_1, \dots, \psi_n\}$ 與結論 $\varphi$,我們說 $\Gamma$ 邏輯蘊涵(entails)$\varphi$,記作

$$\Gamma \models \varphi$$

意思是:凡是讓所有前提同時為真的賦值,也一定讓結論為真。沒有任何「前提全真、結論卻假」的反例賦值存在。 這就是「有效論證(valid argument)」的定義——注意它不要求前提真的為真,只要求「如果前提真,結論必真」。

$\models$ 這個符號叫雙橫槓(double turnstile)。當前提集合是空的,$\models \varphi$ 就退化成「$\varphi$ 是恆真式」。 而對有限的前提,有個好用的橋樑:

$$\{\psi_1, \dots, \psi_n\} \models \varphi \quad\iff\quad (\psi_1 \wedge \cdots \wedge \psi_n) \rightarrow \varphi \ \text{是恆真式}.$$

同理,邏輯等價$\varphi \equiv \psi$ 也能翻成語意:它等於說 $\varphi \leftrightarrow \psi$ 是恆真式,也等於「$\varphi \models \psi$ 且 $\psi \models \varphi$」。 我們寫一個 $\models$ 檢查器——它的精神就是「找反例」: 

import itertools

def entails(premises, conclusion, variables):
    """檢查 premises ⊨ conclusion:有沒有『前提全真、結論卻假』的反例賦值?"""
    for vals in itertools.product([False, True], repeat=len(variables)):
        env = dict(zip(variables, vals))
        if all(p(**env) for p in premises) and not conclusion(**env):
            return False        # 抓到反例 → 論證無效
    return True                 # 找不到反例 → 論證有效

# 範例一:Modus Ponens(肯定前件){P, P → Q} ⊨ Q ?
print(entails([lambda P, Q: P, lambda P, Q: (not P) or Q],
              lambda P, Q: Q, ["P", "Q"]))          # True ✔ 有效

# 範例二:肯定後件的謬誤 {Q, P → Q} ⊨ P ?
print(entails([lambda P, Q: Q, lambda P, Q: (not P) or Q],
              lambda P, Q: P, ["P", "Q"]))          # False ✘ 無效!

# 範例三:否定後件 Modus Tollens {¬Q, P → Q} ⊨ ¬P ?
print(entails([lambda P, Q: not Q, lambda P, Q: (not P) or Q],
              lambda P, Q: not P, ["P", "Q"]))      # True ✔ 有效

範例二抓到了一個經典謬誤:「地濕了(Q),而且『下雨則地濕』(P→Q),所以剛剛下過雨(P)」——無效, 因為地濕也可能是有人灑水(這正是第 13 章「逆命題不成立」的語意版)。$\models$ 把「論證對不對」變成一件可以機械檢查的事, 再也不靠直覺或口才。

🔍
$\models$ 是「語意」的,它要看遍所有世界

留意 entails 怎麼運作:它窮舉每一種賦值,逐一檢查。$\models$ 的本質,就是「在所有可能的解釋下都成立」。 這很可靠,但也很「貴」——它要把整個 $2^n$ 的世界看過一遍。有沒有辦法不窮舉所有世界,而是像下棋一樣純靠符號規則一步步推出結論? 有。那就是下一節的主角,也是現代邏輯真正的分水嶺。

✏️ 練習 56.1 實作

用真值表(可借上面的 classify)判斷下列三個式子各屬於恆真式、矛盾式、還是適然式:

(a) $(P \rightarrow Q) \vee (Q \rightarrow P)$    (b) $(P \leftrightarrow Q) \wedge (P \wedge \neg Q)$    (c) $(P \wedge Q) \rightarrow (P \vee Q)$

看解答

三個答案分別是:(a) 恆真式、(b) 矛盾式、(c) 恆真式。直接交給程式:

print(classify(lambda P, Q: ((not P) or Q) or ((not Q) or P), ["P", "Q"]))   # 恆真式
print(classify(lambda P, Q: (P == Q) and (P and (not Q)), ["P", "Q"]))       # 矛盾式
print(classify(lambda P, Q: (not (P and Q)) or (P or Q), ["P", "Q"]))        # 恆真式

直覺檢查:(a) 很反直覺地恆真——任兩個命題之間,「P 則 Q」與「Q 則 P」至少有一個成立(因為若 P→Q 為假,必是 P 真 Q 假,那 Q→P 就空真)。 (b) 要求「P、Q 真假一致」又「P 真而 Q 假」,自相矛盾。(c) 「兩個都真」當然「至少一個真」,顯然恆真。

$\models$ 與 $\vdash$:現代邏輯的大分野

現在來到本章的樞紐。請慢讀這一節——讀懂它,後面的健全性、完備性、哥德爾定理才會像骨牌一樣自然倒下。

我們有了一種判斷「論證有效」的方法:$\models$,語意的、看遍所有世界的。但人類數學家證明定理時,並不是真的去檢查無窮多個世界—— 我們是從前提出發,套用一條條推理規則,一步步推導到結論。這種「純符號操作」的可推導性,用另一個符號表示:

$$\Gamma \vdash \varphi \qquad \text{讀作「從 } \Gamma \text{ 可以推導出 } \varphi\text{」}$$

$\vdash$ 叫單橫槓(turnstile)。$\Gamma \vdash \varphi$ 的意思是:存在一個形式證明, 從 $\Gamma$ 裡的前提開始,每一步都嚴格遵守事先講好的推理規則,最後寫出 $\varphi$。 這完全是語法層次的——它跟「真假」「世界」一個字都沒提,只關心「符號能不能照規則接龍接到結論」。

$\models$(語意 / semantic)$\vdash$(語法 / syntactic)
問的是在所有解釋/世界下都成立嗎?能用規則一步步推導出來嗎?
工具真值賦值、模型、真值表公理、推理規則、形式證明
本質關於意義與真理關於符號與遊戲規則
比喻「這步棋會不會輸?」(看遍所有後續)「這步棋合不合規則?」(只看規則本)
🔑
兩個世界,兩個橫槓

$\models$ 與 $\vdash$ 是現代邏輯的兩根柱子,初學者最該分清的就是它們:

$\Gamma \models \varphi$(語意蘊涵):「$\varphi$ 真的是 $\Gamma$ 的結論」——在每個讓前提成立的世界裡,$\varphi$ 都成立。這關乎真理。

$\Gamma \vdash \varphi$(語法可導):「$\varphi$ 能被證出來」——存在一串合規則的符號推導抵達 $\varphi$。這關乎證明。

整個二十世紀邏輯,可以說就是在追問一個問題:這兩者,到底是不是同一回事?

為什麼要費這麼大勁把「真」和「可證」分開?因為它們看起來該一樣,卻沒有理由必然一樣。 $\models$ 活在「意義」的世界,$\vdash$ 活在「符號」的世界。一個關於無窮多個世界的真理,憑什麼保證能被一串有限的符號操作捕捉到? 反過來,一串看似合規則的符號推導,憑什麼保證不會推出假話?

這兩個「憑什麼」,正是健全性與完備性兩大定理要回答的。但要談論 $\vdash$,我們得先有一套具體的「推理規則」。 最優雅、也最接近人類實際推理方式的那一套,叫自然演繹。

自然演繹:把證明寫成符號接龍

一個證明系統(proof system),就是「一組玩 $\vdash$ 的規則」。歷史上有好幾種風味:有的偏好少少幾條公理加一條規則(希爾伯特式公理系統,簡潔但寫起證明來像酷刑); 有的把規則設計得貼近人類思路。我們選後者——根岑(Gerhard Gentzen,1934)發明的自然演繹(natural deduction)。

它的精神出奇地對稱漂亮:每個連接詞,都配一條「引入(introduction)」規則和一條「消去(elimination)」規則。 引入規則告訴你「怎麼造出一個帶這個連接詞的式子」;消去規則告訴你「手上有這個連接詞,能拆出什麼」。

連接詞引入規則(怎麼得到它)消去規則(它能給你什麼)
$\wedge$有 $\varphi$、有 $\psi$ $\Rightarrow$ 得 $\varphi \wedge \psi$有 $\varphi \wedge \psi$ $\Rightarrow$ 得 $\varphi$(或 $\psi$)
$\vee$有 $\varphi$ $\Rightarrow$ 得 $\varphi \vee \psi$有 $\varphi \vee \psi$、$\varphi \rightarrow \chi$、$\psi \rightarrow \chi$ $\Rightarrow$ 得 $\chi$(分情況)
$\rightarrow$假設 $\varphi$ 能導出 $\psi$ $\Rightarrow$ 得 $\varphi \rightarrow \psi$(卸下假設)有 $\varphi$、有 $\varphi \rightarrow \psi$ $\Rightarrow$ 得 $\psi$(就是 Modus Ponens!)
$\neg$假設 $\varphi$ 導出矛盾 $\bot$ $\Rightarrow$ 得 $\neg\varphi$有 $\varphi$、有 $\neg\varphi$ $\Rightarrow$ 得 $\bot$
$\bot$(無)有 $\bot$ $\Rightarrow$ 得任意 $\varphi$(爆炸律 ex falso)

最妙的是 $\rightarrow$ 的引入規則,它捕捉了人類「假設性推理」的精髓:想證「如果 $\varphi$ 那麼 $\psi$」,就暫時假設 $\varphi$ 成立, 在這個假設底下推出 $\psi$,然後「卸下(discharge)」這個假設,得到 $\varphi \rightarrow \psi$。這正是你寫數學證明時說「假設 x 是偶數……因此……所以若 x 偶則……」的形式化。

我們用 Fitch 式排版來寫證明:每行一個式子加上「它的理由」,用直槓 │ 圈出「假設的有效範圍(子證明)」。先看一個簡單的——交換律 $P \wedge Q \vdash Q \wedge P$: 

證明:P ∧ Q ⊢ Q ∧ P
─────────────────────────────────
1.  P ∧ Q          前提 (premise)
2.  P              ∧消去 (∧E), 1
3.  Q              ∧消去 (∧E), 1
4.  Q ∧ P          ∧引入 (∧I), 3, 2
─────────────────────────────────
得證。每一行都標明「用哪條規則、引用哪幾行」——
一台不懂任何意義的機器,光對照規則就能逐行驗證它合法。

再看一個用到「假設與卸下」的——傳遞律(假言三段論)$P \rightarrow Q,\ Q \rightarrow R \vdash P \rightarrow R$。 要證一個 $\rightarrow$,就用 $\rightarrow$ 引入:假設前件 $P$,設法推出後件 $R$: 

證明:P → Q,  Q → R  ⊢  P → R
─────────────────────────────────
1.  P → Q          前提
2.  Q → R          前提
3.  │ P            假設 (為了證 P → R 而暫時假設 P)
4.  │ Q            →消去 (MP), 1, 3
5.  │ R            →消去 (MP), 2, 4
6.  P → R          →引入 (→I), 卸下假設 3–5
─────────────────────────────────
第 3~5 行是「假設 P」底下的子證明(用 │ 框住);
第 6 行卸下這個假設,把「P 推得出 R」收成一個 P → R。

感受到了嗎?整個過程沒有真值表、沒有「世界」、沒有真假——純粹是符號照規則接龍。這就是 $\vdash$ 的世界。 而那個「爆炸律」($\bot$ 消去:從矛盾可推出任何東西)看似荒謬,卻正是第 13 章「反證法」的引擎: 假設了 $\neg\varphi$ 卻撞出矛盾 $\bot$,系統就允許你「反彈」回 $\varphi$。

📐
古典 vs 直覺:一條規則的世界觀之爭

上面那套引入/消去規則,其實是直覺主義邏輯(intuitionistic logic)的核心。要得到我們熟悉的古典邏輯(classical logic), 還得補一條:反證法(RAA)——假設 $\neg\varphi$ 導出 $\bot$,則得 $\varphi$;或等價地,雙重否定消去 $\neg\neg\varphi \vdash \varphi$,或排中律 $\varphi \vee \neg\varphi$。 直覺主義者(布勞威爾 Brouwer)拒絕這條:他們認為「證明 $\varphi$ 為真」必須真的建構出 $\varphi$,不能只靠「$\neg\varphi$ 不成立」反推。 這場哲學爭論看似抽象,卻在 CS 裡開花結果——第九節的 Curry–Howard 同構會告訴你:直覺主義證明 = 真的能跑的程式。

✏️ 練習 56.2 思考

用自然演繹寫出 $\vdash P \rightarrow (Q \rightarrow P)$ 的完整證明(這是個沒有前提的定理,意思是它恆成立)。提示:要證最外層的 $\rightarrow$,先假設 $P$;在那底下要證 $Q \rightarrow P$,再假設 $Q$。會用到兩層巢狀假設。

看解答 
證明: ⊢ P → (Q → P)
─────────────────────────────────
1.  │ P                假設(為證外層 P → …)
2.  │ │ Q              假設(為證內層 Q → P)
3.  │ │ P              重述 (reiteration), 1
4.  │ Q → P            →引入, 卸下假設 2–3
5.  P → (Q → P)        →引入, 卸下假設 1–4
─────────────────────────────────

關鍵在第 3 行的「重述」:在內層假設 $Q$ 底下,外層假設的 $P$ 仍然有效,可以直接搬下來用。 卸下內層假設($Q$ 能推出 $P$)得到 $Q \rightarrow P$;再卸下外層假設($P$ 能推出 $Q \rightarrow P$)得到整個定理。

順帶體會它的語意:$P \rightarrow (Q \rightarrow P)$ 說「一個已經為真的 $P$,不管前面加什麼前提 $Q$,都還是會被推出來」—— 這跟蘊含的「空真」性質一脈相承,你也可以用 classify 驗證它是恆真式。能 $\vdash$ 證出來、又是 $\models$ 恆真——這個巧合,正是下一節的主題。

健全性與完備性:讓 $\vdash$ 與 $\models$ 完美重合

我們手上現在有兩套完全不同的機器:

  • $\models$:語意機器。窮舉所有世界,判斷「真的成立」。
  • $\vdash$:語法機器。照規則接龍,判斷「推導得出來」。

一個自然到讓人坐立難安的問題:這兩台機器,會給出一樣的答案嗎? 它們的設計動機天差地別(一個關於真理,一個關於符號),沒有任何「先驗」理由保證它們一致。 要它們一致,需要兩個方向各自成立,而這正是兩條以「方向」命名的定理:

健全性(soundness):證明系統「不說謊」

$$\Gamma \vdash \varphi \quad\Longrightarrow\quad \Gamma \models \varphi$$

健全性說:凡是你推導得出來的,都真的成立。($\vdash$ 蘊含 $\models$。)這是對證明系統最起碼的要求—— 如果一個系統能「證出」假話,那它一文不值。健全性保證你的符號遊戲不會騙你:每一條推理規則都「保真」(前提真則結論真), 所以一整串合規則的推導,絕不會從真前提走到假結論。沒有健全性,證明毫無意義。

完備性(completeness):證明系統「夠強」

$$\Gamma \models \varphi \quad\Longrightarrow\quad \Gamma \vdash \varphi$$

完備性說:凡是真的成立的,你都推導得出來。($\models$ 蘊含 $\vdash$。)這是個強得多、也驚人得多的保證—— 它說你那有限的幾條規則,強到足以捕捉每一個語意真理,不會有「明明是對的,卻怎麼都證不出來」的漏網之魚。

健全性通常「容易」(逐條檢查規則保真即可);完備性則「困難」且絕非理所當然——憑什麼一串有限符號操作,能追上無窮多個世界裡的所有真理? 但對命題邏輯與一階邏輯,它居然成立。後者就是大名鼎鼎的——

🔑
哥德爾完備性定理(1929)

哥德爾(Kurt Gödel)在他二十三歲的博士論文裡證明:一階邏輯是健全且完備的。也就是說,對一階邏輯,

$$\Gamma \vdash \varphi \quad\Longleftrightarrow\quad \Gamma \models \varphi.$$

「可被證明」與「邏輯上為真」恰好是同一件事。命題邏輯也享有同樣的好性質(且更簡單)。這是現代邏輯的奠基石之一: 它保證我們的證明系統「剛剛好」——健全讓它不多證(不證假話),完備讓它不漏證(所有真理都搆得著)。 請務必記住這個名字:這是「完備性」定理。兩年後,同一個哥德爾會再丟出一個「不完備」定理——名字像,意思卻幾乎相反,是全章最常被搞混的地方。

有了完備性,一件很實際的事成立了:既然「為真 = 可證」,而證明是一串有限符號,我們原則上可以寫程式去「枚舉所有可能的證明」, 一個一個試,只要 $\varphi$ 為真,遲早會撞到它的證明。記住這個「枚舉證明」的能力——它在第八節判定一階邏輯時,會帶來一個既美好又殘酷的結論。

✏️ 練習 56.3 思考

用你自己的話,清楚說明「健全性」和「完備性」差在哪。然後回答:如果一個證明系統健全但不完備,會發生什麼壞事?如果完備但不健全呢?哪一種更糟?

看解答

一句話分辨:健全 = $\vdash \Rightarrow \models$(推得出的都為真,系統不說謊);完備 = $\models \Rightarrow \vdash$(為真的都推得出,系統不遺漏)。 一個管「品質」(別證出假的),一個管「涵蓋」(別漏掉真的)。

健全但不完備:系統證出來的東西全都對(可信),但有些真理它永遠證不出來。你會卡在某些對的命題前面,束手無策。可惜,但至少你「相信的都是真的」。

完備但不健全:系統能證出所有真理——但它也能證出假話!這是災難:既然真假它都「證」得出來,「被證明」這件事就失去了一切意義,系統徹底沒用。

所以:不健全比不完備糟糕得多。健全性是底線、是必須的;完備性是夢寐以求的加分。 (預告:第九節的哥德爾不完備定理會告訴我們,對於「夠強到能描述算術」的系統,我們被迫放棄完備性——但絕不放棄健全性。)

一階邏輯:給世界一個「結構」

到目前為止我們玩的都是命題邏輯——它把整個句子當成一個不可分的真假原子($P$、$Q$)。但第 13 章已經提醒過你它的天花板: 它說不出「所有」與「存在」。「蘇格拉底是人」「所有人都會死」——命題邏輯只能把它們各記成一個字母 $P$、$Q$, 完全看不出兩者之間「所以蘇格拉底會死」的邏輯關聯。要捕捉這種推理,我們得升級到一階邏輯(first-order logic,FOL), 又叫述詞邏輯(predicate logic)。

一階邏輯把句子「拆開」,引進四種新零件(第 13 章見過量詞與述詞,這裡補上完整的家族):

  • 個體變數與常數(variables / constants):$x, y$ 指「某個東西」,$socrates$ 指「某個特定的東西」。
  • 述詞(predicates):$Human(x)$(「$x$ 是人」)、$Loves(x, y)$(「$x$ 愛 $y$」)——填入個體後變成有真假的句子。
  • 函數(functions):$father(x)$(「$x$ 的父親」)——吃個體、吐個體,本身沒有真假。
  • 量詞(quantifiers):$\forall$(對所有)、$\exists$(存在)。

於是「所有人都會死」「蘇格拉底是人」可以寫成:

$$\forall x\,(Human(x) \rightarrow Mortal(x)), \qquad Human(socrates).$$

而且這次,$\vdash$ 真的推得出 $Mortal(socrates)$(用 $\forall$ 消去把 $x$ 代成 $socrates$,再 Modus Ponens)——命題邏輯做不到的事,一階邏輯做到了。

語意升級:從「真值賦值」到「模型」

命題邏輯的語意只需要一個真值賦值(給每個原子一個真假)。一階邏輯複雜得多,因為它談論「東西」「東西的性質」「東西之間的關係」。 它的語意,需要一整個模型 / 結構(model / structure) $\mathcal{M}$,包含:

  • 一個論域(domain) $D$:我們在談論「哪一堆東西」(所有整數?所有人?某張圖的頂點?)。
  • 每個常數指到 $D$ 裡的哪個元素、每個函數是 $D$ 上的什麼運算、每個述詞對應 $D$ 上的哪個關係。

關鍵在這裡:同一個一階公式,在不同模型裡可以一真一假。例如 $\forall x\, \exists y\,(y \gt x)$(「每個東西都有比它大的」), 在「整數」這個模型裡為真,在「$0$ 到 $100$ 的整數」這個模型裡卻假(最大的 $100$ 沒有比它更大的)。 於是 $\models$ 的意思就變成:「在每一個模型裡都為真」才叫邏輯有效(valid)。 命題邏輯只有 $2^n$ 個「世界」可數;一階邏輯的「世界(模型)」卻有無窮多種、大小各異——這個躍升,正是一階邏輯威力與麻煩的共同來源。

🐾
為什麼叫「一階」?還有沒有「二階」?

「一階」指的是:量詞只能對「個體」說「所有 / 存在」($\forall x$,$x$ 是一隻狗、一個數)。 二階邏輯(second-order logic)更狂,允許量詞對「性質 / 集合」說話($\forall P$,$P$ 是任意一種性質)。 二階邏輯表達力更強(例如能一句話定義自然數),但它失去了完備性——這正是「一階邏輯」成為數學與 CS 標準工作語言的原因: 它在「表達力」與「乖巧可控」之間,站到了那個甜蜜點上。

和 Prolog 的血緣

如果一階邏輯讓你想起第 45 章的 Prolog,那是因為它們本就是一家人。Prolog 直接「用邏輯來寫程式」: 你寫下的事實與規則,本質上是一階邏輯公式的一個受限片段(霍恩子句 Horn clauses);你下的查詢,是要系統證明一個目標; 而 Prolog 引擎的執行,就是一套自動化的證明搜尋(歸結 resolution 加上 合一 unification)。 

% Prolog:這三行其實就是一階邏輯
human(socrates).                    % Human(socrates).
mortal(X) :- human(X).              % ∀X (Human(X) → Mortal(X)).

% 查詢 ?- mortal(socrates).  引擎會「證明」它 → true
% 你寫的是邏輯;「執行程式」=「找一個證明」。這就是宣告式程式設計的極致。

這是第 1 章「宣告式知識 vs 命令式知識」的最深刻迴響:在 Prolog 裡,你只宣告「什麼是真的」, 讓機器自己去推導你要的答案。一階邏輯,就是這種思路的數學底座。

✏️ 練習 56.4 思考

設論域為「所有人」,述詞 $Loves(x, y)$ 表示「$x$ 愛 $y$」。把下列三句中文形式化成一階邏輯,並特別注意量詞順序:

(a) 每個人都愛著某個人。
(b) 有某個人,被所有人愛著。
(c) 沒有人愛自己。

看解答

(a) $\forall x\, \exists y\, Loves(x, y)$——對每個 $x$,都存在某個他愛的 $y$(各愛各的即可)。

(b) $\exists y\, \forall x\, Loves(x, y)$——存在某個固定的 $y$,被所有 $x$ 愛(一個萬人迷)。

(a) 與 (b) 只是量詞順序對調,意思天差地別——這正是第 13 章「巢狀量詞順序不能換」的核心:(a) 裡的 $y$ 可以隨 $x$ 變;(b) 裡的 $y$ 必須先固定、對所有 $x$ 通用。

(c) $\neg\,\exists x\, Loves(x, x)$,等價於 $\forall x\, \neg Loves(x, x)$(用第 13 章的否定量詞規則:$\neg\exists \equiv \forall\neg$)。 中文:「不存在愛自己的人」=「每個人都不愛自己」。

三句都帶出一階邏輯的威力:這些「所有 / 存在」交織的句子,命題邏輯一句都寫不出來。

可判定性:邏輯要花多少「計算」?

現在把邏輯和計算接起來。既然 $\vdash$、$\models$ 都是機械化的判斷,自然要問:有沒有演算法能自動回答「這個式子有效嗎?可滿足嗎?」? 要花多少代價?答案隨著邏輯的「表達力」拾級而上,一路從「好算」滑向「永遠算不出來」。

命題邏輯:可判定,但是 SAT 很硬

命題邏輯可判定(decidable):給任意公式,真值表窮舉 $2^n$ 列,一定能在有限步內回答「恆真 / 可滿足 / 矛盾」。 其中「給定公式,存在讓它為真的賦值嗎?」這個問題有個專屬名字——SAT(布林可滿足性問題,Boolean satisfiability)。

而 SAT,正是第 33 章那位主角:史上第一個被證明是 NP-complete 的問題(庫克–列文定理,Cook–Levin,1971)。 這意味著:雖然 SAT 可判定,但目前沒人知道有沒有快速(多項式時間)演算法解它——它是整個 NP 難度世界的「源頭」。 前面那條對偶關係在這裡發揮作用:判斷論證有效(恆真)= 判斷其否定不可滿足(UNSAT),所以「驗證邏輯論證」和「解 SAT」根本是一體兩面。

我們寫一個最樸素的 SAT 求解器——就是枚舉真值賦值(把第八節跟第 33 章接起來):

import itertools

def naive_sat(clauses, variables):
    """樸素 SAT 求解器(CNF 合取範式)。
       clauses:子句的 list;每個子句是 (變數名, 取正?) 的 list,代表「析取(OR)」。
       回傳一組可滿足的賦值 dict,或 None(表示 UNSAT)。"""
    for vals in itertools.product([False, True], repeat=len(variables)):
        env = dict(zip(variables, vals))
        # 整個式子可滿足 ⟺ 每個子句都至少有一個 literal 為真
        if all(any(env[v] == positive for (v, positive) in clause)
               for clause in clauses):
            return env          # 找到一組解
    return None                 # 跑遍 2^n 種賦值都不行 → 不可滿足

# (P ∨ Q) ∧ (¬P ∨ R) ∧ (¬Q ∨ ¬R)
clauses = [
    [("P", True),  ("Q", True)],     #  P ∨ Q
    [("P", False), ("R", True)],     # ¬P ∨ R
    [("Q", False), ("R", False)],    # ¬Q ∨ ¬R
]
print(naive_sat(clauses, ["P", "Q", "R"]))
# {'P': False, 'Q': True, 'R': False}  ← 一組可滿足解

這支程式正確,但「枚舉 $2^n$」在 $n$ 大時會慢到天荒地老。神奇的是:現實中的 SAT 求解器(SAT solver) 靠著 DPLL、CDCL 等聰明搜尋技巧,能在幾秒內解開含數百萬個變數的工業級實例。 它們已經是晶片驗證、軟體分析、AI 規劃背後的隱形主力——「理論上很難」的問題,在「實務上常常很快」。

一階邏輯:有效性「不可判定」

升到一階邏輯,風景突變。一階邏輯的「有效性」問題——史稱判定問題(Entscheidungsproblem),是希爾伯特在 1928 年公開懸賞的大哉問: 「有沒有一個演算法,輸入任意一階公式,輸出『它是否邏輯有效』?」

1936 年,邱奇(Alonzo Church)與圖靈(Alan Turing)各自獨立給了同一個答案:沒有。一階邏輯的有效性是不可判定的(undecidable)。 (這正是第 49 章「停機問題」的孿生兄弟——圖靈正是為了解決這個判定問題,才發明了圖靈機!)

但這裡有個微妙又美麗的非對稱,它直接源自上一節的完備性:

  • 若 $\varphi$ 確實有效:由完備性,它有一個有限證明。我們可以寫程式枚舉所有可能證明,遲早會找到它、回答「是」。
  • 若 $\varphi$ 無效:枚舉證明會永遠找不到——但程式無法判斷「是還沒找到,還是根本不存在」,只能無止盡跑下去。

所以一階邏輯的有效性是「半可判定(semi-decidable)」:對的會說「對」,錯的卻可能讓你永遠等不到答案。 這跟停機問題一模一樣——而這個「枚舉所有證明」的程式,正是上一節埋的伏筆。

⚠️
別把「不可判定」和「不完備」搞混

它們是兩件不同的壞消息。不可判定是說「沒有演算法能自動判斷有效性」(關於計算);不完備是說「有些真理無法被某個系統證明」(關於證明)。 一階邏輯本身是完備的(哥德爾完備性定理!),只是有效性不可判定。 而下一節的「不完備」,談的不是一階邏輯這個邏輯,而是建立在它之上、描述算術的那套公理理論。慢慢往下讀,你會看清這個分界。

希爾伯特的大夢

1900 年,當代最偉大的數學家大衛・希爾伯特(David Hilbert)在巴黎國際數學家大會上,列出 23 個指引二十世紀的難題。其中第二題, 是要替整個算術建立一個「絕對的一致性(consistency)證明」。這滋長成野心勃勃的希爾伯特綱領(Hilbert's program): 把全部數學建立在一組形式公理上,並用嚴格的有限方法,證明這套系統(1)一致(永遠推不出矛盾)、(2)完備(每個數學真理都證得出來)、(3)可判定(有演算法判斷任意命題的真假)。 1930 年 9 月,希爾伯特在退休演說中喊出那句名言:「Wir müssen wissen, wir werden wissen.(我們必須知道,我們必將知道。)」 就在同一場柯尼斯堡會議上,一位 24 歲、沉默寡言的年輕人,正準備宣布一個將永遠粉碎這個夢的結果。他叫庫爾特・哥德爾(Kurt Gödel)。

定理在說什麼

1931 年,哥德爾證明了兩件事,合稱不完備定理(incompleteness theorems):

🔑
哥德爾第一不完備定理(直覺版)

任何一致、公理可被機械列舉、且強到足以描述基本算術(加法與乘法)的形式系統 $F$, 一定存在一個命題 $G$,使得 $F$ 既證不出 $G$,也證不出 $\neg G$。而且——只要 $F$ 一致,這個 $G$ 其實是真的。

換句話說:真理,永遠溢出任何一套固定的證明系統。你無法用一組規則,把所有算術真理一網打盡。「為真」嚴格大於「可證」。

這恰恰打碎了希爾伯特綱領的「完備」支柱:不存在一套能證出所有算術真理的公理系統。 而第二不完備定理更狠:這樣的系統 $F$,連「自己是一致的」這件事都無法在自身內部證明(除非它其實不一致)。 希爾伯特原本想用算術自己去擔保算術不矛盾——哥德爾證明這條路原則上走不通。大夢就此終結。

它是怎麼做到的:自我指涉與對角線

哥德爾的核心手法,精巧得令人屏息,而且你已經見過它的近親——第 13 章預告、第 49 章主打的對角論證(diagonal argument)。三步:

  1. 哥德爾編碼(Gödel numbering):把每個符號、每個公式、甚至每個「證明」,都編成一個獨一無二的自然數。於是「關於證明的陳述」,變成了「關於數字的陳述」——系統獲得了談論自己的能力。
  2. 自我指涉(對角線):利用這個編碼,哥德爾構造出一個命題 $G$,它(翻譯成白話)正好說:「$G$ 這個命題,在 $F$ 裡無法被證明。」一個會「指著自己」的句子。
  3. 引爆矛盾:現在問 $F$ 能不能證明 $G$?
    • 若 $F$ 能證明 $G$:那 $G$ 說的「我證不出來」就成了假話——$F$ 證出了假話,$F$ 不健全 / 不一致。✗
    • 所以(只要 $F$ 一致)$F$ 證不出 $G$。但這正是 $G$ 所宣稱的!於是 $G$ 是真的——一個為真、卻無法被證明的命題,活生生出現了。

看出那個熟悉的影子了嗎?這跟「說謊者悖論(這句話是假的)」同源,跟康托爾證明「實數不可數」、圖靈證明「停機問題不可解」用的是同一招對角線。它們是一窩三胞胎:

事實上,有了圖靈,我們能用停機問題反過來推出不完備,而且更簡潔:假設存在一套一致、可機械列舉、又完備的算術系統, 那我們就能判定停機問題了——對任意程式,把「它會停」與「它不會停」兩句話之一去枚舉證明,完備性保證其中一句必有證明,我們遲早找到,於是判定了停機。 但停機問題不可判定(第 49 章),矛盾!所以那樣的完備系統不存在。「不可計算」與「不可證明」,是同一個深淵的兩面。

⚠️
不完備定理「沒有」說的那些事(務必澄清)

這條定理是史上最常被誤用、被神祕化的科學結果。請守住以下幾條:

  • 它沒有說「數學壞了 / 不可靠」。數學照常運轉。它只說:沒有單一一套有限公理能證出所有算術真理。
  • 它沒有說「我們無法知道 $G$ 的真假」。恰恰相反——我們從系統外面清楚看出 $G$ 是真的。「不可證」≠「不可知」。
  • 它不適用於所有形式系統。夠弱的系統反而又完備又可判定:命題邏輯、一階邏輯(完備!)、只含加法的 Presburger 算術都沒事。要「中招」,必須強到能同時表達加法與乘法。
  • 它需要前提:一致 + 公理可機械列舉 + 夠強。少一個都不成立。(一個不一致的系統反而「完備」——因為它什麼都證得出來,包括所有假話。)
  • 別過度延伸到「心靈勝過機器」「上帝存在」「真理是相對的」。這類引申(如 Lucas–Penrose 論證)在哲學上高度有爭議,不是定理本身的內容。誠實的態度是:定理講的是形式系統,別替它加戲。
✏️ 練習 56.5 挑戰

解釋哥德爾第一不完備定理與停機問題的關聯:為什麼「若存在一套完備、一致、可機械列舉的算術系統」就會導致「停機問題可判定」?(把它寫成一段清楚的反證論證。)

看解答

反證。 假設真有一套這樣的系統 $F$:一致、公理可被程式機械列舉、而且完備(每個算術命題 $\varphi$,$F$ 不是證得出 $\varphi$、就是證得出 $\neg\varphi$)。我們要用它造出一台停機判定器,從而導致矛盾。

關鍵前提:「程式 $M$ 對輸入 $w$ 會停機」這件事,可以寫成一個算術命題 $\varphi_{M,w}$(因為程式的執行步驟,終究是對自然數狀態的加減判斷,能在算術裡編碼)。而且「$M$ 停」為真 $\iff \varphi_{M,w}$ 為真。

現在造判定器:給定 $M$、$w$,我們同時枚舉 $F$ 的所有證明(因為公理可機械列舉,所有合法證明也能逐一生成)。一旦看到 $\varphi_{M,w}$ 的證明,就回答「會停」;一旦看到 $\neg\varphi_{M,w}$ 的證明,就回答「不會停」。

由完備性,這兩者必有其一存在;由一致性,不會兩者都出現(否則矛盾)。所以這個枚舉一定會在有限步內停下並給出正確答案——我們判定了停機問題!

但圖靈已證明停機問題不可判定(第 49 章)。矛盾。因此一開始的假設錯誤:不存在這樣一套完備、一致、可機械列舉的算術系統。它若一致且夠強,就必然不完備——這正是哥德爾第一不完備定理。$\blacksquare$

(體會這個論證的核心:完備性把「無界的搜尋」變成「保證會停的搜尋」,於是不可判定性立刻反推出不完備。「不可計算」與「不可證明」,在這裡握手。)

邏輯,無所不在:CS 應用速覽

講了這麼多理論,別忘了:邏輯是 CS 最「實用」的數學之一。它不是博物館裡的古董,而是天天在發電的引擎。快速巡禮一遍——

  • 電路設計(第 19 章):命題邏輯就是布林代數,而每個邏輯閘(AND/OR/NOT)是一條連接詞規則的矽晶片化身。CPU 的正確性驗證,大量依賴 SAT 求解器做「等價性檢查」。
  • 型別系統與 Curry–Howard 同構(第 9 章):這是全章最美的迴響——命題即型別,證明即程式。一個型別 A -> B 對應命題 $A \rightarrow B$;一個該型別的函式,就是一個「從 $A$ 的證明造出 $B$ 的證明」的構造。編譯器做型別檢查,本質上就是在檢查一個證明。直覺主義邏輯與帶型別的 λ 演算,竟是同一個結構的兩種讀法。
  • 形式驗證(formal verification):用邏輯證明軟硬體正確,而非僅僅測試。模型檢驗(model checking)用時序邏輯自動驗證協定與電路(此貢獻拿下 2007 年圖靈獎);定理證明器(theorem prover)如 Coq、Lean、Isabelle,讓人機協作證出機器逐行檢查過的證明——已用於驗證 C 編譯器(CompCert)、作業系統核心(seL4),甚至形式化了四色定理與當代前沿數學。
  • 資料庫查詢(第 37 章):關聯式資料庫的理論基礎是關係演算(relational calculus),本質就是一階邏輯的一個片段。你寫的 SQL WHERE 條件,骨子裡是一階公式;查詢的執行,是在「找出讓公式為真的所有資料列」。
  • AI 知識表示與推理:從早期專家系統、描述邏輯(Description Logic)與本體論(ontology),到現代用 SAT/SMT 求解器做規劃與驗證——「讓機器自動推理」始終是 AI 的一條主軸,而它的語言就是邏輯。

Curry–Howard:證明就是程式

這值得再強調一次,因為它太深刻了。在 Curry–Howard 同構下:

命題 ⟷ 型別;  證明 ⟷ 程式;  證明化簡 ⟷ 程式執行。

於是「寫一個正確的程式」和「證明一個定理」變成字面上同一件事。這不是比喻,而是 Coq、Lean、Agda 這些工具賴以運作的數學事實—— 你在它們裡頭「寫程式」,就是在「做證明」;它們替你「檢查型別」,就是在「驗證證明無誤」。第 1 章的「宣告式 vs 命令式」、第 9 章的型別、第 13 章的證明,在這裡合流成同一條大河。

✏️ 練習 56.6 挑戰

把「驗證一個論證是否有效」完全轉化成 SAT(UNSAT) 問題,並用前面的 naive_sat 實作出來。 具體地:寫一個函式 is_valid(premises, conclusion, variables),利用「論證有效 $\iff$ 『前提全成立但結論不成立』不可滿足」這條對偶關係。用它驗證 Modus Ponens $\{P,\ P \rightarrow Q\} \models Q$。

看解答

核心觀念:論證 $\{\psi_1,\dots,\psi_n\} \models \varphi$ 有效,等價於合取式 $\psi_1 \wedge \cdots \wedge \psi_n \wedge \neg\varphi$ 不可滿足—— 因為「可滿足」就剛好是「前提全真、結論卻假」的那個反例。所以「有效」= 找不到反例 = UNSAT:

import itertools

def is_valid(premises, conclusion, variables):
    """論證有效 ⟺ (所有前提 ∧ ¬結論) 不可滿足。
       這裡 premises / conclusion 都是吃 **env、回傳 bool 的函式。"""
    for vals in itertools.product([False, True], repeat=len(variables)):
        env = dict(zip(variables, vals))
        # 找「前提全真、結論卻假」的反例賦值
        if all(p(**env) for p in premises) and not conclusion(**env):
            return False        # 反例存在 → 可滿足 → 論證無效
    return True                 # 找不到反例 → UNSAT → 論證有效

# Modus Ponens:{P, P → Q} ⊨ Q ?
print(is_valid([lambda P, Q: P, lambda P, Q: (not P) or Q],
               lambda P, Q: Q, ["P", "Q"]))     # True ✔

# 對照組:肯定後件謬誤 {Q, P → Q} ⊨ P ?
print(is_valid([lambda P, Q: Q, lambda P, Q: (not P) or Q],
               lambda P, Q: P, ["P", "Q"]))     # False ✘

你會發現 is_valid 跟前面的 entails 幾乎一模一樣——這不是巧合,而是「邏輯蘊涵」「恆真」「UNSAT」三者本是同一枚硬幣的三面。 這也解釋了為什麼一個夠快的 SAT 求解器,就能拿來自動驗證邏輯論證、檢查電路等價、甚至做程式分析:現實世界把一大票「推理問題」全都翻譯成了 SAT。

把這一章串起來

我們把「推理」搬上了手術台,看清了它的解剖構造。回顧這趟旅程的路標:

  • 語法 vs 語意是現代邏輯的第一刀:一個式子既是「一串符號(WFF)」,又「在所有世界裡有它的真假命運」。由此分出恆真式、矛盾式、可滿足,以及「驗證有效 = 證明恆真 = 證 UNSAT」這條對偶。
  • $\models$ 與 $\vdash$ 是兩個世界:$\models$ 關乎真理(看遍所有解釋),$\vdash$ 關乎證明(純符號接龍)。自然演繹用引入/消去規則把證明寫成可機械檢查的接龍。
  • 健全性 + 完備性讓這兩個世界完美重合:健全(推得出的都為真,系統不說謊)、完備(為真的都推得出,系統不漏接)。哥德爾完備性定理保證一階邏輯「剛剛好」。
  • 一階邏輯給世界一個「模型」,用量詞、述詞、函數談「所有」與「存在」,是數學、Prolog、資料庫的共同工作語言。
  • 可判定性:命題邏輯可判定(但 SAT 是 NP-complete),一階邏輯有效性不可判定(只半可判定),與停機問題同源。
  • 哥德爾不完備定理是高潮:夠強且一致的系統,必有「真而不可證」的命題;真理永遠溢出任何固定的證明系統。它用的對角線,與康托爾、圖靈是一窩三胞胎——別把它神祕化、別過度延伸。

最後,請帶走這個對稱之美。整本書有兩條「疆界」反覆出現:第 49 章的計算理論劃出「可計算」的邊界(有些問題,沒有任何演算法能解); 而這一章的形式邏輯劃出「可證明」的邊界(有些真理,沒有任何系統能證)。 哥德爾與圖靈用的是同一把對角線鑰匙,從兩個方向打開了同一道深淵。

邏輯把「推理」本身變成了研究對象;而它最偉大的發現,是替推理找到了它自己的極限。

懂了「可證明」與「可計算」這兩道互為鏡像的疆界,你就握住了二十世紀數學與電腦科學最深的那個祕密。
