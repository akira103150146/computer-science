為什麼會有這本書?

網路上從來不缺免費的計算機科學資源。MIT、史丹佛、哈佛把整門課放上網; OSSU(Open Source Society University) 甚至把這些課程編成一份結構嚴謹、媲美正規學位的課程地圖(curriculum)。 問題從來不是「資源不夠」,而是——

面對一份列著三十幾門課、上百個連結的清單,大多數人撐不過第三週。

為什麼?因為一份連結清單是地圖,不是嚮導。它告訴你「該去哪」, 卻沒有牽著你的手走過去。每換一門課,你就要重新適應一位教授的口音、一套新的符號、 一種陌生的講法。知識被切成互不相連的碎片,你很難看見它們如何拼成一張完整的圖。

這本書就是那位嚮導。我們以 OSSU 的課程大綱為骨架,用一致的敘事口吻、 一致的符號、一致的中英對照,把整套 CS 必修知識重新講一遍——從「計算是什麼」這種最素樸的問題出發, 一路講到演算法、作業系統、編譯器、機器學習與計算理論。你只需要打開第一章,然後一直往下讀。

ℹ️
本書與 OSSU 的關係

本書不取代原始課程,而是當你的「主線敘事」。我們把每門課最核心的理論講透、 給你直覺與程式碼;當你想更深入(尤其是想拿證書、做完整作業)時,書中會指向對應的原始課程。 把這本書當成「一本讀得完的教科書」,把 OSSU 課程當成「延伸的習題與實驗室」。

本書的編排哲學

每一章,我們都遵守同一套「由淺入深」的節奏。理解這個節奏,你會讀得更順:

1. 先問「為什麼」,再講「是什麼」

我們幾乎不會劈頭就丟給你一個定義或公式。相反地,我們會先設計一個你會卡住的情境, 讓你親身感受「啊,這裡真的需要一個更好的辦法」,然後那個概念才登場——這時它不再是天上掉下來的術語, 而是一個你打從心底需要的工具。

2. 中英對照,術語不迷路

所有重要概念第一次出現時,都會標註英文原文,例如遞迴(recursion)、 攤銷分析(amortized analysis)。因為日後你查資料、讀論文、看 Stack Overflow, 九成是英文。我們希望中文幫你建立直覺,英文幫你接軌世界。

3. 理論立刻配上可執行的程式碼

每個抽象概念,我們都用程式碼讓它「活」起來。原則是:

  • Python:用來快速驗證直覺。語法乾淨,讓你專注在「想法」本身。
  • C / C++:用來看見「底層真相」。當我們談記憶體、指標、效能時,C 不會幫你隱藏任何東西。

書中每一段程式碼都經過設計,讓你可以直接複製、貼上、執行。把手弄髒,是學會的唯一方法。

4. 每章都有練習,而且附解答

看懂,和會做,是兩件事。每章都穿插思考題與程式題, 並附上可展開的解答。請務必先自己想,卡住了再看。底下示範練習區塊長什麼樣子:

✏️ 練習 0.1 暖身

不寫任何程式,純用文字回答:當你在瀏覽器網址列輸入 google.com 並按下 Enter, 到你看到搜尋頁面為止,中間「大概」發生了哪些事?盡量寫出你想得到的步驟。

看看參考方向(讀完整本書你會給出更精確的答案)

這是經典的面試題,牽涉本書好幾個部分:

  • DNS 查詢把 google.com 轉成 IP 位址(第五部:網路)。
  • 瀏覽器透過 TCP/IP 與伺服器建立連線,用 HTTP/HTTPS 發出請求(第五部、第七部)。
  • 作業系統處理這些網路封包、行程與系統呼叫(第五部:作業系統)。
  • 伺服器端可能查詢資料庫、跑搜尋演算法並排序結果(第六、八部)。
  • 回傳的 HTML/CSS/JS 由瀏覽器(本身是個複雜的程式)解析與渲染。

現在你或許只能寫出幾句模糊的描述——這完全正常。讀完本書再回來看這題,你會驚訝於自己的進步。

你也會看到各種「提示框」,幫你區分資訊的輕重緩急:

💡
小技巧實用的撇步或捷徑,讓你少走冤枉路。
⚠️
常見陷阱初學者最容易在這裡犯錯,讀到請放慢速度。
🔑
核心觀念整節最重要的一句話。如果只能記住一件事,就記這個。
📜
歷史小注這個想法從哪來、是誰想出來的——脈絡會讓知識更立體。

學習路線圖:從這裡到 CS 學位

本書分成十五部,對應一個完整 CS 學位的學習順序。你不必嚴格照順序讀, 但若你是初學者,強烈建議從第一部開始,因為後面的內容會一層一層疊在前面之上。

三個階段

  • 入門(第一部):確認 CS 適不適合你。學會用程式表達想法。
  • 核心(第二～九部):相當於 CS 大學前三年的必修。每一部都建議讀完,這是你的內功。
  • 進階(第十～十四部):相當於大四的選修。挑你感興趣的領域深入即可,不必全讀。
💡
建議的修課節奏

數學(第三部)可以和程式設計(第一、二部)並行:每天讀一點數學、寫一點程式, 避免單一主題讀到膩。但「計算機系統」與「演算法」(第五、六部)請等程式設計與數學打好底再開始。

開始之前:先備知識

這本書假設你具備:

  • 高中數學:代數、基本幾何、函數的概念。微積分我們會在第三部從頭教,不用擔心。
  • 基本電腦操作:會安裝軟體、會用瀏覽器、知道「檔案」與「資料夾」是什麼。
  • 英文閱讀的意願:不需要很強,但要願意查單字。CS 的世界以英文為母語。

就這樣。不需要任何程式設計經驗——我們從 print("Hello") 開始。

準備你的工具:開發環境設定

動手寫程式之前,先把兩套環境裝好:Python(主力)與 C/C++ 編譯器(看底層)。

安裝 Python

到 python.org 下載 Python 3。 Windows 使用者請務必勾選「Add Python to PATH」。裝好後,打開終端機(Windows 是 PowerShell,macOS/Linux 是 Terminal),輸入: 

python --version
# 或某些系統上是
python3 --version

看到類似 Python 3.12.x 就成功了。我們再寫個經典的第一支程式:

# hello.py
print("Hello, 計算機科學!")

存成 hello.py,在終端機執行 python hello.py,看到輸出就代表 Python 已就緒。

安裝 C / C++ 編譯器

我們用 GCC(GNU Compiler Collection)這套自由、跨平台的標準編譯器:

  • Windows:安裝 MSYS2,再用它安裝 mingw-w64-gcc;或直接裝 Visual Studio 的 C++ 工具。
  • macOS:在終端機執行 xcode-select --install。
  • Linux:sudo apt install build-essential(Debian/Ubuntu)。

驗證安裝並編譯第一支 C 程式:

/* hello.c */
#include <stdio.h>

int main(void) {
    printf("Hello, 底層世界!\n");
    return 0;
}
gcc hello.c -o hello   # 編譯,產生執行檔 hello
./hello                # 執行(Windows 為 .\hello.exe)
🌐
還不想安裝?先用線上環境

想立刻開始而不想處理安裝,可以先用線上編譯器,例如 online-python 或 Compiler Explorer(C/C++)。 但長遠來說,把本機環境裝好是值得的——這本身就是 CS 技能的一部分(見第四部)。

如何「真正」學會:給自學者的方法論

方法不對,再好的教材也是白搭。以下幾點來自認知科學,也來自無數自學者的血淚:

主動回想,而不是被動重讀

讀完一節,闔上書,試著用自己的話複述剛剛學到什麼。這個動作叫 主動回想(active recall),它逼迫大腦「提取」記憶,遠比反覆重讀有效。 本書每章的練習,就是為此設計的。

間隔重複,對抗遺忘

人會遺忘,這是生理事實。對抗的辦法是間隔重複(spaced repetition): 今天學的東西,明天、一週後、一個月後各複習一次。把重要的定義與直覺做成卡片(例如用 Anki), 效果驚人。

動手做,而且要做到「會痛」

CS 是一門「動詞」,不是「名詞」。你不會因為「讀過」排序演算法就學會它, 你會因為「親手寫出來、寫錯、除錯、再寫對」而學會。本書的程式碼請你親手打一遍, 不要只是讀過去——打字的過程中,你的手會教會你的腦。

🔑
核心觀念

學習不是發生在「你讀懂的那一刻」,而是發生在「你卡住、掙扎、最後突破的那一刻」。 所以,當你覺得某個練習很難——恭喜,那正是你成長最快的地方。不要跳過難的部分。

費曼技巧:能教,才是真懂

物理學家費曼(Richard Feynman)的學習法:挑一個概念,假裝你要教給一個完全不懂的人, 用最白話的方式講出來。一旦你卡住、講不下去,那個卡點就是你其實沒搞懂的地方——回去補,再講一次。 如果你身邊沒有聽眾,講給橡皮鴨聽也行(這在程式設計界叫 小黃鴨除錯法)。

關於時間:這要學多久?

誠實地說:如果照 OSSU 的完整規劃,每週投入約 20 小時,大約兩年能走完核心與部分進階。 這聽起來很長,但請記得兩件事:

  1. 你不是從零到全部一次學完才有用。每讀完一部,你的能力都會實質提升,而且能立刻用上。
  2. 本書是「主線敘事」,讀的速度會比修完整門課快很多。你可以先用本書建立全貌, 再針對最感興趣的部分回頭深入原始課程。

重點從來不是「多快讀完」,而是「有沒有持續前進」。每天 30 分鐘、風雨無阻, 勝過心血來潮的一天 8 小時。

🐢
慢即是快

自學最大的敵人不是難度,是放棄。把目標訂得小一點、可持續一點: 「今天讀完一節、寫完一題」就好。讓學習變成習慣,剩下的交給時間。

準備好了嗎?

如果你讀到這裡還沒跑掉,那麼你已經具備了最稀缺的特質:把一件難事讀完的耐心。 這份耐心,會帶你走完整本書。
