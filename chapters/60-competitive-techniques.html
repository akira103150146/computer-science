競賽的真相:先暴力,再優化

如果你剛從「會寫程式」走進「競賽程式設計」,大概會有個錯覺:高手都是一眼就看穿某個華麗演算法。 其實恰恰相反。絕大多數題目,正解的第一步都是先想出一個一定對、但很慢的暴力解, 然後問自己一句話:「它慢在哪?哪個步驟被我重複做了太多次?」——接著用對的工具,把那個瓶頸抹掉。

舉個最小的例子。給你一個長度 $n$ 的陣列,要回答 $q$ 筆「區間 $[l,r]$ 的總和是多少」。 暴力解直白到不行:每次詢問就從 $l$ 加到 $r$。可是每筆詢問最壞要加 $n$ 個數,$q$ 筆就是 $O(nq)$; 當 $n=q=2\times10^5$,那是 $4\times10^{10}$ 次運算,鐵定逾時。但只要先花 $O(n)$ 算一張前綴和表, 之後每筆詢問都能 $O(1)$ 回答,總共 $O(n+q)$。同一個問題,從「跑不完」變成「眨眼就好」。

這一章的每一把刀,本質上都在做同一件事:把「重複的計算」預先算好、或把「搜尋的範圍」聰明地砍掉。 它們不華麗,卻是 APCS 第三、四題與各大線上裁判最高頻的得分點。我們從最樸素的「枚舉」開始, 一路把工具箱補滿。

🔑
競賽解題的心法:暴力 → 找瓶頸 → 換工具

先寫出「一定正確」的暴力解,用它確認你真的懂題目;再用複雜度估算判斷它會不會逾時; 最後針對最慢的那一步,換上前綴和、差分、二分搜、雙指針或離散化。 這套流程比「背一堆演算法」可靠得多——因為它教你怎麼想,而不只是記得什麼。

枚舉與狀態壓縮:把暴力寫得有章法

枚舉(enumeration),又叫暴力搜尋(brute force),就是把所有可能的答案 一一列出來,逐個檢查哪個對。它的優點是直觀又萬無一失:只要可能的解是有限的,枚舉一定找得到。 缺點也很明顯——慢。所以用枚舉前,第一件事永遠是估算:「可能的解有幾個?跑得完嗎?」

先學會「數」:可能性有多少?

要枚舉,得先知道枚舉對象的數量級。這裡的算術,正是第 26 章計算複雜度的實戰應用—— 一秒鐘大約能做 $10^7 \sim 10^8$ 次基本運算,超過就要換方法:

枚舉對象數量大致可行的 $n$典型場景
所有數值 / 單一迴圈$O(n)$$n \le 10^8$找最大、加總
所有數對 $(i,j)$$O(n^2)$$n \le 5000$兩兩比較
所有子集合$O(2^n)$$n \le 20\!\sim\!25$選或不選
所有排列$O(n!)$$n \le 10\!\sim\!11$順序問題
折半枚舉(meet in the middle)$O(2^{n/2})$$n \le 40$子集合和 / 乘積

看出規律了嗎?$2^n$ 與 $n!$ 長得飛快,$n$ 稍微一大就「指數爆炸」。所以當你看到題目限制 $n \le 20$,那幾乎是在對你大喊:「這題可以枚舉子集合!」限制的大小,常常就是解法的提示。

用位元枚舉所有子集合

「選或不選」是競賽裡最常見的枚舉。$n$ 個元素,每個都有「選」與「不選」兩種狀態, 所以共有 $2^n$ 種組合(子集合)。怎麼把它們全部跑一遍?最漂亮的寫法, 是把每個子集合對應成一個 $n$ 位的二進位數——這正是第 59 章位元運算的拿手好戲, 也叫狀態壓縮(bitmask):用一個整數的每個位元,記錄一個元素「在不在集合裡」。

以 3 個元素 $\{a_0,a_1,a_2\}$ 為例,從 $0$ 數到 $2^3-1=7$,每個整數的二進位剛好對應一種選法:

mask二進位第 2 / 1 / 0 位選到的子集合
0000不 / 不 / 不$\{\,\}$(空集合)
1001不 / 不 / 選$\{a_0\}$
2010不 / 選 / 不$\{a_1\}$
3011不 / 選 / 選$\{a_0,a_1\}$
5101選 / 不 / 選$\{a_0,a_2\}$
7111選 / 選 / 選$\{a_0,a_1,a_2\}$

判斷「第 $j$ 個元素在不在 mask 裡」,就看 mask 的第 $j$ 位是不是 1: 用 mask & (1 << j)。整段子集合枚舉,於是就是一個外層 $2^n$、內層 $n$ 的雙迴圈: 

#include <bits/stdc++.h>
using namespace std;

int main() {
    vector<int> a = {3, 1, 4, 1, 5};   // 5 個元素
    int n = a.size();
    int target = 8;

    // 枚舉 0 ~ 2^n - 1,每個整數代表一種「選 / 不選」組合
    for (int mask = 0; mask < (1 << n); mask++) {
        int sum = 0;
        for (int j = 0; j < n; j++)
            if (mask & (1 << j))       // 第 j 位是 1 → 選 a[j]
                sum += a[j];
        if (sum == target) {           // 找到一組和為 target 的子集合
            for (int j = 0; j < n; j++)
                if (mask & (1 << j)) cout << a[j] << ' ';
            cout << '\n';
        }
    }
    return 0;
}

Python 寫起來更短,概念完全一樣:

a = [3, 1, 4, 1, 5]
n = len(a)
target = 8

for mask in range(1 << n):                       # 0 ~ 2^n - 1
    subset = [a[j] for j in range(n) if mask >> j & 1]
    if sum(subset) == target:
        print(subset)
💡
位元枚舉 vs 遞迴 DFS

枚舉子集合有兩種寫法:位元迴圈(上面這種)與遞迴 DFS(每個元素「選 / 不選」兩條分支)。 位元版短、沒有遞迴開銷,還能直接把子集合當成整數拿去當索引或雜湊; 但它很難剪枝。遞迴版能剪枝——一旦發現某條路不可能成功(例如和已超過目標), 就能砍掉它「後面所有」的分支,效率常常好得多。$n$ 小就用位元版圖個方便,需要剪枝就用遞迴。

聰明的枚舉:換個對象、少枚一層

枚舉的藝術,是挑對枚舉對象。看這個經典小題:求 $\max(a_j - a_i)$,其中 $i \lt j$ (買低賣高的最大價差)。直覺是枚舉所有數對 $(i,j)$,$O(n^2)$。但仔細想:對固定的 $j$, 我們要的 $a_i$ 永遠是「$j$ 左邊出現過的最大值」。於是只要枚舉 $j$,順手維護一個「目前看過的最大值」就好, 一層迴圈 $O(n)$ 搞定: 

int ans = INT_MIN, premax = INT_MIN;
for (int j = 0; j < n; j++) {
    if (premax != INT_MIN)
        ans = max(ans, a[j] - premax);  // 和「左邊最大值」配對
    premax = max(premax, a[j]);         // 更新左邊最大值
}

「少枚一層」的念頭——把一個維度用已知資訊替換掉——正是接下來所有技巧的共同靈魂。 前綴和、雙指針、二分搜答案,全都是這個念頭的不同化身。

前綴和:把「區間和」變成一次減法

回到開場那個「查區間和」的問題。前綴和(prefix sum)的點子簡單到近乎作弊: 先算好「從頭到每個位置」的累積和,那麼任何一段區間的和,都能用兩個前綴和相減得到。

正式一點:給定陣列 $A$(這裡用 1-indexed,$A_1, A_2, \dots, A_n$),定義前綴和 $S_0 = 0$、$S_i = S_{i-1} + A_i$。那麼區間 $[l, r]$ 的和就是

$$ \sum_{k=l}^{r} A_k = S_r - S_{l-1}. $$

為什麼?$S_r$ 是「$A_1$ 到 $A_r$」的總和,$S_{l-1}$ 是「$A_1$ 到 $A_{l-1}$」的總和, 兩者一減,前面那段 $A_1 \dots A_{l-1}$ 剛好抵銷,只剩下 $A_l \dots A_r$。用 AP325 的經典例子 $A = [1,5,3,6,2,4]$ 來看: 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int main() {
    int n;
    cin >> n;
    vector<LL> S(n + 1, 0);             // S[0] = 0 當哨兵
    for (int i = 1; i <= n; i++) {
        LL x; cin >> x;
        S[i] = S[i - 1] + x;           // 前綴和遞推
    }

    int q;
    cin >> q;
    while (q--) {
        int l, r;
        cin >> l >> r;
        cout << S[r] - S[l - 1] << '\n';  // O(1) 回答區間和
    }
    return 0;
}
⚠️
前綴和最愛的兩個坑:溢位與差一

第一,前綴和會越加越大。$2\times10^5$ 個各 $10^9$ 的數,總和可達 $2\times10^{14}$, 遠超 32 位元 int 的上限——請用 long long。 第二,務必用 $S_0 = 0$ 的哨兵與 1-indexed,否則查 $l=1$ 時會碰到 $S_{l-1}=S_0$ 這個邊界。把這兩件事變成肌肉記憶,能省下無數次 WA。

二維前綴和:矩形區域和

一維能做的,二維照樣能做——只是要小心「重疊」。定義 $P_{i,j}$ 為「從左上角 $(1,1)$ 到 $(i,j)$ 這塊矩形」 的所有元素和。建表時,每格等於「上面那塊 + 左邊那塊 − 重複算到的左上角 + 自己」:

$$ P_{i,j} = P_{i-1,j} + P_{i,j-1} - P_{i-1,j-1} + A_{i,j}. $$

而要查「左上角 $(x_1,y_1)$、右下角 $(x_2,y_2)$」這塊子矩形的和,用的是容斥原理: 拿大矩形,扣掉上方長條、扣掉左方長條,再把被扣兩次的左上角補回來:

$$ \text{sum} = P_{x_2,y_2} - P_{x_1-1,y_2} - P_{x_2,y_1-1} + P_{x_1-1,y_1-1}. $$

// A、P 皆 1-indexed,P 的第 0 列、第 0 行全為 0
vector<vector<LL>> P(n + 1, vector<LL>(m + 1, 0));
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++)
        P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + A[i][j];

// 左上 (x1,y1)、右下 (x2,y2) 的矩形和
LL rect = P[x2][y2] - P[x1-1][y2] - P[x2][y1-1] + P[x1-1][y1-1];

差分:把「區間加值」變成一次加減

前綴和解決了「重複查區間和」。那「重複改一整段」呢?設想:你要對陣列做 $m$ 次操作, 每次把區間 $[l,r]$ 的每個元素都加上 $x$。暴力做法是每次跑一個 for 迴圈,$O(nm)$——又逾時了。 這時換差分(difference array)上場。

差分陣列定義為「相鄰兩項的差」:$D_1 = A_1$,$D_i = A_i - A_{i-1}$。它有個關鍵性質: 對 $D$ 做前綴和,就會還原回 $A$。理解了這點,神奇的事就來了—— 要把 $A$ 的區間 $[l,r]$ 整段加 $x$,只需要動 $D$ 的兩個位置:

$$ D_l \mathrel{+}= x, \qquad D_{r+1} \mathrel{-}= x. $$

為什麼?$D_l$ 加了 $x$,做前綴和時,從位置 $l$ 開始的每一項都會多 $x$; 而 $D_{r+1}$ 減了 $x$,讓「從 $r+1$ 開始」的多餘部分被抵銷掉。一加一減之間, 剛好只有 $[l,r]$ 這段被墊高了 $x$。每次操作 $O(1)$,全部改完最後做一次前綴和 $O(n)$, 總共 $O(n+m)$。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int n, m;                              // n 個元素,m 筆區間加值
vector<LL> D;                          // 差分陣列(多開一格放 r+1)

void range_add(int l, int r, LL x) {   // 區間 [l,r] 加 x,O(1)
    D[l]     += x;
    D[r + 1] -= x;
}

int main() {
    cin >> n >> m;
    D.assign(n + 2, 0);
    while (m--) {
        int l, r; LL x;
        cin >> l >> r >> x;
        range_add(l, r, x);
    }
    // 對差分取前綴和,得到最終陣列
    LL cur = 0;
    for (int i = 1; i <= n; i++) {
        cur += D[i];
        cout << cur << " \n"[i == n];
    }
    return 0;
}
🔑
前綴和與差分,互為逆運算

這是本章最該記住的一句話:差分是前綴和的反操作。對 $A$ 做差分得到 $D$, 再對 $D$ 做前綴和,就完整還原回 $A$——一來一回,什麼都沒變。所以兩者是一體兩面: 前綴和把「逐點的值」變成「累積量」,擅長 $O(1)$ 查詢區間和; 差分把「累積的修改」推遲到最後,擅長 $O(1)$ 完成區間加值。 看到「多次查區間和」想前綴和,看到「多次改一整段」想差分。

二維差分:對整塊矩形加值

同樣能升到二維。要對矩形 $[(x_1,y_1),(x_2,y_2)]$ 整塊加 $x$,在二維差分陣列上動四個角 (又是容斥的味道): 

void rect_add(int x1, int y1, int x2, int y2, LL x) {
    D[x1][y1]         += x;
    D[x1][y2 + 1]     -= x;
    D[x2 + 1][y1]     -= x;
    D[x2 + 1][y2 + 1] += x;
}
// 全部操作做完,對 D 做「二維前綴和」即還原成加值後的矩陣

是不是越來越熟悉?一維動兩點、二維動四點,背後都是同一個「差分埋點、前綴和結算」的思想。 把它和上一節的二維前綴和對照,你會更深刻地體會兩者的對稱之美。

二分搜進階:對「答案」二分搜

二分搜你在第 5、28 章已經很熟了:在已排序的陣列裡找一個值,每次看中間、砍掉一半, $O(\log n)$。但競賽裡二分搜真正的威力,藏在一個更抽象的招式——二分搜答案 (binary search on the answer)。它的舞台不是「現成的陣列」,而是「答案本身的值域」。

關鍵前提:答案具有單調性

很多最佳化問題長這樣:「求最小的 $m$,使得某條件成立」或「求最大的 $m$,使得……」。 如果這個條件具有單調性——也就是「一旦某個 $m$ 可行,所有比它更寬鬆的 $m$ 也都可行」—— 那麼把「可行 / 不可行」沿著 $m$ 由小到大排開,一定長成「不不不…可可可」這種一刀切的形狀。 而我們要的答案,正是那道分界線:第一個可行的 $m$。既然有序,就能二分!

於是套路固定成三步:

  1. 確定值域:答案 $m$ 的下界 lo 與上界 hi。
  2. 寫可行性檢查 feasible(m):給定 $m$,判斷「能不能達成」——通常是個貪心或模擬。
  3. 二分:在 [lo, hi] 上猜 mid,可行就往小縮、不可行就往大找。

注意這個漂亮的轉換:原本難解的最佳化問題(「最小的 $m$ 是多少」), 被換成了一連串好解的判定問題(「$m$ 行不行」)。這往往讓難題瞬間崩解。

經典例:最小化「最大段和」

來個課本級的例子(它是 APCS 與各大裁判的常客,別名「抄書問題」「分配問題」):

有 $n$ 本書,第 $i$ 本有 $a_i$ 頁(順序固定)。要把它們分給 $K$ 個人抄,每人負責連續的一段。如何分,才能讓「抄最多頁的那個人」抄得最少?也就是,最小化各段頁數和的最大值。

直接想怎麼分很燒腦。但換成二分搜答案,就清爽了。設答案(那個「最大段和」)為 $m$,問: 「每段和都不超過 $m$ 的前提下,最少要切成幾段?」用貪心一路塞,塞不下就開新段。 若需要的段數 $\le K$,代表 $m$ 這個上限夠寬鬆、可行;否則太緊、不可行。 而 $m$ 越大越容易可行——單調性成立,可以二分!

值域怎麼定?$m$ 至少要能裝下最厚的那一本書(否則那本書放不進任何段),所以 $\text{lo} = \max a_i$;$m$ 最多就是全部頁數加起來(全部塞一段),$\text{hi} = \sum a_i$。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int n, K;
vector<LL> a;

// 每段和都 ≤ limit 時,最少需要幾段?能否 ≤ K 段?
bool feasible(LL limit) {
    int groups = 1;
    LL cur = 0;
    for (LL x : a) {
        if (x > limit) return false;     // 單一本就超過上限,絕不可能
        if (cur + x > limit) {           // 這段放不下了 → 開新的一段
            groups++;
            cur = x;
        } else {
            cur += x;
        }
    }
    return groups <= K;
}

LL solve() {
    LL lo = *max_element(a.begin(), a.end());       // 下界:最厚的一本
    LL hi = accumulate(a.begin(), a.end(), 0LL);    // 上界:全部一段
    while (lo < hi) {
        LL mid = lo + (hi - lo) / 2;
        if (feasible(mid)) hi = mid;     // 可行 → 試更小的上限
        else               lo = mid + 1; // 不可行 → 上限得更大
    }
    return lo;   // lo == hi,即第一個可行的 m
}

用 Python 表達同一套邏輯:

def feasible(a, K, limit):
    groups, cur = 1, 0
    for x in a:
        if x > limit:
            return False
        if cur + x > limit:      # 放不下,開新段
            groups += 1
            cur = x
        else:
            cur += x
    return groups <= K

def solve(a, K):
    lo, hi = max(a), sum(a)
    while lo < hi:
        mid = (lo + hi) // 2
        if feasible(a, K, mid):
            hi = mid             # 可行,往小縮
        else:
            lo = mid + 1         # 不可行,往大找
    return lo

複雜度是 $O(n \log(\sum a_i))$:每次檢查掃一遍 $O(n)$,二分把值域砍 $\log$ 次。 相較於枚舉所有切法的天文數字,這是壓倒性的勝利。「最小化最大值」「最大化最小值」 這類字眼,幾乎就是二分搜答案的招牌——看到它們,先想單調性。

⚠️
二分搜答案的三個邊界陷阱

(1) 收斂條件:用 while (lo < hi) 配 hi = mid / lo = mid + 1 找「第一個可行」,迴圈結束時 lo == hi 就是答案,不會無窮迴圈。 若改成在浮點數值域上二分(例如答案是小數),則用「跑固定 100 次」或 while (hi - lo > eps)。 (2) 值域方向:想清楚到底是「越大越可行」還是「越小越可行」,別把縮放方向寫反。 (3) 可行性檢查要正確且夠快——它會被呼叫 $O(\log)$ 次,本身別藏一個 $O(n^2)$。

雙指針與滑動視窗:兩個只往右走的指標

雙指針(two pointers)是另一種「少枚一層」的魔法。考慮這題:給定一個正整數陣列, 求「總和不超過 $S$」的最長連續子陣列有多長。暴力枚舉所有 $(l, r)$ 再求和是 $O(n^2)$ (即使配前綴和也要 $O(n^2)$ 枚舉端點)。但這裡有個可以利用的單調性: 當左端固定,右端越往右,區間和只會越大(因為都是正數)。

於是我們讓兩個指標 l、r 都只往右移動,維護一個「視窗」$[l, r]$: 右端 r 不斷吃進新元素;一旦視窗和超過 $S$,就讓左端 l 往右縮,直到合法為止。 因為兩個指標各自最多走 $n$ 步,總共只 $O(n)$——這就是滑動視窗(sliding window)。 

// a 為正整數,求總和 ≤ S 的最長連續子陣列長度
int longest_le(const vector<int>& a, LL S) {
    int l = 0, best = 0;
    LL sum = 0;
    for (int r = 0; r < (int)a.size(); r++) {
        sum += a[r];                 // 右端吃進新元素
        while (sum > S) {            // 太大 → 左端往右縮
            sum -= a[l];
            l++;
        }
        best = max(best, r - l + 1); // 此刻的合法視窗長度
    }
    return best;
}
ℹ️
雙指針與前綴和是親戚

同一個「區間和」問題,前綴和靠的是「先全部算好、再相減」,適合離線、任意端點的查詢; 雙指針靠的是「視窗連續滑動、增量維護」,適合找滿足某條件的最佳區間。 當題目的條件具有單調性(右端右移使某量單調變化),雙指針常能把前綴和 + 二分的 $O(n\log n)$ 再壓成 $O(n)$。兩者都建立在「不要重算已知資訊」這個核心信念上。

離散化:把稀疏的大數值壓進小編號

最後一把刀,專治一種惱人情況:數值範圍大到不像話(例如座標到 $10^9$),但實際出現的個數卻很少 (例如只有 $2\times10^5$ 個)。這時你沒辦法開一個 $10^9$ 大小的陣列(記憶體爆掉), 但其實你只在乎這些數的相對大小,不在乎它們的絕對值。

離散化(coordinate compression)就是把這些零散的大數值,換成它們的排名。 例如 $[88, 7, 9999, 666]$ 排序後是 $[7, 88, 666, 9999]$,於是 $7\to0$、$88\to1$、$666\to2$、$9999\to3$, 原陣列就壓成了 $[1, 0, 3, 2]$——大小關係完全不變,值域卻從 $10^4$ 縮到 $\{0,1,2,3\}$。 做法是「排序 → 去重 → 用二分搜查排名」三件套: 

vector<int> vals = a;                              // 複製一份
sort(vals.begin(), vals.end());
vals.erase(unique(vals.begin(), vals.end()),
           vals.end());                            // 去掉重複,只留相異值

// 把每個 a[i] 換成它在 vals 裡的排名(0-indexed)
for (int& x : a)
    x = lower_bound(vals.begin(), vals.end(), x) - vals.begin();
// 之後 a[i] 都落在 [0, 相異值個數) 之間,可安心當陣列索引

Python 版本同樣三步:

import bisect
vals = sorted(set(a))                  # 排序 + 去重
a = [bisect.bisect_left(vals, x) for x in a]   # 換成排名

離散化常常是其他資料結構的前置處理:當你想用一個「以數值當索引」的工具 (例如第 61 章的樹狀數組(BIT)或線段樹)去處理大值域資料時, 先離散化把值域壓小,工具才開得起。它也常和差分搭配——下一節的綜合練習就會看到。

工具箱:把優化暴力的技巧串起來

到這裡,你的競賽瑞士刀已經配齊了。它們看似各管各的,骨子裡卻共享同一個信念: 「不要重複計算已知的東西,也不要搜尋已知不可能的範圍。」 遇到題目逾時,就拿這張對照表掃一遍,問問哪把刀切得進去:

看到這種瓶頸……就想到這把刀把複雜度從……變成
反覆查「區間 $[l,r]$ 的和」前綴和每次 $O(n)$ → $O(1)$
反覆查「矩形區域和」二維前綴和每次 $O(nm)$ → $O(1)$
反覆對「整段 / 整塊」加值差分每次 $O(n)$ → $O(1)$
求「最小化最大值 / 最大化最小值」二分搜答案枚舉 → $O(\text{檢查}\times\log)$

$n \le 20\!\sim\!25$ 的「選或不選」位元枚舉子集合(本來就只能枚舉)$O(2^n)$

更棒的是,它們能互相疊加:離散化後再差分、前綴和後再二分、雙指針裡藏前綴和…… 高分題往往就是把兩三把刀組合起來。記住流程:先暴力確認懂題、用複雜度判斷逾不逾時、 再針對瓶頸換上對的工具。這就是競賽選手「優化暴力」的全部祕密。

✏️ 練習 60.1 實作

給長度 $n$ 的整數序列與 $q$ 筆詢問,每筆問區間 $[l,r]$(1-indexed)的總和。 $n, q \le 2\times10^5$,元素絕對值 $\le 10^9$。請在總時間 $O(n+q)$ 內回答。

看解答

標準一維前綴和。注意總和會超過 32 位元,必須用 long long;並用 $S_0=0$ 哨兵避免 $l=1$ 的邊界。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int main() {
    int n; scanf("%d", &n);
    vector<LL> S(n + 1, 0);
    for (int i = 1; i <= n; i++) {
        LL x; scanf("%lld", &x);
        S[i] = S[i - 1] + x;
    }
    int q; scanf("%d", &q);
    while (q--) {
        int l, r; scanf("%d %d", &l, &r);
        printf("%lld\n", S[r] - S[l - 1]);
    }
    return 0;
}
import sys
input = sys.stdin.readline
n = int(input())
A = list(map(int, input().split()))
S = [0] * (n + 1)
for i in range(1, n + 1):
    S[i] = S[i - 1] + A[i - 1]
q = int(input())
out = []
for _ in range(q):
    l, r = map(int, input().split())
    out.append(str(S[r] - S[l - 1]))
print('\n'.join(out))
✏️ 練習 60.2 實作

給一個 $n\times m$ 的整數矩陣與 $q$ 筆詢問,每筆給左上角 $(x_1,y_1)$、右下角 $(x_2,y_2)$, 問該矩形區域內所有元素的和。請每筆 $O(1)$ 回答。

看解答

二維前綴和。建表時容斥「上 + 左 − 左上 + 自己」,查詢時容斥「大塊 − 上條 − 左條 + 左上」。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int n, m;
LL P[1005][1005];
int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= m; j++) {
            LL a; cin >> a;
            P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + a;
        }
    int q; cin >> q;
    while (q--) {
        int x1, y1, x2, y2;
        cin >> x1 >> y1 >> x2 >> y2;
        cout << P[x2][y2] - P[x1-1][y2] - P[x2][y1-1] + P[x1-1][y1-1] << '\n';
    }
    return 0;
}
n, m = map(int, input().split())
P = [[0]*(m+1) for _ in range(n+1)]
for i in range(1, n+1):
    row = list(map(int, input().split()))
    for j in range(1, m+1):
        P[i][j] = P[i-1][j] + P[i][j-1] - P[i-1][j-1] + row[j-1]
q = int(input())
for _ in range(q):
    x1, y1, x2, y2 = map(int, input().split())
    print(P[x2][y2] - P[x1-1][y2] - P[x2][y1-1] + P[x1-1][y1-1])
✏️ 練習 60.3 實作

一條長 $n$ 的路、初始每格高度為 0。有 $m$ 筆操作「把區間 $[l,r]$ 整段升高 $x$」。 所有操作做完後,輸出每一格的最終高度。$n, m \le 2\times10^5$。

看解答

一維差分:每筆操作只動兩格 $D_l\mathrel{+}=x$、$D_{r+1}\mathrel{-}=x$,$O(1)$;最後對 $D$ 做前綴和還原,$O(n)$。總計 $O(n+m)$。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int main() {
    int n, m; cin >> n >> m;
    vector<LL> D(n + 2, 0);
    while (m--) {
        int l, r; LL x;
        cin >> l >> r >> x;
        D[l] += x;
        D[r + 1] -= x;
    }
    LL cur = 0;
    for (int i = 1; i <= n; i++) {
        cur += D[i];                 // 差分的前綴和 = 最終高度
        cout << cur << " \n"[i == n];
    }
    return 0;
}
n, m = map(int, input().split())
D = [0] * (n + 2)
for _ in range(m):
    l, r, x = map(int, input().split())
    D[l]   += x
    D[r+1] -= x
cur, out = 0, []
for i in range(1, n + 1):
    cur += D[i]
    out.append(str(cur))
print(' '.join(out))
✏️ 練習 60.4 挑戰

把長度 $n$ 的正整數陣列(順序固定)切成恰好 $K$ 段連續子陣列, 使「各段和的最大值」最小。求這個最小的最大段和。例如 $a=[7,2,5,10,8]$、$K=2$,答案是 $18$ (切成 $[7,2,5]$ 與 $[10,8]$,兩段和分別 $14$、$18$,最大為 $18$,已是所有切法裡最小的)。

看解答

二分搜答案。設答案為 $m$,feasible(m) 用貪心算「每段和 $\le m$ 時最少切幾段」, 能 $\le K$ 段即可行。$m$ 越大越可行,具單調性。值域 $[\max a_i,\ \sum a_i]$。 注意題目要求「恰好 $K$ 段」,但只要能用「$\le K$ 段」達成,必能把某段再切開湊到剛好 $K$ 段而不增加最大值,所以判斷 $\le K$ 即可。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int n, K;
vector<LL> a;
bool feasible(LL limit) {
    int groups = 1; LL cur = 0;
    for (LL x : a) {
        if (x > limit) return false;
        if (cur + x > limit) { groups++; cur = x; }
        else                  cur += x;
    }
    return groups <= K;
}
int main() {
    cin >> n >> K;
    a.resize(n);
    LL lo = 0, hi = 0;
    for (auto &x : a) { cin >> x; lo = max(lo, x); hi += x; }
    while (lo < hi) {
        LL mid = lo + (hi - lo) / 2;
        if (feasible(mid)) hi = mid;
        else               lo = mid + 1;
    }
    cout << lo << '\n';
    return 0;
}
def feasible(a, K, limit):
    groups, cur = 1, 0
    for x in a:
        if x > limit:
            return False
        if cur + x > limit:
            groups += 1; cur = x
        else:
            cur += x
    return groups <= K

n, K = map(int, input().split())
a = list(map(int, input().split()))
lo, hi = max(a), sum(a)
while lo < hi:
    mid = (lo + hi) // 2
    if feasible(a, K, mid):
        hi = mid
    else:
        lo = mid + 1
print(lo)
✏️ 練習 60.5 實作

給長度 $n$ 的陣列,求最長的、元素互不重複的連續子陣列長度。 例如 $[1,2,1,3,4,3]$ 的答案是 $4$(子陣列 $[2,1,3,4]$ 或 $[1,3,4,3]$… 取 $[2,1,3,4]$,長度 4)。$n \le 2\times10^5$。

看解答

同向雙指針 + 雜湊表記「每個值最後出現的位置」。右端 r 推進時, 若 a[r] 上次出現的位置 $\ge l$,就把左界 l 跳到「上次出現處的右邊一格」, 確保視窗內無重複。每個指標只前進,$O(n)$。 

#include <bits/stdc++.h>
using namespace std;
int main() {
    int n; cin >> n;
    vector<int> a(n);
    for (auto &x : a) cin >> x;
    unordered_map<int,int> last;   // 值 → 最後出現的索引
    int l = 0, best = 0;
    for (int r = 0; r < n; r++) {
        auto it = last.find(a[r]);
        if (it != last.end() && it->second >= l)
            l = it->second + 1;       // 把左界跳過上一次出現處
        last[a[r]] = r;
        best = max(best, r - l + 1);
    }
    cout << best << '\n';
    return 0;
}
n = int(input())
a = list(map(int, input().split()))
last = {}
l = best = 0
for r in range(n):
    if a[r] in last and last[a[r]] >= l:
        l = last[a[r]] + 1     # 左界跳過上次出現處
    last[a[r]] = r
    best = max(best, r - l + 1)
print(best)
✏️ 練習 60.6 挑戰

有 $n$ 個區間 $[l_i, r_i]$,座標可大到 $10^9$。求「被最多區間同時覆蓋」的位置,被幾個區間覆蓋? $n \le 2\times10^5$。(這是離散化 + 差分思想的綜合題。)

看解答

座標到 $10^9$,不能直接開差分陣列——這正是離散化的場景。把每個區間拆成兩個事件: 在 $l_i$ 處 +1(進入)、在 $r_i+1$ 處 -1(離開)。這 +1 / -1 正是差分的精神; 把所有事件按座標排序後掃一遍、累加計數,過程中的最大值就是答案。排序本身就完成了「離散化」—— 我們只在「有事件發生」的座標上結算,完全不碰中間的空白。總複雜度 $O(n\log n)$。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
int main() {
    int n; cin >> n;
    vector<pair<LL,int>> ev;          // (座標, +1 / -1)
    for (int i = 0; i < n; i++) {
        LL l, r; cin >> l >> r;
        ev.push_back({l,     +1});   // 進入
        ev.push_back({r + 1, -1});   // 離開(r+1 才減,等同差分端點)
    }
    sort(ev.begin(), ev.end());      // 按座標排序;同座標時 -1 排在 +1 前(因 -1 < +1)
    int cur = 0, best = 0;
    for (auto [x, d] : ev) {
        cur += d;
        best = max(best, cur);
    }
    cout << best << '\n';
    return 0;
}
n = int(input())
ev = []
for _ in range(n):
    l, r = map(int, input().split())
    ev.append((l, 1))        # 進入
    ev.append((r + 1, -1))   # 離開
ev.sort()                    # 同座標時 -1 在 +1 前,避免高估
cur = best = 0
for _, d in ev:
    cur += d
    best = max(best, cur)
print(best)

小提醒:同一座標若同時有區間結束與另一區間開始,把 -1 排在 +1 之前, 才不會在那個瞬間虛報覆蓋數(上面用 (座標, 增量) 排序,$-1 \lt +1$ 自然就對了)。 若題目把「閉端點接觸也算重疊」,則改成在 $r_i$(而非 $r_i+1$)結束,並讓 +1 排在前。

把這一章串起來

這一章我們配齊了競賽選手的核心瑞士刀。它們各有專長,但共享同一條哲學—— 把重複的計算預先存好,把不可能的範圍提早砍掉:

  • 枚舉與位元狀態壓縮:暴力的有章法寫法;先用複雜度判斷「枚不枚得完」,$n\le20$ 就想子集合枚舉。
  • 前綴和:$O(1)$ 查區間和 / 矩形區域和——對付「反覆查詢」。
  • 差分:$O(1)$ 完成區間 / 矩形加值——對付「反覆修改」;它與前綴和互為逆運算。
  • 二分搜答案:當答案單調,把「最佳化」變「判定」,「最小化最大值 / 最大化最小值」的招牌題。
  • 雙指針 / 滑動視窗:兩個只往右走的指標,把區間掃描從 $O(n^2)$ 壓到 $O(n)$。
  • 離散化:大值域、少相異值時,壓成連續小編號,常為 BIT / 線段樹鋪路。

往前看,這些工具會在後續章節反覆現身:它們依賴的複雜度直覺來自第 26 章;位元枚舉接續第 59 章的位元運算; 而離散化正是第 61 章樹狀數組與線段樹的標準前置處理。掌握這套「優化暴力」的方法論, 你面對 APCS 第三、四題與線上裁判難題時,就有了一條清晰可循的進攻路線: 先暴力、估複雜度、找瓶頸、換對的刀。
