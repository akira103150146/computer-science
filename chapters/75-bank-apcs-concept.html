導讀:APCS 觀念題在考什麼

APCS(大學程式設計先修檢測)的程式設計觀念題是選擇題:給你一段 C / C++ 程式(偶爾是虛擬碼),問你「輸出是什麼」「回傳值多少」「哪個選項填空才對」「哪一行有 bug」「哪組測資能測出錯誤」。它不要求你從零寫程式,卻要求你在腦中精準地執行程式——一個整數除法捨去、一個 switch 漏寫 break、一個遞迴的去程/回程順序,都足以讓四個選項天差地遠。

把歷年題目攤開,你會發現它反覆圍繞幾個主題:進位與記憶體、運算子與型別轉換、流程控制、陣列定址、傳值與傳址、遞迴追蹤、複雜度估算、字串與 ASCII、除錯與測資設計。其中份量最重、最會卡人的是遞迴追蹤——本題庫光是遞迴一章就有 30 題,幾乎是其他章的兩三倍,值得你花最多時間。

C1-01 · 程式錯誤類型（105/3#18） ⭐

程式編譯器可以發現下列哪種錯誤？

(A) 語法錯誤

(B) 語意錯誤

(C) 邏輯錯誤

C1-02 · 陣列記憶體位址（row-major）（105/3#19） ⭐⭐

大部分程式語言以列為主（row-major）儲存陣列。在 8×4 陣列 A 中，若每個元素需 2 單位記憶體，且 A[0][0] 的位址為 108（十進制），則 A[1][2] 的位址為何？

(A) 120

(B) 124

(C) 128

(D) 以上皆非

看答案與詳解

答案:(A)　　標籤:C1-P4　　難度:⭐⭐

詳解:row-major 位址 = base + (i×行數N + j)×元素大小 = 108 + (1×4 + 2)×2 = 108 + 12 = 120。

C1-03 · 進位轉換（105/10#22） ⭐⭐⭐

若 Xₙ 代表 X 為 n 進位，請問 D02A₁₆ + 5487₁₀ 等於多少？

(A) 1100 0101 1001 1001₂

(B) 162631₈

(C) 58787₁₆

(D) F599₁₆

看答案與詳解

答案:(B)　　標籤:C1-P3　　難度:⭐⭐⭐

詳解:D02A₁₆ = 13×4096 + 2×16 + 10 = 53290；53290 + 5487 = 58777。58777 轉八進位 = 162631₈（驗算：1×8⁵+6×8⁴+2×8³+6×8²+3×8+1 = 58777）。

C1-04 · 變數儲存位置（106/3#23） ⭐

(A) 記憶體

(B) 硬碟

答案:(A)　　標籤:C1-P1　　難度:⭐

詳解:執行期的變數值存在主記憶體（RAM）。硬碟存的是檔案／程式本體。

C1-05 · 溢位 overflow（106/3#24） ⭐⭐

程式執行過程中，若變數發生溢位（overflow），其主要原因為何？

(A) 以有限數目的位元儲存變數值

(B) 電壓不穩定

(C) 作業系統與程式不甚相容

C2-01 · 整數除法（範例#36） ⭐

給定下列程式，輸出結果為何？

int a=2, b=3;
int c=4, d=5;
int val;

val = b/a + c/b + d/b;
printf("%d\n", val);

(A) 3

(B) 4

(C) 5

(D) 6

看答案與詳解

答案:(A)　　標籤:C2-P1　　難度:⭐

詳解:整數除法無條件捨去：3/2=1、4/3=1、5/3=1，1+1+1=3。

C2-02 · 浮點數與 floor（範例#9） ⭐⭐⭐

f(a) 回傳小於浮點數 a 的最大整數（型態仍為浮點數）。程式執行後輸出為何？

int main() {
    float x=10, y=3;
    if ((0.5*x/y - f(0.5*x/y)) == 0.5) {
        printf("%f \n", f(0.5*x/y) - 1);
    }
    else if ((0.5*x/y - f(0.5*x/y)) < 0.5) {
        printf("%f \n", f(0.5*x/y));
    }
    else
        printf("%f \n", f(0.5*x/y) + 1);
    return 0;
}

(A) 0.000000

(B) 1.000000

(C) 1.666667

(D) 2.000000

看答案與詳解

答案:(D)　　標籤:C2-P2　　難度:⭐⭐⭐

詳解:0.5×10/3 ≈ 1.6667，f=1.0，小數部分 0.6667。不等於 0.5、也不小於 0.5，走 else：印 f+1 = 1.0+1 = 2.000000。

C2-03 · 運算子優先順序（106/3#25） ⭐

若 a, b, c, d, e 均為整數變數，下列哪個算式計算結果與 a+b*c-e 相同？

(A) (((a+b)*c)-e)

(B) ((a+b)*(c-e))

(C) ((a+(b*c))-e)

(D) (a+((b*c)-e))

看答案與詳解

答案:(C)　　標籤:C2-P3　　難度:⭐

詳解:* 優先於 +、-；+、- 同級由左而右。即 ((a+(b*c))-e)。

C2-04 · 布林運算（範例#44） ⭐⭐

設 x=TRUE, y=TRUE, z=FALSE。下列各布林運算式的真假值依序為何？

  • !(y || z) || x
  • !y || (z || !x)
  • z || (x && (y || z))
  • (x || x) && z

(A) TRUE FALSE TRUE FALSE

(B) FALSE FALSE TRUE FALSE

(C) FALSE TRUE TRUE FALSE

(D) TRUE TRUE FALSE TRUE

看答案與詳解

答案:(A)　　標籤:C2-P4　　難度:⭐⭐

詳解:①!(T)||T=T ②F||(F||F)=F ③F||(T&&T)=T ④T&&F=F → T F T F。

C2-05 · 字元陣列與 '\0'（105/10#13） ⭐⭐

若宣告 char str[20] = "Hello world!";，該陣列 str[12] 的值為何？

(A) 未宣告

(B) \0

(C) !

(D) \n

看答案與詳解

答案:(B)　　標籤:C2-P6　　難度:⭐⭐

詳解:"Hello world!" 共 12 個字元（索引 0~11），第 13 個（str[12]）是字串結束字元 '\0'。

C2-06 · 亂數範圍（範例#65） ⭐⭐

若 rand() 回傳介於 0 和 10000 之間的亂數，下列哪個運算式可產生介於 100 和 1000 之間（含端點）的任意數？

(A) rand() % 900 + 100

(B) rand() % 1000 + 1

(C) rand() % 899 + 101

(D) rand() % 901 + 100

看答案與詳解

答案:(D)　　標籤:C2-P5　　難度:⭐⭐

詳解:欲取 [100,1000] 共 901 個值。rand()%901 → 0~900，加 100 → 100~1000。

C2-07 · 布林運算反推（範例#75） ⭐

若邏輯判斷式 !(X1 || X2) 計算結果為真（True），則 X1 與 X2 的值分別應為何？

(A) X1 為 False，X2 為 False

(B) X1 為 True，X2 為 True

(C) X1 為 True，X2 為 False

(D) X1 為 False，X2 為 True

看答案與詳解

答案:(A)　　標籤:C2-P4　　難度:⭐

詳解:!(X1||X2) 為真 → (X1||X2) 為假 → X1、X2 皆為 False（笛摩根定律）。

C2-08 · 型別混合連除（@apcser） ⭐⭐

a=7,b=3。問 w,x,y,z 何者異於其他三者？

int a=7, b=3;
float w = a/2/b*1.0;
float x = a/b/2.0;
float y = a/2/b;
float z = 1.0*a/2/b;

(A) w

(B) x

(C) y

(D) z

看答案與詳解

答案:(D)　　標籤:C2-P1　　難度:⭐⭐

詳解:整數先除：w,x,y 皆為 1.0；z 先轉 float→7.0/2/3=1.167。注意整數除法與型別轉換時機。

C2-09 · #define 巨集展開陷阱（@apcser） ⭐⭐

C2-10 · ASCII 與 int 除法（109/10） ⭐⭐

C3-01 · 巢狀 if-else 追蹤（範例#26） ⭐⭐

下列程式執行過後所輸出數值為何？

void main() {
    int count = 10;
    if (count > 0)  { count = 11; }
    if (count > 10) {
        count = 12;
        if (count % 3 == 4) { count = 1; }
        else { count = 0; }
    }
    else if (count > 11) { count = 13; }
    else { count = 14; }
    if (count) { count = 15; }
    else       { count = 16; }
    printf("%d\n", count);
}

(A) 11

(B) 13

(C) 15

(D) 16

看答案與詳解

答案:(D)　　標籤:C3-P1　　難度:⭐⭐

詳解:count 10→11；11>10 成立→12，12%3=0≠4→count=0（else if/else 不執行）；最後 if(count) 即 if(0) 為假→count=16。

C3-02 · switch 改寫 if-else（範例#34） ⭐⭐

下列 switch 可如何以 if-else 改寫？

switch (x) {
    case 10: y = 'a'; break;
    case 20:
    case 30: y = 'b'; break;
    default: y = 'c';
}

(A) if (x==10) y='a'; if (x==20||x==30) y='b'; y='c';

(B) if (x==10) y='a'; else if (x==20||x==30) y='b'; else y='c';

(C) if (x==10) y='a'; if (x>=20 && x<=30) y='b'; y='c';

(D) if (x==10) y='a'; else if (x>=20 && x<=30) y='b'; else y='c';

看答案與詳解

答案:(B)　　標籤:C3-P2　　難度:⭐⭐

詳解:case 有 break → 互斥，須用 else if／else 串接；case 20/30 連寫表「等於 20 或 30」，非區間 20~30，故排除 (C)(D)。

C10-08 · 二分搜『找不到』的 bug 測資（110/11） ⭐⭐⭐

(B) x=s[i];　→ 輸出 abaca

(C) s[i]='x';　→ 輸出 xbxcx

看答案與詳解

答案:(A)　　標籤:C11-P2　　難度:⭐⭐
