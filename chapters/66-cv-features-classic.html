特徵與傳統視覺

在深度學習席捲一切之前,有整整數十年,人類是靠「親手設計特徵」教電腦看世界的。那些聰明絕頂的點子——把邊緣連成形狀、在影像裡找出獨一無二的角點、用一個 3×3 矩陣把兩張照片縫成全景——至今仍跑在你的手機、你的車、你家的監視器裡。這一章,我們把這套經典功夫一層一層拆給你看。

在深度學習之前,人類怎麼教電腦「看」

上一章(第 65 章「影像處理」)我們把影像當成一堆數字來「整理」:調亮度、去雜訊、找邊緣、做形態學。 那些操作很有用,但它們有個共同的天花板——電腦處理完之後,還是不知道畫面裡「有什麼」。 它知道某些像素比較亮、某些地方有一條邊,卻不知道「那是一個杯子」「這兩張照片拍的是同一棟樓」 「那台車正從左往右開過去」。

從「看到像素」跨到「看懂內容」,是電腦視覺(computer vision)真正的難題。今天我們會直覺地說: 「丟給神經網路就好啦。」但深度學習要到 2012 年才真正引爆;在那之前的數十年裡,一代又一代研究者 靠的是一個樸實卻威力強大的策略——手工設計特徵(hand-crafted features): 人類用幾何、用數學,親手定義「什麼樣的東西值得電腦注意」,再把這些線索組合起來做判斷。

這條路線聽起來「土法煉鋼」,但你別小看它。你手機的全景模式、相機的鏡頭校正、監視器的移動偵測、 擴增實境(AR)把虛擬物件擺到桌上、SLAM 讓掃地機器人知道自己在哪——這些至今仍大量倚賴本章的技術, 因為它們快、省電、可解釋、不需要海量標註資料。傳統視覺不是被淘汰的古董,而是深度學習旁邊 一套始終在線的可靠工具。

🔑
這一章的一條主線

傳統電腦視覺,本質上是一場「用人類的幾何直覺,把像素提煉成可比對、可量測的線索」的工程。 無論是輪廓、角點、描述子、單應矩陣還是光流,背後都是同一個追求:找出對「光線、角度、大小變化」 盡量穩健、又足夠獨特的特徵,讓電腦能在不同影像之間認出同一個東西。把這句話放在心裡,後面五個主題就會串成一條線。

從邊緣到形狀:輪廓分析

我們從階梯的第一階開始。第 65 章學過邊緣偵測,但邊緣的輸出是「一堆零散的白點」——它告訴你 「這裡顏色變化很劇烈」,卻沒把這些點組織成「一個物件」。輪廓(contour)就是下一步: 把這些邊緣點連成一條封閉曲線,代表一個物件完整的外形。一旦有了封閉曲線, 我們就能算它的面積、周長、形狀——電腦第一次能用「幾何」來描述一個東西。

取輪廓:findContours

OpenCV 用 cv2.findContours 從二值圖(慣例:白色=前景)抓出所有輪廓。 最常用兩個參數:RETR_EXTERNAL 只取最外層輪廓(忽略物件內部的洞),RETR_TREE 則取完整層級(連洞都找出來);CHAIN_APPROX_SIMPLE 會壓縮輪廓點(一段直線只存兩端點)省記憶體。 

import cv2

# bw 是二值圖(白=前景)。findContours 回傳「每個物件一條輪廓」的清單
cnts, hierarchy = cv2.findContours(bw, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)
print(f"找到 {len(cnts)} 條輪廓")          # 每條輪廓 = 一個物件的外形(一串點)

for c in cnts:                            # 一條一條處理
    print(c.shape)                        # (N, 1, 2):N 個點,每點一組 (x, y)
⚠️
最常見的「版本坑」

不同 OpenCV 版本,findContours 回傳 2 個或 3 個值(早期版本多回一個影像)。 要跨版本不出錯,慣用寫法是取最後兩個:cnts, hier = cv2.findContours(...)[-2:]。 這個小技巧能省下你大量「為什麼別人能跑我不行」的除錯時間。

輪廓的「尺規」:面積、周長、外接形狀

有了一條輪廓,我們就能拿出各種「尺規」去量它。這些量測值,正是下一步判斷「它是什麼形狀」的依據: 

import cv2

# 假設 cnts 是 findContours 的輸出
c = max(cnts, key=cv2.contourArea)        # 挑面積最大的輪廓(常用來鎖定主物件)

area = cv2.contourArea(c)                 # 面積(輪廓圍住的像素數)
peri = cv2.arcLength(c, True)             # 周長;True = 封閉曲線

x, y, w, h = cv2.boundingRect(c)          # 正立外接矩形(畫框 / 裁切常用)
(cx, cy), r = cv2.minEnclosingCircle(c)   # 最小外接圓:圓心 (cx,cy)、半徑 r
hull = cv2.convexHull(c)                  # 凸包:像用橡皮筋套住物件的外框

max(cnts, key=cv2.contourArea) 是個很 Python 的寫法:key=函式 表示 「不是比清單元素本身,而是比每個元素丟進這個函式後的結果」——這裡就是「比面積、回傳面積最大的那條輪廓」。 凸包(convex hull)則可以想成:用一條橡皮筋從外面套住整個物件,凹進去的地方會被拉直, 得到的外框就是凸包。它在等一下「數手指」時會派上大用場。

形狀分類:用頂點數就能認出三角形、矩形、圓

這是輪廓分析裡最漂亮的小魔術。核心點子很簡單:把彎彎曲曲的輪廓,簡化成幾條直線, 再數剩下幾個頂點。三角形會剩 3 個、矩形 4 個、五邊形 5 個,而圓會逼近出一大堆頂點—— 靠頂點數,電腦就能分辨形狀。負責「簡化」的函式是 cv2.approxPolyDP: 

import cv2

def classify(c):
    """用多邊形逼近的頂點數,判斷形狀名稱。"""
    peri = cv2.arcLength(c, True)
    # 0.04*peri 是容許誤差:越大越粗略(頂點越少)、越小越貼合(頂點越多)
    approx = cv2.approxPolyDP(c, 0.04 * peri, True)
    v = len(approx)                        # 數頂點個數
    return {3: "三角形", 4: "矩形", 5: "五邊形"}.get(v, "圓形")  # 找不到就當圓
💡
想要「旋轉、縮放都不變」的形狀指紋?

頂點數會受雜訊影響。若要更穩健地比對形狀,OpenCV 提供 cv2.moments 算質心、 cv2.HuMoments 算出 7 個對旋轉與縮放不變的形狀描述子(Hu 矩), 再用 cv2.matchShapes 比兩個形狀有多像。這就像給每個形狀一張不受擺放角度影響的「身分證」。

應用一瞥:物件計數與凸缺陷數手指

輪廓分析最直接的應用就是物件計數:二值化 → 找輪廓 → 過濾掉太小的雜訊輪廓 → 數剩幾條。 工廠數零件、生物實驗數細胞、農業數果實,都是這個套路。再進一步,凸包還能玩出「數手指」這種手勢辨識: 把手用凸包(橡皮筋)套住,手指之間的凹谷就是凸包與真實輪廓的落差,叫做 凸缺陷(convexity defect)。數出夠深的凹谷個數,就能推算張開幾根手指——剪刀石頭布就靠這招。

什麼是「好的特徵點」?角點的故事

輪廓分析很強,但它有個罩門:它要求你先把物件乾淨地切出來(二值化、去背)。 一旦背景雜亂、物件互相重疊、光線飄忽,輪廓就崩了。更要命的是,如果同一個東西換個角度、換個距離拍, 輪廓的形狀就完全變樣——你沒辦法靠輪廓在兩張不同照片裡認出「這是同一個杯子」。

我們需要一種更局部、更穩健的線索:不依賴整個物件被切乾淨,而是在影像裡找出一些 獨特、好認、而且換個角度還認得出來的小地方。這就是特徵點(feature point / keypoint)的核心思想,也是本章的重頭戲。問題只剩一個:什麼樣的點才「好認」?

平坦、邊緣、角點:誰最好定位?

想像你拿一個小視窗在影像上滑動,看看視窗內的內容會不會變。分三種情況:

  • 平坦區(flat):視窗往任何方向移動,看到的都一樣(一片白牆)。完全無法定位——你根本不知道自己移動了沒。
  • 邊緣(edge):沿著邊的方向移動,看起來一樣;只有垂直跨過邊時才會變。能定位一個方向,另一個方向仍模稜兩可。
  • 角點(corner):往任何方向移動,視窗內容都會明顯改變。這是唯一能被「釘死」在精確位置的點——所以角點是最好的特徵。

Harris 角點:把直覺寫成公式

Harris 角點偵測(1988)把上面的直覺數學化。它衡量「視窗移動 $(u,v)$ 後,內容變化有多大」:

$$E(u,v)=\sum_{x,y} w(x,y)\,\big[\,I(x+u,\,y+v)-I(x,y)\,\big]^2$$

把這個變化量做泰勒近似後,核心會濃縮成一個 $2\times2$ 的結構矩陣(structure tensor) $M$, 它由影像在 $x$、$y$ 方向的梯度組成。$M$ 的兩個特徵值 $\lambda_1,\lambda_2$ 告訴我們這一點的「方向性」: 兩個都小 → 平坦;一大一小 → 邊緣;兩個都大 → 角點(呼應第 54 章:特徵值描述一個線性映射在各方向的「伸縮強度」)。 Harris 用一個不必真的解特徵值的便宜公式來判斷:

$$R=\det(M)-k\,\big(\operatorname{trace}(M)\big)^2,\qquad k\approx 0.04\text{–}0.06$$

$R$ 很大代表角點、$R$ 為負代表邊緣、$|R|$ 很小代表平坦。在 OpenCV 裡,你甚至不必碰這些公式, 一行就能拿到「最好的 N 個角點」(Shi-Tomasi 版,Harris 的改良): 

import cv2
import numpy as np

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
# 找最多 100 個品質夠好、彼此至少隔 10 像素的角點
corners = cv2.goodFeaturesToTrack(gray, maxCorners=100,
                                  qualityLevel=0.01, minDistance=10)
for x, y in corners.reshape(-1, 2):
    cv2.circle(img, (int(x), int(y)), 4, (0, 255, 0), -1)   # 標出每個角點

從「點」到「指紋」:特徵描述子

光知道角點在哪還不夠。要在另一張照片裡認出「同一個角點」,我們需要描述這個點 周圍長什麼樣子——這就是描述子(descriptor):一串數字,像是這個點的「指紋」。 兩張照片裡的兩個點,只要指紋夠像,就判定它們是同一個東西。

好的描述子要對「干擾」免疫:換個角度、拉遠拉近、轉個方向、光線變一點,指紋都要盡量不變。 兩個經典名字你一定會遇到:

  • SIFT(Scale-Invariant Feature Transform,1999):電腦視覺史上影響最深遠的演算法之一。 它對縮放、旋轉、光照都極穩健,代價是計算較慢。(因專利曾被收費,如今已開放免費使用。)
  • ORB(Oriented FAST and Rotated BRIEF,2011):免費、極快、實務首選。 它產生的是二進位描述子(一串 0/1),比對時用「位元差幾個」的 Hamming 距離, 在手機等資源有限的裝置上特別划算。
import cv2

orb = cv2.ORB_create(nfeatures=1000)          # 最多找 1000 個關鍵點
# detectAndCompute 一次做完「找關鍵點」+「算描述子」;None 是遮罩(不限範圍)
kp, des = orb.detectAndCompute(gray, None)
print(f"找到 {len(kp)} 個關鍵點;描述子矩陣 {des.shape}")  # 每列一個 32 位元組指紋

vis = cv2.drawKeypoints(img, kp, None, color=(0, 255, 0),
                        flags=cv2.DrawMatchesFlags_DRAW_RICH_KEYPOINTS)  # 圓圈含方向
ℹ️
關鍵點 vs 描述子,別搞混

關鍵點(keypoint)是「在哪裡」——影像中一個獨特的位置(通常是角點), 還帶有尺度與方向資訊。描述子(descriptor)是「長什麼樣」——把那個點周圍的樣貌 編碼成一串數字。匹配時比對的是描述子,而關鍵點告訴你比中的東西座落在哪。

特徵匹配:在兩張圖裡認出同一個東西

有了兩張圖各自的描述子,匹配就是「幫每個描述子,在對面找最像的那個」。 最直接的是暴力匹配器(BFMatcher):把一邊的每個描述子,跟另一邊全部比一輪。 這裡有個一定要記住的細節——ORB 的二進位描述子要用 Hamming 距離,SIFT 的浮點描述子要用 L2 距離, 用錯距離匹配會爛掉。

但暴力匹配會產生大量錯配。救星是 David Lowe 提出的比值測試(ratio test): 對每個點找最近與次近兩個對象,只有當「最近」明顯比「次近」更近(小於 0.75 倍)才採信。 直覺是:如果最近和次近差不多近,代表這個點不夠獨特、容易認錯,乾脆別要它。 

import cv2

orb = cv2.ORB_create(1000)
kp1, des1 = orb.detectAndCompute(img1, None)
kp2, des2 = orb.detectAndCompute(img2, None)

bf = cv2.BFMatcher(cv2.NORM_HAMMING)          # ORB 用 Hamming;SIFT 改 NORM_L2
matches = bf.knnMatch(des1, des2, k=2)        # 每個點找「最近 2 個」對象

# Lowe 比值測試:最近(m)要明顯比次近(n)更近,才算可信
good = [m for m, n in matches if m.distance < 0.75 * n.distance]
print(f"原始配對 {len(matches)} → 過濾後好匹配 {len(good)}")

vis = cv2.drawMatches(img1, kp1, img2, kp2, good[:30], None,
                      flags=cv2.DrawMatchesFlags_NOT_DRAW_SINGLE_POINTS)

[m for m, n in matches if ...] 是串列生成式;for m, n in 一次把每組 (最近, 次近) 拆成兩個變數,if 只留通過測試的最近匹配 m。 實務上,務必把「不過濾」與「過濾後」的連線圖並排看一次——前者亂連如毛線球,後者乾淨筆直,對比強烈到你會立刻記住比值測試的價值。

影像拼接:把特徵匹配變成全景

現在來看特徵匹配最迷人的一個應用——全景拼接(panorama stitching)。 你手機的「全景模式」、Google 街景、空拍地圖,本質上都在做同一件事:把多張有重疊的照片, 對齊後縫成一張大圖。而它用到的積木,你已經全部學過了。

單應矩陣:把一個平面映到另一個平面

關鍵的數學工具叫單應矩陣(homography),記作 $H$,是一個 $3\times3$ 矩陣。 它描述「同一個平面,從一個視角看 vs 從另一個視角看」之間的對應關係。用齊次座標表示 (把 2D 點 $(x,y)$ 補成 $(x,y,1)$,呼應第 54 章的座標技巧),映射就是一次矩陣乘法:

$$\begin{bmatrix}x'\\y'\\w'\end{bmatrix}\;\sim\;H\begin{bmatrix}x\\y\\1\end{bmatrix}, \qquad H=\begin{bmatrix}h_{11}&h_{12}&h_{13}\\h_{21}&h_{22}&h_{23}\\h_{31}&h_{32}&1\end{bmatrix}$$

算出 $(x',y',w')$ 後,再除以 $w'$ 得到真正的影像座標 $(x'/w',\,y'/w')$——這個「除以最後一維」就是 透視(perspective)的來源,它讓 $H$ 能表達「近大遠小」的傾斜效果,比第 54 章的純線性變換更廣。 $H$ 有 8 個自由度,而每組對應點提供 2 條方程式,所以至少要 4 組對應點才解得出 $H$。 這些對應點,正是上一節的特徵匹配幫我們找到的!

RANSAC:在一堆錯配裡,撈出正確的變換

理論上 4 組點就能求 $H$,但實務上特徵匹配總會混進一些錯配(離群點 outlier)。如果用全部的點去硬解, 幾個錯配就能把結果帶歪。解法是 RANSAC(隨機抽樣一致性),它的點子簡單到不可思議:

  1. 隨機抽最少數量的點(求 $H$ 需 4 組),算出一個候選 $H$。
  2. 看看其餘所有點裡,有多少「同意」這個 $H$(投影後落在誤差範圍內,稱為內群 inlier)。
  3. 重複很多次,最後採用「贊成票最多」的那個 $H$,再用所有內群點精算一次。

因為正確的對應彼此一致、錯配各走各的,所以少數錯配永遠湊不出多數同意—— RANSAC 自然就把它們排除了。這是電腦視覺裡最重要的「抗離群」工具之一。

手刻拼接:其實是 M9 + 第 54 章的組合

把上面全部串起來,兩張圖拼接的程式短得驚人——而且你會發現,「新東西其實是舊積木拼的」: 

import cv2
import numpy as np

# 1) 兩張圖各偵測 ORB 特徵,匹配 + 比值測試,得到 good(見前一節)
orb = cv2.ORB_create(2000)
k1, d1 = orb.detectAndCompute(left, None)
k2, d2 = orb.detectAndCompute(right, None)
bf = cv2.BFMatcher(cv2.NORM_HAMMING)
# 求「右圖 → 左圖」的變換,所以 query 用右圖、train 用左圖
good = [m for m, n in bf.knnMatch(d2, d1, k=2) if m.distance < 0.75 * n.distance]

if len(good) >= 4:                         # 至少 4 組對應才求得出 H
    src = np.float32([k2[m.queryIdx].pt for m in good]).reshape(-1, 1, 2)  # 右圖點
    dst = np.float32([k1[m.trainIdx].pt for m in good]).reshape(-1, 1, 2)  # 左圖點
    # 2) RANSAC 求單應矩陣(自動剔除錯配);5.0 是容許誤差(像素)
    H, mask = cv2.findHomography(src, dst, cv2.RANSAC, 5.0)
    # 3) 把右圖 warp 到左圖座標系,畫布加寬才放得下
    h1, w1 = left.shape[:2]
    pano = cv2.warpPerspective(right, H, (w1 + right.shape[1], h1))
    pano[0:h1, 0:w1] = left                # 左圖原封不動貼到左邊 → 兩張接起來
    cv2.imwrite("pano.jpg", pano)

這就是 M9 的「特徵匹配 + 求 $H$」加上第 54 章的「透視變換把座標搬家」。直接覆蓋會在重疊處留下明顯接縫與曝光差, 實務上會再做融合(blending)把接縫抹平。如果你只想要結果,OpenCV 還有一鍵全景的高階介面: 

import cv2

stitcher = cv2.Stitcher_create(cv2.Stitcher_PANORAMA)
status, pano = stitcher.stitch([img1, img2, img3])   # 一串圖,順序不限
if status == cv2.Stitcher_OK:                        # 一定要檢查 status!
    cv2.imwrite("panorama.jpg", pano)
else:
    print("拼接失敗,常因重疊不足(需 ≥30%)或畫面無紋理")
⚠️
拼接拼不起來?八成是這兩件事

第一,重疊不足或畫面太空(白牆、藍天無紋理)→ 找不到足夠的對應點。 相鄰兩張至少要重疊 30%。第二,沒檢查 status——高階 Stitcher 失敗時回傳非 0,你卻拿了個空的結果去用。記得永遠先確認 status == cv2.Stitcher_OK。

相機標定與多視幾何:把 2D 連回 3D

到目前為止我們都在 2D 像素的世界裡打轉。但很多應用需要回答 3D 問題:這個零件實際多大?那台車離我多遠? 要把 AR 的虛擬茶杯穩穩擺在真實桌面上,得先知道相機是怎麼把 3D 世界「拍扁」成 2D 影像的——這就是 多視幾何(multi-view geometry)的領域,而一切的起點是搞懂相機本身。

針孔相機模型

最簡單也最核心的相機模型是針孔相機(pinhole camera):想像一個密閉盒子, 正面戳一個小孔,3D 世界的光線穿過小孔,在後方的成像平面上投出一個上下顛倒的像。 這個「3D 點 → 2D 像素」的投影,可以用一條優雅的矩陣式表達:

$$s\begin{bmatrix}u\\v\\1\end{bmatrix} =\underbrace{\begin{bmatrix}f_x&0&c_x\\0&f_y&c_y\\0&0&1\end{bmatrix}}_{K\ (\text{內參})} \;\underbrace{\big[\,R\,\mid\,t\,\big]}_{\text{外參}} \begin{bmatrix}X\\Y\\Z\\1\end{bmatrix}$$

  • 內參矩陣 $K$(intrinsics):相機自身的性質——焦距 $f_x,f_y$(畫面被「放大」多少)、 主點 $c_x,c_y$(光軸打在感光元件上的位置,通常接近影像中心)。同一台相機這組值固定不變。
  • 外參 $[R\mid t]$(extrinsics):相機擺在哪——它在世界座標中的旋轉 $R$ 與平移 $t$。 相機每動一下,外參就變。

為什麼要標定?——找出 K 與畸變係數

每一台相機、每一顆鏡頭,內參 $K$ 都不一樣,而且真實鏡頭還會帶來畸變(distortion): 廣角鏡頭讓直線往外凸(桶狀畸變)、某些鏡頭讓直線往內凹(枕狀畸變)。 只要你想用畫面來量測或做 3D/AR,就必須先把這些參數量出來——這個過程叫 相機標定(camera calibration)。

最經典的做法是拍棋盤格:它的格點間距已知、又好偵測。拍 15~20 張不同角度的棋盤, 收集每張的「角點 3D 座標 ↔ 角點 2D 影像座標」對應,丟給 calibrateCamera 一次解出 $K$ 與畸變係數: 

import cv2
import numpy as np

# 對每張棋盤照片:找內角點(9×6 = 每邊格數 − 1)
found, corners = cv2.findChessboardCorners(gray, (9, 6))
# ... 收集多張的 objpoints(角點 3D 座標)與 imgpoints(角點 2D 座標) ...

# 用所有對應解出內參與畸變;gray.shape[::-1] 把 (高,寬) 反成 (寬,高)
ret, K, dist, rvecs, tvecs = cv2.calibrateCamera(
    objpoints, imgpoints, gray.shape[::-1], None, None)

undist = cv2.undistort(img, K, dist)      # 用標定結果把彎掉的直線拉回直的

立體視覺:用兩隻眼睛量出深度

單一張照片丟失了深度資訊(你沒辦法只看一張平面照就確定遠近)。但兩台略為分開的相機—— 就像你的兩隻眼睛——能把深度算回來。做個小實驗:伸出手指放在面前,左右眼交替閉, 你會看到手指相對背景「跳」得很厲害;再看遠處的牆,幾乎不跳。這個左右影像之間的位移, 就叫視差(disparity),而且——越近的東西,視差越大。

深度與視差成反比,公式漂亮得很:

$$Z=\frac{f\cdot B}{d}$$

其中 $Z$ 是深度、$f$ 是焦距、$B$ 是兩相機間距(基線 baseline)、$d$ 是視差。 把畫面每個像素的視差都算出來、再上色,就得到一張深度圖(depth map)—— 自駕車、3D 掃描、體感遊戲(早期的 Kinect 概念)都建立在這之上。 

import cv2

# 立體匹配:numDisparities 須為 16 倍數;blockSize 是比對區塊(奇數)
stereo = cv2.StereoSGBM_create(numDisparities=64, blockSize=9)
disparity = stereo.compute(grayL, grayR)      # 吃左右兩張灰階圖,算每像素視差
# 視差越大 = 越近;把 disparity 正規化上色,就是一張深度圖

加入時間:物件追蹤與光流

前面都在處理靜態影像。但真實世界會動,而影片就是一連串隨時間排列的影像(影格 frame)。 一旦加入時間維度,就能問新問題:畫面裡什麼東西在動?它往哪邊移動? 這就是物件追蹤(tracking)與光流(optical flow)的主場——交通計數、賣場動線分析、運動科技、保全偵測,全靠它。

背景相減:抓出「會動的東西」

最樸素也最常用的運動偵測,是背景相減(background subtraction)。直覺很簡單: 固定鏡頭拍一條馬路,「不動的」是背景(馬路),「在動的」就是前景(車)。OpenCV 的背景相減器會 持續學習背景模型,你每餵一幀,它就回傳一張前景遮罩——移動的東西標白、背景標黑。 

import cv2

bs = cv2.createBackgroundSubtractorMOG2()     # 會持續學習背景的物件
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

while True:
    ok, frame = cap.read()
    if not ok:
        break
    fg = bs.apply(frame)                       # 前景遮罩:移動物體=白
    fg = cv2.morphologyEx(fg, cv2.MORPH_OPEN, kernel)   # 開運算去雜點(第 65 章)
    cnts, _ = cv2.findContours(fg, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)[-2:]
    for c in cnts:
        if cv2.contourArea(c) > 500:           # 過濾太小的雜訊
            x, y, w, h = cv2.boundingRect(c)
            cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 255, 0), 2)

光流:像素如何在影格之間移動

光流(optical flow)回答更細的問題:同一個東西,從前一幀到這一幀, 移動了多少、往哪移?它有兩種風味:

  • 稀疏光流(sparse):只追你指定的少數幾個點(通常是角點)。經典演算法是 Lucas-Kanade。它假設一個小鄰域內的像素一起移動,從而解出每個點的位移。 把「前一幀位置 → 這一幀位置」連起來,就是運動軌跡。
  • 稠密光流(dense):算每一個像素的位移向量,得到整張「運動場」。 常用 Farnebäck 演算法,把方向轉成色相、大小轉成亮度,畫成一張彩色運動圖,哪裡在動、往哪動一目了然。
import cv2
import numpy as np

# 稀疏光流(Lucas-Kanade):追蹤上一幀的角點 p0 在這一幀的新位置
p0 = cv2.goodFeaturesToTrack(prev_gray, 100, 0.01, 10)   # 先找一批角點來追
p1, st, err = cv2.calcOpticalFlowPyrLK(prev_gray, gray, p0, None)
good_new = p1[st == 1]                          # st=1 表示成功追到的點
good_old = p0[st == 1]
for (a, b), (c, d) in zip(good_new, good_old):
    cv2.line(frame, (int(a), int(b)), (int(c), int(d)), (0, 255, 0), 2)  # 畫軌跡
# 下一輪:把 p1 當成新的 p0 繼續追

物件追蹤器:給一框,之後它自己跟

如果你只想鎖定一個目標(一台車、一張臉、一顆球),OpenCV 內建的追蹤器最方便: 第一幀你框一個目標,之後它每一幀自動回報新位置。 

import cv2

tracker = cv2.TrackerCSRT_create()             # CSRT 較準;KCF 較快
tracker.init(frame, bbox)                       # bbox = (x, y, w, h),告訴它要跟誰

while True:
    ok, frame = cap.read()
    if not ok:
        break
    success, bbox = tracker.update(frame)       # 每幀回報:還跟得到嗎、新位置在哪
    if success:
        x, y, w, h = map(int, bbox)
        cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 255, 0), 2)
    else:
        cv2.putText(frame, "Lost!", (20, 40), cv2.FONT_HERSHEY_SIMPLEX, 1, (0, 0, 255), 2)

目標被長時間遮住、或外觀變化太大時會跟丟(success=False)。實務上常配 卡爾曼濾波(Kalman filter):用運動模型預測目標下一刻的位置,抗短暫遮擋與抖動, 並在多目標時維持穩定的 ID。把「偵測 + 追蹤」結合起來,就是現代多目標追蹤(MOT)的雛形—— 而「偵測」這一塊,正是下一章深度學習要登場的地方。

💡
一個經典應用:跨線計數

車流/人流計數的核心技巧很巧妙:畫一條虛擬計數線,記錄每個目標上一幀與 這一幀的質心在線的哪一側。當它從一側跨到另一側時 +1。判斷「跨線」只要看 (上一幀質心 − 線) × (這一幀質心 − 線) < 0——兩者異號就代表跨過去了。 記得要存上一幀的位置,否則會重複計數。

傳統視覺的價值與限制

我們爬完了整座階梯:輪廓 → 特徵點 → 匹配與幾何 → 時間。現在退一步問:這套「手工特徵」的功夫, 到底好在哪、又卡在哪?

它的價值,在受控場景裡至今無可取代。當光線穩定、背景單純、物件外觀變化有限—— 產線檢測、棋盤標定、文件掃描、固定鏡頭的監視器——傳統視覺又快、又省電、又精準,而且完全可解釋。 你能清楚說出「它是因為數到 4 個頂點才判斷成矩形」「它是靠這 50 組對應點求出 H」。 它不需要海量標註資料,一兩張範本就能上工;它能在沒有 GPU 的微控制器上即時運行。 當問題定義清楚,殺雞何必用牛刀?

它的限制,在複雜、開放的場景裡會原形畢露。手工特徵是人類「猜」出來的規則—— 角點、邊緣、Hu 矩——它們對幾何變化穩健,卻擋不住真實世界的混亂:嚴重遮擋、劇烈光影、 形變的物體(一隻擺各種姿勢的貓)、抽象的語意(「這是不是一張快樂的臉?」)。 你沒辦法手寫一條規則去定義「貓的特徵」,因為貓有千萬種樣子。

這正是深度學習帶來的典範轉移:與其讓人類絞盡腦汁設計特徵,不如餵給神經網路海量資料, 讓它自己學出該注意什麼特徵。下一章(第 67 章),我們就會看到卷積神經網路(CNN) 如何把本章「人工設計濾波器/特徵」的工作,變成「從資料中自動學習」——但你會發現, CNN 學到的低層濾波器,長得竟然很像本章的邊緣與角點偵測器。傳統視覺不是被丟掉的舊知識, 而是理解深度學習為何有效的最佳起點。

以下練習由淺入深,涵蓋本章五個主題。強烈建議先自己寫,再展開解答對照。所有程式都用 Python + OpenCV。

✏️ 練習 66.1 實作

給一張有多個白色物件(深色背景)的影像,用輪廓數出有幾個物件, 並在每個物件上畫出外接矩形與編號。記得過濾掉太小的雜訊輪廓。

看解答

標準套路:灰階 → 二值化 → findContours → 過濾小面積 → 數數量並標號。

import cv2

img = cv2.imread("objects.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
_, bw = cv2.threshold(gray, 60, 255, cv2.THRESH_BINARY)   # 白物件、深背景

cnts, _ = cv2.findContours(bw, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)[-2:]

count = 0
for c in cnts:
    if cv2.contourArea(c) < 200:          # 過濾雜訊輪廓
        continue
    count += 1
    x, y, w, h = cv2.boundingRect(c)
    cv2.rectangle(img, (x, y), (x + w, y + h), (0, 255, 0), 2)
    cv2.putText(img, str(count), (x, y - 6),
                cv2.FONT_HERSHEY_SIMPLEX, 0.6, (0, 255, 255), 2)

print(f"共偵測到 {count} 個物件")
cv2.imwrite("counted.jpg", img)

若物件彼此相黏導致少算,可在二值化後加一次形態學開運算,或改用 cv2.RETR_TREE 搭配層級分析;若想更穩,可加 distance transform + watershed 把黏住的物件切開。

✏️ 練習 66.2 實作

在一張影像上找出特徵點:(a) 用 goodFeaturesToTrack 找出最好的 50 個角點並畫綠點; (b) 改用 ORB 偵測關鍵點並用 drawKeypoints 畫出(含方向)。比較兩者抓到的點有何不同。

看解答 
import cv2

img = cv2.imread("scene.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# (a) Shi-Tomasi 角點(Harris 的改良)
corners = cv2.goodFeaturesToTrack(gray, maxCorners=50,
                                  qualityLevel=0.01, minDistance=10)
a = img.copy()
for x, y in corners.reshape(-1, 2):
    cv2.circle(a, (int(x), int(y)), 4, (0, 255, 0), -1)
cv2.imwrite("corners.jpg", a)

# (b) ORB:關鍵點 + 描述子
orb = cv2.ORB_create(nfeatures=500)
kp, des = orb.detectAndCompute(gray, None)
b = cv2.drawKeypoints(img, kp, None, color=(0, 255, 0),
                      flags=cv2.DrawMatchesFlags_DRAW_RICH_KEYPOINTS)
cv2.imwrite("orb.jpg", b)
print(f"角點 {len(corners)} 個;ORB 關鍵點 {len(kp)} 個,描述子 {des.shape}")

觀察重點:goodFeaturesToTrack 只給「位置」,適合拿去餵光流追蹤; ORB 的關鍵點還帶尺度與方向(畫出來的圓圈大小與線段),並附帶描述子, 才能做跨圖匹配。兩者都集中在紋理豐富、角點多的地方,平坦區幾乎沒有點。

✏️ 練習 66.3 挑戰

「以圖找物」:給一張範本(例如某本書的封面)與一張場景照片(書放在桌上的一角), 用 ORB 特徵匹配 + 比值測試 + RANSAC 求單應矩陣,把範本的四個角投影到場景上,畫框標出物件位置。

看解答

這是 M9 旗艦專題的完整流程:偵測 → 匹配 → 濾錯配 → 求 H → 投影四角。

import cv2
import numpy as np

tmpl = cv2.imread("book_cover.jpg")    # 範本
scene = cv2.imread("desk.jpg")         # 場景

orb = cv2.ORB_create(1000)
k1, d1 = orb.detectAndCompute(tmpl, None)
k2, d2 = orb.detectAndCompute(scene, None)

bf = cv2.BFMatcher(cv2.NORM_HAMMING)
good = [m for m, n in bf.knnMatch(d1, d2, k=2) if m.distance < 0.75 * n.distance]
print(f"好匹配 {len(good)} 組")

if len(good) >= 4:
    # queryIdx=範本端、trainIdx=場景端
    src = np.float32([k1[m.queryIdx].pt for m in good]).reshape(-1, 1, 2)
    dst = np.float32([k2[m.trainIdx].pt for m in good]).reshape(-1, 1, 2)
    H, mask = cv2.findHomography(src, dst, cv2.RANSAC, 5.0)

    h, w = tmpl.shape[:2]
    corners = np.float32([[0, 0], [w, 0], [w, h], [0, h]]).reshape(-1, 1, 2)
    proj = cv2.perspectiveTransform(corners, H)         # 範本四角投影到場景
    cv2.polylines(scene, [np.int32(proj)], True, (0, 255, 0), 3)
    cv2.imwrite("located.jpg", scene)
else:
    print("好匹配不足 4 組,無法定位(換紋理更豐富的範本或更清晰的場景)")

常見坑:範本太平、無紋理(找不到特徵);距離度量用錯(ORB 必須 NORM_HAMMING); 忘了比值測試導致錯配淹沒正確匹配。把 drawMatches 畫出來檢查連線,是最快的除錯方式。

✏️ 練習 66.4 思考

用你自己的話解釋:單應矩陣(homography)如何把一個平面映射到另一個平面? 為什麼至少需要 4 組對應點?為什麼它能表達「近大遠小」的透視效果,而第 54 章的一般 $2\times2$ 線性變換不行?

看解答

怎麼映射:把 2D 點寫成齊次座標 $(x, y, 1)$,單應矩陣 $H$(3×3)做一次矩陣乘法 得到 $(x', y', w')$,再除以 $w'$ 還原成真正的影像座標 $(x'/w',\, y'/w')$。 這個「先矩陣乘、再除以最後一維」就是它的全部。

為什麼要 4 組點:$H$ 有 9 個元素,但整體可以乘上任意非零常數而不改變映射(因為最後會除以 $w'$), 所以實際只有 8 個自由度。每組對應點 $(x,y)\to(x',y')$ 提供 2 條方程式($x'$ 一條、$y'$ 一條), 因此 $8\div2 = 4$ 組點剛好把 $H$ 釘死。(點愈多愈好,可配 RANSAC 抗錯配。)

為什麼能表達透視:關鍵在「除以 $w'$」這一步。$H$ 的最後一列 $(h_{31}, h_{32}, 1)$ 讓 $w'$ 隨著 $(x, y)$ 改變,於是同樣的位移在不同位置被「除以不同的數」—— 這就製造出近大遠小。第 54 章的 $2\times2$ 線性變換沒有這個除法(它把平行線永遠保持平行), 所以只能做旋轉、縮放、剪切,變不出梯形般的透視。單應矩陣正是線性變換往「投影幾何」的自然延伸。

✏️ 練習 66.5 實作

用幀差法(frame differencing)偵測影片中的運動:比較連續兩幀的差異, 把變動夠大的區域框出來。(進階:改用稀疏光流追蹤角點並畫出運動軌跡。)

看解答

幀差法是最輕量的運動偵測:相鄰兩幀相減 → 二值化 → 形態學清理 → 找輪廓框出。

import cv2

cap = cv2.VideoCapture("traffic.mp4")
ok, prev = cap.read()
prev_gray = cv2.cvtColor(prev, cv2.COLOR_BGR2GRAY)
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))

while True:
    ok, frame = cap.read()
    if not ok:
        break
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
    diff = cv2.absdiff(prev_gray, gray)              # 兩幀相減(絕對值)
    _, motion = cv2.threshold(diff, 25, 255, cv2.THRESH_BINARY)
    motion = cv2.morphologyEx(motion, cv2.MORPH_OPEN, kernel)   # 去雜點

    cnts, _ = cv2.findContours(motion, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)[-2:]
    for c in cnts:
        if cv2.contourArea(c) > 500:                 # 過濾太小的變動
            x, y, w, h = cv2.boundingRect(c)
            cv2.rectangle(frame, (x, y), (x + w, y + h), (0, 255, 0), 2)

    cv2.imshow("motion", frame)
    if cv2.waitKey(30) & 0xFF == ord('q'):
        break
    prev_gray = gray                                  # 這一幀成為下一輪的「上一幀」

cap.release()
cv2.destroyAllWindows()

幀差法很省、但只看得到「正在動」的邊緣(物件中央若顏色均勻會被當成沒動)。 想要完整的前景遮罩,改用本章的背景相減 createBackgroundSubtractorMOG2; 想知道「往哪動」,則用稀疏光流 calcOpticalFlowPyrLK 追角點、把 p0→p1 連線即得軌跡。

把這一章串起來

我們沿著一座「資訊階梯」,看完了深度學習之前人類教電腦看世界的整套功夫。回顧路標:

  • 輪廓與形狀:把零散邊緣連成封閉曲線,用面積、周長、多邊形逼近頂點數做形狀分類與物件計數;凸缺陷還能數手指。
  • 特徵點:角點是最好的特徵,因為它在任何方向移動都會變;Harris 用結構矩陣的特徵值量化這件事。
  • 描述子與匹配:SIFT/ORB 把特徵點周圍編成「指紋」,Lowe 比值測試濾掉模稜兩可的錯配。
  • 單應矩陣與 RANSAC:用 4 組以上對應點把一個平面映到另一個平面(透視來自「除以 $w'$」),RANSAC 從錯配中撈出正確的 $H$——影像拼接就是這些積木的組合。
  • 相機標定與多視幾何:針孔模型用內參 $K$ + 外參 $[R\mid t]$ 描述投影;標定量出 $K$ 與畸變並校正;雙眼視差由 $Z=fB/d$ 還原深度。
  • 追蹤與光流:背景相減抓會動的東西,光流量出像素位移,追蹤器鎖定單一目標——加上時間維度就能做運動偵測與計數。

最重要的體悟也許是:這些技術沒有一個是魔法,全是樸實的幾何與線性代數疊出來的—— 把邊緣連成形狀、用特徵值判斷角點、用矩陣乘法搬動平面、用相似三角形換算深度。 它們在受控場景至今好用得很;而當場景變得雜亂、開放、需要高層語意時,我們就需要一種能 「自己學出特徵」的方法。下一章(第 67 章),我們走進深度學習的電腦視覺, 看卷積神經網路如何接手這份工作——並驚訝地發現,它在底層學到的東西,竟和本章的邊緣與角點如此相似。
