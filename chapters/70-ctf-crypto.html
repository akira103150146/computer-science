古典與現代密碼破譯

同一把 AES,有人拿它守護銀行,有人卻把它用成形同虛設。CTF 的密碼旗子,幾乎都藏在「用錯、用弱了」的密碼系統背後——這一章,我們翻到攻擊者那一側,親手把它們一一拆開,從而真正讀懂第 35 章那句鐵律的份量。

📌 讀完這一章,你會知道
  • CTF 密碼題到底在玩什麼:它考的不是「破解 AES」,而是抓出「被用錯、用弱」的密碼系統,還原明文或金鑰
  • 破譯者的萬用配方:暴力枚舉把所有可能的金鑰跑一遍,再用評分函數讓程式自動挑出「最像英文」的那一個
  • 古典密碼怎麼破:凱撒(枚舉 26 種位移)、單表替換(頻率分析 + 爬山法)、維吉尼亞(用重合指數找金鑰長度)、單位元組 XOR(枚舉 256 把金鑰)
  • 現代密碼怎麼破:雜湊不可逆卻擋不住字典攻擊、為何 MD5/SHA 不該存密碼、加鹽如何救命;以及當 RSA 的 $n$ 分解得開時,如何一路算回私鑰 $d$ 解密
  • 常見的 RSA CTF 弱點:小 $e$、共模、$p$ 與 $q$ 太近(Fermat 分解)
  • 每一種破譯,背後都對應一條「該怎麼正確做」——攻擊者思維,正是設計者最好的老師
  • 實戰工具箱:CyberChef、RsaCtfTool、factordb、hashcat / john

一道躺著旗子的密碼題

你打開一個 CTF 的 Crypto 題,附件只有一行十六進位字串,題目敘述寫著一句欠揍的話:「我自己寫了個加密,保證沒人破得了。」 你的任務,是從這串亂碼裡撈出一面旗子(flag)——通常長得像 FLAG{s0me_secret_here} 的字串, 交上去就得分。這就是密碼類 CTF 題的日常。

先把一個天大的誤會講清楚。CTF 的 Crypto 題,幾乎從不是要你去破解 AES、RSA 這些演算法本身—— 那些演算法經過全世界密碼學家二十幾年圍攻都沒倒(這正是第 35 章反覆強調的)。真正會出現在題目裡、而且真的能解開的, 是另一類東西:

一個「用錯了、或用弱了」的密碼系統。
——金鑰空間太小、金鑰重複使用、參數選得太爛、把雜湊當加密、自己手刻的「土砲加密」⋯⋯

換句話說,CTF 密碼題是一座「反面教材博物館」:把第 35 章警告過的每一個坑,做成一道道關卡讓你親手踩。 而破題的目標只有兩個:還原明文(讀出旗子),或還原金鑰(往往就能讀出旗子)。這一章, 我們就站到攻擊者那一側,把這些坑一個個拆開——因為唯有真正理解一個系統怎麼被攻破,你才知道該怎麼把它設計得攻不破。

🔑
攻擊是最深刻的學習

第 35 章從「設計者/使用者」的角度,教你密碼學該怎麼用;這一章從「攻擊者」的角度,教你密碼學被用錯時怎麼倒。 兩者是同一枚硬幣的兩面。Kerckhoffs 原則說「演算法可以公開,安全只能靠金鑰」——本章你會親眼看到, 一旦某個環節(金鑰、參數、模式、流程)出包,公開的演算法就成了攻擊者的施工藍圖。把這一章讀完, 你回頭看第 35 章那句「別自己造密碼系統」,會從「一句忠告」變成「一道道血淋淋的傷口」。

⛔
先講規矩:只破解「給你破的」東西

本章所有技術,都只能用在授權的學習環境:CTF 平台的題目、你自己出的練習、合法靶場(picoCTF、CryptoHack)。 這些密文是「故意設弱、放在那裡等你破」的教學材料。絕對不要把這些手法指向任何真實系統、別人的帳號、 未經授權的資料——那在《刑法》妨害電腦使用罪等規範下是犯罪,和「學習」無關。技術中立,用途有責。 這一章的每一行程式,都是為了讓你成為「更會防守的人」而存在。

破譯者的萬用配方:枚舉 + 評分函數

在跳進各種密碼之前,先送你一把貫穿整章的萬能鑰匙。回想第 35 章練習 35.1 那個小頓悟:破解不需要「試到對」的運氣, 而是把人類的直覺(「哪一行讀起來像英文」)變成一個可計算的評分函數。把這句話展開,就是幾乎所有古典密碼破譯共用的兩步配方:

  1. 枚舉(brute-force enumeration):當金鑰空間夠小時(凱撒只有 26 種、單位元組 XOR 只有 256 種),把每一把可能的金鑰都拿來解一次,得到一大堆候選明文。
  2. 評分(scoring):用一個「這段文字有多像英文」的評分函數,替每個候選明文打分,分數最高的那個,十之八九就是答案。

那「像不像英文」要怎麼算?最簡單實用的做法,是給常見字母與空白加分、給亂碼(不可列印字元)重重扣分。 英文裡最常出現的字母是 e t a o i n s h r d l u,再加上空白——一段真正的英文,這些東西的密度會很高; 而一段「解錯了」的亂碼,則充滿了奇怪的符號。把這個直覺寫成函式: 

# 「這段文字有多像英文」的評分函數 —— 整章共用的核心武器
COMMON = "etaoinshrdlu "          # 最常見的英文字母 + 空白

def score_english(text):
    s = 0
    for c in text.lower():
        if c in COMMON:
            s += 2                 # 常見字母與空白:大大加分
        elif c.isalnum() or c in "_{}":
            s += 1                 # 其他英數字、底線、大括號(旗子常用):小加分
        elif not c.isprintable():
            s -= 5                 # 出現亂碼(不可列印字元):重罰
    return s

就這麼簡單的一個函式,卻是整章的主角。它把「人眼掃過去覺得對」這件主觀的事,變成一個冰冷的數字—— 而電腦最擅長的,就是在幾百萬個數字裡瞬間挑出最大值。接下來每一種古典密碼,本質上都是「換一種枚舉方式,但用同一個 score_english 收尾」。

💡
更講究的評分:卡方與 n-gram

上面這個「加加減減」的評分很粗暴但夠用。要更精準,密碼分析常改用兩種統計量: 卡方檢定(chi-squared) $\chi^2 = \sum_i \dfrac{(O_i - E_i)^2}{E_i}$ 衡量「候選明文的字母分布」和「標準英文分布」差多少(越小越像英文); 或用 n-gram 對數機率——把常見的雙字母(th、he)、四字母(tion、that)出現頻率取對數加總,越大越像英文。 待會破維吉尼亞用卡方、破單表替換用雙字母,你會看到它們的威力。

古典密碼破譯(L1)

古典密碼是密碼學的童年,也是 CTF 的入門關。它們的共同死穴,第 35 章已經點破:不是金鑰空間太小(凱撒、XOR), 就是擋不住統計(替換、維吉尼亞)。我們就照這條線,把四種最常見的古典密碼依序拆開。

凱撒/移位:程式替你看 25 種可能

凱撒密碼(Caesar cipher)把每個字母往後推固定格數,金鑰就是那個「位移量」。第 35 章說過它的致命傷: 金鑰只有 25 種可能(在 CTF 裡這類「把字母轉一圈」的也常叫 ROT-n,如著名的 ROT13)。 既然金鑰空間這麼小,我們連肉眼都不必動——把 0 到 25 全試一遍,交給 score_english 挑冠軍: 

def caesar_decrypt(text, shift):
    out = []
    for ch in text:
        if 'a' <= ch <= 'z':
            out.append(chr((ord(ch) - 97 - shift) % 26 + 97))
        elif 'A' <= ch <= 'Z':
            out.append(chr((ord(ch) - 65 - shift) % 26 + 65))
        else:
            out.append(ch)                 # 非字母(空白、標點、數字)原樣保留
    return ''.join(out)

def caesar_encrypt(text, shift):
    return caesar_decrypt(text, -shift)    # 加密 = 反方向解密

def break_caesar(cipher):
    # 對 26 種位移各解一次,挑 score_english 分數最高的
    best = max(range(26), key=lambda s: score_english(caesar_decrypt(cipher, s)))
    return best, caesar_decrypt(cipher, best)

secret = caesar_encrypt("The flag is CTF{ROT_is_not_crypto}", 7)
print("密文:", secret)          # Aol mshn pz JAM{YVA_pz_uva_jyfwav}
print("破解:", break_caesar(secret))
# (7, 'The flag is CTF{ROT_is_not_crypto}') —— 程式自己找出位移是 7

注意 break_caesar 連「位移 7」這個金鑰是怎麼被找出來的:它不靠任何聰明, 只是把 26 個候選明文丟給評分函數,回傳分數最高的那一個。金鑰空間小到能枚舉,等於沒上鎖—— 這是凱撒(以及所有 ROT 系)的原罪,也是你在 CTF 裡看到「一串只有字母被打亂、空白標點都還在」的密文時,該第一個嘗試的手法。

✏️ 練習 70.1 實作

實作 caesar_decrypt(text, shift) 與 break_caesar(cipher),讓程式自動(不靠肉眼逐行看) 破解一段凱撒密文,回傳「找到的位移」與「還原的明文」。請用本節的 score_english 當評分函數, 並用一段你自己加密的句子驗證它真的找得到正確位移。

單表替換:頻率分析 + 爬山法

「金鑰太少」很好補:讓每個字母隨機對應到另一個字母,而不是整齊地推固定格數,這就是單表替換密碼(monoalphabetic substitution)。 金鑰是一張 26 個字母的對照表,排列數高達 $26! \approx 4 \times 10^{26}$——暴力枚舉徹底沒戲。可是第 35 章請出了九世紀的金迪(Al-Kindi): 每個字母換了張臉,但它出現的「頻率」沒變。

這就是頻率分析(frequency analysis)。密文裡最常出現的符號,八成對應明文的 e;次常出現的對應 t……。 但光靠單字母頻率,只能猜對幾個高頻字母,j、q、x、z 這些低頻字母會張冠李戴。怎麼自動化、又補上單字母頻率的不足? 答案是爬山法(hill climbing):先用頻率猜一個起始對照表,然後不斷「隨機交換表裡兩個字母的對應」, 只要交換後「更像英文」就保留,否則還原——像爬山一樣,每一步都往分數更高處走。 

import random, string
from collections import Counter

ENG_FREQ = {  # 各字母在英文中的出現頻率(%)
    'a':8.2,'b':1.5,'c':2.8,'d':4.3,'e':12.7,'f':2.2,'g':2.0,'h':6.1,
    'i':7.0,'j':0.15,'k':0.77,'l':4.0,'m':2.4,'n':6.7,'o':7.5,'p':1.9,
    'q':0.095,'r':6.0,'s':6.3,'t':9.1,'u':2.8,'v':0.98,'w':2.4,'x':0.15,
    'y':2.0,'z':0.074}
COMMON_BIGRAMS = {  # 常見的雙字母組合,出現就加分
    'th':3,'he':3,'in':2,'er':2,'an':2,'re':2,'on':2,'at':1,'en':1,'nd':1,
    'ti':1,'es':1,'or':1,'te':1,'of':1,'ed':1,'is':1,'it':1,'al':1,'ng':1}

def decrypt_sub(cipher, key):
    # key:字典「密文字母 → 猜測的明文字母」(皆小寫)
    out = []
    for c in cipher:
        low = c.lower()
        if low in key:
            p = key[low]
            out.append(p.upper() if c.isupper() else p)
        else:
            out.append(c)
    return ''.join(out)

def fitness(text):
    # 評分:字母頻率越接近英文越好(差距當扣分)+ 常見雙字母加分
    t = [c for c in text.lower() if c.isalpha()]
    n = len(t)
    if n == 0:
        return -1e9
    s = 0.0
    for ch, f in Counter(t).items():
        s -= abs(ENG_FREQ.get(ch, 0) - 100 * f / n)
    low = ''.join(t)
    for i in range(len(low) - 1):
        s += COMMON_BIGRAMS.get(low[i:i+2], 0)
    return s

def break_substitution(cipher, restarts=20, steps=2000):
    letters = list(string.ascii_lowercase)
    best_key, best_score = None, -1e18
    for _ in range(restarts):               # 多次隨機重啟,避免卡在「半山腰」
        vals = letters[:]
        random.shuffle(vals)
        key = dict(zip(letters, vals))
        score = fitness(decrypt_sub(cipher, key))
        for _ in range(steps):
            a, b = random.sample(letters, 2)
            key[a], key[b] = key[b], key[a]      # 隨機交換兩個對應
            new = fitness(decrypt_sub(cipher, key))
            if new > score:
                score = new                       # 變好就保留
            else:
                key[a], key[b] = key[b], key[a]   # 沒變好就還原
        if score > best_score:
            best_score, best_key = score, dict(key)
    return best_key, decrypt_sub(cipher, best_key)

ℹ️
為什麼不是百分百還原?——因為這是「玩具評分」

用「字母頻率 + 雙字母」當評分,通常能還原六、七成,讓你一眼認出內容、補完剩下幾個字母。 想要程式 100% 自動破出,實務做法是把評分換成 quadgram(四連字母)對數機率: 事先用一大堆英文語料統計每個四字母組合的機率,評分 = 候選明文裡所有四字母對數機率的總和。 這個評分對「哪裡像英文、哪裡不像」極度敏感,配合爬山法,幾句話的密文就能完美還原。 這正是線上工具(如 quipqiup)背後的原理。但核心精神不變:枚舉/搜尋 + 一個夠好的評分函數。

✏️ 練習 70.2 挑戰

用頻率分析破單表替換。先寫最基本的一招:統計密文裡每個字母的出現頻率, 把「最常出現的密文字母」對應到 e、次常的對應 t……(依英文頻率排序 etaoinshrdlu...), 產生一個「起始對照表」並解密看看。然後(進階)接上本節的爬山法 break_substitution, 觀察它如何從這個粗糙的起點,一步步把明文磨清楚。

看解答 
from collections import Counter

ENG_ORDER = "etaoinshrdlcumwfgypbvkjxqz"   # 英文字母依頻率由高到低

def frequency_guess(cipher):
    # 密文字母依出現次數由高到低排序
    counts = Counter(c for c in cipher.lower() if c.isalpha())
    cipher_order = [c for c, _ in counts.most_common()]
    # 補齊沒出現的字母,湊滿 26 個
    for c in "abcdefghijklmnopqrstuvwxyz":
        if c not in cipher_order:
            cipher_order.append(c)
    # 最常見密文字母 → e,次常 → t,以此類推
    key = {cipher_order[i]: ENG_ORDER[i] for i in range(26)}
    return key

def decrypt_sub(cipher, key):
    return ''.join(key.get(c.lower(), c) for c in cipher)

cipher = "rth vczwn flauq yam scpgd abhl rth xjke iao"
key0 = frequency_guess(cipher)
print("頻率分析起始解:", decrypt_sub(cipher, key0))
# 起點通常已能看出 'the' 等高頻字;再餵給上一節的 break_substitution() 爬山,
# 就能把剩下的字母逐步喬對 —— 頻率分析給「起點」,爬山法給「收斂」。

洞見:頻率分析單獨用,只在密文很長時準;但它是爬山法的絕佳起跑點。 「先用統計給個好起點,再用局部搜尋優化」——這個套路,在密碼分析、機器學習、最佳化裡反覆出現。

維吉尼亞:先找出金鑰長度

單表替換敗在「同一個字母永遠換成同一個臉」,頻率藏不住。維吉尼亞密碼(Vigenère cipher)想補這個洞: 用一個關鍵字當金鑰,循環地對每個字母施以「不同的凱撒位移」。比方金鑰是 LEMON, 第 1 個字母位移 L(=11)、第 2 個位移 E(=4)、第 3 個位移 M(=12)……到第 6 個又繞回 L。 這叫多表替換(polyalphabetic):同一個明文字母,在不同位置會變成不同的密文,單純的頻率分析就此失靈。

但它有個關鍵破綻:金鑰會循環。一旦你知道金鑰長度是 $L$,整個維吉尼亞就「碎成」 $L$ 個獨立的凱撒密碼—— 第 1、$1{+}L$、$1{+}2L$… 個字母用的是同一個位移,把它們抽出來,就是一段純凱撒密文,而凱撒你已經會破了! 所以破維吉尼亞的關鍵,全在第一步:找出金鑰長度。

歷史上有兩種找金鑰長度的方法。Kasiski 檢測:找密文裡重複出現的字母片段,量它們之間的距離, 金鑰長度往往是這些距離的公因數。更穩、更好寫成程式的是重合指數(Index of Coincidence, IC):

IC 衡量「隨手抓兩個字母,它們剛好相同」的機率。一段自然英文因為字母分布不均(e 特別多),IC 約 0.066; 而一段接近隨機的文字,IC 約 0.038。公式是:

$$\text{IC} = \frac{\sum_{i=\text{a}}^{\text{z}} n_i(n_i-1)}{N(N-1)}$$

其中 $n_i$ 是字母 $i$ 出現的次數,$N$ 是總字母數。妙用在這:如果我們猜對了金鑰長度 $L$, 那麼每一欄都是「同一個凱撒位移」的英文(只是整體平移),字母分布依然不均,IC 會接近 0.066; 但如果猜錯,欄內混了不同位移,分布被抹平,IC 會掉向 0.038。所以——逐一試各種長度,哪個長度讓「各欄平均 IC」最接近 0.066,它就是金鑰長度。 

import string
from collections import Counter

def clean(text):
    return [c for c in text.lower() if c.isalpha()]

def index_of_coincidence(letters):
    n = len(letters)
    if n < 2:
        return 0.0
    counts = Counter(letters)
    return sum(c * (c - 1) for c in counts.values()) / (n * (n - 1))

def guess_key_length(text, max_len=15):
    letters = clean(text)
    scores = []
    for klen in range(1, max_len + 1):
        # 把字母依「位置 mod klen」分成 klen 欄,各算 IC 取平均
        ics = [index_of_coincidence(letters[i::klen]) for i in range(klen)]
        scores.append((sum(ics) / klen, klen))
    scores.sort(reverse=True)        # 平均 IC 越接近 0.066 越可能是對的長度
    return scores

ENG_FREQ = {'a':8.2,'b':1.5,'c':2.8,'d':4.3,'e':12.7,'f':2.2,'g':2.0,'h':6.1,
    'i':7.0,'j':0.15,'k':0.77,'l':4.0,'m':2.4,'n':6.7,'o':7.5,'p':1.9,'q':0.095,
    'r':6.0,'s':6.3,'t':9.1,'u':2.8,'v':0.98,'w':2.4,'x':0.15,'y':2.0,'z':0.074}

def best_shift(col):
    # 對單一欄(等於一段凱撒)用卡方找最佳位移
    best_s, best_chi = 0, 1e18
    n = len(col)
    for s in range(26):
        dec = [chr((ord(c) - 97 - s) % 26 + 97) for c in col]
        cnt = Counter(dec)
        chi = sum((cnt.get(ch, 0) - ENG_FREQ[ch] / 100 * n) ** 2 / (ENG_FREQ[ch] / 100 * n)
                  for ch in string.ascii_lowercase)
        if chi < best_chi:
            best_chi, best_s = chi, s
    return best_s

def break_vigenere(text, klen):
    letters = clean(text)
    shifts = [best_shift(letters[i::klen]) for i in range(klen)]
    key = ''.join(chr(97 + s) for s in shifts)
    # 用求出的位移把整段(連標點)解回來
    out, idx = [], 0
    for c in text:
        if c.isalpha():
            s = shifts[idx % klen]
            out.append(chr((ord(c.lower()) - 97 - s) % 26 + 97))
            idx += 1
        else:
            out.append(c)
    return key, ''.join(out)
def vigenere_encrypt(text, key):
    out, idx = [], 0
    for c in text:
        if c.isalpha():
            s = ord(key[idx % len(key)]) - 97
            out.append(chr((ord(c.lower()) - 97 + s) % 26 + 97))
            idx += 1
        else:
            out.append(c)
    return ''.join(out)

# 一段用金鑰 LEMON 加密的維吉尼亞密文(夠長,統計才穩):
plaintext = ("the quick brown fox jumps over the lazy dog and then the dog runs "
             "back to the warm house where it sleeps all afternoon while the "
             "children play in the green garden near the river bank") * 2
cipher = vigenere_encrypt(plaintext, "lemon")

print(guess_key_length(cipher)[:4])
# [(0.0739, 15), (0.0726, 10), (0.0680, 5), (0.0487, 9)] ← 高分都是 5 的倍數!
key, plain = break_vigenere(cipher, 5)   # 挑「最小」的強候選 5
print("金鑰:", key)        # lemon
print(plain[:40])          # the quick brown fox jumps over the lazy  完整還原
⚠️
小心:金鑰長度的「倍數」也會得高分

上面結果裡,15、10、5 的 IC 都很高——因為金鑰長度的整數倍也能讓每一欄落在「同一個位移」上(只是切得更細)。 所以挑金鑰長度時,要選那群高分裡最小的(這裡是 5),它才是真正的週期。這個「真實週期 vs 其倍數」的現象, 在訊號處理、找重複圖樣時也會遇到,記起來很有用。

單位元組 XOR:枚舉 256 把金鑰

第 69 章(CTF 編碼基礎)與第 35 章都見過 XOR:它是對稱加密最底層的原子,性質是「同一個值連續 XOR 兩次會變回自己」。 CTF 最常見的入門變體是單位元組 XOR(single-byte XOR):把整段資料的每一個位元組,都和「同一個位元組金鑰」做 XOR。 金鑰只是 0 到 255 之間的一個數——金鑰空間只有 256,比凱撒還小。破法你已經完全猜得到:全試一遍,評分挑冠軍。 

def break_single_xor(data):
    best = (None, -1e9, "")           # (金鑰, 分數, 明文)
    for k in range(256):              # 枚舉全部 256 個可能的位元組金鑰
        cand = bytes(b ^ k for b in data)
        # 凡是出現「不可列印字元」的候選,直接淘汰
        if any(b < 9 or (13 < b < 32) or b > 126 for b in cand):
            continue
        text = cand.decode("ascii", "replace")
        sc = score_english(text)
        if sc > best[1]:
            best = (k, sc, text)
    return best[0], best[2]

# 出題者把旗子用金鑰 0x3D 做單位元組 XOR(這把金鑰是「祕密」):
plain  = b"CTF{single_byte_xor_falls_in_256_tries}"
cipher = bytes(b ^ 0x3D for b in plain)
print("密文(hex):", cipher.hex())
# 7e697b464e54535a5158625f4449586245524f625b5c51514e625453620f080b62494f54584e40

key, recovered = break_single_xor(cipher)
print(f"破出金鑰 = {key} (0x{key:02x})")   # 破出金鑰 = 61 (0x3d)
print(recovered)                            # CTF{single_byte_xor_falls_in_256_tries}

注意那行「淘汰含不可列印字元的候選」——它是評分前的硬性過濾:正確的明文是一段乾淨的 ASCII 文字, 而錯誤的金鑰幾乎一定會解出一堆控制字元或高位位元組。先砍掉這些明顯不可能的候選,剩下的再交給 score_english 精挑, 破解既快又準。當你在 CTF 拿到一串 hex、嗅到「可能是單位元組 XOR」時,這 20 行就是你的開罐器。

ℹ️
那「多位元組 XOR」呢?——回到維吉尼亞

如果金鑰是好幾個位元組、循環重複地 XOR(repeating-key XOR),它其實就是「位元組版的維吉尼亞」! 破法一模一樣:先用 IC(或漢明距離)猜出金鑰長度 $L$,把資料每隔 $L$ 個位元組抽成一欄,每一欄就退化成單位元組 XOR, 各自枚舉 256 把金鑰破掉,再拼回完整金鑰。這正是著名的 Cryptopals 挑戰第 6 題。 看出來了嗎?古典密碼的破法是會「複用」的——維吉尼亞 ≈ 多重凱撒,repeating-XOR ≈ 多重單位元組 XOR。

現代密碼的破譯(L2)

古典密碼是「演算法本身就弱」;現代密碼(雜湊、RSA)的演算法很強,CTF 考的是「用法」或「參數」出了包。 這一節我們看兩個最經典的現代破譯場景:雜湊的字典攻擊,以及 RSA 的弱參數分解。

雜湊破解:單向,但擋不住字典攻擊

第 35 章說過,密碼學雜湊(MD5、SHA-256)是單向的:給輸入算指紋很容易,從指紋反推輸入「在計算上不可行」。 那 CTF 給你一個雜湊值要你「破解」,不是矛盾嗎?關鍵在:「不可逆」不等於「猜不到」。 如果原文是個常見密碼(像 sunshine、123456),攻擊者根本不必反推—— 他把一大堆常見密碼正著算一遍雜湊,和目標比對,中了就是它。這就是字典攻擊(dictionary attack)。 

import hashlib

def crack_hash(target_hex, wordlist, algo="md5"):
    h = getattr(hashlib, algo)            # 支援 md5 / sha1 / sha256 ...
    for w in wordlist:
        if h(w.encode()).hexdigest() == target_hex:
            return w                       # 找到原文!
    return None

# 假設這是外洩資料庫裡的一條雜湊(攻擊者只看得到這串):
target = hashlib.md5(b"sunshine").hexdigest()
print("外洩雜湊:", target, "(長度", len(target), "→ 32 個 hex,典型 MD5)")

# 攻擊者拿一份常見密碼字典來逐一比對:
wordlist = ["123456", "password", "qwerty", "sunshine", "letmein", "dragon"]
print("破出原文:", crack_hash(target, wordlist, "md5"))   # sunshine

真實世界的字典不是這 6 個字,而是像 rockyou.txt 這種收錄了上千萬個外洩密碼的清單; 搭配 GPU,每秒能算上百億次雜湊。這就是為什麼第 35 章說「絕不能只存密碼的雜湊」: MD5/SHA 設計得飛快(對校驗檔案是優點,對存密碼是災難),又沒加鹽,等於把使用者的密碼裝在玻璃箱裡。

💡
看長度認雜湊

拿到一串十六進位,先數長度就能猜演算法:32 個 hex 字元 = 128 bit = MD5; 40 = 160 bit = SHA-1;64 = 256 bit = SHA-256; 以 $2y$、$argon2 開頭的則是 bcrypt / Argon2(有加鹽、故意很慢,字典攻擊會慢到崩潰)。 線上工具 hash-identifier、或直接觀察長度,是 CTF 雜湊題的第一步。

而加鹽(salt)正是字典/彩虹表攻擊的剋星(第 35 章詳述過)。看一眼它為什麼有效: 

import hashlib, os

# 沒加鹽:同樣的密碼 → 同樣的雜湊 → 攻擊者可「預先算好一張大表」一次查光
print(hashlib.sha256(b"sunshine").hexdigest())
print(hashlib.sha256(b"sunshine").hexdigest())   # 兩行完全一樣 → 彩虹表的天堂

# 加鹽:每個使用者配一段隨機鹽,拼進去再雜湊 → 同密碼也算出不同雜湊
def hash_with_salt(password):
    salt = os.urandom(16)
    digest = hashlib.sha256(salt + password.encode()).hexdigest()
    return salt.hex(), digest

print(hash_with_salt("sunshine"))
print(hash_with_salt("sunshine"))   # 兩行的 digest 完全不同 → 預算好的彩虹表全部作廢

加了鹽,攻擊者就不能「算一張表查所有人」,只能針對每一份鹽、每一個使用者,重新硬算; 再配上 bcrypt / Argon2 這類故意很慢的雜湊,字典攻擊的成本被推高好幾個數量級。 實戰裡跑大規模字典攻擊的專業工具是 hashcat 與 john(John the Ripper)—— 一行 hashcat -m 0 hash.txt rockyou.txt 就能對一個 MD5 雜湊跑字典(-m 0 指定 MD5); 我們上面那 10 行 Python,正是它們的「原理縮小版」。

加鹽為何讓它失效?因為被雜湊的不再是 password,而是 salt + password。 我們字典裡算的是 md5("sunshine"),但資料庫存的是 md5(salt + "sunshine")——兩者對不上。 攻擊者必須先知道那份鹽(鹽通常和雜湊一起存,所以拿得到),再把每個候選密碼都「拼上這份鹽」重算; 這雖然仍可行,但同一張表不能再套用到別的使用者(每人鹽不同),攻擊成本因此暴增—— 彩虹表(預先算好的巨表)更是直接報廢。這就是加鹽的價值。

RSA 弱參數:當 n 分解得開

第 15 章與第 35 章都拼過玩具 RSA,複習它的命根子:公鑰是 $(e, n)$,私鑰指數 $d$ 滿足 $e \cdot d \equiv 1 \pmod{\varphi(n)}$, 其中 $\varphi(n) = (p-1)(q-1)$,而 $n = p \times q$。RSA 的安全,全押在「把 $n$ 分解回 $p, q$ 極其困難」這個信念上—— 真實世界的 $n$ 有 2048 位元以上,分解它比宇宙年齡還久。

但 CTF 出題者(以及現實中的粗心工程師)常犯一個錯:$n$ 用了太小、或太好分解的質數。 一旦你能分解 $n$,整座城堡瞬間倒塌——因為有了 $p, q$,你就能算出 $\varphi(n)$,進而算出私鑰 $d$,然後像合法收信人一樣解密。 攻擊鏈是這樣的:

用 Python 把這條鏈走一遍。出題者刻意把質數設小(這裡約 21 位元),試除法(trial division)幾乎瞬間就分解掉:

def factor(n):
    # 試除法:從 2 開始找最小質因數。n 由小質數組成時秒解;
    # 真實 RSA 的 n 有 600+ 位數,這招要算到天荒地老 —— 這正是 RSA 安全的來源。
    i = 2
    while i * i <= n:
        if n % i == 0:
            return i, n // i
        i += 1
    return None

def rsa_break(n, e, c):
    p, q = factor(n)                 # ① 分解 n
    phi = (p - 1) * (q - 1)          # ② 算 φ(n)
    d = pow(e, -1, phi)              # ③ 求私鑰 d(Python 3.8+ 內建模反元素)
    m = pow(c, d, n)                 # ④ 解密 m = c^d mod n
    return m.to_bytes((m.bit_length() + 7) // 8, "big")   # 整數轉回位元組

# ---- 出題者(用了弱參數):n 只用兩個小質數 ----
p, q = 1299709, 2750021
n   = p * q                          # 3574227043889
e   = 65537
c   = pow(int.from_bytes(b"Eul3r", "big"), e, n)   # 把旗子核心加密成密文
print("給攻擊者的:n =", n, " e =", e, " c =", c)

# ---- 攻擊者(只拿到 n, e, c)----
inner = rsa_break(n, e, c)
print("解出:", inner)                # b'Eul3r'
print("旗子: FLAG{" + inner.decode() + "}")   # FLAG{Eul3r}

在我的機器上,factor 大約 0.3 秒就把 $n$ 分解掉了。這就是弱 RSA 的全部祕密:不是 RSA 演算法壞了, 而是有人把「天文數字的大質數」換成了「秒解的小質數」。當你在 CTF 看到一個小小的 $n$,別猶豫,直接分解。

💡
不必自己算分解:factordb 與 RsaCtfTool

很多 CTF 的 $n$,早就被世界上某個人分解過、上傳到公開資料庫 factordb.com——你把 $n$ 貼上去查, 可能直接拿到 $p, q$,連跑程式都省了。要更全自動,有神兵 RsaCtfTool:餵它公鑰或 $(n, e, c)$, 它會把幾十種已知 RSA 攻擊(下面要講的小 $e$、共模、Fermat⋯⋯)全部自動試一輪。 但工具是「加速器」不是「替代品」:你得先懂下面這些弱點長什麼樣,才知道工具在替你做什麼、結果對不對。

m = pow(c, d, n)
inner = m.to_bytes((m.bit_length() + 7) // 8, "big")
print("旗子: FLAG{" + inner.decode() + "}")   # FLAG{Eul3r}

整題的「難」只在第一步分解;因為 $n$ 被設小,試除法瞬殺。把這四步背起來——它是 RSA 類 CTF 出現頻率最高的解法骨架。 延伸想一想:如果 $n$ 是 2048 位元的兩個大質數乘積,factor 會卡在哪?(答案:第一個 while 迴圈, 要跑到約 $\sqrt{n} \approx 2^{1024}$ 次,地老天荒。)

常見的 RSA CTF 弱點

「$n$ 太小可分解」只是最基本的一種。RSA 的數學很美,但只要任何一個參數選錯,就有對應的攻擊。 認識它們,既能讓你在 CTF 快速辨題,也能讓你日後設計系統時知道「哪些洞絕對不能挖」:

弱點出包原因攻擊概念
小 $e$ + 短訊息 用 $e=3$ 又不做填充(padding),且明文 $m$ 很小 若 $m^e$ 比 $n$ 還小,加密時根本沒「模到」,$c = m^e$ 是直接的乘冪;攻擊者對 $c$ 開 $e$ 次方根($m = \sqrt[e]{c}$)就還原明文,連 $n$ 都不必碰。
共模(common modulus) 同一個 $n$,用兩個互質的 $e_1, e_2$ 加密了同一則訊息 用擴展歐幾里得找 $a, b$ 使 $a e_1 + b e_2 = 1$,則 $c_1^{a} \cdot c_2^{b} \equiv m \pmod n$。不必分解 $n$ 就還原明文。
$p, q$ 太近 兩個質數選得太接近(差距小) Fermat 分解:把 $n$ 寫成 $a^2 - b^2 = (a-b)(a+b)$,從 $a = \lceil\sqrt{n}\,\rceil$ 開始往上試,$p, q$ 越近,$b$ 越小,幾步就分解出來。
$n$ 在 factordb 裡 用了別人用過、或太常見的質數 直接上 factordb.com 查 $n$,$p, q$ 可能已被別人算好公開。

其中 Fermat 分解最值得親手寫一次,它把「$p, q$ 太近」這個看不見的疏忽,變成幾行就能利用的攻擊:

from math import isqrt

def fermat_factor(n):
    # n = a^2 - b^2 = (a-b)(a+b);p,q 越接近,a 從 sqrt(n) 往上沒幾步就中
    a = isqrt(n)
    if a * a < n:
        a += 1
    while True:
        b2 = a * a - n
        b = isqrt(b2)
        if b * b == b2:           # b2 剛好是完全平方 → 找到 a, b
            return a - b, a + b
        a += 1

# 兩個只差 30 的質數(刻意選太近):
n = 1000003 * 1000033
print(fermat_factor(n))           # (1000003, 1000033) —— 瞬間分解
⚠️
這些攻擊不是「RSA 壞了」,是「RSA 被用壞了」

每一條弱點,病根都在參數或用法,而非演算法:小 $e$ 沒填充、共用模數、質數太近或重複。 正確實作的 RSA(2048 位元以上、隨機獨立的大質數、$e=65537$、OAEP 填充)至今穩如泰山。 這完美呼應第 35 章的鐵律——演算法很少是被攻破的點,「怎麼用」的細節才是。所以,別自己拼裝 RSA。

✏️ 練習 70.6 挑戰

(任選一題)(a) Vigenère:用本章的 guess_key_length 與 break_vigenere, 寫一個「丟進密文 → 自動猜金鑰長度 → 還原明文與金鑰」的完整破譯器,並用一段你自己用某關鍵字加密的長文驗證。 (b) Fermat:實作 fermat_factor(n),並設計一個小實驗:固定一個質數 $p$, 讓 $q$ 從「離 $p$ 很近」逐漸變「離 $p$ 很遠」,記錄 fermat_factor 各花了幾次迴圈, 用數據說明「為什麼 $p, q$ 不能選太近」。

看解答(b) 
from math import isqrt

def is_prime(x):
    if x < 2: return False
    for d in range(2, isqrt(x) + 1):
        if x % d == 0: return False
    return True

def fermat_factor(n):
    a = isqrt(n)
    if a * a < n:
        a += 1
    steps = 0
    while True:
        steps += 1
        b2 = a * a - n
        b = isqrt(b2)
        if b * b == b2:
            return (a - b, a + b), steps
        a += 1

p = 1000003
for gap in [10, 100, 10_000, 1_000_000]:
    # 找一個離 p 大約 gap 遠的質數 q
    q = next(x for x in range(p + gap, p + gap + 5000) if is_prime(x))
    (f1, f2), steps = fermat_factor(p * q)
    print(f"q - p ≈ {q - p:>8} → Fermat 花了 {steps} 次迴圈")
# 你會看到:p、q 越近,steps 越小(常常 1～2 次);拉遠後 steps 暴增。
# 結論:Fermat 分解的成本約和 (q - p)^2 成正比 → 質數必須「隨機且相距夠遠」。

這個實驗把一句抽象的安全守則(「質數別選太近」)變成你親眼看到的曲線。 這正是攻擊者思維的價值:你不是背一條規則,而是理解規則背後那道「越近越好攻」的數學斜坡。

攻擊者思維,讓你成為更好的設計者

現在把整章倒過來看。我們拆開的每一個系統,其實都在無聲地教我們「該怎麼正確做」——這正是 CTF 之於資安教育最珍貴的地方。 把破譯與防禦並排,你會發現它們是同一句話的正反兩面(全部呼應第 35 章的正確用法):

我們怎麼破的弱在哪該怎麼正確做(第 35 章)
凱撒:枚舉 26 種位移金鑰空間太小用 AES,金鑰空間 $2^{128}$ 起跳,枚舉到宇宙毀滅也試不完
單表替換:頻率分析 + 爬山統計特徵藏不住用現代對稱密碼,密文要和隨機亂數無法區分(沒有可被統計的規律)
維吉尼亞 / repeating-XOR:找週期再各個擊破金鑰循環重複使用金鑰流不可重複;一次性密本的金鑰「只能用一次」;對稱加密每則訊息配獨立的隨機 nonce/IV
雜湊:字典攻擊正著查快、無鹽,密碼又是常見字存密碼用 加鹽 + 慢雜湊(bcrypt / scrypt / Argon2);絕不用 MD5/SHA 裸存

把這一章串起來

我們翻到攻擊者那一側,把第 35 章警告過的每一個坑,親手踩了一遍、也拆了一遍:

  • CTF Crypto 的本質:不是破解強演算法,而是抓出「用錯、用弱」的系統,還原明文或金鑰。
  • 萬用配方:暴力枚舉 + 評分函數(score_english / 卡方 / n-gram),把「像不像英文」變成可計算的數字。
  • 古典四破:凱撒(枚舉 26)、單表替換(頻率分析 + 爬山法)、維吉尼亞(IC 找金鑰長度,再各欄當凱撒破)、單位元組 XOR(枚舉 256)。破法會互相複用。
  • 現代兩破:雜湊用字典攻擊正著查(故加鹽 + 慢雜湊才安全);RSA 在 $n$ 分解得開時,沿「分解 → φ → d → 解密」的攻擊鏈一路推倒,並認識小 $e$、共模、$p,q$ 太近(Fermat)等經典弱點。
  • 攻防一體:每一種破譯,都對應一條「該怎麼正確做」;理解攻擊,正是設計安全系統的前提。
  • 工具箱:CyberChef、factordb、RsaCtfTool、hashcat/john——先懂原理,再用它們加速。

密碼破譯最迷人的地方,是它讓「安全」這個抽象詞,變成你指尖能驗證的東西:你親手讓一面藏了三千年的旗子現形, 也親眼看見一個聰明的系統如何因為一個小疏忽而全盤崩塌。帶著這雙「破譯者的眼睛」,你回頭再讀第 35 章那句 「永遠別自己造密碼系統」,終於懂了它不是膽小,而是看過太多屍體後的智慧。 下一章,我們會把戰場從「數學」搬到「服務與檔案」——看 Web 攻防、封包分析與檔案鑑識, 如何用同樣的攻擊者思維,在真實的協定與位元組之間奪旗。
