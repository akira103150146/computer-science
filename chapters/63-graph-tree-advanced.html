圖的建模與表示:競賽寫法

競賽裡的第一道關卡,往往不是演算法,而是「怎麼把題目讀進來、存成圖」。第 15、30 章說過, 圖有兩種主要存法:鄰接矩陣(adjacency matrix)與鄰接串列(adjacency list)。 在競賽中,除非點數很少($V$ 大約 $\le 500$)或你要查「全點對」,否則答案幾乎永遠是鄰接串列—— 因為現實的圖通常稀疏(sparse):邊數遠小於 $V^2$。

鄰接串列:C++ 的 vector 寫法

競賽 C++ 的鄰接串列就是「一個 vector 陣列」:g[u] 存著 u 的所有鄰居。 習慣上開全域固定大小陣列(避免每次 new 的開銷),點通常編號 $1 \sim n$。 

#include <bits/stdc++.h>   // 競賽常用:一次引入所有標準函式庫
using namespace std;

const int N = 1e5 + 5;       // 最大點數,多開幾格保險
vector<int> g[N];            // g[u] = u 的鄰居清單

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);  // I/O 加速,競賽必備
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        g[u].push_back(v);
        g[v].push_back(u);   // 無向圖:兩個方向都加。有向圖只加這行上面那一條
    }
    // 想列舉 u 的鄰居:for (int v : g[u]) ...
    return 0;
}

帶權圖與有向圖

若邊有權重(weight)(距離、費用、時間),鄰居清單裡每一項就要同時記「鄰居是誰」和「邊權多少」。 最簡潔的寫法是 vector<pair<int,int>>,用 {鄰居, 邊權} 表示; 若還需要更多欄位,就改用 struct。 

// 帶權圖:g[u] 存 {鄰居 v, 邊權 w}
vector<pair<int,int>> g[N];

// 讀入一條無向帶權邊
int u, v, w;
cin >> u >> v >> w;
g[u].push_back({v, w});
g[v].push_back({u, w});

// 列舉 u 的帶權鄰居(C++17 結構化綁定,乾淨好讀)
for (auto [v, w] : g[u]) {
    // v 是鄰居,w 是這條邊的權重
}

隱式圖:把問題「翻譯」成圖

競賽圖論真正的門檻,常常不在演算法,而在建模(modeling)——看出「這題其實是一張圖」。 很多題目表面上和圖毫無關係,但只要你定義好「什麼是點、什麼算相鄰」,它就瞬間塌縮成一個 BFS 或 DFS。 這種「點與邊沒有真的存進記憶體,而是需要時即時推算」的圖,叫隱式圖(implicit graph)。

三個經典的「翻譯」例子,值得你刻進直覺:

  • 單字接龍(word ladder):每個單字是一個點,「只差一個字母」就算一條邊。問「最少改幾次能從 A 變到 B」 ——就是隱式圖上的 BFS 最短步數。沒有人會真的把所有單字對建成鄰接串列,而是查字典即時找鄰居。
  • 網格 / 迷宮:每個格子是一個點,「上下左右相鄰且不是牆」算一條邊。求最少步數就是 BFS。 格子的鄰居用座標位移即時算出,不必建圖。(這就是下一節「圖外搜尋」的主場。)
  • 狀態空間:魔術方塊的每個轉法、推箱子的每個盤面、八數碼的每種排列,都是一個「狀態點」, 「一步操作可達」就是邊。求最少步數,還是 BFS。

AP325 教材裡的「銀河帝國旅行社」把星球的從屬關係翻譯成父子節點(於是變成一棵樹), 「邏輯電路」把邏輯閘當點、電線當邊(於是延遲計算變成拓樸排序),都是同一種思維。 記住這句口訣:看到「最少步數」「能不能到」「分成幾群」「先後順序」,先問自己——點是什麼?邊是什麼?

建模優先於演算法

本章後面所有演算法,你在第 30 章其實都見過。競賽真正考你的,是「認出它」: 把一個披著文字外衣的問題,剝開成「這是一張圖,我要在上面跑 BFS / 最短路 / 拓樸排序」。 一旦建模成功,寫程式只是把模板貼上去。所以解競賽圖論題,八成的力氣花在想「點與邊」,兩成花在打字。

BFS 與 DFS:競賽應用全家桶

BFS 與 DFS 是所有圖演算法的根。第 30 章講透了它們的直覺,這裡我們直接看 C++ 模板,然後一口氣示範 四個競賽最常考的應用:連通元件、洪水填充、網格最短步數、二分圖判定。

C++ 模板:DFS(遞迴)與 BFS(佇列)

vector<int> g[N];
bool vis[N];

// DFS:遞迴,簡潔。注意 C++ 預設遞迴深度有限,點太多(深鏈)可能爆 stack
void dfs(int u) {
    vis[u] = true;
    for (int v : g[u])
        if (!vis[v]) dfs(v);
}

// BFS:佇列,順便求無權最短步數
int dist[N];
void bfs(int s) {
    queue<int> q;
    q.push(s); vis[s] = true; dist[s] = 0;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (int v : g[u]) if (!vis[v]) {
            vis[v] = true;            // 關鍵:入列時就標記(第 30 章的鐵律)
            dist[v] = dist[u] + 1;
            q.push(v);
        }
    }
}

兩者都是 $O(V+E)$。差別只在那個「待辦容器」是佇列(BFS,先進先出,層層擴散)還是堆疊/遞迴 (DFS,後進先出,一路到底)。看到「最少步數」用 BFS;看到「找一條路、連通塊、判環」用 DFS——這條取捨在第 30 章詳述過。

應用一:連通元件——這張圖分成幾塊?

「圖分成幾個互不相連的塊?」是最基本的圖論題。做法:對每個還沒被走過的點發動一次 DFS/BFS, 每發動一次就代表發現了一座新的「島」。 

int components = 0;
for (int i = 1; i <= n; i++) {
    if (!vis[i]) {
        components++;     // 找到一塊新的連通元件
        dfs(i);           // 把這一整塊走光、標記掉
    }
}
cout << components << '\n';

這個 $O(V+E)$ 的小迴圈是無數題目的骨架。AP325 的「開車蒐集寶物」(找最大權重連通塊)、 CSES 的「Building Roads」(問還要補幾條邊才全連通——答案就是連通元件數 $- 1$)都是它的變形。 順帶一提,連通性也能用並查集解(本章稍後的 MST 會用到),兩種工具都要會。

圖上 vs 圖外(網格):搜尋的兩種長相

這是競賽圖論一個非常重要、卻常被忽略的觀念分水嶺。BFS/DFS 的「骨架」永遠一樣, 差別只在「怎麼找到一個點的鄰居」:

圖上(explicit graph)圖外(網格 / 隱式圖)
圖怎麼來讀入邊、建成鄰接串列 g[]沒有建圖,鄰居「即時算」
怎麼找鄰居for (int v : g[u])用方向陣列 nx=x+dx[k], ny=y+dy[k]
「點」是什麼頂點編號格子座標 (x, y)
典型題城市道路、社群關係、相依迷宮、數島、淹水、棋盤

「圖外」搜尋的核心工具是方向陣列(direction array):把「上下左右」四個移動打包成兩個小陣列, 用一個迴圈就能列舉四個鄰居。要對角線就用八方向版。 

// 四方向:下、上、右、左
int dx[4] = {1, -1, 0, 0};
int dy[4] = {0, 0, 1, -1};

// 列舉格子 (x, y) 的四個鄰居,標準三件套:位移 → 邊界檢查 → 可走檢查
for (int k = 0; k < 4; k++) {
    int nx = x + dx[k], ny = y + dy[k];
    if (nx >= 0 && nx < n && ny >= 0 && ny < m && grid[nx][ny] == 0) {
        // (nx, ny) 在界內、而且可以走
    }
}

// 八方向(含對角線):多四組

應用二:洪水填充(flood fill)——數島嶼

經典題:給一個 0/1 網格,上下左右相連的 1 算同一座島,問有幾座島? 做法和「連通元件」一模一樣,只是搬到網格上:每碰到一個還是 1 的格子,答案 +1, 再用 DFS 把整座島「沉掉」(全部改成 0)以免重複數。把走過的格子改成 0, 等於免費省下一個 visited[][] 陣列——這是網格題的招牌技巧。 

#include <bits/stdc++.h>
using namespace std;

int n, m;
int grid[1005][1005];
int dx[4] = {1, -1, 0, 0}, dy[4] = {0, 0, 1, -1};

void flood(int x, int y) {
    grid[x][y] = 0;                       // 「沉島」:把這格洗成海,當作已訪
    for (int k = 0; k < 4; k++) {
        int nx = x + dx[k], ny = y + dy[k];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m && grid[nx][ny] == 1)
            flood(nx, ny);
    }
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    cin >> n >> m;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            cin >> grid[i][j];

    int islands = 0;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == 1) {        // 發現一座沒沉過的島
                islands++;
                flood(i, j);              // 整片沉掉
            }
    cout << islands << '\n';
    return 0;
}

應用三:網格最短步數(BFS)

把「數島」的 DFS 換成 BFS,就能求網格上的最少步數(每步代價都是 1,所以 BFS 第一次到達即最短)。 座標用 pair<int,int> 放進佇列。一個漂亮的省記憶體技巧:用 dist 陣列身兼「已訪標記」與「步數紀錄」 ——初始化成 $-1$ 代表未訪,被賦值就代表訪過。 

int bfs_grid(int sx, int sy, int tx, int ty) {
    vector<vector<int>> dist(n, vector<int>(m, -1));  // -1 = 未訪
    queue<pair<int,int>> q;
    q.push({sx, sy});
    dist[sx][sy] = 0;
    while (!q.empty()) {
        auto [x, y] = q.front(); q.pop();
        if (x == tx && y == ty) return dist[x][y];   // 抵達終點
        for (int k = 0; k < 4; k++) {
            int nx = x + dx[k], ny = y + dy[k];
            if (nx >= 0 && nx < n && ny >= 0 && ny < m
                && grid[nx][ny] == 0 && dist[nx][ny] == -1) {
                dist[nx][ny] = dist[x][y] + 1;        // 比上一格多走一步
                q.push({nx, ny});
            }
        }
    }
    return -1;   // 走不到
}

網格題的兩個常見陷阱

第一,邊界檢查要寫全:四個條件 nx>=0 && nx<n && ny>=0 && ny<m 缺一不可, 否則陣列越界,RE(Runtime Error)或更糟——讀到垃圾值給出 WA(Wrong Answer)。 第二,起點本身可能就是終點(或起點就是答案格),要先特判,別讓 BFS 從第一步才開始算。 AP325 的「WAKUWAKU 尋找興奮源」就是栽在「起點剛好是興奮源」這個邊界。

應用四:二分圖判定(染色 BFS)

二分圖(bipartite graph):能不能把所有點塗成兩色,讓每條邊的兩端顏色不同? 它等價於「圖中沒有奇數長度的環」,在配對問題(學生選社團、工作分派)裡是核心結構。 判定法:BFS 時把鄰居塗成相反色,一旦發現「相鄰卻同色」就不是二分圖。 

int color[N];   // -1 = 未染色,0 / 1 = 兩種顏色

bool is_bipartite(int n) {
    for (int i = 1; i <= n; i++) color[i] = -1;
    for (int s = 1; s <= n; s++) {       // 圖可能不連通,每塊都要查
        if (color[s] != -1) continue;
        queue<int> q;
        q.push(s); color[s] = 0;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : g[u]) {
                if (color[v] == -1) {
                    color[v] = color[u] ^ 1;   // 塗相反色(0↔1)
                    q.push(v);
                } else if (color[v] == color[u]) {
                    return false;              // 相鄰同色 → 不是二分圖
                }
            }
        }
    }
    return true;
}

注意 color[u] ^ 1 這個位元小技巧:0 ^ 1 = 1、1 ^ 1 = 0,一行就翻色。 AP325 的「小寶的著色問題」正是這題,並特別提醒:圖可能不連通,每個未染色的點都要當起點跑一次。

拓樸排序與 DAG 上 DP

當圖是有向無環圖(DAG, Directed Acyclic Graph)時,我們可以把所有點排成一條線, 讓每條有向邊都「從前指向後」——這就是拓樸排序(topological sort)。它是處理「相依關係」的萬用工具: 課程先修、建置系統、邏輯電路的訊號傳遞。

Kahn 演算法:看「入度」(C++ 版)

Kahn 的直覺:入度(in-degree,指向我的箭頭數)為 0 的點可以馬上處理。處理完它, 把它指向的點入度各減一;誰減到 0,就輪到誰。用佇列驅動,本質是一個 BFS。 

vector<int> g[N];
int indeg[N];

vector<int> topo_sort(int n) {
    queue<int> q;
    for (int i = 1; i <= n; i++)
        if (indeg[i] == 0) q.push(i);   // 所有入度 0 的點先入列
    vector<int> order;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : g[u])
            if (--indeg[v] == 0)         // 拔掉 u 的出邊,v 入度若歸零就入列
                q.push(v);
    }
    return order;   // 若 order.size() < n,代表圖中有環(無法拓樸排序)
}

最後那句註解很重要:Kahn 順手就能偵測有向環——環上的點入度永遠減不到 0,進不了佇列, 所以 order 會少幾個。若 order.size() == n 才是合法 DAG。

DAG 上 DP:最長路徑

DAG「無環」的性質,讓動態規劃(DP)在它身上特別好用——因為沿著拓樸序處理,每個點被算到時, 它的所有前驅都已經算完了,不會循環依賴。最經典的就是DAG 最長路徑 (專案管理裡叫「關鍵路徑」,決定整個工程最短要多久)。

設 dp[v] = 以 v 結尾的最長路徑長度。沿拓樸序處理每個 u,用它去更新所有後繼 v: 

// g[u] 存 {鄰居 v, 邊權 w};order 是 topo_sort 算出的拓樸序
vector<pair<int,int>> g[N];
long long dp[N];   // dp[v] = 到 v 的最長路徑

long long longest_path(vector<int>& order, int n) {
    for (int u : order)
        for (auto [v, w] : g[u])
            dp[v] = max(dp[v], dp[u] + w);   // 鬆弛:從 u 接一條邊到 v
    long long best = 0;
    for (int i = 1; i <= n; i++) best = max(best, dp[i]);
    return best;
}

AP325 的「DAG 的最長與最短路徑」正是這題;把 max 換成 min(並小心初始化)就是最短路徑版。 這種「在 DAG 上沿拓樸序做 DP」的模式,是第 62 章競賽 DP 的重要橋樑——很多看似複雜的 DP, 本質都是在一張隱式 DAG(「狀態」當點、「轉移」當邊)上跑最長/最短路。

🔑
拓樸序 = DP 的合法計算順序

DP 的本質是「大問題拆成小問題,小問題先算」。當這些子問題的依賴關係構成一張 DAG, 拓樸排序就是那個「先算誰、後算誰」的合法順序。所以「DAG 上 DP」不是一個特殊技巧, 而是 DP 與圖論交會的根本真相。第 30 章偷瞄到的 Floyd-Warshall、本章的最長路徑,都是這個思想的化身。

最短路徑:競賽三巨頭(C++)

BFS 解的是「無權」最短路。一旦邊有了權重,就要請出三巨頭。第 30 章講了它們的直覺與 Python 版, 這裡給出競賽 C++ 寫法,並把「何時用哪個」釘死成一張表。三者的共同核心都是 鬆弛(relaxation):if (dist[u] + w < dist[v]) dist[v] = dist[u] + w;

Dijkstra:priority_queue 版

Dijkstra 貪心地「每次取出目前距離最小的未定點」。這個「取最小」正是優先佇列(priority_queue) ——也就是第 29 章的最小堆積——的拿手好戲。C++ 的 priority_queue 預設是最大堆, 要用 greater 改成最小堆。要求:邊權非負。 

typedef pair<long long,int> pli;   // {距離, 頂點}
vector<pair<int,int>> g[N];         // {鄰居, 邊權}
long long dist[N];

void dijkstra(int s, int n) {
    for (int i = 1; i <= n; i++) dist[i] = LLONG_MAX;
    priority_queue<pli, vector<pli>, greater<pli>> pq;  // 最小堆
    dist[s] = 0;
    pq.push({0, s});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;          // 過期紀錄(同一點被推過多次),跳過
        for (auto [v, w] : g[u]) {
            if (dist[u] + w < dist[v]) {     // 鬆弛成功 → 更新並重新入堆
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}

複雜度 $O((V+E)\log V)$。注意那行 if (d > dist[u]) continue;——這是 priority_queue 版 Dijkstra 的靈魂:我們不刪除堆裡的舊紀錄(STL 堆不支援高效刪除),而是允許同一個點被推進去多次, 靠這行「發現拿到的是過期距離就略過」來保證正確。這個「過期跳過」技巧在 第 61 章 講競賽資料結構時會再深入。

⚠️
Dijkstra 不能有負權邊

Dijkstra 的貪心建立在「一旦取出某點就定案、不會再變短」之上,這在邊權非負時成立。 負邊會打破它(第 30 章圖 30-9 那個例子)。看到負權,改用 Bellman-Ford。 另外,距離用 long long:$10^5$ 條邊、每條權重 $10^9$,加起來會爆 int。

Bellman-Ford 與 SPFA:慢,但吃負權

Bellman-Ford 不貪心,改走暴力:把每一條邊都鬆弛,重複 $V-1$ 輪。它能處理負權, 還能偵測負環(跑完 $V-1$ 輪後再跑一輪,若還能鬆弛就有負環)。複雜度 $O(VE)$。 

struct Edge { int u, v, w; };
vector<Edge> edges;
long long dist[N];

bool bellman_ford(int s, int n) {
    for (int i = 1; i <= n; i++) dist[i] = LLONG_MAX;
    dist[s] = 0;
    for (int i = 1; i < n; i++)                   // 鬆弛 V-1 輪
        for (auto& e : edges)
            if (dist[e.u] != LLONG_MAX && dist[e.u] + e.w < dist[e.v])
                dist[e.v] = dist[e.u] + e.w;
    for (auto& e : edges)                          // 第 V 輪:還能鬆弛 → 有負環
        if (dist[e.u] != LLONG_MAX && dist[e.u] + e.w < dist[e.v])
            return false;
    return true;                                   // 無負環
}

SPFA(Shortest Path Faster Algorithm)是 Bellman-Ford 的佇列優化版:只把「距離剛被更新」的點 重新入列,而不是每輪盲目掃所有邊。在隨機圖上快很多,但最壞情況仍是 $O(VE)$,且容易被出題者「卡」(hack) ——所以非負權時請優先用 Dijkstra,SPFA 留給「有負權又想偷懶」的場合。 

// SPFA:Bellman-Ford 的佇列優化
vector<pair<int,int>> g[N];
long long dist[N];
bool inq[N];

void spfa(int s, int n) {
    for (int i = 1; i <= n; i++) dist[i] = LLONG_MAX;
    queue<int> q;
    dist[s] = 0; q.push(s); inq[s] = true;
    while (!q.empty()) {
        int u = q.front(); q.pop(); inq[u] = false;
        for (auto [v, w] : g[u])
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                if (!inq[v]) { q.push(v); inq[v] = true; }  // 不重複入列
            }
    }
}

Floyd-Warshall:三層迴圈,全點對

想一次算出每一對頂點之間的最短距離?用 Floyd-Warshall。它是個漂亮的 DP:逐一允許「經過頂點 $k$ 當中繼」。 程式只有三層迴圈,複雜度 $O(V^3)$,只適合 $V$ 小(約 $\le 500$)的情形。 

const long long INF = 1e18;
long long d[505][505];   // d[i][j]:初始為直達邊權,自己到自己為 0,無邊為 INF

void floyd(int n) {
    for (int k = 1; k <= n; k++)              // 中繼站 k(必須在最外層!)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                if (d[i][k] < INF && d[k][j] < INF
                    && d[i][k] + d[k][j] < d[i][j])
                    d[i][j] = d[i][k] + d[k][j];
}
⚠️
Floyd-Warshall 的 k 一定要在最外層

三層迴圈的順序不能調換:中繼站 k 必須是最外層的迴圈。 原因是 DP 的定義是「只允許用前 $k$ 個點當中繼時的最短路」,必須一層一層「解鎖」中繼站。 把 k 寫進內層是初學者最致命、又最難看出的 bug——程式照跑、答案卻錯。

最小生成樹(MST)

最小生成樹(Minimum Spanning Tree, MST):用「最便宜的一組邊」把所有頂點連起來、不留環。 想像用最少的纜線總長串連所有城市,或最低成本鋪設電網。一棵生成樹恰好有 $n-1$ 條邊。 這是第 31 章貪心演算法的招牌應用,有兩個經典解法。

並查集複習(Kruskal 的引擎)

Kruskal 需要一個能快速回答「這兩點已經連通了嗎?」的工具,那就是第 29 章的 並查集(Union-Find / Disjoint Set Union)。配上「路徑壓縮」與「按大小合併」兩個優化, 每次操作近乎 $O(1)$。這個結構在 第 61 章 還會被反覆使用。 

int par[N], sz[N];
void dsu_init(int n) {
    for (int i = 1; i <= n; i++) { par[i] = i; sz[i] = 1; }
}
int find(int x) {                       // 路徑壓縮
    return par[x] == x ? x : par[x] = find(par[x]);
}
bool unite(int a, int b) {              // 合併;回傳 false 表示原本就同一集合
    a = find(a); b = find(b);
    if (a == b) return false;
    if (sz[a] < sz[b]) swap(a, b);      // 按大小合併:小的併到大的
    par[b] = a; sz[a] += sz[b];
    return true;
}

Kruskal:由小到大加邊

Kruskal 的貪心極其直白:把所有邊按權重由小到大排序,依序嘗試加入;若這條邊的兩端已經連通(會成環)就跳過。 加滿 $n-1$ 條邊就完成。它用的是「邊列表」表示法。 

struct Edge { int u, v, w; };

long long kruskal(vector<Edge>& edges, int n) {
    sort(edges.begin(), edges.end(),
         [](const Edge& a, const Edge& b){ return a.w < b.w; });  // 邊權由小到大
    dsu_init(n);
    long long total = 0;
    int used = 0;
    for (auto& e : edges) {
        if (unite(e.u, e.v)) {          // 兩端原本不連通 → 加這條邊不成環
            total += e.w;
            if (++used == n - 1) break; // 湊滿 n-1 條,完成
        }
    }
    return total;   // 若 used < n-1,代表圖不連通,沒有生成樹
}

複雜度 $O(E \log E)$,瓶頸在排序。

Prim:從一點長出

Prim 的策略像 Dijkstra:從一個點出發,維護一群「已在樹裡」的點,每次用優先佇列挑出「連接樹與樹外、最便宜的那條邊」, 把對應的新點納入。適合稠密圖(用鄰接串列 + 堆積時複雜度 $O(E \log V)$)。 

typedef pair<int,int> pii;          // {邊權, 頂點}
vector<pair<int,int>> g[N];          // {鄰居, 邊權}

long long prim(int s, int n) {
    vector<bool> inMST(n + 1, false);
    priority_queue<pii, vector<pii>, greater<pii>> pq;
    pq.push({0, s});
    long long total = 0;
    int cnt = 0;
    while (!pq.empty() && cnt < n) {
        auto [w, u] = pq.top(); pq.pop();
        if (inMST[u]) continue;         // 已在樹裡,跳過(過期紀錄)
        inMST[u] = true;
        total += w;
        cnt++;
        for (auto [v, wt] : g[u])
            if (!inMST[v]) pq.push({wt, v});
    }
    return total;
}
💡
Kruskal vs Prim:一句話選

競賽裡 Kruskal 用得更多:它寫起來短(排序 + 並查集),又天然適合「邊列表」輸入,稀疏圖尤佳。 當圖很稠密(邊接近 $V^2$)或題目本來就給鄰接串列時,Prim 可能更順手。 兩者答案相同(MST 唯一性在邊權互異時成立)。注意它們和 Dijkstra 長得很像—— 差別只在 Prim 的 key 是「到樹的邊權」,Dijkstra 是「到起點的總距離」。

樹上演算法:本章的重頭戲

終於來到壓軸。樹(tree)是一種特別的圖:$n$ 個點、恰 $n-1$ 條邊、連通、無環。 正因為這些限制,樹擁有一票普通圖沒有的漂亮性質,讓許多在一般圖上很難的問題,在樹上有又快又優雅的解。 這一節整合 AP325 第 8 章,是 APCS 進階與競賽的高頻考點。

樹的性質:為什麼樹這麼好用

  • $n$ 點、$n-1$ 邊、連通、無環——這四個條件任三個都能推出第四個。
  • 任兩點之間恰有唯一一條路徑。這是樹最珍貴的性質:不必擔心「繞路」,路徑是唯一的。
  • 任意加一條邊就產生環;任意刪一條邊就斷成兩棵樹。
  • 除了根之外,每個點恰有一個父節點;沒有父節點的就是根(root),沒有子節點的就是葉(leaf)。

這帶來一個解題上的大紅利:在樹上 DFS,常常不需要 visited 陣列——只要記住「父節點是誰」, 不要走回頭路就行。這是樹 DFS 的招牌寫法: 

vector<int> g[N];   // 無根樹用無向鄰接串列
int sz[N];          // 子樹大小

// 傳入 parent,用「不等於父節點」取代 visited 陣列
void dfs(int u, int parent) {
    sz[u] = 1;                          // 自己算一個
    for (int v : g[u]) {
        if (v == parent) continue;      // 不走回父節點 → 不會繞回去
        dfs(v, u);
        sz[u] += sz[v];                 // 把子樹大小累加上來
    }
}
// 指定 1 為根:dfs(1, 0);  之後 sz[u] 就是「以 u 為根的子樹」有幾個點

上面這個 sz[](子樹大小)是「子樹資訊由下往上合併」最樸素的例子。把它換成別的資訊 (子樹高度、子樹點權和……),骨架完全一樣。這個「DFS 回傳子樹資訊、父節點合併」的模式, 就是樹形 DP 的心臟。

ℹ️
深度用 BFS,高度用 DFS

一個好記的對比:深度(depth)(根到某點的距離)是「由上往下」的資訊, BFS 一層一層走最自然;高度(height)(某點到最遠子孫的距離)是「由下往上」的資訊, 需要子節點先回傳數值給父節點,所以要靠 DFS 遞迴的「回傳」特性。記住:往下的量用 BFS,往上的量用 DFS。

樹形 DP:子樹資訊合併(最大獨立集)

樹形 DP(tree DP)的精髓:每個點的答案,由它各個子樹的答案合併而來。 我們用一個經典題把它講透——樹的最大權重獨立集(maximum weight independent set)。

題目:每個點 $u$ 有一個權重 $w_u$。請選出一組點,使得沒有任何兩個被選的點相鄰(這叫「獨立集」), 並讓被選點的權重總和最大。經典背景故事是「公司派對」:每個員工有歡樂值,但員工和他的直屬主管不能同時出席 (太尷尬),問派對最大歡樂值。

關鍵觀察:對每個點 $u$,只有兩種狀態——選 $u$ 或 不選 $u$。我們對每個點記兩個值:

  • dp[u][0] = 在 u 的子樹裡,不選 u 時的最大權重和。此時每個子節點 v 可選可不選,取較大者。
  • dp[u][1] = 在 u 的子樹裡,選 u 時的最大權重和。此時每個子節點 v 都不能選。

於是合併公式($v$ 跑遍 $u$ 的所有子節點):

$$dp[u][0] = \sum_{v} \max\big(dp[v][0],\, dp[v][1]\big)$$

$$dp[u][1] = w_u + \sum_{v} dp[v][0]$$

最後答案是 $\max\big(dp[\text{root}][0],\, dp[\text{root}][1]\big)$。

vector<int> g[N];
long long dp[N][2];
int w[N];               // 點權

void dfs(int u, int parent) {
    dp[u][0] = 0;        // 不選 u 的基底
    dp[u][1] = w[u];     // 選 u 的基底:先把自己的權重算進來
    for (int v : g[u]) {
        if (v == parent) continue;
        dfs(v, u);                                   // 先算完子樹
        dp[u][0] += max(dp[v][0], dp[v][1]);         // 不選 u:子節點隨意
        dp[u][1] += dp[v][0];                        // 選 u:子節點必不選
    }
}

// 主程式:dfs(1, 0); 答案 = max(dp[1][0], dp[1][1]);
🔑
樹形 DP 的通用配方

幾乎所有樹形 DP 都長這樣:(1) 為每個點定義「以它為根的子樹」上、若干種狀態的最佳值 (例如「選/不選這個點」);(2) 用一次 DFS,在「子節點全算完之後」把它們的狀態合併成父節點的狀態; (3) 根節點的狀態取最佳就是答案。同一個配方能解最大獨立集、最小頂點覆蓋、最小管轄集、 樹上選點/染色……換的只是「狀態定義」與「合併公式」。AP325 第 8 章一整章都在玩這個配方。

樹的直徑:兩次 BFS 的魔法

樹的直徑(diameter):樹上距離最遠的兩個點之間的距離。 它有一個美得不像話的解法——只要兩次 BFS:

  1. 從任意一點(例如 1)BFS,找到離它最遠的點 $a$。
  2. 再從 $a$ BFS,找到離 $a$ 最遠的點 $b$。則 $a$ 到 $b$ 的距離就是直徑。

為什麼第一步隨便選一點,找到的最遠點 $a$ 就一定是某條直徑的端點? 直覺:樹上「最遠」具有傳遞性——從任何點出發走到最遠處,必然停在直徑的一端。 (嚴格證明用反證法,可參考延伸資源。)這招在 AP325 叫「farthest of farthest」。 

vector<int> g[N];

// 從 s 做 BFS,回傳 {最遠的點, 該距離}
pair<int,int> bfs_far(int s, int n) {
    vector<int> dist(n + 1, -1);
    queue<int> q;
    q.push(s); dist[s] = 0;
    int far = s;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        if (dist[u] > dist[far]) far = u;
        for (int v : g[u]) if (dist[v] == -1) {
            dist[v] = dist[u] + 1;
            q.push(v);
        }
    }
    return {far, dist[far]};
}

int diameter(int n) {
    int a = bfs_far(1, n).first;        // 第一次:任取一點找最遠 a
    auto [b, d] = bfs_far(a, n);        // 第二次:從 a 找最遠 b,距離 d 即直徑
    return d;
}

另一種解法是一次 DFS:對每個點求「往下兩條最長路徑」相加(它們會經過該點,構成一條候選直徑), 取所有點的最大值。這個寫法在帶權樹上一樣適用,而且能順便求出更多資訊: 

int ans = 0;
// 回傳「u 往下能延伸的最長路徑(邊數)」,同時更新全域直徑 ans
int dfs_diam(int u, int parent) {
    int max1 = 0, max2 = 0;             // 最長、次長的「往下深度」
    for (int v : g[u]) {
        if (v == parent) continue;
        int h = dfs_diam(v, u) + 1;
        if (h > max1) { max2 = max1; max1 = h; }
        else if (h > max2) max2 = h;
    }
    ans = max(ans, max1 + max2);        // 經過 u 的最長路徑 = 兩條最長下行路相加
    return max1;
}

最近共同祖先 LCA:倍增法

最後一個重頭戲:最近共同祖先(Lowest Common Ancestor, LCA)。 在一棵有根樹裡,兩點 $u$、$v$ 的 LCA 是「同時是兩者祖先、且離它們最近」的那個點。 它能解一票問題:樹上兩點距離 = $depth[u] + depth[v] - 2 \times depth[\text{LCA}(u,v)]$; 判斷祖孫關係;樹上路徑查詢……

最暴力的做法是「讓較深的點先往上爬到同深度,再兩點一起往上爬,直到相遇」,單次查詢 $O(h)$(樹高)。 鏈狀樹會退化成 $O(n)$,大量查詢就爆了。競賽標準解是倍增法(binary lifting): 預處理每個點「往上跳 $2^k$ 步」的祖先,之後每次查詢只要 $O(\log n)$。

核心是一張表 up[v][k] = 「從 v 往上跳 $2^k$ 步」到達的祖先。它有個漂亮的遞推: 往上跳 $2^k$ 步 = 先跳 $2^{k-1}$ 步,再從那裡跳 $2^{k-1}$ 步:

$$up[v][k] = up[\,up[v][k-1]\,][k-1]$$

const int LOG = 17;        // 2^17 = 131072 > 1e5,夠用
vector<int> g[N];
int up[N][LOG];            // up[v][k] = v 往上跳 2^k 步的祖先
int depth[N];

// 預處理:DFS 算 depth 與 up 表。根 r 呼叫 dfs(r, r)
void dfs(int u, int parent) {
    up[u][0] = parent;                            // 往上 1 步 = 父節點
    for (int k = 1; k < LOG; k++)
        up[u][k] = up[ up[u][k-1] ][k-1];         // 倍增遞推
    for (int v : g[u]) if (v != parent) {
        depth[v] = depth[u] + 1;
        dfs(v, u);
    }
}

int lca(int a, int b) {
    if (depth[a] < depth[b]) swap(a, b);          // 保證 a 較深
    int diff = depth[a] - depth[b];
    for (int k = 0; k < LOG; k++)                  // 把 a 提到和 b 同深度
        if (diff & (1 << k)) a = up[a][k];
    if (a == b) return a;                          // b 本來就是 a 的祖先
    for (int k = LOG - 1; k >= 0; k--)             // 兩點一起往上,跳到 LCA 正下方
        if (up[a][k] != up[b][k]) {
            a = up[a][k];
            b = up[b][k];
        }
    return up[a][0];                               // 再跳一步就是 LCA
}

// 樹上兩點距離 = depth[a] + depth[b] - 2 * depth[lca(a,b)]

預處理 $O(n \log n)$,每次查詢 $O(\log n)$。理解兩個關鍵步驟:第一個迴圈用 diff 的二進位位元, 把較深的點「精準地」提升到與另一點同深度;第二個迴圈是從大步到小步嘗試一起跳, 「能跳就跳,但永遠不跳到共同祖先之上」,最後兩點會停在 LCA 的正下方,再跳一步(up[a][0])就是答案。 這個「從高位往低位試跳」的逆序迴圈是倍增法的精髓,務必想清楚。

ℹ️
倍增法不只用在 LCA

「預存 $2^k$ 步的結果,查詢時用二進位拼湊」這個倍增(binary lifting)思想威力極廣: 求樹上「往上第 $k$ 個祖先」、稀疏表(sparse table)做區間最值查詢、快速冪($a^n$ 用 $O(\log n)$ 次乘法) ——都是同一招的變奏。看到「重複很多次的同質操作 + 大量查詢」,就該想到倍增。

✏️ 練習 63.1 實作

洪水填充:數島嶼。給一個 $n \times m$ 的 0/1 網格,上下左右相連的 1 算同一座島。輸出島嶼數量。

看解答 
#include <bits/stdc++.h>
using namespace std;

int n, m, grid[1005][1005];
int dx[4] = {1, -1, 0, 0}, dy[4] = {0, 0, 1, -1};

void flood(int x, int y) {
    grid[x][y] = 0;                       // 沉島當作已訪
    for (int k = 0; k < 4; k++) {
        int nx = x + dx[k], ny = y + dy[k];
        if (nx >= 0 && nx < n && ny >= 0 && ny < m && grid[nx][ny] == 1)
            flood(nx, ny);
    }
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    cin >> n >> m;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++) cin >> grid[i][j];
    int islands = 0;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            if (grid[i][j] == 1) { islands++; flood(i, j); }
    cout << islands << '\n';
    return 0;
}

重點:把走過的格子改成 0,等於免費省掉 visited[][] 陣列。若網格很大(深鏈), 遞迴 DFS 可能爆 stack,可改成用 queue<pair<int,int>> 的 BFS 版,邏輯完全一樣。

✏️ 練習 63.2 實作

二分圖判定。給一張無向圖(可能不連通),判斷它是否為二分圖。輸出 YES / NO。

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
vector<int> g[N];
int color[N];

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v; cin >> u >> v;
        g[u].push_back(v); g[v].push_back(u);
    }
    for (int i = 1; i <= n; i++) color[i] = -1;
    bool ok = true;
    for (int s = 1; s <= n && ok; s++) {
        if (color[s] != -1) continue;
        queue<int> q; q.push(s); color[s] = 0;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (int v : g[u]) {
                if (color[v] == -1) { color[v] = color[u] ^ 1; q.push(v); }
                else if (color[v] == color[u]) { ok = false; break; }
            }
        }
    }
    cout << (ok ? "YES" : "NO") << '\n';
    return 0;
}

核心是 color[u] ^ 1 一行翻色。別忘了「圖可能不連通」,每個未染色點都要當起點。 二分圖 ⇔ 沒有奇數長度的環。

✏️ 練習 63.3 實作

拓樸排序。給一張有 $n$ 點 $m$ 邊的有向圖,輸出一個合法的拓樸序;若有環則輸出 IMPOSSIBLE。 (例:課程先修排序。)

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
vector<int> g[N];
int indeg[N];

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v; cin >> u >> v;   // u 必須排在 v 前面
        g[u].push_back(v);
        indeg[v]++;
    }
    queue<int> q;
    for (int i = 1; i <= n; i++) if (indeg[i] == 0) q.push(i);
    vector<int> order;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        order.push_back(u);
        for (int v : g[u]) if (--indeg[v] == 0) q.push(v);
    }
    if ((int)order.size() != n) { cout << "IMPOSSIBLE\n"; return 0; }
    for (int x : order) cout << x << ' ';
    cout << '\n';
    return 0;
}

Kahn 順手偵測有向環:若有環,環上的點入度永遠減不到 0,order 會少幾個, 所以 order.size() != n 就代表無解。

✏️ 練習 63.4 實作

Dijkstra 最短路。給帶權有向圖($1 \le w \le 10^9$,非負),求從點 1 到所有點的最短距離。 走不到輸出 -1。

看解答 
#include <bits/stdc++.h>
using namespace std;
typedef pair<long long,int> pli;

const int N = 1e5 + 5;
vector<pair<int,int>> g[N];
long long dist[N];

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    for (int i = 0; i < m; i++) {
        int u, v, w; cin >> u >> v >> w;
        g[u].push_back({v, w});
    }
    for (int i = 1; i <= n; i++) dist[i] = LLONG_MAX;
    priority_queue<pli, vector<pli>, greater<pli>> pq;
    dist[1] = 0; pq.push({0, 1});
    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (d > dist[u]) continue;            // 過期紀錄
        for (auto [v, w] : g[u])
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
    }
    for (int i = 1; i <= n; i++)
        cout << (dist[i] == LLONG_MAX ? -1 : dist[i]) << " \n"[i == n];
    return 0;
}

用 long long 防溢位;if (d > dist[u]) continue; 是 STL 堆 Dijkstra 的靈魂 (不刪舊紀錄,出堆時跳過過期者)。想輸出實際路徑就再記一個 parent[],鬆弛成功時更新。

✏️ 練習 63.5 實作

Kruskal 最小生成樹。給 $n$ 點 $m$ 條帶權無向邊,求 MST 的總權重;若圖不連通輸出 IMPOSSIBLE。

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
int par[N], sz[N];
int find(int x) { return par[x] == x ? x : par[x] = find(par[x]); }
bool unite(int a, int b) {
    a = find(a); b = find(b);
    if (a == b) return false;
    if (sz[a] < sz[b]) swap(a, b);
    par[b] = a; sz[a] += sz[b];
    return true;
}

struct Edge { int u, v, w; };

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m;
    cin >> n >> m;
    vector<Edge> edges(m);
    for (auto& e : edges) cin >> e.u >> e.v >> e.w;
    sort(edges.begin(), edges.end(),
         [](const Edge& a, const Edge& b){ return a.w < b.w; });
    for (int i = 1; i <= n; i++) { par[i] = i; sz[i] = 1; }
    long long total = 0; int used = 0;
    for (auto& e : edges)
        if (unite(e.u, e.v)) { total += e.w; used++; }
    if (used != n - 1) cout << "IMPOSSIBLE\n";
    else cout << total << '\n';
    return 0;
}

Kruskal = 排序 + 並查集。unite 回傳 false 代表兩端已連通(加了會成環),自動跳過。 最後若選到的邊不足 $n-1$ 條,代表圖不連通、沒有生成樹。

✏️ 練習 63.6 挑戰

樹的直徑。給一棵 $n$ 點的樹($n-1$ 條無向邊),求它的直徑(最遠兩點的邊數距離)。 用「兩次 BFS」實作。(背景:AP325「血緣關係」。)

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
vector<int> g[N];

pair<int,int> bfs_far(int s, int n) {   // 回傳 {最遠點, 距離}
    vector<int> dist(n + 1, -1);
    queue<int> q; q.push(s); dist[s] = 0;
    int far = s;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        if (dist[u] > dist[far]) far = u;
        for (int v : g[u]) if (dist[v] == -1) {
            dist[v] = dist[u] + 1; q.push(v);
        }
    }
    return {far, dist[far]};
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n; cin >> n;
    for (int i = 0; i < n - 1; i++) {
        int u, v; cin >> u >> v;
        g[u].push_back(v); g[v].push_back(u);
    }
    int a = bfs_far(1, n).first;     // 任取一點找最遠 a
    int d = bfs_far(a, n).second;    // 從 a 找最遠,距離即直徑
    cout << d << '\n';
    return 0;
}

為什麼第一次「隨便選一點」找到的 $a$ 一定是直徑端點?因為樹上「最遠」有傳遞性: 從任何點走到最遠處,必落在某條直徑的一端。第二次從 $a$ 出發,自然量到整條直徑。

✏️ 練習 63.7 挑戰

樹形 DP:最大權重獨立集。給一棵 $n$ 點的樹,每點有權重 $w_i$。選一組互不相鄰的點,使權重和最大。 (背景:AP325「公司派對」。)

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5;
vector<int> g[N];
long long dp[N][2];
int w[N];

void dfs(int u, int parent) {
    dp[u][0] = 0;          // 不選 u
    dp[u][1] = w[u];       // 選 u
    for (int v : g[u]) {
        if (v == parent) continue;
        dfs(v, u);
        dp[u][0] += max(dp[v][0], dp[v][1]);   // 不選 u → 子節點隨意
        dp[u][1] += dp[v][0];                   // 選 u → 子節點不能選
    }
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n; cin >> n;
    for (int i = 1; i <= n; i++) cin >> w[i];
    for (int i = 0; i < n - 1; i++) {
        int u, v; cin >> u >> v;
        g[u].push_back(v); g[v].push_back(u);
    }
    dfs(1, 0);
    cout << max(dp[1][0], dp[1][1]) << '\n';
    return 0;
}

樹形 DP 的通用配方:每點兩個狀態(選/不選),DFS 在「子節點全算完」後合併。 想練變形?把規則改成「最小頂點覆蓋」(每條邊至少一端被選)或「最小管轄集」,狀態與公式換掉、骨架不變。 注意若樹是一條深鏈,遞迴可能爆 stack——競賽中可手動加大堆疊或改成迭代式 DFS。

✏️ 練習 63.8 挑戰

進階:LCA 與樹上距離查詢。給一棵 $n$ 點有根樹(根為 1)與 $q$ 筆查詢,每筆問兩點 $u$、$v$ 之間的距離。 用倍增法 LCA,讓每筆查詢 $O(\log n)$。

看解答 
#include <bits/stdc++.h>
using namespace std;

const int N = 1e5 + 5, LOG = 17;
vector<int> g[N];
int up[N][LOG], dep[N];

void dfs(int u, int parent) {
    up[u][0] = parent;
    for (int k = 1; k < LOG; k++)
        up[u][k] = up[ up[u][k-1] ][k-1];
    for (int v : g[u]) if (v != parent) {
        dep[v] = dep[u] + 1;
        dfs(v, u);
    }
}

int lca(int a, int b) {
    if (dep[a] < dep[b]) swap(a, b);
    int diff = dep[a] - dep[b];
    for (int k = 0; k < LOG; k++)
        if (diff & (1 << k)) a = up[a][k];
    if (a == b) return a;
    for (int k = LOG - 1; k >= 0; k--)
        if (up[a][k] != up[b][k]) { a = up[a][k]; b = up[b][k]; }
    return up[a][0];
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, q;
    cin >> n >> q;
    for (int i = 0; i < n - 1; i++) {
        int u, v; cin >> u >> v;
        g[u].push_back(v); g[v].push_back(u);
    }
    dep[1] = 0;
    dfs(1, 1);                       // 根的父節點設為自己,跳出界仍停在根
    while (q--) {
        int u, v; cin >> u >> v;
        int w = lca(u, v);
        cout << dep[u] + dep[v] - 2 * dep[w] << '\n';   // 樹上距離公式
        // 樹上距離 = dep[u] + dep[v] - 2*dep[LCA(u,v)]
    }
    return 0;
}

把根的父節點設成自己(dfs(1, 1)),這樣 up[1][k] 永遠是 1,往上跳超過根也安全停在根。 距離公式的直覺:$u$ 到 LCA、$v$ 到 LCA 兩段路相加,而它們共用的「深度」被算了兩次,所以減 $2 \times dep[\text{LCA}]$。

把這一章串起來

我們把第 30 章的圖演算法,從 Python 直覺一路鍛造成競賽級的 C++ 兵器庫:

  • 建模與表示:鄰接串列(vector)是稀疏圖的預設;真正的功夫在隱式圖——把文字題翻譯成「點與邊」。
  • BFS/DFS 四大應用:連通元件、洪水填充數島、網格最短步數、二分圖判定。關鍵分水嶺是 「圖上(查 g[])vs 圖外(方向陣列即時算鄰居)」。
  • 拓樸排序與 DAG 上 DP:Kahn(入度 + 佇列)排出相依順序、順手偵測有向環;沿拓樸序做 DP 解最長路徑——這是通往第 62 章競賽 DP 的橋。
  • 最短路三巨頭:Dijkstra(priority_queue、非負權、最常用)、Bellman-Ford / SPFA(可負權、偵測負環)、Floyd-Warshall(全點對、$k$ 在最外層)。
  • 最小生成樹:Kruskal(排序 + 並查集)與 Prim(優先佇列),呼應第 31 章貪心。
  • 樹上演算法(重頭戲):樹的性質讓 DFS 可省 visited;樹形 DP(子樹資訊合併,最大獨立集); 樹直徑(兩次 BFS 的魔法);LCA 倍增法($O(\log n)$ 查祖先與距離)。

你會發現,這一章幾乎沒有發明全新的「資料結構」——佇列、堆積、並查集都是第 29 章的老朋友 (它們在第 61 章競賽資料結構裡會再被深挖)。競賽的真正功夫,在於把這些零件, 對著一個被你成功「翻譯成圖」的問題,組裝成解法。下一步,你該做的就是打開評測系統, 把本章的八題模板親手敲過、送出、被 AC 一次——演算法是手指的肌肉記憶,讀懂永遠不等於寫得出來。
