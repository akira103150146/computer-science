一個垃圾郵件過濾器的故事

假設老闆要你寫一個垃圾郵件過濾器。第一天,你信心滿滿地用傳統方法:寫規則。 「信件裡有『中獎』就標成垃圾。」上線。隔天垃圾信改寫成「中.獎」,規則破功。 你再加一條:「有『免費』也算垃圾。」結果把同事發的「免費咖啡券領取通知」也誤殺了。 你加規則、垃圾信變招、你再加規則……你發現自己掉進一個永遠追不完的軍備競賽。

問題出在哪?垃圾郵件的「規則」太複雜、太多變,複雜到人類根本寫不完、也寫不準。 但有件事你做得到:你手上有十萬封信,每一封都已經被標好了「是垃圾 / 不是垃圾」。 如果……有一種方法,能讓電腦自己從這十萬個例子裡,歸納出「什麼樣的信像垃圾」的規律, 那該有多好?

這,就是機器學習(machine learning,簡稱 ML)。

翻轉:從「寫規則」到「學規則」

傳統程式設計與機器學習,差別可以濃縮成一張圖。傳統程式裡,規則是輸入:你(人)把規則想清楚寫成程式, 程式吃進資料、吐出答案。機器學習把這件事上下顛倒:你把資料和答案一起餵進去, 讓機器吐出規則(我們之後會叫它「模型」)。

機器學習把這件事上下顛倒:規則變成了輸出。

🔑
機器學習一句話

機器學習,是讓電腦從資料中自動學出「規則」(模型)的方法——而不是由人把規則寫死進程式。 學出的模型,目標是對從未見過的新資料也能做出好的預測,這個能力叫泛化(generalization)。 整本機器學習,講的其實就是兩件事:怎麼學、以及怎麼確定它真的「學會」而不是「背答案」。

兩個經典定義

早在 1959 年,IBM 的亞瑟・山繆爾(Arthur Samuel)寫了一個會下西洋跳棋的程式, 透過跟自己對弈幾千局,棋力竟然超過了他本人。他給機器學習下了一個常被引用的定義:

機器學習,是一門讓電腦「不必被明確編寫程式」就能學習的研究領域。
(原文:the field of study that gives computers the ability to learn without being explicitly programmed.)

這個定義很有畫面,但不夠精確。1997 年,卡內基美隆大學的湯姆・米切爾(Tom Mitchell) 給了一個工程師更喜歡、可操作的版本:

如果一個電腦程式,在某類任務 T 上、用表現指標 P 衡量的成績, 會隨著經驗 E 增加而進步,我們就說它從經驗 E 中「學習」了。

套回垃圾郵件過濾器:任務 T 是「把信分成垃圾/正常」,表現 P 是「分對的比例」, 經驗 E 是「越來越多被標好的信」。看出來了嗎?Mitchell 的定義其實偷偷告訴你: 做 ML 一定要先想清楚「T、P、E 各是什麼」——這正是每個 ML 專案的起手式。

📜
「機器學習」這個詞與它的起伏

「machine learning」一詞正是 Arthur Samuel 在 1959 年那篇跳棋論文裡推廣開的。 這個領域並非一帆風順:1950–60 年代的樂觀(感知器 Perceptron)在 1969 年被 Minsky & Papert 指出致命限制後跌入「AI 寒冬」; 1986 年反向傳播被重新發揚、1990 年代統計學習(SVM)興起;直到 2012 年深度學習在影像辨識競賽 ImageNet 上大勝, 才迎來今天這波爆發。這段歷史提醒我們:技術的潮起潮落,往往不是因為點子變了,而是因為資料與算力終於追上了點子。

AI、機器學習、深度學習,到底誰包含誰?

這三個詞常被混用,但它們是層層包含的同心圓關係。人工智慧(artificial intelligence,AI) 是最大的圈,泛指「讓機器展現智慧行為」的所有努力(包括早年那種人手寫規則的「專家系統」)。 機器學習是 AI 的一個子集,專指「從資料中學」這條路線。 而深度學習(deep learning)又是機器學習的一個子集,特指用「深層神經網路」來學習的那一類方法。

三大類型:機器「學」的三種方式

在跳進細節前,先有張地圖。機器學習依「給機器什麼樣的回饋」分成三大類:

  • 監督式學習(supervised learning):給機器「題目 + 標準答案」(有標籤的資料),讓它學會「看到題目猜答案」。垃圾郵件分類、房價預測都屬此類——這是本章的主軸,也是入門最該打穩的地基。
  • 非監督式學習(unsupervised learning):只給「題目」、沒有答案,讓機器自己找出資料裡的結構或群組。例如把顧客自動分成幾群(分群 clustering)、把高維資料壓縮(降維)。
  • 強化學習(reinforcement learning):沒有標準答案,只有「做得好給獎勵、做不好給懲罰」,讓機器透過反覆試錯,學出一套「在環境中拿最高分」的策略。AlphaGo、自駕車、機器人走路都靠它。

三者裡,監督式學習應用最廣、概念也最清楚,所以我們花最多篇幅在它身上。 把監督式的框架吃透,你會發現另外兩類其實是同一套思想的變奏。

監督式學習的框架:特徵、標籤、模型、損失

所有監督式學習,都長一個樣子。我先把這套「共同骨架」立起來,之後每個演算法都只是往這個骨架填血肉。

特徵 X 與標籤 y

每一筆資料,我們用一組數字來描述它,這些數字叫特徵(feature),通常記成向量 $\mathbf{x}$。 例如要預測房價,一間房子的特徵可能是 $\mathbf{x}=(\text{坪數},\ \text{屋齡},\ \text{到捷運站距離})$。 而我們想預測的那個目標(答案),叫標籤(label),記成 $y$——這裡就是「房價」。

把很多筆資料疊起來,特徵就排成一個矩陣 $X$(每列一筆樣本、每行一個特徵),標籤排成一個向量 $\mathbf{y}$。 監督式學習要做的,就是找一個函數 $h$,讓 $h(\mathbf{x})$ 盡量接近真正的 $y$。這個 $h$ 就是我們的模型(model), 也常被叫做假設函數(hypothesis)。

依照標籤 $y$ 的型態,監督式學習又分兩種,這個分法你會用一輩子:

類型標籤 y 長什麼樣例子
迴歸(regression)連續的數值房價、氣溫、明天的股價、考試分數
分類(classification)離散的類別垃圾/正常、貓/狗/鳥、良性/惡性腫瘤

訓練集與測試集:絕對不能偷看的考卷

這裡有個 ML 初學者最常犯、後果最慘的錯。我們手上的資料,絕不能全部拿去訓練。 必須切成兩塊:訓練集(training set)拿給模型學;測試集(test set)藏起來, 等模型學完,才拿出來考它——而且測試集在訓練過程中,模型連看都不能看一眼。

為什麼?因為我們真正在乎的,不是模型「背題庫」背得多熟,而是它面對沒見過的新資料表現如何。 這就像準備考試:你用考古題(訓練集)練習,但真正能證明你學會的,是你在從沒看過的新題目(測試集)上的成績。 如果你拿著答案去「練習」,考前自我感覺良好,上考場卻一塌糊塗——這在 ML 裡叫過擬合(overfitting),稍後我們會用一整節對付它。

損失函數:把「錯多少」變成一個數字

模型要「學」,得先知道自己現在「錯多少」。我們用一個損失函數(loss function,也叫成本函數 cost function) 把模型在訓練集上的總誤差,量化成一個數字:預測得越準,損失越小;預測得越爛,損失越大。

於是,「學習」這件聽起來很玄的事,被翻譯成了一個再具體不過的數學問題:

🔑
學習 = 最小化損失

監督式學習的本質,就是調整模型的參數,讓損失函數的值變到最小。 這正是第 12 章講的最佳化(optimization)問題! 而我們在那章手刻過的梯度下降(gradient descent),就是達成它最通用的工具。 換句話說:你早在學微積分時,就已經把機器學習的引擎組裝好了——這一章,我們只是把它發動起來。

線性迴歸:把整個 ML 流程走透一遍

我們用最簡單、卻五臟俱全的模型——線性迴歸(linear regression)——把上面那套框架完整跑一遍。 麻雀雖小,它包含了「假設函數 → 損失函數 → 用梯度下降最佳化 → 評估」的全部環節。 把這一節真正吃透,你就懂了 90% 的監督式學習。

問題與假設函數

情境:我們蒐集了八位學生「每天唸書時數 $x$」與「考試分數 $y$」的資料,想學出一條規律, 好預測「唸 5 小時大概考幾分」。最簡單的假設:分數和時數呈直線關係。於是假設函數寫成:

$$h(x) = w x + b$$

這就是國中學的 $y=ax+b$ 換了套衣服。$w$ 叫權重(weight)或斜率(每多唸一小時,分數漲多少), $b$ 叫偏置(bias)或截距(完全不唸的基本盤)。$w$ 和 $b$ 就是這個模型的參數(parameter)—— 機器要「學」的,正是這兩個數字的最佳值。

損失函數:均方誤差 MSE

怎麼衡量「這條線畫得好不好」?看它離資料點有多遠。對第 $i$ 筆資料, 預測值 $h(x_i)$ 和真實值 $y_i$ 的差距叫殘差。我們把所有殘差平方後取平均, 得到最常用的迴歸損失——均方誤差(mean squared error,MSE):

$$L(w,b) = \frac{1}{n}\sum_{i=1}^{n}\big(h(x_i) - y_i\big)^2 = \frac{1}{n}\sum_{i=1}^{n}\big(w x_i + b - y_i\big)^2$$

為什麼要平方?三個理由:(1) 平方讓正負誤差不會互相抵消;(2) 它對「大錯」懲罰得特別重(錯 2 分罰 4,錯 10 分罰 100), 逼模型不敢犯大錯;(3) 平方是平滑可微的,等一下用梯度下降時,我們需要對它求導——這點至關重要。

注意 $L$ 是誰的函數:資料 $x_i,y_i$ 都是固定的,真正能動的是 $w$ 和 $b$。 所以 $L(w,b)$ 是一個「以參數為自變數」的函數,它的圖形是一個漂亮的碗(凸函數)。 我們要找的,就是這個碗的最低點——那組讓損失最小的 $(w,b)$。聽起來很熟?對,這就是找極值。

用梯度下降學參數

碗的最低點,我們在第 12 章學過兩種找法:解析地令導數為零,或——當變數一多就解不動時——用梯度下降一步步走下山。 ML 一律用後者,因為真實模型動輒百萬參數。回憶更新規則:沿著「損失對參數的導數」的反方向,踏一小步。 對 $w$ 和 $b$ 各算一條偏導數(對 MSE 套鏈鎖法則,很好算):

$$\frac{\partial L}{\partial w} = \frac{2}{n}\sum_{i=1}^{n}\big(h(x_i)-y_i\big)\,x_i,\qquad \frac{\partial L}{\partial b} = \frac{2}{n}\sum_{i=1}^{n}\big(h(x_i)-y_i\big)$$

然後同步更新兩個參數($\eta$ 是學習率):

$$w \leftarrow w - \eta\,\frac{\partial L}{\partial w},\qquad b \leftarrow b - \eta\,\frac{\partial L}{\partial b}$$

反覆做這件事,$w$ 和 $b$ 就會一路滑進碗底,損失越來越小。這就是「學習」發生的那一刻。

特徵縮放:先讓資料站在同一個起跑點

動手寫程式前,有個實務細節不能跳過:特徵縮放(feature scaling)。 如果一個特徵的數值範圍是 $1\sim 8$(時數),另一個是 $0\sim 100000$(收入),梯度下降會走得歪七扭八、極慢。 標準做法是標準化(standardization):把每個特徵減掉平均、除以標準差,讓它變成「平均 0、標準差 1」: $x' = \dfrac{x - \mu}{\sigma}$。縮放後梯度下降會順暢得多,學習率也好挑。

從零實作(只用 numpy)

理論講完,動手。下面這段程式不依賴 scikit-learn,純 numpy 把上面每一條公式都實作出來。 這幾十行,就是一個完整的機器學習訓練迴圈: 

import numpy as np

# --- 資料:唸書時數 X(小時) -> 考試分數 y ---
X = np.array([1., 2., 3., 4., 5., 6., 7., 8.])
y = np.array([35., 42., 50., 55., 63., 70., 78., 85.])

# --- 特徵縮放:標準化(平均 0、標準差 1)---
X_mean, X_std = X.mean(), X.std()
Xn = (X - X_mean) / X_std

# --- 初始化參數與超參數 ---
w, b = 0.0, 0.0          # 從零開始猜
lr = 0.1                 # 學習率 η
n = len(X)
losses = []              # 記錄每一步的損失,之後畫圖

# --- 訓練迴圈:梯度下降 ---
for it in range(1, 1001):
    y_hat = w * Xn + b               # 假設函數 h(x)=wx+b
    err = y_hat - y                  # 殘差 (h(x) - y)
    loss = np.mean(err ** 2)         # MSE 損失
    losses.append(loss)

    dw = (2 / n) * np.sum(err * Xn)  # 損失對 w 的偏導數
    db = (2 / n) * np.sum(err)       # 損失對 b 的偏導數
    w -= lr * dw                     # 往下坡走一步
    b -= lr * db

    if it in (1, 5, 20, 100):
        print(f"iter {it:4d}: loss={loss:8.4f}  w={w:.4f}  b={b:.4f}")

# iter    1: loss=3836.5000  w=3.2624  b=11.9500
# iter    5: loss= 643.9601  w=10.9667 b=40.1711
# iter   20: loss=   1.1598  w=16.1237 b=59.0611
# iter  100: loss=   0.3631  w=16.3118 b=59.7500   ← 損失幾乎不動了,收斂

# --- 把參數從「標準化空間」轉回「原始時數」空間 ---
w_orig = w / X_std
b_orig = b - w * X_mean / X_std
print(f"原始空間:分數 ≈ {w_orig:.2f} × 時數 + {b_orig:.2f}")
# 原始空間:分數 ≈ 7.12 × 時數 + 27.71

# --- 用學好的模型預測:唸 5 小時 ---
print("預測(唸 5 小時):", w_orig * 5 + b_orig)   # ≈ 63.31

看那行損失:從第 1 步的 3836,到第 20 步的 1.16,到第 100 步的 0.36—— 模型在我們眼前「學會」了。最後它得出「分數 ≈ 7.12 × 時數 + 27.71」,並預測唸 5 小時約得 63 分。 把每一步的 losses 畫出來,你會看到經典的損失下降曲線:

⚠️
學習率:太大太小都出事

學習率 $\eta$ 是你最常要調的「超參數(hyperparameter)」。太小,損失曲線降得像蝸牛,訓練到天荒地老; 太大,每步跨過頭,損失會劇烈震盪甚至爆炸成 NaN。實務上常從 0.1、0.01、0.001 試起, 看損失曲線挑一個「降得快又穩」的。記住:參數(w, b)是模型學的;超參數(η、訓練步數)是你調的,別搞混。

ℹ️
其實線性迴歸有「公式解」

純線性迴歸是少數有解析解(正規方程式 normal equation,$\mathbf{w}=(X^\top X)^{-1}X^\top \mathbf{y}$)的模型,不必梯度下降也能一步算出。 那我們幹嘛還用梯度下降?因為絕大多數模型沒有公式解(邏輯迴歸、神經網路都沒有), 梯度下降才是通用的萬能鑰匙。這裡用它解線性迴歸,是為了讓你在最簡單的場景裡看清楚這把鑰匙怎麼轉。

邏輯迴歸:從「預測數值」到「做出分類」

線性迴歸吐出的是連續數值。但很多問題要的是「是/否」:這封信是垃圾嗎?這顆腫瘤是惡性嗎? 這類分類問題,主角換成邏輯迴歸(logistic regression)——別被名字騙了, 它名字裡有「迴歸」,做的卻是分類。

問題:直線跑出 0~1 之外了

分類的標籤是 0 或 1(例如 1=垃圾、0=正常)。如果硬用線性迴歸 $wx+b$ 去配, 它會吐出像 $-3.7$ 或 $58$ 這種值——可是「機率」只能介於 0 和 1 之間啊。 我們需要一個「閘門」,把任意實數壓進 $[0,1]$ 區間,好解讀成「是垃圾的機率」。這個閘門就是 sigmoid 函數。

sigmoid:把整條實數線壓進 (0, 1)

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$

它的圖形是一條優雅的 S 形曲線:$z$ 很負時趨近 0,$z$ 很正時趨近 1,$z=0$ 時剛好是 0.5。 無論輸入多大多小,輸出永遠卡在 0 與 1 之間——完美符合「機率」的要求。

於是邏輯迴歸的假設函數,就是「線性部分外面套一層 sigmoid」:

$$h(x) = \sigma(wx + b) = \frac{1}{1 + e^{-(wx+b)}}\quad\in(0,1)$$

$h(x)$ 解讀成「樣本屬於正類(例如垃圾信)的機率」。實作 sigmoid 只要一行,我們親眼看它的輸出: 

import numpy as np

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

for z in [-4, -2, -1, 0, 1, 2, 4]:
    print(f"sigmoid({z:>2}) = {sigmoid(z):.4f}")

# sigmoid(-4) = 0.0180   ← 很負,幾乎判為「否」
# sigmoid(-2) = 0.1192
# sigmoid(-1) = 0.2689
# sigmoid( 0) = 0.5000   ← 正中間,模稜兩可
# sigmoid( 1) = 0.7311
# sigmoid( 2) = 0.8808
# sigmoid( 4) = 0.9820   ← 很正,幾乎判為「是」

決策邊界

有了機率,怎麼做「決定」?設一個門檻,通常是 0.5:$h(x) \ge 0.5$ 就判正類,否則判負類。 因為 $\sigma(z)\ge 0.5$ 恰好等價於 $z\ge 0$,所以「$wx+b=0$」這條線(或在高維是一個平面), 就把空間切成兩半——這就是決策邊界(decision boundary)。一邊判「是」,另一邊判「否」。

交叉熵損失:分類問題的「錯多少」

分類不能用 MSE 當損失(套上 sigmoid 後 MSE 不再是漂亮的碗,梯度下降容易卡住)。 分類用的是交叉熵損失(cross-entropy loss,又叫 log loss):

$$L = -\frac{1}{n}\sum_{i=1}^{n}\Big[\,y_i\log h(x_i) + (1-y_i)\log\big(1-h(x_i)\big)\Big]$$

式子看起來嚇人,直覺其實很美。當真實標籤 $y_i=1$,只剩前半項 $-\log h(x_i)$:模型越篤定地猜對(機率 $h$ 越接近 1), $-\log h$ 越接近 0(損失小);模型越篤定地猜錯(機率接近 0),$-\log h$ 衝向無限大(重罰)。 $y_i=0$ 時對稱地用後半項。一句話:交叉熵會狠狠懲罰「很有信心卻錯得離譜」的預測——這正是我們要的。

有了損失,訓練流程跟線性迴歸一模一樣:算梯度、梯度下降、更新 $w,b$。 實務上你通常不會自己手刻,而是交給 scikit-learn,三行搞定: 

from sklearn.linear_model import LogisticRegression

model = LogisticRegression()
model.fit(X_train, y_train)          # 訓練(內部就是在最小化交叉熵)
proba = model.predict_proba(X_test)  # 預測「屬於各類別的機率」
pred  = model.predict(X_test)        # 直接給出 0/1 判定
🔑
迴歸與分類,是同一個框架的兩個樣子

看出來了嗎?線性迴歸與邏輯迴歸,骨架完全相同:都是「線性組合 $wx+b$ → 損失函數 → 梯度下降」。 差別只在兩處:(1) 分類在線性部分外套一層 sigmoid 把輸出變成機率;(2) 損失從 MSE 換成交叉熵。 記住這個「換零件」的觀念——稍後的神經網路,不過是把這些零件堆成好幾層而已。

過擬合與正則化:ML 的第一道難關

現在進入機器學習最核心、也最容易被忽略的觀念。前面我們一直追求「損失越小越好」, 但這裡有個驚人的轉折:損失太小,反而是壞事。

欠擬合 vs 剛好 vs 過擬合

想像你要用一條曲線去配一堆有雜訊的資料點,有三種結局:

  • 欠擬合(underfitting):模型太簡單(例如硬用直線配彎曲的資料),連訓練資料的大趨勢都抓不住。訓練誤差大,測試誤差也大。這叫高偏差(high bias)。
  • 剛好(good fit):模型複雜度恰當,抓住了真正的規律、忽略了雜訊。訓練誤差小,測試誤差也小。這是我們的目標。
  • 過擬合(overfitting):模型太複雜,複雜到把訓練資料的每一個雜訊點都硬背了下來。訓練誤差小到誇張(甚至為 0),但一碰到新資料就崩潰。這叫高變異(high variance)。

過擬合不是抽象的恐嚇,我們用程式親眼看它。下面用不同次數的多項式去配同一組(帶雜訊的)資料, 同時看訓練誤差與測試誤差: 

import numpy as np
rng = np.random.default_rng(42)
x_tr = np.linspace(0, 1, 12)
y_tr = np.sin(2*np.pi*x_tr) + rng.normal(0, 0.2, 12)   # 真規律是 sin,加雜訊
x_te = np.linspace(0, 1, 50)
y_te = np.sin(2*np.pi*x_te) + rng.normal(0, 0.2, 50)   # 沒見過的測試資料

for deg in [1, 3, 9, 11]:
    c = np.polyfit(x_tr, y_tr, deg)            # 配 deg 次多項式
    train_mse = np.mean((np.polyval(c, x_tr) - y_tr) ** 2)
    test_mse  = np.mean((np.polyval(c, x_te) - y_te) ** 2)
    print(f"degree {deg:2d}: 訓練 MSE={train_mse:.4f}  測試 MSE={test_mse:.4f}")

# degree  1: 訓練 MSE=0.3056  測試 MSE=0.2461   ← 欠擬合:兩者都大
# degree  3: 訓練 MSE=0.0307  測試 MSE=0.0405   ← 剛好:兩者都小,最佳!
# degree  9: 訓練 MSE=0.0051  測試 MSE=0.0609   ← 開始過擬合
# degree 11: 訓練 MSE=0.0000  測試 MSE=0.2173   ← 嚴重過擬合:訓練完美,測試災難

看最後一行:11 次多項式把訓練誤差壓到 0.0000——完美背下了 12 個點。 但測試誤差飆到 0.2173,比簡單的 3 次模型差了五倍!這就是過擬合的鐵證: 訓練分數漂亮得不像話,真本事卻一塌糊塗。模型不是「學會」了,是「背題庫」背過頭了。

偏差-變異權衡

欠擬合與過擬合,是一場拉鋸戰,專業上叫偏差-變異權衡(bias-variance tradeoff)。 偏差(bias)是「模型太死板、抓不住真規律」造成的系統性誤差(欠擬合那端); 變異(variance)是「模型太敏感、隨雜訊起舞」造成的誤差(過擬合那端)。 模型一複雜,偏差就降、變異就升。總誤差是兩者之和,呈一個 U 形——最低點,就是「剛好」的甜蜜點。

三套資料:訓練、驗證、測試

前面說要把資料切成訓練/測試。但實務上要切成三份:

  • 訓練集(training set):用來學參數 $w,b$。
  • 驗證集(validation set):用來挑超參數(多項式次數、學習率、正則化強度等),並偵測過擬合。
  • 測試集(test set):鎖進保險箱,整個開發過程碰都不碰,最後只用一次,給出對「真實泛化能力」的誠實估計。

為什麼驗證集和測試集要分開?因為你會反覆用驗證集挑超參數——挑久了,模型會偷偷對驗證集「過擬合」。 所以需要一個從頭到尾完全沒參與決策的測試集,當最後的判官。資料不夠多時,常用k 折交叉驗證(k-fold cross-validation): 把訓練資料切成 $k$ 份,輪流拿其中 1 份當驗證、其餘 $k-1$ 份訓練,跑 $k$ 次取平均——用同一批資料榨出更穩定的評估。

正則化:給模型「戴上緊箍咒」

除了「換簡單一點的模型」,對付過擬合還有個更精巧的武器:正則化(regularization)。 直覺是:過擬合的模型,參數 $w$ 往往會變得又大又極端(為了硬穿過每個點而劇烈擺動)。 那我們就在損失函數裡加一個懲罰項,逼模型「在配好資料的同時,也要保持參數小而溫和」:

$$L_{\text{new}} = \underbrace{L_{\text{原本}}}_{\text{要配好資料}} + \underbrace{\lambda\sum_j w_j^2}_{\text{懲罰大參數}}$$

這個「懲罰平方和」的版本叫 L2 正則化(Ridge,脊迴歸):它讓所有參數都縮小但不歸零,模型變得平滑。 另一個版本 L1 正則化(Lasso)懲罰的是參數的絕對值和 $\lambda\sum_j |w_j|$:它有個迷人的副作用—— 會把不重要的特徵的權重直接壓成 0,等於自動做「特徵選擇」。$\lambda$ 是強度旋鈕: 太小沒效果(還是過擬合),太大則矯枉過正(變成欠擬合)——又是一個要用驗證集來調的超參數。

🔑
機器學習的第一誡:在測試集上評估

一個模型「好不好」,永遠由它在「從未見過的資料」上的表現決定,而不是訓練集。 訓練誤差低,可能只是背題庫;唯有測試誤差低,才證明它真的學到了規律、能泛化。 一個讓無數人翻車的鐵律是:測試集的任何資訊,都不准在訓練/調參時洩漏出去。 這條誡命你只要違反一次,做出來的「99% 準確率」就全是自欺欺人。我們在實務章節會看到它最陰險的破口——資料洩漏。

評估指標:準確率,常常在騙你

訓練好了模型,怎麼打分數?不同任務有不同的尺。選錯尺,你會對一個爛模型沾沾自喜。

迴歸的指標

指標公式直覺特性
MSE(均方誤差)$\frac{1}{n}\sum(\hat y_i - y_i)^2$重罰大誤差;單位是「原單位的平方」
MAE(平均絕對誤差)$\frac{1}{n}\sum|\hat y_i - y_i|$對離群值較不敏感;單位同原值,好解讀
R²(決定係數)$1 - \frac{\text{模型誤差}}{\text{用平均值猜的誤差}}$$1$=完美,$0$=跟「永遠猜平均」一樣爛,可為負

$R^2$ 特別好用,因為它無單位、好比較:它回答「我的模型,比『無腦永遠猜平均值』好多少?」$R^2=0.85$ 意思是模型解釋了 85% 的變異。

分類:為什麼「準確率」是個陷阱

準確率(accuracy)=「猜對的比例」,聽起來天經地義。但考慮這個場景: 某罕見疾病的盛行率是 1%。我寫一個「模型」,它什麼都不學,永遠predict『沒病』。 它的準確率是多少?99%!因為 100 個人裡 99 個真的沒病,它全猜對了。 但這個模型完全沒用——它一個病人都抓不到。

看出問題了嗎?當類別不平衡(一類遠多於另一類)時,準確率會給你嚴重的假象。 這正是第 14 章講的基率謬誤(base rate fallacy)在 ML 裡的化身: 罕見的正類被龐大的負類淹沒,光看「對的比例」會徹底誤判。我們需要更細緻的尺。

混淆矩陣:把對錯拆成四格

把分類結果攤成一張 $2\times 2$ 的混淆矩陣(confusion matrix),一切就清楚了。 以垃圾郵件為例(垃圾=正類),四種情況:

精確率、召回率、F1

從這四格,衍生出三個關鍵指標:

  • 精確率(precision)$=\dfrac{TP}{TP+FP}$:模型說「是垃圾」的信裡,真的是垃圾的比例。它回答「抓的準不準?」——精確率低代表常誤殺好人。
  • 召回率(recall)$=\dfrac{TP}{TP+FN}$:所有真正的垃圾信裡,被成功抓到的比例。它回答「抓的全不全?」——召回率低代表常放過壞人。
  • F1 分數$=2\cdot\dfrac{\text{precision}\cdot\text{recall}}{\text{precision}+\text{recall}}$:精確率與召回率的調和平均,把兩者綜合成一個數(只有兩者都高,F1 才高)。

用圖 38-10 的數字算一遍:

TP, FP, FN, TN = 18, 7, 2, 73

accuracy  = (TP + TN) / (TP + FP + FN + TN)
precision = TP / (TP + FP)
recall    = TP / (TP + FN)
f1        = 2 * precision * recall / (precision + recall)

print(f"準確率 accuracy  = {accuracy:.2f}")   # 0.91
print(f"精確率 precision = {precision:.2f}")   # 0.72
print(f"召回率 recall    = {recall:.2f}")      # 0.90
print(f"F1 分數          = {f1:.2f}")          # 0.80

精確率與召回率往往此消彼長:你把過濾器調得更兇(寧可錯殺),召回率上升、精確率下降; 調得更鬆,反之。該偏哪邊,取決於業務代價:癌症篩檢寧可錯抓也不能漏掉(重召回率), 垃圾郵件則不能誤殺重要信(重精確率)。ROC 曲線則是把「在各種門檻下,真陽率對偽陽率」畫成一條線, 用線下面積(AUC)總結模型的整體辨識力——一句話帶過,但你會常在報告裡看到它。

⚠️
永遠先問:類別平衡嗎?

拿到分類任務,第一件事先看「各類別各佔多少」。一旦發現嚴重不平衡(例如 99:1), 就立刻把準確率丟掉,改看精確率、召回率、F1,或對少數類別加權、重採樣。 無數「上線後才發現完全沒用」的 ML 系統,死因都是訓練時只盯著那個騙人的高準確率。

其他經典演算法:一人一段

線性/邏輯迴歸是地基,但 ML 的工具箱遠不止於此。以下每個經典演算法給你一段「直覺 + 用途」, 讓你聽到名字時心裡有畫面。它們大多在 scikit-learn 裡都是「換一行」就能用。

k-近鄰(kNN):近朱者赤

k-近鄰(k-nearest neighbors,kNN)是最樸素的想法:要分類一個新樣本,就看「離它最近的 $k$ 個鄰居」 是哪一類,少數服從多數。它幾乎不「訓練」(只是把資料記下來),全部工夫花在預測時的「找鄰居」。 直覺強、好懂,但資料一大就慢,而且維度一高就失靈(維度災難)。適合小資料、低維度的快速基準線。

決策樹與隨機森林:會問問題的二十問遊戲

決策樹(decision tree)像在玩「二十問」:每個節點問一個是非題(「屋齡 > 20 年?」), 順著答案往下走,葉子節點就是預測。它的最大優點是可解釋——你能把整棵樹畫出來給老闆看。 缺點是單棵樹很容易過擬合。解法是隨機森林(random forest):種出成百上千棵「各看一部分資料/特徵」的樹, 讓它們投票。三個臭皮匠勝過諸葛亮,森林比單棵樹穩健準確得多,是表格資料上極受歡迎的強力武器。

支援向量機(SVM):找一條「最寬的街」

支援向量機(support vector machine,SVM)不只想把兩類分開,還想找出那條讓兩類間隔(margin)最寬的分界線; 搭配「核技巧(kernel trick)」還能畫出彎曲的邊界。在深度學習興起前,它是中小型資料上的霸主——這裡一句話帶過。

k-means 分群:非監督的代表

前面都是監督式。k-means是非監督分群的經典:資料沒有標籤,你只想把它們自動分成 $k$ 群。 演算法簡單到迷人,反覆做兩步:(1) 把每個點分給「最近的群心」;(2) 把群心移到「該群所有點的平均位置」。 重複到群心不再移動。我們用純 numpy 實作它,並親眼看它分群: 

import numpy as np

def kmeans(data, k, iters=10, seed=2):
    rng = np.random.default_rng(seed)
    centers = data[rng.choice(len(data), k, replace=False)]   # 隨機選 k 個點當初始群心
    for _ in range(iters):
        # 步驟 1:每個點分給最近的群心
        dist = np.linalg.norm(data[:, None, :] - centers[None, :, :], axis=2)
        labels = dist.argmin(axis=1)
        # 步驟 2:群心移到各群的平均位置
        centers = np.array([data[labels == j].mean(axis=0) for j in range(k)])
    return centers, labels

# 造三坨資料(分別圍繞 (1,1)、(5,5)、(1,5))
rng = np.random.default_rng(0)
data = np.vstack([rng.normal([1,1], 0.4, (30,2)),
                  rng.normal([5,5], 0.4, (30,2)),
                  rng.normal([1,5], 0.4, (30,2))])

centers, labels = kmeans(data, k=3)
print(np.round(centers, 2))
# [[5.04 5.02]
#  [0.95 1.11]
#  [0.95 5.  ]]   ← 漂亮地找回三個中心!

PCA 降維:抓住資料最重要的幾個方向

當特徵有幾百個維度,既難畫圖也容易過擬合。主成分分析(principal component analysis,PCA) 能找出資料「變化最大的幾個方向」,把高維資料投影到少數幾維、盡量保留資訊——這背後正是 線性代數的特徵值與特徵向量(第 54 章會講透),這裡先記住它的用途:壓縮、去噪、視覺化。

神經網路與深度學習:把零件堆成層

終於來到大家最好奇的部分。神經網路聽起來高深,但如果你跟著本章一路走來,你會發現一個令人安心的真相: 它不過是把我們已經學會的零件,堆疊起來而已。

一個神經元,就是一個邏輯迴歸

回想邏輯迴歸:把輸入做線性組合 $z=wx+b$,再過一個 sigmoid。 一個人工神經元(neuron)做的正是這件事:收進好幾個輸入、各乘上一個權重、加總再加偏置、 最後通過一個激活函數(activation function)。沒錯——單一神經元 ≈ 一個邏輯迴歸單元。

魔法發生在「把神經元連成層」。前一層每個神經元的輸出,當成下一層每個神經元的輸入,一層餵一層。 中間那些看不到的層叫隱藏層(hidden layer)。資料從輸入層流到輸出層算出預測,這個過程叫前向傳播(forward propagation)。

激活函數為什麼非要不可?因為若沒有它,「線性組合疊線性組合」永遠還是線性組合,堆再多層也等於一層,白搭。 激活函數(早年用 sigmoid,現在多用更簡單的 ReLU:$\max(0, z)$)注入了非線性, 這才讓網路有能力學習複雜的彎曲關係。

反向傳播:其實就是鏈鎖法則 + 梯度下降

網路有了,怎麼訓練那成千上萬個權重?答案你早就會了。整個訓練,還是那套老把戲: 定義損失 → 算「損失對每個權重的梯度」→ 梯度下降更新。唯一的新問題是:網路這麼多層, 怎麼高效地算出「損失對最前面那層權重」的梯度?

答案是第 12 章的主角——鏈鎖法則(chain rule)。 一個神經網路本質上是「一層套一層的合成函數」$f_L(\cdots f_2(f_1(x)))$。要求最終損失對某個深層權重的導數, 就把「各層的局部導數,由輸出往輸入逐層相乘」。這個「把梯度從後往前一路乘回去」的高效演算法, 就叫反向傳播(backpropagation)。

🔑
反向傳播 = 鏈鎖法則的大規模自動化

別被「反向傳播」這個術語嚇到。它不是什麼新數學,它就是你在微積分學過的鏈鎖法則, 只是被有系統地、自動地套用在百萬層級的合成函數上。前向傳播算出預測與損失; 反向傳播把損失的梯度由後往前沿著每條鏈相乘、分配回每個權重;然後梯度下降照著梯度更新權重。 前向算值、反向算梯度、梯度下降更新——這三拍,就是所有深度學習訓練的心跳。 今天你用 PyTorch 寫 loss.backward(),它在背後做的,正是這件你已經理解的事。

深度學習為什麼是「近年」才爆發?

神經網路的數學在 1980 年代就大致齊備了,為什麼直到 2012 年後才席捲世界?因為要讓「深」的網路真正威力全開, 需要三樣東西同時到位,而它們直到近十幾年才湊齊:

  • 資料(Data):網際網路與智慧型手機,製造出前所未有的海量標註資料(如百萬張標好的 ImageNet 影像)。深層模型是「資料胃口」極大的怪獸,沒有大數據餵不飽。
  • 算力(Compute):GPU(原本為電玩繪圖而生)極擅長平行做矩陣運算——而神經網路的前向/反向傳播,骨子裡全是矩陣乘法。GPU 讓訓練從「幾個月」縮短到「幾天」。
  • 演算法(Algorithms):ReLU、更好的初始化、Dropout、批次正規化、Adam 最佳化器等一連串技巧,解決了深層網路「訓不動」的老毛病。

三者缺一不可。這也呼應了本章開頭那段歷史:好點子常常要等「資料 + 算力」追上來,才會突然引爆。

三大深度學習家族(各一句話)

  • 卷積神經網路(CNN):擅長處理「有空間結構」的資料,如影像——靠「卷積」抓取局部紋理與形狀。人臉辨識、醫學影像的主力。
  • 循環神經網路(RNN):擅長處理「有先後順序」的資料,如文字、語音、時間序列——靠「記憶」把前文資訊帶到後面。
  • Transformer:2017 年問世、靠「注意力機制(attention)」一舉超越 RNN 的架構,是今天所有大型語言模型(如 ChatGPT 背後的 GPT)的基石——也是這波生成式 AI 浪潮的引擎。

ML 實務流程與陷阱

Kaggle 競賽與真實專案的殘酷真相是:頂尖工程師八成的時間,不在調炫炮的模型,而在「搞資料」。 模型只是冰山一角,水面下是又髒又累、卻決定成敗的資料工作。

  • 資料清理(data cleaning):處理缺失值、錯誤值、重複、單位不一。「垃圾進,垃圾出」——再強的模型也救不了爛資料。
  • 特徵工程(feature engineering):把原始資料加工成對模型更有用的特徵(例如把「生日」轉成「年齡」、把「時間戳」拆出「星期幾」)。好的特徵,常比換模型更能提升效果。
⛔
最陰險的殺手:資料洩漏(data leakage)

資料洩漏是指「訓練時,模型偷偷用到了測試時拿不到的資訊」,通常源於測試集的資訊汙染了訓練。 最常見的破口:在切分訓練/測試集之前,就用全部資料去做標準化、填補缺失值或特徵選擇。 這樣一來,測試集的統計量(平均、標準差)就「洩漏」進了訓練流程,你的測試分數會虛高得離譜—— 上線後卻一夕崩盤。鐵律:任何「向資料學習」的前處理步驟,都只能用訓練集去 fit,再套用到測試集上。 這也是 scikit-learn 的 Pipeline 存在的最大理由——它幫你把這件事自動做對。

  • 類別不平衡(class imbalance):如前所述,正負類懸殊時準確率會騙人;對策有重採樣、調整類別權重、改用 F1/AUC 等指標。
  • 過度依賴單一指標:只盯一個數字(尤其準確率)是危險的。要看混淆矩陣的全貌,並回到業務代價去權衡精確率與召回率。

當模型會傷人:ML 的倫理

機器學習不是中立的數學。它從資料中學規律,而資料,承載著人類社會既有的偏見。 一旦疏忽,模型不只會複製這些偏見,還會用「演算法很客觀」的外衣把它們放大、合理化:

  • 偏見與公平性(bias & fairness):用歷史招聘資料訓練的履歷篩選模型,可能學會歧視女性(因為過去就少錄取女性);用過去逮捕資料訓練的再犯預測,可能對特定族群系統性地打高風險分。模型「忠實」地學會了資料裡的不公,後果卻由真實的人承擔。
  • 可解釋性(interpretability):當銀行用模型拒絕你的貸款、醫院用模型決定你的療程,「為什麼?」成了攸關權益的問題。但深層模型常是難以解釋的黑盒。可解釋 AI(XAI)正是要撬開這個盒子。
  • 問責與隱私:模型出錯了誰負責?訓練資料裡的個資如何保護?這些都不是技術細節,而是會影響真實人生的責任問題。

這些議題的份量,值得專門的篇幅——本書第九部會深入探討科技倫理。此刻請先把一句話刻進心裡: 一個有能力的 ML 工程師,必須同時是一個有責任感的人。你寫的模型若被部署,它做的每個決定,都關乎真實的人。

✏️ 練習 38.1 實作

不准用 scikit-learn。只用 numpy,手刻一個線性迴歸的梯度下降,擬合下面這組資料,並印出最終的 $w, b$ 與損失:

X = np.array([1., 2., 3., 4., 5.])
y = np.array([2.2, 4.1, 6.3, 7.9, 10.1])   # 大約是 y = 2x

提示:記得用前面講的 MSE 損失與梯度公式。這組資料範圍小,可以不做特徵縮放。觀察損失有沒有穩定下降。

看解答

核心就是一個訓練迴圈,把「算預測 → 算梯度 → 更新參數」重複夠多次:

import numpy as np

X = np.array([1., 2., 3., 4., 5.])
y = np.array([2.2, 4.1, 6.3, 7.9, 10.1])

w, b = 0.0, 0.0
lr = 0.01            # 沒縮放,學習率取小一點比較穩
n = len(X)

for it in range(1, 2001):
    y_hat = w * X + b
    err = y_hat - y
    dw = (2/n) * np.sum(err * X)
    db = (2/n) * np.sum(err)
    w -= lr * dw
    b -= lr * db
    if it % 500 == 0:
        loss = np.mean(err**2)
        print(f"iter {it}: loss={loss:.5f}  w={w:.4f}  b={b:.4f}")

# 收斂後 w ≈ 1.98, b ≈ 0.13,正好接近資料背後的 y = 2x

關鍵心得:(1) 學習率太大(例如 0.1)在沒縮放時可能震盪甚至發散,可自己改大試試; (2) 兩個參數必須同步更新(都用「更新前」的梯度); (3) 用更多迭代,$w$ 會越來越貼近 2。這短短十幾行,就是一切監督式學習的縮影。

你訓練了一個模型,發現它在訓練集上準確率高達 99%,但在測試集上只有 70%。 (a) 這是什麼現象?(b) 它對應偏差-變異權衡的哪一端?(c) 請提出至少三種具體對策。

看解答

(b) 對應高變異(high variance)那一端——模型太複雜、對訓練資料太敏感。

  • 蒐集更多訓練資料:資料越多,模型越難靠背誦蒙混,被迫去學真正的規律。
  • 降低模型複雜度:用次數更低的多項式、更淺的網路、更少的特徵。

TP, FP, FN, TN = 40, 60, 10, 890
accuracy  = (TP + TN) / 1000              # 0.93
precision = TP / (TP + FP)                # 0.40
recall    = TP / (TP + FN)                # 0.80
f1        = 2*precision*recall/(precision+recall)  # ≈ 0.533
print(accuracy, precision, recall, round(f1,3))
# 0.93 0.4 0.8 0.533

✏️ 練習 38.5 思考

用你自己的話解釋:反向傳播(backpropagation)和鏈鎖法則(chain rule)是什麼關係? 並說明在一次訓練迭代中,「前向傳播」與「反向傳播」各自做了什麼、順序為何。

看解答

關係:反向傳播就是鏈鎖法則的大規模、自動化應用。神經網路是「一層套一層的合成函數」, 要算「最終損失對某個深層權重的導數」,鏈鎖法則告訴我們:把沿路每一層的局部導數相乘即可。 反向傳播就是一套有系統地「由輸出往輸入,逐層把梯度相乘並累積回去」的高效演算法——數學內核完全是鏈鎖法則,沒有新東西。

一次訓練迭代的三拍:

  1. 前向傳播(forward):資料從輸入層流到輸出層,逐層計算,最後算出預測值與損失。
  2. 反向傳播(backward):從損失出發,用鏈鎖法則由後往前逐層算出「損失對每個權重的梯度」。
  3. 梯度下降更新:每個權重沿著自己梯度的反方向走一小步(減去 $\eta \times$ 梯度)。

順序必然是「先前向、後反向」:因為反向傳播需要前向傳播過程中算出的各層中間值,才能計算局部導數。 這三拍不斷重複,損失就一步步下降,網路就「學會了」。你在 PyTorch 寫的 output = model(x)(前向)、 loss.backward()(反向)、optimizer.step()(更新),正是這三拍。

✏️ 練習 38.6 挑戰

用純 numpy 實作 k-means(不准用 scikit-learn)。給定二維資料與 $k$,回傳每個點的群編號與群心。 進階:k-means 對初始群心很敏感,容易卡在「局部最優」。請設計一個簡單機制,讓結果更穩定。

看解答

基本版就是反覆做「指派 → 更新群心」兩步。穩定化的常見招式:用不同隨機起點多跑幾次,挑「群內總距離(inertia)」最小的那次:

import numpy as np

def kmeans_once(data, k, iters=20, seed=0):
    rng = np.random.default_rng(seed)
    centers = data[rng.choice(len(data), k, replace=False)]
    for _ in range(iters):
        dist = np.linalg.norm(data[:, None, :] - centers[None, :, :], axis=2)
        labels = dist.argmin(axis=1)
        new_centers = np.array([data[labels == j].mean(axis=0)
                                if np.any(labels == j) else centers[j]
                                for j in range(k)])
        if np.allclose(new_centers, centers):   # 群心不再動,提早收斂
            break
        centers = new_centers
    inertia = np.sum((data - centers[labels]) ** 2)   # 群內平方距離總和
    return centers, labels, inertia

def kmeans(data, k, n_init=10):
    best = None
    for s in range(n_init):                    # 多個隨機起點
        c, l, inertia = kmeans_once(data, k, seed=s)
        if best is None or inertia < best[2]:  # 留下 inertia 最小的
            best = (c, l, inertia)
    return best

rng = np.random.default_rng(0)
data = np.vstack([rng.normal([1,1],0.4,(30,2)),
                  rng.normal([5,5],0.4,(30,2)),
                  rng.normal([1,5],0.4,(30,2))])
centers, labels, inertia = kmeans(data, 3)
print("群心:\n", np.round(centers,2), "\ninertia =", round(inertia,2))

重點:(1) 處理「某群變空」的邊界情況(否則 mean 會出 NaN);(2) 群心不動就提早收斂; (3) 多起點取最小 inertia 是對抗局部最優的標準做法(scikit-learn 的 n_init 參數就是幹這個); (4) 進一步可用 k-means++ 來挑更聰明的初始群心。

✏️ 練習 38.7 挑戰

你的同事這樣寫訓練流程:「先把整份資料做標準化,再切成訓練集與測試集,然後訓練、評估。」 (a) 指出這段流程的嚴重 bug。(b) 它會造成什麼後果?(c) 寫出正確的流程(可用 scikit-learn 的 Pipeline 概念)。

看解答

(a) Bug:資料洩漏(data leakage)。「先用整份資料做標準化」時,計算平均與標準差用到了測試集, 等於讓測試集的統計資訊偷偷洩漏進了訓練流程。切分必須在任何「向資料學習」的前處理之前。

(b) 後果:測試分數會虛高,讓你對模型的泛化能力產生錯誤的樂觀;一旦上線面對真正全新的資料, 表現會明顯低於測試時看到的數字。更糟的是這種 bug 很隱蔽,不會報錯,常常上線後才被發現。

(c) 正確流程:先切分,再「只用訓練集」fit 前處理器,然後把同一套參數套用到測試集。

from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import make_pipeline
from sklearn.linear_model import LogisticRegression

# 1) 先切分(測試集從此鎖進保險箱)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, random_state=0)

# 2) Pipeline 確保:scaler 只在訓練集上 fit,再套用到測試集
pipe = make_pipeline(StandardScaler(), LogisticRegression())
pipe.fit(X_train, y_train)          # fit 只看 X_train,杜絕洩漏
print("測試集分數:", pipe.score(X_test, y_test))

關鍵:StandardScaler 在 pipe.fit 時只看 X_train 算平均/標準差; 對 X_test 評估時,它套用訓練時學到的同一組參數,測試集的資訊全程沒有外洩。 這就是 Pipeline 最大的價值:把「先 fit 再 transform」這件最容易出錯的事自動做對, 在交叉驗證裡尤其重要。手刻版同理:mean, std = X_train.mean(), X_train.std(), 然後 X_test_scaled = (X_test - mean) / std——永遠別用測試集去算 mean/std。

把這一章串起來

我們從一個追不完規則的垃圾郵件過濾器出發,抵達了現代 AI 的引擎室。回顧這趟旅程的路標:

  • 機器學習的翻轉:不寫死規則,而是餵「資料 + 答案」,讓機器學出規則(模型)。目標是泛化到沒見過的資料。
  • 三大類型:監督式(有標籤)、非監督式(無標籤,找結構)、強化學習(靠獎懲試錯)。
  • 監督式框架:特徵 X、標籤 y、模型 $h$、損失函數。學習 = 用梯度下降最小化損失——這正是第 12 章的最佳化。
  • 線性迴歸把整個流程走透:假設 $h=wx+b$、MSE 損失、梯度下降、特徵縮放、學習率;邏輯迴歸只是外加 sigmoid + 交叉熵,就從迴歸變成了分類。
  • 過擬合與正則化是 ML 的第一道難關:偏差-變異權衡、訓練/驗證/測試三分、L1/L2 正則化,以及鐵律——永遠在測試集上評估。
  • 評估:迴歸看 MSE/MAE/R²;分類別只信準確率(尤其類別不平衡時),要看混淆矩陣、精確率、召回率、F1——這呼應了第 14 章的基率謬誤。
  • 經典演算法:kNN、決策樹/隨機森林、SVM、k-means、PCA,各有擅場。
  • 神經網路不過是把邏輯迴歸單元堆成層;反向傳播 = 鏈鎖法則 + 梯度下降;深度學習近年爆發,靠的是資料 + 算力 + 演算法三者到位。
  • 實務與倫理:八成時間在搞資料;當心資料洩漏與類別不平衡;而模型會學到、甚至放大人類的偏見——技術能力之外,責任感不可或缺。

你現在握有的,不是一堆 API 的使用說明,而是機器到底如何「學」的第一性原理。 往後無論你用 scikit-learn 還是 PyTorch、做的是表格資料還是大型語言模型, 底層流動的,永遠是這一章的這幾條血脈:假設函數、損失、梯度下降、泛化。掌握了它們, 你就不再是面對魔法黑盒的旁觀者,而是能掀開蓋子、知道引擎怎麼轉的工程師。
