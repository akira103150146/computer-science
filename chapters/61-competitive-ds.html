從「通用資料結構」到「競賽資料結構」

在第 29 章,我們親手打造了陣列、鏈結串列、堆疊、佇列、雜湊表、樹、堆積、並查集, 並學會「面對問題挑對結構」。那一章的精神是通用:這些結構在任何程式裡都用得上。 這一章,我們把場景縮到一個特別的戰場——程式競賽與 APCS 實作題。

競賽的世界有它獨特的壓力:題目給你一個 $n$ 可能高達 $2\times10^5$ 甚至 $10^6$ 的序列, 然後丟給你成千上萬筆「查詢」與「修改」,並要求你在一秒內全部答完。 這時候,一個 $O(n^2)$ 的暴力解——哪怕邏輯完全正確——也會無情地 TLE(超時)。 於是競賽選手發展出一套專門的兵器:它們不見得能解決所有問題,但只要對上特定的「區間查詢」「滑動視窗」「動態前綴和」題型, 就能把複雜度從平方壓到對數,甚至線性。

這一章介紹六件這樣的兵器:單調棧、單調隊列、稀疏表、樹狀數組(BIT)、線段樹、競賽版並查集。 我們對每一個都走同一條路線,和第 29 章一致: 它解決什麼問題 → 背後的原理(畫給你看)→ 完整 C++ 實作 → 複雜度 → 用在哪。 程式以 C++ 為主,因為它是競賽的通用語言——快、且有現成好用的 STL。

🔑
這一章的兩條主線

① 單調性(monotonicity):很多時候,資料裡有一大堆「永遠不會是答案」的元素。 只要果斷把它們丟掉、只留下一條單調的序列,複雜度就會崩塌。單調棧、單調隊列都是這個思想。
② 分塊與倍增(blocking & doubling):與其每次查詢都從頭掃,不如預先把資料切成 一塊塊「整理好的小結論」,查詢時把幾塊拼起來就好。稀疏表、BIT、線段樹都是這個思想的不同化身。

暖身:競賽裡的堆疊、佇列與雙端佇列

在動用重兵器之前,先快速複習三個第 29 章的老朋友,並把它們翻譯成 C++ 的講法。 競賽中我們幾乎不會自己手刻這些結構(除非是 APCS 觀念題),而是直接用 STL,省時又不易出錯。

結構規矩STL核心操作
堆疊 stack後進先出 LIFOstd::stackpush / pop / top / empty
佇列 queue先進先出 FIFOstd::queuepush / pop / front / empty
雙端佇列 deque兩端都能進出std::dequepush_back/front / pop_back/front / front / back
 
#include <bits/stdc++.h>   // 競賽常用的萬用標頭(把幾乎所有標準庫都引入)
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);  // 關閉同步,加速 cin/cout

    stack<int> stk;          // 堆疊
    stk.push(1); stk.push(2);
    cout << stk.top() << "\n";   // 2(最後進的在最上面)
    stk.pop();

    queue<int> q;            // 佇列
    q.push(1); q.push(2);
    cout << q.front() << "\n";    // 1(最先進的在最前面)
    q.pop();

    deque<int> dq;           // 雙端佇列,兩端都能動
    dq.push_back(1); dq.push_front(0);
    cout << dq.front() << " " << dq.back() << "\n";   // 0 1
    return 0;
}

這三個容器各對應一類經典應用(第 29 章詳述過):堆疊用於括號配對、後序運算式求值、模擬遞迴; 佇列用於 BFS、流程模擬;deque 則因為兩端都能 $O(1)$ 進出,成了本章單調隊列的硬體基礎。 記住一句話:當你需要「在序列上維護一條會動態增刪的單調序列」,deque 幾乎一定是答案。

單調棧:果斷丟掉「永遠輪不到」的資料

它解決什麼問題

先看一道在 APCS 出現過的題目(改寫自「最接近的高人」):給你 $n$ 個人的身高排成一列, 對每個人,求他右邊第一個比他高的人是誰。這類問題有個響亮的名字: 下一個更大元素(Next Greater Element, NGE)。

最直覺的做法:對每個 $i$,往右一個一個找,直到找到更大的。最壞情況(身高遞減)每個都要掃到底, 總共 $O(n^2)$。當 $n=2\times10^5$,這是 $4\times10^{10}$ 次運算——穩穩超時。我們需要 $O(n)$。

原理:維護一條單調遞減的「待解決」堆疊

關鍵觀察(這也是所有單調結構的靈魂):如果 $a[j] \le a[i]$ 而且 $j < i$,那麼 $a[j]$ 永遠不可能是 「$i$ 之後某人」的下一個更大元素——因為更靠右、又更高(或一樣高)的 $a[i]$ 會先擋住它。 既然 $a[j]$ 已經沒有利用價值,我們就把它丟掉。

於是我們用一個堆疊,只存「還沒找到更大元素」的索引。每讀進一個新元素 $a[i]$, 就把堆疊頂端所有「比 $a[i]$ 小」的元素彈出——對它們而言,$a[i]$ 正是它們等待已久的「下一個更大元素」。 彈完之後,堆疊裡剩下的恰好是一條由底到頂單調遞減的序列,再把 $i$ 推進去。

完整 C++ 實作

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n; cin >> n;
    vector<int> a(n), ans(n, -1);   // ans[i]:右邊第一個更大的值,沒有就 -1
    for (int i = 0; i < n; i++) cin >> a[i];

    stack<int> stk;                 // 存「還沒找到更大元素」的索引,值單調遞減
    for (int i = 0; i < n; i++) {
        // 當前 a[i] 比堆疊頂端的值大 → 它就是頂端那些元素的「下一個更大元素」
        while (!stk.empty() && a[i] > a[stk.top()]) {
            ans[stk.top()] = a[i];
            stk.pop();
        }
        stk.push(i);               // a[i] 自己也加入「等待隊伍」
    }
    for (int i = 0; i < n; i++)
        cout << ans[i] << " \n"[i == n - 1];   // 最後一個換行,其餘空格
    return 0;
}

那個 " \n"[i == n - 1] 是競賽愛用的小技巧:它是一個長度 2 的字串, 當 i == n-1 為真(值 1)時取到 '\n',否則取到 ' '。 效果就是「元素間印空格、最後印換行」。

複雜度:均攤分析的經典範例

乍看那個 while 迴圈藏在 for 裡,像是 $O(n^2)$?其實不是。 關鍵在於:每個索引最多被 push 一次、pop 一次。整個過程中 pop 的總次數, 絕不可能超過 push 的總次數 $n$。所以兩個迴圈合起來,總操作數是 $O(n)$—— 這正是第 26 章講的均攤分析(amortized analysis)的招牌例子。

應用:柱狀圖中的最大矩形

單調棧最漂亮的應用之一:給一排寬度都是 1、高度為 h[i] 的柱子, 求能在裡面框出的最大矩形面積(矩形底邊必須貼齊地面)。 這題在第 27 章「分治」也出現過,但單調棧能直接 $O(n)$ 解決。

核心想法:對每根柱子 $i$,我們想知道「以它的高度 h[i] 當矩形的高,最多能往左右延伸多寬」。 能延伸的範圍,就是左右兩側第一根比它矮的柱子之間。這又是「找下一個(上一個)更小元素」—— 單調棧的拿手好戲。實作上,我們維護一個高度單調遞增的索引堆疊;當一根更矮的柱子到來, 就把比它高的柱子一一彈出,彈出的瞬間正好能算出「那根柱子當高」的最大矩形。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n; cin >> n;
    vector<int> h(n);
    for (int i = 0; i < n; i++) cin >> h[i];

    stack<int> stk;       // 存索引,對應高度單調遞增
    LL best = 0;
    for (int i = 0; i <= n; i++) {
        int cur = (i == n) ? 0 : h[i];   // 末端放一根高 0 的哨兵,逼出剩下所有矩形
        // 來了一根更矮(或一樣)的柱子 → 把比它高的彈出並結算
        while (!stk.empty() && h[stk.top()] >= cur) {
            int height = h[stk.top()];
            stk.pop();
            int left = stk.empty() ? -1 : stk.top();  // 左邊第一根更矮的位置
            LL width = i - left - 1;                   // 這根柱子能延伸的寬度
            best = max(best, (LL)height * width);
        }
        stk.push(i);
    }
    cout << best << "\n";
    return 0;
}
💡
辨識「該用單調棧」的訊號

看到「對每個元素,找它左/右第一個更大/更小的元素」, 或「以每個元素為某種極值,能延伸多遠」,八九不離十就是單調棧。 口訣:找「更大」用遞減棧、找「更小」用遞增棧。

單調隊列:滑動視窗最大值的 $O(n)$ 神技

它解決什麼問題

經典題:給一個長度 $n$ 的陣列,和一個固定寬度 $k$ 的「視窗」。視窗從最左滑到最右,每滑一步, 請回報視窗內的最大值。這就是 滑動視窗最大值(Sliding Window Maximum)。

暴力解:每個視窗位置都重新掃 $k$ 個元素找最大,共 $O(nk)$,當 $n,k$ 都是 $2\times10^5$ 時直接爆炸。 我們要 $O(n)$。

原理:單調棧的「會過期」版本

和單調棧一樣的觀察:如果 $a[j] \le a[i]$ 而且 $j < i$,那 $a[j]$ 一旦和 $a[i]$ 同時在視窗裡, 就永遠輪不到它當最大值——因為 $a[i]$ 更大、又更晚離開視窗。所以可以丟掉。

但這裡比單調棧多了一個麻煩:視窗會從左邊離開元素。最大值可能正好是即將離開視窗的那個, 我們得能從頭部移除過期元素。一個只能單端進出的堆疊辦不到——所以改用 雙端佇列(deque):尾部負責「丟掉比新元素小的」(維持單調),頭部負責「丟掉過期的」。 維護完後,deque 的頭部永遠是當前視窗的最大值。

完整 C++ 實作

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, k; cin >> n >> k;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];

    deque<int> dq;          // 存索引,對應值由前到後單調遞減
    for (int i = 0; i < n; i++) {
        // 1. 頭部過期:索引 ≤ i-k 表示已滑出視窗左界
        if (!dq.empty() && dq.front() <= i - k) dq.pop_front();
        // 2. 尾部清理:比新元素 a[i] 小的都沒用了,從尾部彈掉
        while (!dq.empty() && a[dq.back()] <= a[i]) dq.pop_back();
        // 3. 新元素入隊(從尾部)
        dq.push_back(i);
        // 4. 視窗成形後(i ≥ k-1),頭部就是這個視窗的最大值
        if (i >= k - 1) cout << a[dq.front()] << " ";
    }
    cout << "\n";
    return 0;
}

複雜度與「單調棧 vs 單調隊列」

和單調棧一樣,每個索引最多進 deque 一次、出一次,總共 $O(n)$。 兩者是一對孿生兄弟,差別只在「資料怎麼離場」:

單調棧 (monotonic stack)單調隊列 (monotonic queue)
底層容器stack(只動一端)deque(兩端都動)
誰會被移除只有尾端(被新元素「打敗」者)尾端(被打敗)+ 頭端(過期)
典型問題下一個更大/更小元素、最大矩形固定/滑動視窗的最值、優化 DP 轉移
複雜度$O(n)$$O(n)$
ℹ️
單調隊列也能優化 DP

在第 62 章會看到,許多形如 $dp[i] = \min\{dp[j] : i-k \le j < i\} + c[i]$ 的轉移式,本質就是「在一個滑動視窗裡求最小值」, 可以用單調隊列把 $O(nk)$ 的 DP 壓到 $O(n)$。這是單調隊列在競賽中最值錢的延伸用途。 若不想自己維護單調性,用 multiset 也能做到 $O(n\log n)$,程式更短但較慢。

稀疏表:$O(1)$ 回答「這段區間的最小值是多少」

它解決什麼問題

區間最值查詢(Range Minimum/Maximum Query, RMQ):給定一個不會改變的陣列, 要回答大量「區間 $[l, r]$ 裡的最小值是多少?」。如果每次查詢都掃一遍,單次 $O(n)$,$q$ 筆就是 $O(nq)$,太慢。

當資料靜態(不修改)、查詢的是可重疊運算(最大、最小、gcd、按位 AND/OR 這種「算兩次也沒差」的運算)時, 有一個極漂亮的工具:稀疏表(sparse table)。它 $O(n\log n)$ 預處理後,每筆查詢只要 $O(1)$。

原理:倍增 + 兩塊重疊覆蓋

核心是倍增(binary lifting / doubling)思想。我們預先算好所有「長度是 2 的次方」的區間答案: 令 st[j][i] = 從 $i$ 開始、長度 $2^j$ 的區間(即 $[i,\, i+2^j-1]$)的最小值。 這張表能遞推地建出來,因為一個長度 $2^j$ 的區間,恰好等於兩個長度 $2^{j-1}$ 的區間拼起來:

$$\text{st}[j][i] = \min\Big(\text{st}[j-1][i],\ \ \text{st}[j-1][\,i + 2^{\,j-1}\,]\Big)$$

查詢 $[l, r]$ 時,設區間長度為 $\text{len} = r-l+1$,取 $k = \lfloor \log_2 \text{len} \rfloor$。 一個長度 $2^k$ 的區間可能蓋不滿 $[l,r]$,但兩個長度 $2^k$ 的區間——一個從 $l$ 往右、一個從 $r$ 往左—— 一定能完全覆蓋 $[l,r]$(中間重疊沒關係,因為求 min 重複算不影響答案):

$$\min\Big(\text{st}[k][l],\ \ \text{st}[k][\,r - 2^k + 1\,]\Big)$$

完整 C++ 實作

#include <bits/stdc++.h>
using namespace std;
const int MAXN = 100005, LOG = 17;   // 2^17 > 1e5

int st[LOG][MAXN];   // st[j][i] = a[i .. i+2^j-1] 的最小值
int lg[MAXN];        // 預先算好的 floor(log2)

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, q; cin >> n >> q;
    for (int i = 0; i < n; i++) cin >> st[0][i];   // 長度 2^0=1 的區間就是自己

    // 預處理 log:lg[x] = floor(log2(x))
    lg[1] = 0;
    for (int i = 2; i <= n; i++) lg[i] = lg[i / 2] + 1;

    // 倍增建表:長度 2^j = 兩個長度 2^(j-1) 拼起來
    for (int j = 1; (1 << j) <= n; j++)
        for (int i = 0; i + (1 << j) <= n; i++)
            st[j][i] = min(st[j-1][i], st[j-1][i + (1 << (j-1))]);

    while (q--) {
        int l, r; cin >> l >> r;        // 查詢 [l, r] 最小值(0-indexed)
        int k = lg[r - l + 1];
        cout << min(st[k][l], st[k][r - (1 << k) + 1]) << "\n";
    }
    return 0;
}

複雜度與限制

  • 預處理:表有 $\log n$ 列、每列 $n$ 格,$O(n\log n)$ 時間與空間。
  • 查詢:$O(1)$——只取兩格、算一次 min。這是稀疏表無人能敵的賣點。
  • 致命限制:不支援修改!一旦陣列有任何一格變動,整張表都可能要重建。 而且只適用於「可重疊」的運算(min、max、gcd、AND、OR)。 像「區間和」這種重複算會出錯的運算,稀疏表就不行——那是下面 BIT 與線段樹的舞台。
⚠️
稀疏表只給「靜態」資料用

判準很清楚:資料建好後不再改、查詢的是區間最值(或 gcd)——選稀疏表,$O(1)$ 查到爽。 只要題目出現「修改某個元素」,就必須改用 BIT 或線段樹。別在會修改的題目上硬用稀疏表。

樹狀數組 / BIT:用一個位元技巧維護前綴和

它解決什麼問題

回想第 60 章的前綴和:它能 $O(1)$ 回答區間和, 但有個大弱點——一旦修改某個元素,後面所有前綴和都要重算,$O(n)$。 如果題目同時要「單點修改」和「區間求和」,而且兩者都很多,前綴和就跪了。

樹狀數組(Binary Indexed Tree, BIT;又稱 Fenwick tree、芬威克樹)正是為此而生: 它讓單點修改與前綴和查詢都變成 $O(\log n)$。程式短到只有幾行, 是競賽中 C/P 值最高的資料結構,沒有之一。

原理:lowbit 與「每格管轄一段」

BIT 用一個和原陣列一樣大的陣列 bit[](1-indexed,從 1 開始)。 魔法全在一個運算:lowbit。

$$\text{lowbit}(x) = x \mathbin{\&} (-x)$$

它取出 $x$ 在二進位下最低位的那個 1 所代表的值。例如 $6 = (110)_2$,最低位的 1 在第 1 位, 所以 $\text{lowbit}(6) = 2$;$12 = (1100)_2$,$\text{lowbit}(12) = 4$。 為什麼 x & (-x) 行得通?因為電腦用二補數(two's complement)表示負數, $-x$ 等於「把 $x$ 每個位元反轉再加 1」,結果恰好讓「最低位的 1」對齊、其餘位元歸零(這呼應第 20 章的二補數表示)。

BIT 的精髓:bit[i] 負責管轄 $a[\,i-\text{lowbit}(i)+1\,]$ 到 $a[i]$ 這一段的和, 長度恰好是 $\text{lowbit}(i)$。畫出來就懂了:

兩個核心操作:沿著 lowbit 跳

  • 查詢前綴和 query(i):要算 $a[1..i]$,從 $i$ 出發, 每次加上 bit[i],再令 i -= lowbit(i),直到 $i=0$。 這條路徑把 $[1,i]$ 切成 $O(\log n)$ 段不重疊的 BIT 區間。
  • 單點修改 update(i, delta):把 $a[i]$ 加上 delta, 則所有「管轄範圍涵蓋 $i$」的格子都要跟著加。做法相反:從 $i$ 出發,每次 bit[i] += delta, 再令 i += lowbit(i),直到超出 $n$。

完整 C++ 實作

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int MAXN = 100005;

LL bit[MAXN];   // 1-indexed!
int n;

int lowbit(int x) { return x & (-x); }   // 取最低位的 1

void update(int i, LL delta) {           // a[i] += delta
    for (; i <= n; i += lowbit(i))
        bit[i] += delta;
}

LL query(int i) {                        // 回傳 a[1..i] 的前綴和
    LL sum = 0;
    for (; i > 0; i -= lowbit(i))
        sum += bit[i];
    return sum;
}

LL rangeSum(int l, int r) {              // a[l..r] = 前綴(r) - 前綴(l-1)
    return query(r) - query(l - 1);
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    n = 8;
    int a[9] = {0, 3, 2, 7, 5, 1, 4, 6, 8};   // a[1..8],a[0] 不用
    for (int i = 1; i <= n; i++) update(i, a[i]);   // 逐一建表 O(n log n)

    cout << rangeSum(2, 5) << "\n";   // a[2..5] = 2+7+5+1 = 15
    update(4, 10);                    // a[4] 從 5 變 15(+10)
    cout << rangeSum(2, 5) << "\n";   // 15 + 10 = 25
    return 0;
}

短短十幾行,你就有了一個能在 $O(\log n)$ 內同時「改」和「查」的前綴和。 體會 update 和 query 走的方向恰好相反(一個 +=、一個 -=), 這對稱性正是 lowbit 結構的優雅之處。

應用:配合離散化求逆序對

第 27 章用「魔改合併排序」$O(n\log n)$ 數過逆序對(inversion) ——位置在前、值卻更大的數對 $(i,j)$,$i<j$ 且 $a[i]>a[j]$。BIT 提供了另一條同樣 $O(n\log n)$、且更容易推廣的路:

想法:從左到右掃陣列,用 BIT 把「每個值出現了沒」記在對應位置上。 處理到 $a[i]$ 時,「前面已出現、且比 $a[i]$ 大的數的個數」就是以 $i$ 為右端點新增的逆序對數。 這個數 = (已插入總數 $i$) − (已插入且 ≤ $a[i]$ 的個數) = i - query(rank(a[i]))。

但 $a[i]$ 的值域可能很大(例如到 $10^9$),BIT 開不了那麼大的陣列。解法是 離散化(discretization):把所有出現過的值排序去重, 用它們的排名(第幾小)取代原值。這樣值域就壓縮到 $1 \sim n$,BIT 就裝得下了。 (離散化是第 60 章的基本功,這裡正好派上用場。) 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int n;
LL bit[200005];
int lowbit(int x) { return x & (-x); }
void update(int i, LL v) { for (; i <= n; i += lowbit(i)) bit[i] += v; }
LL query(int i) { LL s = 0; for (; i > 0; i -= lowbit(i)) s += bit[i]; return s; }

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    cin >> n;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];

    // 離散化:排序去重,再用 lower_bound 查每個值的排名(1-indexed)
    vector<int> srt(a);
    sort(srt.begin(), srt.end());
    srt.erase(unique(srt.begin(), srt.end()), srt.end());
    auto rankOf = [&](int x) {
        return int(lower_bound(srt.begin(), srt.end(), x) - srt.begin()) + 1;
    };

    LL inv = 0;
    for (int i = 0; i < n; i++) {
        int r = rankOf(a[i]);
        inv += i - query(r);   // 前面已插入、且比 a[i] 大的個數
        update(r, 1);          // 把 a[i] 記進 BIT
    }
    cout << inv << "\n";
    return 0;
}
💡
BIT 的變奏

BIT 不只能做「單點改 + 前綴和」。配合差分(第 60 章),它還能做「區間加值 + 單點查詢」; 用兩個 BIT 甚至能做「區間加值 + 區間求和」。它也能維護前綴最大值(但不能改小)。 總之,只要操作能拆成「沿 lowbit 跳」,BIT 通常都是最短最快的選擇。

線段樹:區間查詢與區間修改的萬用瑞士刀(重頭戲)

它解決什麼問題

BIT 很香,但它擅長的是「前綴和」這類問題。當題目要的是更刁鑽的組合—— 區間求和 + 區間最大值 + 區間修改,甚至「把一整段區間都加上某個值」「把一整段都改成某個值」—— BIT 就力不從心了。這時候登場的是競賽資料結構的王者:線段樹(segment tree)。

線段樹幾乎能維護任何「可以由左右兩半合併而來」的區間資訊(和、最大、最小、gcd、最大子段和……), 並支援單點修改與(配合懶惰標記的)區間修改,全部 $O(\log n)$。它比 BIT 長、比 BIT 慢一點,但萬用。

原理:把區間遞迴對半切成一棵樹

線段樹是一棵二元樹。根代表整個區間 $[0, n-1]$;每個節點代表一段區間 $[lo, hi]$, 它的兩個小孩各代表左半 $[lo, mid]$ 與右半 $[mid+1, hi]$;一路切到葉節點, 每片葉子代表單一個元素。每個節點存的,是「它所代表那段區間的答案」(這裡以區間和為例)。

三個遞迴:建樹、查詢、單點修改

實作上,我們用一個陣列 seg[] 存整棵樹(開 4 倍空間最保險), 節點編號 1 是根,節點 node 的左小孩是 node*2、右小孩是 node*2+1。 三個操作都是漂亮的遞迴:

  • 建樹 build:葉節點直接放元素值;內部節點遞迴建好左右子樹後,合併兩半。
  • 查詢 query([ql,qr]):對每個節點分三種情況—— 完全不相交(回傳 0)、完全被包含(直接回傳這個節點的答案,不再下探)、 部分重疊(往左右兩個小孩遞迴再合併)。
  • 單點修改 update:往下找到那片葉子改值,回程時順手更新沿路每個祖先的和。
#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int MAXN = 100005;

LL seg[4 * MAXN];   // 線段樹陣列,開 4 倍空間
int a[MAXN];

// 在 node(代表區間 [lo,hi])上建樹
void build(int node, int lo, int hi) {
    if (lo == hi) { seg[node] = a[lo]; return; }   // 葉節點
    int mid = (lo + hi) / 2;
    build(node * 2,     lo,      mid);
    build(node * 2 + 1, mid + 1, hi);
    seg[node] = seg[node * 2] + seg[node * 2 + 1]; // 合併:父 = 左 + 右
}

// 把 a[pos] 改成 val
void update(int node, int lo, int hi, int pos, LL val) {
    if (lo == hi) { seg[node] = val; return; }
    int mid = (lo + hi) / 2;
    if (pos <= mid) update(node * 2,     lo,      mid, pos, val);
    else            update(node * 2 + 1, mid + 1, hi,  pos, val);
    seg[node] = seg[node * 2] + seg[node * 2 + 1]; // 回程更新祖先
}

// 查詢 [ql, qr] 的區間和
LL query(int node, int lo, int hi, int ql, int qr) {
    if (qr < lo || hi < ql) return 0;              // 完全不相交
    if (ql <= lo && hi <= qr) return seg[node];     // 完全被包含
    int mid = (lo + hi) / 2;                       // 部分重疊 → 拆兩半
    return query(node * 2,     lo,      mid, ql, qr)
         + query(node * 2 + 1, mid + 1, hi,  ql, qr);
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n = 4;
    int init[4] = {2, 1, 4, 3};
    for (int i = 0; i < n; i++) a[i] = init[i];

    build(1, 0, n - 1);
    cout << query(1, 0, n - 1, 1, 2) << "\n";   // a[1..2] = 1+4 = 5
    update(1, 0, n - 1, 2, 10);                 // a[2] 從 4 改成 10
    cout << query(1, 0, n - 1, 1, 2) << "\n";   // 1+10 = 11
    return 0;
}

懶惰標記:讓「區間加值」也快起來

如果題目要「把 $[l,r]$ 這一整段每個元素都加上 $v$」,逐一單點修改是 $O(n\log n)$,太慢。 線段樹的殺手鐧是懶惰標記(lazy propagation):當一段區間完全落在修改範圍內時, 我們不真的下去改每個葉子,而是在這個節點上記一張欠條(lazy 標記), 說「我和我底下所有元素,都欠加 $v$」,並立刻更新這個節點的總和。 只有當之後某次查詢或修改需要往下走時,才把欠條下推(push down)給兩個小孩。 這樣「能偷懶就偷懶」,把區間修改也壓到 $O(\log n)$。 

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int MAXN = 100005;

LL seg[4 * MAXN], lazy[4 * MAXN];   // lazy[node]:這段區間「每個元素還欠加多少」
int a[MAXN];

void build(int node, int lo, int hi) {
    lazy[node] = 0;
    if (lo == hi) { seg[node] = a[lo]; return; }
    int mid = (lo + hi) / 2;
    build(node * 2, lo, mid);
    build(node * 2 + 1, mid + 1, hi);
    seg[node] = seg[node * 2] + seg[node * 2 + 1];
}

// 把 node 的欠條下推給兩個小孩
void pushDown(int node, int lo, int hi) {
    if (lazy[node] == 0) return;
    int mid = (lo + hi) / 2;
    LL add = lazy[node];
    // 左孩子 [lo, mid]
    seg[node * 2]  += add * (mid - lo + 1);
    lazy[node * 2] += add;
    // 右孩子 [mid+1, hi]
    seg[node * 2 + 1]  += add * (hi - mid);
    lazy[node * 2 + 1] += add;
    lazy[node] = 0;          // 自己的欠條已交付,清掉
}

// 區間 [ql, qr] 每個元素加上 val
void update(int node, int lo, int hi, int ql, int qr, LL val) {
    if (qr < lo || hi < ql) return;             // 不相交
    if (ql <= lo && hi <= qr) {                   // 完全包含 → 掛欠條,不下探
        seg[node]  += val * (hi - lo + 1);
        lazy[node] += val;
        return;
    }
    pushDown(node, lo, hi);                     // 要往下,先還清欠條
    int mid = (lo + hi) / 2;
    update(node * 2,     lo,      mid, ql, qr, val);
    update(node * 2 + 1, mid + 1, hi,  ql, qr, val);
    seg[node] = seg[node * 2] + seg[node * 2 + 1];
}

LL query(int node, int lo, int hi, int ql, int qr) {
    if (qr < lo || hi < ql) return 0;
    if (ql <= lo && hi <= qr) return seg[node];
    pushDown(node, lo, hi);                     // 查詢往下前也要先下推
    int mid = (lo + hi) / 2;
    return query(node * 2,     lo,      mid, ql, qr)
         + query(node * 2 + 1, mid + 1, hi,  ql, qr);
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n = 6;
    int init[6] = {1, 2, 3, 4, 5, 6};
    for (int i = 0; i < n; i++) a[i] = init[i];
    build(1, 0, n - 1);

    cout << query(1, 0, n - 1, 1, 4) << "\n";   // a[1..4] = 2+3+4+5 = 14
    update(1, 0, n - 1, 1, 4, 10);              // a[1..4] 各 +10
    cout << query(1, 0, n - 1, 1, 4) << "\n";   // 14 + 10×4 = 54
    cout << query(1, 0, n - 1, 0, 2) << "\n";   // a[0..2] = 1 + 12 + 13 = 26
    return 0;
}

線段樹 vs BIT:該選哪一個?

樹狀數組 BIT線段樹 Segment Tree
程式長度極短(約 6 行)較長(建/查/改三個遞迴)
常數 / 速度快、省記憶體($n$ 格)慢一點($4n$ 格)
單點改 + 前綴/區間和✅ 拿手✅ 可以
區間最大/最小值❌(只能勉強做前綴極值)✅ 拿手
區間修改(加值/賦值)△(需差分技巧、較受限)✅ 懶惰標記,通吃
各種奇怪的區間資訊❌✅ 只要能「左右合併」就行
🔑
一句話的選擇法則

只要 BIT 能做的,就用 BIT——短、快、不易錯(尤其「單點改 + 區間和」)。 一旦遇到區間最值、區間修改、或其他需要自訂合併方式的需求, 就果斷上線段樹。線段樹是「能力上界」,BIT 是「特定情況下更省事的特化版」。

競賽風格的並查集

第 29 章已經講透了並查集(union-find / disjoint set)的原理: 每個集合是一棵樹、根當代表,靠路徑壓縮與按秩(或按大小)合併把每次操作壓到近乎常數 $O(\alpha(n))$($\alpha$ 是增長極慢的反阿克曼函數,實務上 ≤ 4)。這裡只補上競賽中最常見的「短打」寫法。

競賽選手追求又短又快。find 用一行遞迴加路徑壓縮;合併用「按大小」並順手維護每個集合的人數 (很多題目會問「這群有幾個人」)。注意這裡用 0-indexed,且通常用全域陣列省事: 

#include <bits/stdc++.h>
using namespace std;
const int MAXN = 100005;

int par[MAXN], sz[MAXN];   // par[x]:父節點;sz[x]:以 x 為根的集合大小

void init(int n) {
    for (int i = 0; i < n; i++) { par[i] = i; sz[i] = 1; }
}

int find(int x) {          // 路徑壓縮:回傳根,並把沿路節點直接接到根
    return par[x] == x ? x : par[x] = find(par[x]);
}

bool unite(int x, int y) {       // 合併;回傳 false 表示本來就同一群
    x = find(x); y = find(y);
    if (x == y) return false;
    if (sz[x] < sz[y]) swap(x, y);   // 按大小:把小的掛到大的下面
    par[y] = x;
    sz[x] += sz[y];
    return true;
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m; cin >> n >> m;          // n 個點、m 條「朋友關係」
    init(n);
    int components = n;              // 一開始每人自成一群
    for (int i = 0; i < m; i++) {
        int a, b; cin >> a >> b;
        if (unite(a, b)) components--;   // 每成功合併一次,群數 -1
    }
    cout << components << "\n";       // 連通分量(朋友圈)的數目
    return 0;
}
ℹ️
並查集在競賽的主戰場

① 動態連通性:不斷加邊,隨時問「兩點連通了嗎」。② Kruskal 最小生成樹: 邊由小到大加,用並查集判斷「會不會成環」(第 63 章會用到)。③ 連通分量計數(上面這題)。 ④ 進階技巧如虛擬根 / 帶權並查集(處理「敵人的敵人是朋友」這類關係)。 它的程式短、效率高,是競賽必背的基本功。

一張總表:操作、複雜度,何時選哪個

把這一章(加上第 60 章的前綴和)的兵器全列出來。挑工具時先問三件事: 「資料會不會修改?」「我要查的是和、還是最值、還是別的?」「修改是單點還是整段?」

結構預處理查詢修改最適合的場景
前綴和
(第 60 章)		 $O(n)$ 區間和 $O(1)$ 不支援(改一格 $O(n)$) 靜態資料的區間和
稀疏表 $O(n\log n)$ 區間最值 $O(1)$ 不支援 靜態資料的 RMQ / gcd
樹狀數組 BIT $O(n\log n)$ 前綴 / 區間和 $O(\log n)$ 單點 $O(\log n)$ 單點改 + 區間和(短、快首選)
線段樹 $O(n)$ 任意可合併資訊 $O(\log n)$ 單點 / 區間 $O(\log n)$ 區間最值、區間修改、複雜區間資訊
單調棧 — 整趟掃描 $O(n)$ 下一個更大/更小、最大矩形
單調隊列 — 整趟掃描 $O(n)$ 滑動視窗最值、優化 DP 轉移
並查集 $O(n)$ 連通性 $O(\alpha(n))$ 合併 $O(\alpha(n))$ 動態連通性、Kruskal MST
🔑
三步驟選擇法

① 不修改 → 區間和用前綴和、區間最值用稀疏表(都 $O(1)$)。
② 會修改 → 單點改+區間和用 BIT;要區間改、區間最值、或奇怪的合併用線段樹。
③ 單向掃描求極值(下一個更大、滑動視窗)→ 單調棧 / 單調隊列,$O(n)$; 動態分組 / 連通 → 並查集。

動手練習

這些結構,看懂程式碼只是第一步;真正內化,要靠親手打。下面七題覆蓋本章每個主角, 建議到 ZeroJudge、AP325 judge 或 OJ 上找對應題目實際提交(AC 才是真懂)。請先自己寫,卡住再看解答。

✏️ 練習 61.1 實作

給一個長度 $n$ 的陣列,對每個元素輸出它右邊第一個比它大的元素(沒有則輸出 -1)。 要求 $O(n)$。例如 [2, 1, 2, 4, 3] → [4, 2, 4, -1, -1]。

看解答

標準單調棧:維護一個「值單調遞減」的索引堆疊;新元素比堆疊頂端大時, 就是頂端那些元素的答案,彈出並填答案。

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n; cin >> n;
    vector<int> a(n), ans(n, -1);
    for (int i = 0; i < n; i++) cin >> a[i];

    stack<int> stk;                 // 存索引,值單調遞減
    for (int i = 0; i < n; i++) {
        while (!stk.empty() && a[i] > a[stk.top()]) {
            ans[stk.top()] = a[i];   // a[i] 是它的「下一個更大元素」
            stk.pop();
        }
        stk.push(i);
    }
    for (int i = 0; i < n; i++)
        cout << ans[i] << " \n"[i == n - 1];
    return 0;
}

每個索引進出堆疊各一次,均攤 $O(n)$。把不等號反向、或改成「找右邊第一個更小」, 就能解一大票變形(例如「每日溫度」「接雨水」)。

✏️ 練習 61.2 實作

給陣列 a 與視窗寬度 $k$,輸出每個長度 $k$ 的滑動視窗內的最大值,$O(n)$。 例如 a=[1,3,-1,-3,5,3]、k=3 → [3, 3, 5, 5]。

看解答

單調隊列:deque 存索引、對應值由前到後遞減。每步先丟過期的頭、再從尾部丟掉比新元素小的, 然後新元素入隊;視窗成形後頭部即最大值。

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, k; cin >> n >> k;
    vector<int> a(n);
    for (int i = 0; i < n; i++) cin >> a[i];

    deque<int> dq;                       // 索引,值單調遞減
    for (int i = 0; i < n; i++) {
        if (!dq.empty() && dq.front() <= i - k) dq.pop_front(); // 過期
        while (!dq.empty() && a[dq.back()] <= a[i]) dq.pop_back(); // 沒用的
        dq.push_back(i);
        if (i >= k - 1) cout << a[dq.front()] << " ";
    }
    cout << "\n";
    return 0;
}

把 <= 改成 >=、deque 維持遞增,就是滑動視窗最小值。 要同時求最大與最小(例如「固定長度區間的最大區段差」),就開兩個 deque。

✏️ 練習 61.3 實作

給一個長度 $n$、值域可能很大的整數陣列,求逆序對的數目 (位置 $i<j$ 但 $a[i]>a[j]$ 的對數)。用 BIT + 離散化,$O(n\log n)$。

看解答

先把值離散化成排名(壓進 $1\sim n$),再從左到右掃:處理 a[i] 時, 「前面已插入、且比它大的個數」= i - query(rank),累加後把它記入 BIT。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;

int n;
LL bit[200005];
int lowbit(int x) { return x & (-x); }
void update(int i, LL v) { for (; i <= n; i += lowbit(i)) bit[i] += v; }
LL query(int i) { LL s = 0; for (; i > 0; i -= lowbit(i)) s += bit[i]; return s; }

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    cin >> n;
    vector<int> a(n), srt;
    for (int i = 0; i < n; i++) { cin >> a[i]; srt.push_back(a[i]); }
    sort(srt.begin(), srt.end());
    srt.erase(unique(srt.begin(), srt.end()), srt.end());

    LL inv = 0;
    for (int i = 0; i < n; i++) {
        int r = int(lower_bound(srt.begin(), srt.end(), a[i]) - srt.begin()) + 1;
        inv += i - query(r);   // 已插入、且比 a[i] 大的個數
        update(r, 1);
    }
    cout << inv << "\n";
    return 0;
}

這和第 27 章「合併排序數逆序對」殊途同歸,但 BIT 法更容易推廣到「二維偏序」「動態加入」等變形。

✏️ 練習 61.4 實作

實作一棵區間和線段樹,支援兩種操作:U p v(把 $a[p]$ 改成 $v$)、 Q l r(輸出 $a[l..r]$ 的和),各 $O(\log n)$。

看解答

三個遞迴:build、單點 update、區間 query。query 的三分支(不相交 / 完全包含 / 部分重疊)是關鍵。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int MAXN = 100005;
LL seg[4 * MAXN];
int a[MAXN], n;

void build(int nd, int lo, int hi) {
    if (lo == hi) { seg[nd] = a[lo]; return; }
    int mid = (lo + hi) / 2;
    build(nd * 2, lo, mid); build(nd * 2 + 1, mid + 1, hi);
    seg[nd] = seg[nd * 2] + seg[nd * 2 + 1];
}
void update(int nd, int lo, int hi, int pos, LL val) {
    if (lo == hi) { seg[nd] = val; return; }
    int mid = (lo + hi) / 2;
    if (pos <= mid) update(nd * 2, lo, mid, pos, val);
    else            update(nd * 2 + 1, mid + 1, hi, pos, val);
    seg[nd] = seg[nd * 2] + seg[nd * 2 + 1];
}
LL query(int nd, int lo, int hi, int ql, int qr) {
    if (qr < lo || hi < ql) return 0;
    if (ql <= lo && hi <= qr) return seg[nd];
    int mid = (lo + hi) / 2;
    return query(nd * 2, lo, mid, ql, qr) + query(nd * 2 + 1, mid + 1, hi, ql, qr);
}

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int q; cin >> n >> q;
    for (int i = 0; i < n; i++) cin >> a[i];
    build(1, 0, n - 1);
    while (q--) {
        char op; int x, y; cin >> op >> x >> y;
        if (op == 'U') update(1, 0, n - 1, x, y);
        else cout << query(1, 0, n - 1, x, y) << "\n";
    }
    return 0;
}

把合併方式 seg[nd*2] + seg[nd*2+1] 改成 max(...) 或 min(...) (查詢初值與「不相交」回傳值也要跟著改成 $\pm\infty$),立刻變成區間最值線段樹。

✏️ 練習 61.5 實作

給一個不會修改的陣列與 $q$ 筆查詢,每筆問 $[l,r]$ 的最小值。 用稀疏表,預處理 $O(n\log n)$、單筆查詢 $O(1)$。

看解答

建表用倍增遞推;查詢用兩塊長度 $2^k$ 的區間覆蓋 $[l,r]$,取 min。

#include <bits/stdc++.h>
using namespace std;
const int MAXN = 100005, LOG = 17;
int st[LOG][MAXN], lg[MAXN];

int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, q; cin >> n >> q;
    for (int i = 0; i < n; i++) cin >> st[0][i];
    lg[1] = 0;
    for (int i = 2; i <= n; i++) lg[i] = lg[i / 2] + 1;
    for (int j = 1; (1 << j) <= n; j++)
        for (int i = 0; i + (1 << j) <= n; i++)
            st[j][i] = min(st[j-1][i], st[j-1][i + (1 << (j-1))]);
    while (q--) {
        int l, r; cin >> l >> r;
        int k = lg[r - l + 1];
        cout << min(st[k][l], st[k][r - (1 << k) + 1]) << "\n";
    }
    return 0;
}

注意:稀疏表只對「可重疊」運算(min/max/gcd)有效。若改成區間和,因為重疊會重複計入而出錯—— 那種情況請改用 BIT 或線段樹。

✏️ 練習 61.6 挑戰

實作一棵帶懶惰標記的線段樹,支援:A l r v(把 $a[l..r]$ 每個元素加上 $v$)、 Q l r(輸出 $a[l..r]$ 的和),兩者皆 $O(\log n)$。

看解答

關鍵在 pushDown:每次要往下走(無論查詢或修改)之前,先把欠條傳給小孩。 區間加值時,若節點完全被包含,就只更新自己的和並累加 lazy,不再下探。

#include <bits/stdc++.h>
using namespace std;
typedef long long LL;
const int MAXN = 100005;
LL seg[4 * MAXN], lazy[4 * MAXN];
int a[MAXN], n;

void build(int nd, int lo, int hi) {
    lazy[nd] = 0;
    if (lo == hi) { seg[nd] = a[lo]; return; }
    int mid = (lo + hi) / 2;
    build(nd * 2, lo, mid); build(nd * 2 + 1, mid + 1, hi);
    seg[nd] = seg[nd * 2] + seg[nd * 2 + 1];
}
void pushDown(int nd, int lo, int hi) {
    if (!lazy[nd]) return;
    int mid = (lo + hi) / 2; LL add = lazy[nd];
    seg[nd*2]   += add * (mid - lo + 1);  lazy[nd*2]   += add;
    seg[nd*2+1] += add * (hi - mid);      lazy[nd*2+1] += add;
    lazy[nd] = 0;
}
void update(int nd, int lo, int hi, int ql, int qr, LL v) {
    if (qr < lo || hi < ql) return;
    if (ql <= lo && hi <= qr) { seg[nd] += v * (hi - lo + 1); lazy[nd] += v; return; }
    pushDown(nd, lo, hi);
    int mid = (lo + hi) / 2;
    update(nd*2, lo, mid, ql, qr, v); update(nd*2+1, mid+1, hi, ql, qr, v);
    seg[nd] = seg[nd*2] + seg[nd*2+1];
}
LL query(int nd, int lo, int hi, int ql, int qr) {
    if (qr < lo || hi < ql) return 0;
    if (ql <= lo && hi <= qr) return seg[nd];
    pushDown(nd, lo, hi);
    int mid = (lo + hi) / 2;
    return query(nd*2, lo, mid, ql, qr) + query(nd*2+1, mid+1, hi, ql, qr);
}
int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int q; cin >> n >> q;
    for (int i = 0; i < n; i++) cin >> a[i];
    build(1, 0, n - 1);
    while (q--) {
        char op; cin >> op;
        if (op == 'A') { int l, r; LL v; cin >> l >> r >> v; update(1, 0, n-1, l, r, v); }
        else { int l, r; cin >> l >> r; cout << query(1, 0, n-1, l, r) << "\n"; }
    }
    return 0;
}

最容易錯的兩個地方:① 忘了在 query 裡也 pushDown;② lazy 累加要用 += 而非 = (因為一個節點可能累積多次未下推的加值)。把這兩點顧好,懶惰標記就穩了。

✏️ 練習 61.7 實作

有 $n$ 個人(編號 $0 \sim n-1$)與 $m$ 條「朋友關係」。朋友的朋友也是朋友。 用並查集求最後總共有幾個朋友圈(連通分量),並回答若干筆「$x$ 和 $y$ 是不是同一圈」。

看解答

路徑壓縮 + 按大小合併。每成功合併一次,連通分量數就 −1;同圈判斷就是比較兩者的根。

#include <bits/stdc++.h>
using namespace std;
const int MAXN = 100005;
int par[MAXN], sz[MAXN];

int find(int x) { return par[x] == x ? x : par[x] = find(par[x]); }
bool unite(int x, int y) {
    x = find(x); y = find(y);
    if (x == y) return false;
    if (sz[x] < sz[y]) swap(x, y);
    par[y] = x; sz[x] += sz[y];
    return true;
}
int main() {
    ios::sync_with_stdio(false); cin.tie(nullptr);
    int n, m, q; cin >> n >> m >> q;
    for (int i = 0; i < n; i++) { par[i] = i; sz[i] = 1; }
    int comp = n;
    for (int i = 0; i < m; i++) {
        int a, b; cin >> a >> b;
        if (unite(a, b)) comp--;
    }
    cout << comp << "\n";                 // 朋友圈數目
    while (q--) {
        int x, y; cin >> x >> y;
        cout << (find(x) == find(y) ? "YES" : "NO") << "\n";
    }
    return 0;
}

進階變形:用「虛擬根」(把編號 $i$ 的敵人記在 $i+n$)可以處理「敵人的敵人是朋友」這類題; 要支援刪點時,常用離線反向(把刪除倒過來看成加入)。這些都是並查集的常見競賽套路。

把這一章串起來

這一章把第 29 章的通用資料結構,升級成競賽戰場上能扛住 $2\times10^5$ 級資料、一秒內答完的高效兵器。 回顧整趟旅程,你會發現它們全圍繞著兩個核心思想打轉:

  • 單調棧:果斷丟掉「永遠輪不到」的元素,只留單調序列。$O(n)$ 解「下一個更大元素」「柱狀圖最大矩形」。
  • 單調隊列:單調棧的「會過期」版本,用 deque 兩端維護。$O(n)$ 解滑動視窗最值,也能優化 DP。
  • 稀疏表:倍增預處理、兩塊重疊覆蓋,$O(1)$ 回答靜態 RMQ——代價是不能修改。
  • 樹狀數組 BIT:lowbit 讓每格管轄一段,$O(\log n)$ 同時支援單點改與前綴和;配離散化求逆序對。又短又快。
  • 線段樹:把區間遞迴對半切成樹,$O(\log n)$ 駕馭任何「能左右合併」的區間資訊;懶惰標記讓區間修改也變 $O(\log n)$。萬用王者。
  • 並查集:路徑壓縮 + 按大小合併,近乎常數時間處理動態連通性。Kruskal 與連通分量的好搭檔。

選擇的心法不變:不修改就用前綴和 / 稀疏表($O(1)$);要修改,單點改+區間和用 BIT、其餘用線段樹; 單向掃描求極值用單調棧 / 隊列;動態分組用並查集。沒有最強的結構,只有最貼合手邊那道題的結構。

有了這些兵器,你已經能拆解一大票 APCS 實作題第 3、4 題與 OJ 中等難度題。 接下來的第 62 章,我們要把焦點從「資料結構」轉向「演算法範式」—— 暴搜、分治、貪心、DP——並看到單調隊列、線段樹如何反過來成為這些範式的加速引擎。
