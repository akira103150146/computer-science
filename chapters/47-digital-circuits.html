數位電路與有限狀態機

第 19、20 章把邏輯閘當成乾淨俐落的理想黑盒子。但真實世界裡,閘是用電晶體做的,訊號是會抖動的電壓,而且——每一次計算,都要花時間。這一章,我們掀開那層蓋子,看看真正的數位電路,以及它如何長出「記憶與行為」:有限狀態機。

掀開閘的蓋子:從抽象回到物理

在第 19、20 章,我們做了一件非常「電腦科學」的事:把電晶體抽象成邏輯閘,然後就再也不去想電晶體了。 AND、OR、NOT、NAND 對我們來說是完美的數學函數——你餵它 0 和 1,它立刻、精準、毫不猶豫地吐出正確答案。 我們甚至只用一顆理想的 NAND,就在紙上蓋出了加法器、ALU、暫存器、RAM,一路堆到 CPU 的雛形。

這套抽象漂亮、好用,而且該這樣用——蓋大樓的時候,你不需要一邊想著水泥裡每顆砂石的位置。 但是,當你想真正理解一台電腦為什麼能可靠地運作、為什麼有速度上限、為什麼會有那些詭異的時序 bug, 你就得回頭,掀開那層蓋子,看看閘底下究竟發生了什麼事。

這一章,我們要從電腦科學家,暫時變回電子工程師。你會看到:所謂的「0 和 1」,其實是兩段電壓區間; 所謂的閘,其實是幾顆電晶體排成的小開關電路;而所謂「立刻」吐出答案,其實要花上幾十皮秒(picosecond)。 最後,我們會把這些真實的零件——會算的組合邏輯、會記的暫存器、滴答作響的時脈——組裝成這一章的重頭戲: 有限狀態機(finite state machine,FSM),也就是讓硬體能「依當下狀態做出不同行為」的通用模型。

🔑
這一章站在抽象塔的哪一格

第 1 章說過,整個電腦世界是一座抽象高塔:電晶體 → 邏輯閘 → CPU → 組合語言 → C → Python。 第 19、20 章站在「邏輯閘」那一格往上看。這一章反過來,往下看一格:我們把「邏輯閘」這塊磚敲開, 看清它是怎麼由電晶體與電壓砌成的,以及這塊磚其實有重量(延遲)。 理解了下面這一層,你對上面所有層次的直覺都會更紮實。

數位抽象:為什麼堅持只用 0 和 1

第 19 章給過一個直覺答案:用兩種狀態(高電壓/低電壓)最不容易被雜訊搞混。現在我們把這件事講精確—— 因為這正是「數位」之所以能擊敗「類比」、之所以能堆疊到幾百億顆電晶體還不崩潰的根本原因。

一條電線上的電壓是連續的:它可以是 0V、0.31V、2.97V……任何值。所謂數位抽象(digital abstraction), 就是我們人為地規定:「低於某個門檻就算 0,高於另一個門檻就算 1,中間那段不准用。」 我們把無限多種電壓,硬是壓縮成兩個符號。問題是:電子世界很吵——電源會波動、相鄰的線會互相干擾、 電阻會發熱產生熱雜訊。一個本來是「乾淨的 1」的訊號,傳著傳著就被推高推低了零點幾伏。怎麼確保它不會被誤讀成 0?

四個門檻與雜訊邊際

關鍵的巧思是:對輸出端和輸入端採取「不對稱」的標準。每個邏輯閘都遵守一份合約,用四個電壓門檻定義:

  • $V_{OL}$(output low):閘輸出 0 時,保證電壓不高於這個值(壓得很低)。
  • $V_{OH}$(output high):閘輸出 1 時,保證電壓不低於這個值(拉得很高)。
  • $V_{IL}$(input low):閘讀到的電壓只要不高於這個值,就一律當成 0。
  • $V_{IH}$(input high):閘讀到的電壓只要不低於這個值,就一律當成 1。

魔法藏在它們的大小關係:$V_{OL} < V_{IL} < V_{IH} < V_{OH}$。也就是說, 輸出被要求得比輸入嚴格:輸出 1 一定拉得比「輸入願意接受為 1」的最低標準更高;輸出 0 一定壓得比「輸入願意接受為 0」的最高標準更低。 這中間多出來的兩段緩衝,就是雜訊邊際(noise margin):

$$NM_{H} = V_{OH} - V_{IH}, \qquad NM_{L} = V_{IL} - V_{OL}.$$

舉個具體數字:假設 $V_{OH}=2.4\text{V}$、$V_{IH}=2.0\text{V}$,那麼高電壓的雜訊邊際就是 $0.4\text{V}$—— 一個閘吐出 $2.4\text{V}$ 的「1」,就算路上被雜訊扣掉 $0.39\text{V}$,到達下一個閘還有 $2.01\text{V}$,仍穩穩在「讀成 1」的範圍內。 雜訊只要不超過這個邊際,就完全無害。

靜態規範:數位之所以可堆疊的祕密

把上面這份合約寫成一句話,就是靜態規範(static discipline):

只要你餵給一個元件的輸入是合法的(低於 $V_{IL}$ 或高於 $V_{IH}$),它就保證產生合法的輸出(低於 $V_{OL}$ 或高於 $V_{OH}$)。

這條規範帶來一個驚人的後果,叫做訊號還原(signal restoration):每經過一個閘,訊號就被「重新拉滿、推到位」一次。 一個被雜訊弄髒、降到 $2.05\text{V}$ 的「1」,進入下一個閘後,輸出又是漂亮的 $2.4\text{V}$。 於是雜訊不會累積——這正是類比訊號做不到的事。

🔑
這就是「數位」能贏的根本理由

類比電路裡,每經過一級放大,雜訊與失真就累積一點;串個幾十級,訊號早就面目全非。 但數位電路因為有雜訊邊際 + 靜態規範,每一級都把訊號「清乾淨、還原成標準的 0 或 1」。 所以你可以把幾百億顆電晶體串起來,訊號穿過成千上萬層閘,到最後一層依然是乾淨的 0 與 1。 「離散化」看似損失了資訊,換來的卻是無限可堆疊的可靠性——這是整個數位文明的地基。

這也回扣第 19 章那句話:用 0/1 不是因為簡單,而是因為能抗雜訊。現在你看到的是它精確的工程版本。

CMOS:用電晶體當開關

閘底下是什麼?是電晶體(transistor)。今天幾乎所有數位晶片都用一種叫 CMOS(Complementary Metal-Oxide-Semiconductor,互補式金屬氧化物半導體)的技術。 我們不需要鑽進半導體物理,只要抓住一個直覺就夠了:

🔑
電晶體 = 一個用電壓控制的開關

一顆 MOS 電晶體有三隻腳:閘極(gate,控制端)、源極、汲極(兩端)。 閘極上的電壓,決定另外兩端之間是「通」還是「斷」——就像一個由電壓遙控的水龍頭。 數位電路裡的一切,歸根究柢都是這些小開關在「導通」與「截止」之間切換。

CMOS 用的是兩種互補的開關,個性正好相反:

  • NMOS:閘極為 1(高電壓)時導通,為 0 時截止。它擅長把輸出往下拉到 0(GND),是好的「下拉(pull-down)」開關。
  • PMOS:閘極為 0(低電壓)時導通,為 1 時截止(行為跟 NMOS 完全相反)。它擅長把輸出往上拉到 1(電源 $V_{DD}$),是好的「上拉(pull-up)」開關。

用兩顆電晶體做一個反相器(NOT)

最小的 CMOS 閘是反相器(inverter,也就是 NOT),只要兩顆電晶體:一顆 PMOS 接在電源 $V_{DD}$ 與輸出之間, 一顆 NMOS 接在輸出與接地 GND 之間,兩顆的閘極接同一條輸入 $A$。

注意一個關鍵:無論輸入是 0 還是 1,上下兩顆電晶體永遠「剛好一開一關」。 於是在穩定狀態下,從 $V_{DD}$ 到 GND絕不會有一條直接導通的路徑。這帶來 CMOS 最大的優點——

🔑
CMOS 為什麼省電:互補的代價是「幾乎不耗電」

因為上拉(PMOS)與下拉(NMOS)網路互補,穩態下總有一邊是斷的,電源到地之間沒有短路電流。 理想上,一個閘「靜靜待著不切換」時幾乎不耗電;只有在輸入翻轉的那一瞬間, 為了替線路上的電容充放電,才會流過電流。這就是為什麼你的手機能塞進幾十億顆電晶體卻不會立刻燒掉: 大部分電晶體在大部分時間都是「省電待命」的。也因此,晶片的功耗大致正比於時脈頻率(切換越頻繁越耗電)—— 這也解釋了為什麼狂飆時脈會讓 CPU 發燙。

NAND 在 CMOS 裡天生就比 AND 便宜

把反相器的概念推廣一下:任何 CMOS 閘都是「一個 PMOS 上拉網路 + 一個 NMOS 下拉網路」,兩者互補。 做一個 NAND 只要四顆電晶體:下拉網路是兩顆 NMOS 串聯,上拉網路是兩顆 PMOS 並聯。

為什麼「下拉串聯、上拉並聯」?因為串聯要「全部導通」才通(對應 AND 的「都是 1」), 並聯只要「一個導通」就通(對應 OR)——而 NMOS 在輸入為 1 時導通。把這兩件事拼起來,下拉網路在「$A$ 且 $B$」時導通、把輸出拉成 0, 正是 $\neg(A\cdot B)$。上拉網路必須是下拉的互補(對偶),於是用 PMOS 並聯—— 你看,這個「串聯 ↔ 並聯、AND ↔ OR」的對調,骨子裡就是第 19 章的 De Morgan 定律長在矽晶片上的樣子。

💡
原來 NAND 的萬能,不只是數學上的

第 19 章說 NAND 是「萬能閘」——只用它就能組出一切。現在你多懂一層:在 CMOS 裡, NAND 與 NOR 是「天生」的閘,反而 AND 與 OR 比較貴。因為 CMOS 的下拉/上拉結構天生會反相, 所以「先算邏輯、再反相」的 NAND/NOR 各只要 4 顆電晶體;而 AND 必須是「NAND 之後再接一個反相器」, 反而要 6 顆。難怪晶圓廠超愛 NAND——它既在數學上萬能,又在物理上最省。 (你能想想 NOR 的 CMOS 接法嗎?那是本章的練習。)

組合邏輯的化簡:卡諾圖

我們已經知道(第 19 章):任何布林函數都能寫成積之和(SOP)——把真值表裡每個輸出為 1 的列寫成一個最小項,再 OR 起來。 這保證正確,但常常臃腫。化簡一條布林式,在硬體上就是少用幾顆閘、省電晶體、省電、跑更快。 第 19 章用代數恆等式(交換律、分配律、De Morgan……)手動化簡;但代數化簡很吃靈感、容易漏。 有沒有一個機械化、用看的就能化簡的方法?有,它叫卡諾圖(Karnaugh map,K-map)。

卡諾圖的核心洞見是:布林代數有一條「合併律」——$X\cdot Y + X\cdot \neg Y = X$(不管 $Y$ 是 0 是 1,只要 $X$ 成立就成立,所以 $Y$ 可以消掉)。 卡諾圖把真值表重新排成一張方格圖,而且刻意讓相鄰格子只差一個變數。 於是「可以套用合併律的項」在圖上就是挨在一起的 1——你只要把它們圈起來,就完成了化簡。

實例:三人多數表決,再來一次

還記得第 19 章那個「三人投票,過半就亮燈」的多數函數 $M(A,B,C)$ 嗎?當時我們用代數把它從 4 個最小項硬湊成 $M = AB + BC + AC$。現在用卡諾圖,看它一眼就化出來。先把真值表填進方格:橫軸是 $BC$(用 格雷碼順序 00、01、11、10,確保相鄰只差一位),縱軸是 $A$。

規則就這麼幾條:(1) 圈起來的 1 必須是矩形,且格數是 2 的次方(1、2、4、8…); (2) 圈越大越好(消去越多變數);(3) 圈可以重疊(同一個 1 可被多個圈覆蓋); (4) 圖的邊緣會繞回去(最左欄與最右欄相鄰、最上列與最下列相鄰,因為格雷碼是環狀的)。 每個圈讀出一個 AND 項:圈內始終不變的變數保留(是 1 寫原樣、是 0 加否定),有變動的變數消去。

ℹ️
卡諾圖只是把代數「畫出來」

卡諾圖沒有任何新魔法——它只是把布林合併律 $XY + X\neg Y = X$ 變成「相鄰的兩個 1 可以合併」的視覺規則。 人腦很不擅長盯著一長串符號做代數,卻很擅長在圖上一眼看出「哪些方塊連在一起」。卡諾圖就是把化簡這件事, 從「考驗代數技巧」變成「考驗畫圈圈的眼力」。變數一多(超過 4、5 個)圖就畫不下了,那時就交給電腦—— 晶片設計軟體裡跑的是 Quine–McCluskey、Espresso 這類演算法,本質上做的是同一件事:找最少、最大的「圈」。

稍後你會看到,卡諾圖在 FSM 裡會立刻派上用場:設計狀態機時,「次態邏輯」就是一坨組合邏輯,正好用卡諾圖化簡。

時間維度:傳播延遲、毛刺與關鍵路徑

到目前為止,我們(還有第 19、20 章)都假裝閘是「瞬間」運作的:輸入一變,輸出立刻變。 但真實電晶體要替線路上的電容充放電,需要時間。這個「輸入改變」到「輸出穩定」之間的延遲, 叫傳播延遲(propagation delay,$t_{pd}$),通常以皮秒($10^{-12}$ 秒)計。 一旦把時間考慮進來,兩個重要、且常常咬人的現象就冒出來了。

毛刺(glitch):輸出短暫地「說錯話」

一個訊號若經由長短不一的兩條路徑抵達同一個閘,它們到達的時間不同,輸出就可能在穩定下來之前, 先短暫地閃出一個錯誤的值——這個轉瞬即逝的錯誤脈衝叫毛刺(glitch),也叫冒險(hazard)。

毛刺不代表電路設計錯了——只要你等得夠久,組合邏輯最終一定會穩定到正確值。問題在於:在它穩定之前, 輸出是垃圾。如果某個記憶元件在「還沒穩定」時就去採樣它,就會存進錯誤的值。這正是時脈存在的理由 (第 20 章已經埋下這個伏筆):給組合邏輯足夠的時間把結果算穩,等穩了,才在時脈邊緣一次採樣。 問題是——「足夠的時間」到底是多久?這就帶出關鍵路徑。

關鍵路徑(critical path):決定你能跑多快

一塊組合邏輯裡,訊號從輸入到輸出有很多條路徑,每條經過的閘數、延遲都不同。其中延遲最長的那一條, 叫關鍵路徑(critical path)。整塊電路必須等到最慢的路徑也算完,輸出才算全部穩定。 所以,關鍵路徑的延遲,就決定了「一拍最少要多長」。

一拍(時脈週期 $T$)至少要容納下面這串時間,訊號才來得及從一個暫存器穩穩送到下一個暫存器:

$$T \;\ge\; t_{pcq} \;+\; t_{pd}(\text{關鍵路徑}) \;+\; t_{setup}.$$

其中 $t_{pcq}$ 是暫存器「時脈邊緣到輸出穩定」的延遲(clock-to-Q),$t_{setup}$ 是下一個暫存器要求資料提前穩定的時間(下一節談)。 最高時脈頻率就是 $f_{max} = 1/T$。以圖中的數字:$T = 40 + 140 + 20 = 200\text{ps}$,所以 $f_{max} = 1/(200\times 10^{-12}\text{s}) = 5\text{ GHz}$。

🔑
為什麼電路有速度上限

你常聽到「這顆 CPU 是 5 GHz」,意思就是它的時脈每秒跳 50 億次,而它跳這麼快的上限,正是被關鍵路徑卡住的。 你不能把時脈調得比「最慢那條路徑算完所需的時間」還快,否則暫存器會在訊號還是垃圾時就採樣,整台機器算出亂碼。 這就是為什麼晶片設計師會拼命去縮短關鍵路徑(例如第 20 章提過的前瞻進位加法器,就是為了縮短加法的進位鏈), 以及為什麼超頻會不穩定——你把時脈逼到比關鍵路徑還短,訊號就來不及穩定了。計算需要時間,而最慢的那一步,定義了你的極速。

找最大的圈,只需要兩個 4 格的圈:

  • 四個角(m0、m2、m8、m10):這四格繞回相鄰,構成一個 $2\times 2$ 的圈。它們共同點是 $B=0$ 且 $D=0$($A$、$C$ 都在變)→ 項為 $\neg B\cdot\neg D$。
  • 正中央的四格(m5、m7、m13、m15):共同點是 $B=1$ 且 $D=1$ → 項為 $B\cdot D$。

所以化簡結果是:

$$F = \neg B\cdot \neg D \;+\; B\cdot D.$$

$A$ 和 $C$ 完全被消掉了——它們對結果毫無影響!這個式子其實就是「$B$ 與 $D$ 相等時輸出 1」,也就是 $B$ 與 $D$ 的 XNOR(同或)。 原本 8 個四輸入最小項(每個要 3 個 AND + 一堆反相器),化簡後只剩 2 個兩輸入 AND 加一個 OR。用 Python 驗證一下:

def F_raw(a, b, c, d):           # 原始:列出 8 個最小項
    m = (a<<3) | (b<<2) | (c<<1) | d
    return 1 if m in (0,2,5,7,8,10,13,15) else 0

def F_simplified(a, b, c, d):    # 化簡後:¬B·¬D + B·D
    return ((1-b) & (1-d)) | (b & d)

ok = all(F_raw(a,b,c,d) == F_simplified(a,b,c,d)
         for a in (0,1) for b in (0,1) for c in (0,1) for d in (0,1))
print(ok)    # True ← 16 種輸入全部吻合

循序邏輯深化:latch、flip-flop 與時序限制

第 20 章把正反器(flip-flop,DFF)當成「循序邏輯的原語」直接用,記住它「在時脈邊緣鎖存一個位元」就好。 現在掀開它的蓋子,看看記憶元件在真實時間裡的細節——這些細節,正是真實電路最常出 bug 的地方。

latch vs flip-flop:準位觸發 vs 邊緣觸發

記憶元件有兩大類,差別在於「什麼時候聽輸入的話」:

  • 閂鎖(latch):準位觸發(level-sensitive)。當致能訊號為高的一整段期間,它是「透明」的——輸出會跟著輸入一起變;致能變低後,才把當下的值「鎖住」。
  • 正反器(flip-flop):邊緣觸發(edge-triggered)。它只在時脈跳變的那一瞬間(通常是上升邊緣)採樣一次輸入,其餘時間對輸入完全充耳不聞。

一個 D flip-flop 通常由兩個 latch 串接而成(主從架構,master-slave):前一個 latch 在時脈為低時接收輸入,後一個在時脈為高時放行—— 兩者交棒的結果,就是「只有在邊緣那一刻,輸入才被放行一次」。為什麼工程上偏愛邊緣觸發的 flip-flop? 因為邊緣是一個明確的時間點,整台機器可以約定「大家都在這一刻、同時更新」,時序好分析得多; 而透明的 latch 在致能期間若輸入晃動,麻煩會傳得到處都是。本書其餘地方講「暫存器」「狀態」,指的都是邊緣觸發的 flip-flop。

建立時間與保持時間(setup / hold time)

flip-flop 要能在邊緣「乾淨地採樣」,有個前提:被採樣的資料,必須在邊緣前後一小段時間內保持穩定、不准亂動。 這段「禁止變動」的視窗由兩個參數界定:

  • 建立時間(setup time,$t_{su}$):資料必須在時脈邊緣到來之前就穩定至少 $t_{su}$。
  • 保持時間(hold time,$t_{h}$):資料必須在時脈邊緣過去之後繼續維持至少 $t_{h}$,才准改變。

違反 setup 或 hold 會發生什麼?flip-flop 可能進入亞穩態(metastability):輸出卡在 0 與 1 之間的模糊地帶, 既不是乾淨的高、也不是乾淨的低,搖擺不定,而且要花不可預測的時間才(可能)塌縮到某一邊。 下游邏輯讀到這種值,有的判成 0、有的判成 1,整台機器的狀態就分裂錯亂——這是硬體裡最陰險的一種錯誤。

⚠️
setup 違反可以靠「放慢時脈」救,hold 違反不行

注意這兩種違反的本質不同。setup 違反是「資料來得太晚」——通常是關鍵路徑太長, 資料在邊緣前還沒算穩;這可以靠把時脈調慢(拉長週期)來解決,給訊號更多時間。 但 hold 違反是「資料變得太早」——在邊緣剛過、舊值還沒被穩穩鎖住時,新資料就竄了進來; 這跟時脈快慢無關(週期再長也沒用),只能靠改電路(例如在太短的路徑上故意加緩衝延遲)來修。 這也是為什麼晶片設計裡 hold 違反特別讓人頭痛。

時脈偏移(clock skew):節拍器不同步

第 20 章說「時脈是全電路的共同節拍」,理想上所有暫存器都在同一瞬間聽到那一聲滴答。 但時脈訊號也是電,得靠實體的線送到晶片各個角落,而線有長有短——於是時脈到達不同暫存器的時間會有微小差異, 這叫時脈偏移(clock skew)。偏移會吃掉原本的時序餘裕:如果接收端的時脈來得比較晚, 資料就得更早穩定(setup 更緊張);如果來得比較早,又更容易踩到 hold 違反。所以真實晶片會花大量心力設計 「時脈樹(clock tree)」,想盡辦法讓那一聲滴答盡量同時抵達每個角落。把時脈偏移納入後,前面的時序不等式會更嚴格:

$$T \;\ge\; t_{pcq} + t_{pd}(\text{關鍵路徑}) + t_{setup} + t_{skew}.$$

✏️ 練習 47.2 思考

某條管線級在兩個 D flip-flop 之間夾著一塊組合邏輯。已知:flip-flop 的 clock-to-Q 延遲 $t_{pcq}=40\text{ps}$、建立時間 $t_{setup}=20\text{ps}$; 各類閘的傳播延遲為 NOT $=10\text{ps}$、AND $=50\text{ps}$、OR $=30\text{ps}$、XOR $=60\text{ps}$。 組合邏輯裡有兩條從輸入到輸出的路徑:

  • 路徑 1:NOT → AND → OR
  • 路徑 2:XOR → AND → OR

(a) 哪一條是關鍵路徑?它的延遲多少?
(b) 這級電路的最小時脈週期與最高頻率是多少?
(c) 假設又量到時脈偏移 $t_{skew}=10\text{ps}$,最高頻率會掉到多少?

看解答

(a) 路徑 1 = $10+50+30 = 90\text{ps}$;路徑 2 = $60+50+30 = 140\text{ps}$。 路徑 2 是關鍵路徑,延遲 140 ps(整塊組合邏輯必須等最慢的它算完)。

(b) $T_{min} = t_{pcq} + t_{pd} + t_{setup} = 40 + 140 + 20 = \mathbf{200\,ps}$。 所以 $f_{max} = 1/200\text{ps} = 1/(200\times 10^{-12}\text{s}) = \mathbf{5\,GHz}$。

(c) 加入偏移:$T_{min} = 40 + 140 + 20 + 10 = 210\text{ps}$, $f_{max} = 1/210\text{ps} \approx \mathbf{4.76\,GHz}$。短短 10 ps 的偏移,就讓最高頻率掉了約 5%——這就是為什麼大廠要拼命壓低時脈偏移。

✏️ 練習 47.3 思考

用你自己的話解釋:(a) 違反 setup time 與違反 hold time 各自代表「資料相對於時脈邊緣」發生了什麼? (b) 為什麼「把時脈調慢」能救 setup 違反,卻救不了 hold 違反?(c) 「亞穩態」為什麼比「單純存錯一個值」更危險?

看解答

(a) setup 違反 = 資料來得太晚:在時脈邊緣前 $t_{su}$ 的瞬間,資料還沒穩定(通常因為關鍵路徑太長)。 hold 違反 = 資料變得太早:邊緣剛過、舊值還沒被穩穩鎖進去,新資料就竄進來把它覆蓋了(通常因為某條路徑太短、或時脈偏移)。

(b) 把時脈調慢 = 拉長週期 $T$。setup 的問題是「資料在下一個邊緣前來不及穩定」,週期拉長就給了訊號更多時間,自然解決。 但 hold 看的是「同一個邊緣之後資料有沒有撐住 $t_h$」——這完全發生在邊緣的瞬間附近,跟下一拍多久才來毫無關係,所以拉長週期一點忙也幫不上;只能改電路(例如在過快的路徑上加延遲緩衝)。

(c) 存錯一個值,至少它是個確定的 0 或 1,後續行為雖然錯但一致、可重現、可除錯。 亞穩態則是輸出卡在 0 與 1 之間的模糊電壓,不同的下游閘可能把它判讀成不同的值(有的看成 0、有的看成 1), 於是系統狀態當場分裂;而且它要多久才塌縮到某一邊是機率性、不可預測的。一個本該同步的系統,從此各說各話——這種「薛丁格的位元」遠比一個明確的錯誤難抓。

有限狀態機:讓硬體「依狀態行動」

現在,所有零件都到齊了:會算的組合邏輯(還能用卡諾圖化簡)、會記的暫存器(由 flip-flop 組成)、 當節拍器的時脈。把它們組合起來,就能造出這一章的重頭戲,也是所有循序數位系統的通用模型—— 有限狀態機(finite state machine,FSM)。

FSM 的概念其實很生活化。想想一個十字路口的紅綠燈:它有幾個狀態(綠燈、黃燈、紅燈), 會依某些條件(時間到了、或偵測到車)轉移到下一個狀態,並在每個狀態做出對應的輸出(亮哪個燈)。 一台自動販賣機、一個轉門、一個電梯控制器、甚至你 App 裡的登入流程,本質上都是 FSM。

🔑
FSM 的四個要素

一台有限狀態機由四樣東西定義:
① 一組有限的狀態(states):機器「記得」自己現在處於哪一種情境。
② 一個起始狀態(initial state):開機時從哪裡開始。
③ 轉移函數(transition): 給定「現在的狀態 + 現在的輸入」,決定「下一拍的狀態(次態)」。
④ 輸出函數(output): 決定每個情境下機器對外吐出什麼。
「有限」二字是重點:狀態的數目是固定、有限的——這正是它能用固定數量的 flip-flop 實作出來的原因。

Moore 機 vs Mealy 機

依「輸出取決於什麼」,FSM 分成兩種風格,值得分清楚:

  • Moore 機:輸出只取決於當前狀態。輸出寫在狀態(節點)上。好處是輸出穩定、不受輸入瞬間晃動影響;代價是有時需要多一些狀態。
  • Mealy 機:輸出取決於當前狀態 + 當前輸入。輸出寫在轉移(箭頭)上。好處是常常用更少的狀態就能達成同樣行為,反應也快一拍;代價是輸出可能跟著輸入毛刺、時序較難分析。

兩者沒有絕對好壞,是工程取捨。下面我們用同一個任務,先做 Moore 版,再(在練習裡)做 Mealy 版,你就能體會差異。

實例:偵測序列 "101"

任務:一個位元一個位元地讀入一串 0/1,每當最近讀進的三個位元剛好是 1 0 1 時,就輸出 1(其餘時候輸出 0)。 而且要支援重疊:像 10101 裡有兩個 "101"(位置 1–3 與 3–5),兩個都要抓到。

我們用 Moore 機,設計四個狀態,每個狀態代表「目前已經吻合到 '101' 的多長前綴」:

  • S0:還沒有任何有用的前綴(起始)。
  • S1:剛看到 "1"。
  • S2:剛看到 "10"。
  • S3:剛看到 "101" → 偵測成功!(Moore:這個狀態的輸出 = 1)

把狀態圖整理成精確的狀態轉移表(state transition table)——這是真正拿去做電路的依據:

現態含意輸入 = 0 → 次態輸入 = 1 → 次態輸出(Moore)
S0起始 / 無前綴S0S10
S1看到 "1"S2S10
S2看到 "10"S0S30
S3看到 "101"S2S11

怎麼把 FSM 變成電路

這是把第 20 章的「算」與「記」正式合體的時刻。任何 FSM 都用同一個通用架構實作,只有三個方塊:

  1. 一個狀態暫存器(state register)(由 flip-flop 組成),用來「記住現態」。$n$ 個狀態就用 $\lceil\log_2 n\rceil$ 個 flip-flop 把狀態編碼成二進位。
  2. 一塊次態邏輯(next-state logic):純組合邏輯,吃「現態 + 輸入」,算出「次態」。
  3. 一塊輸出邏輯(output logic):純組合邏輯,算出輸出(Moore 只看現態;Mealy 還看輸入)。

把狀態編碼成 2 個位元($Q_1Q_0$):S0 = 00、S1 = 01、S2 = 10、S3 = 11。代進轉移表,次態邏輯與輸出邏輯就變成 以 $Q_1, Q_0, x$(輸入)為變數的布林函數——而你已經會化簡布林函數了:用卡諾圖。 例如 Moore 輸出在這個編碼下特別漂亮:只有 S3(11)輸出 1,所以 輸出 $= Q_1 \cdot Q_0$,一個 AND 就搞定。 次態的兩個位元 $Q_1', Q_0'$ 各是 $(Q_1,Q_0,x)$ 的布林函數,把它們填成卡諾圖、圈一圈,就得到要接的閘。 FSM 設計,到頭來就是「畫狀態圖 → 列轉移表 → 編碼 → 用卡諾圖化簡次態/輸出邏輯 → 接上狀態暫存器」這條流水線。

📜
FSM 就是 DFA——只是這次它是硬體

如果你覺得這個「狀態 + 轉移 + 輸入」的模型有點眼熟,那是因為它和計算理論裡的 確定有限自動機(deterministic finite automaton,DFA)是同一個東西。 DFA 是計算理論用來定義「正規語言(regular language)」的抽象機器,我們會在第十二部第 49 章正式研究它; 它也正是你寫正規表示式(regex)時,引擎底層在跑的東西。換句話說,本章畫的 "101" 偵測器, 就是 regex .*101 的硬體化身。同一個數學模型,一邊是純理論的自動機,一邊是矽晶片上真實的電路—— 這正是電腦科學最迷人的地方:抽象與實體,常常是同一枚硬幣的兩面。

用 Python 模擬這台 FSM

在動手接電路前,先用 Python 把這台狀態機跑一遍驗證邏輯,是工程上的好習慣。最直接的寫法,就是把 狀態轉移表原封不動翻成一個字典——你會發現程式碼幾乎就是表格本身: 

# 把「狀態轉移表」直接寫成資料:transitions[現態][輸入] = 次態
transitions = {
    'S0': {'0': 'S0', '1': 'S1'},
    'S1': {'0': 'S2', '1': 'S1'},
    'S2': {'0': 'S0', '1': 'S3'},
    'S3': {'0': 'S2', '1': 'S1'},
}
output = {'S0': 0, 'S1': 0, 'S2': 0, 'S3': 1}   # Moore:輸出只看現態

def run_fsm(bits):
    state = 'S0'                       # 起始狀態
    hits = []
    for i, b in enumerate(bits):
        state = transitions[state][b] # 時脈邊緣:依(現態,輸入)更新狀態
        if output[state] == 1:        # Moore 輸出:讀新的現態
            hits.append(i)            # 在讀完第 i 個位元後,偵測到 "101"
    return hits

stream = "1101011010101"
print(run_fsm(stream))   # [3, 5, 8, 10, 12] ← 每個索引都對應一個(可重疊的)"101"

輸出 [3, 5, 8, 10, 12] 告訴我們:在讀完第 3、5、8、10、12 個位元時各偵測到一次 "101"。 你可以自己對著字串 1101011010101 數一數,連重疊的那幾個都漂亮地抓到了——因為 S3 的轉移特意設計成「把最後的 '1' 留作下一輪的開頭」。

再往前一步:把它寫成一個通用的 FSM 引擎,任何 Moore 機都能丟進去跑。這其實就是圖 47-9 那個架構的軟體版:

class MooreFSM:
    """通用 Moore 有限狀態機:對應圖 47-9 的硬體架構"""
    def __init__(self, transitions, output, start):
        self.transitions = transitions   # 次態邏輯(查表)
        self.output = output             # 輸出邏輯(查表)
        self.state = start               # 狀態暫存器

    def tick(self, inp):                 # 等同一個時脈邊緣
        self.state = self.transitions[self.state][inp]   # 算次態並鎖存
        return self.output[self.state]   # 回傳這一拍的輸出

fsm = MooreFSM(transitions, output, 'S0')
result = [fsm.tick(b) for b in "10101"]
print(result)         # [0, 0, 1, 0, 1] ← "10101" 中兩個重疊的 101 都被抓到

從 FSM 到「資料路徑 + 控制」

"101" 偵測器很小,狀態少、輸出簡單。但真實系統(一台洗衣機、一顆 CPU)的「資料」動輒幾十上百個位元, 運算也複雜得多。難道要把每種情況都塞進一張巨大的狀態表嗎?那會爆炸。工程上的解法,是把系統拆成兩半:

  • 資料路徑(datapath):負責實際的計算與搬運。它由第 20 章那些零件組成——ALU、暫存器、多工器、匯流排。它力氣大、會算會存,但自己不知道「現在該做哪一步」。
  • 控制(control):一台FSM,負責指揮。它依當前狀態(與資料路徑回報的狀態旗標,例如「結果是不是 0」),每一拍送出一組控制訊號,告訴資料路徑:「這一拍,ALU 做減法、把結果寫進 R2、多工器選這條線。」

這個「資料路徑 + 控制」的拆分,是所有複雜數位系統的組織原則——而它最重要的應用,你其實已經見過了:

🔑
這正是 CPU 的組織方式

第 20、21 章那台 CPU,骨子裡就是這個架構。資料路徑是 ALU + 暫存器堆 + RAM + PC; 而那個指揮一切的控制單元(control unit)——第 21 章說它負責「取指令 → 解碼 → 執行」—— 本質上就是一台 FSM:它的狀態大致是「取指令」「解碼」「執行」「寫回」這些步驟, 每一拍依當前步驟與指令內容,送出一組控制訊號驅動資料路徑。 你之所以覺得 CPU 神奇,是因為它把「資料路徑會算」和「FSM 會依序指揮」這兩件你現在都懂的事, 合在一起放大到極致。我們會在第 48 章(計算機結構)把這台機器的組織與效能再往下挖一層。

✏️ 練習 47.4 實作

把本章的 "101" 偵測器改寫成 Mealy 機。Mealy 的輸出寫在轉移上(看現態 + 當下輸入)。 請:(a) 設計狀態(提示:只需要 3 個),畫出狀態圖的轉移(用「輸入 / 輸出」標在箭頭上); (b) 列出 Mealy 的轉移表;(c) 說明為什麼 Mealy 版能比 Moore 版少一個狀態。

看解答

(a) 三個狀態: A(起始)、B(看到 "1")、C(看到 "10")。「偵測成功」不再需要一個獨立狀態, 而是發生在「現態 C、讀到輸入 1」的那條轉移上(此時輸出 1,並回到 B,因為最後讀到的 '1' 可當下一個 "101" 的開頭)。

(b) Mealy 轉移表(格式:次態 / 輸出):

現態含意輸入 = 0(次態 / 輸出)輸入 = 1(次態 / 輸出)
A起始 / 無前綴A / 0B / 0
B看到 "1"C / 0B / 0
C看到 "10"A / 0B / 1

那條粗體的 C --1/1--> B 就是「偵測到 101」的瞬間:輸出 1。

(c) 為什麼少一個狀態: Moore 機因為「輸出只能看現態」,必須專門騰出一個狀態 S3 來代表「剛剛達成 101,該輸出 1」。 Mealy 機允許「輸出看現態 + 輸入」,於是可以在 C 讀到 1 的轉移上直接吐出 1, 不必為這個結果單獨設一個狀態——省下的正是 S3。代價是 Mealy 的輸出和輸入綁在一起,輸入一抖,輸出可能跟著抖(毛刺),時序較難分析。用 Python 驗證 Mealy 版:

mealy = {                       # mealy[現態][輸入] = (次態, 輸出)
    'A': {'0': ('A', 0), '1': ('B', 0)},
    'B': {'0': ('C', 0), '1': ('B', 0)},
    'C': {'0': ('A', 0), '1': ('B', 1)},
}
def run_mealy(bits):
    state, outs = 'A', []
    for b in bits:
        state, y = mealy[state][b]   # 輸出在轉移當下就決定
        outs.append(y)
    return outs

print(run_mealy("10101"))   # [0, 0, 1, 0, 1] ← 與 Moore 版抓到相同的兩個重疊 101
✏️ 練習 47.5 挑戰

設計並用 Python 實作一個交通號誌控制器的 FSM(這是「控制 FSM」的經典範例)。 規格:燈號循環 綠 → 黃 → 紅 → 綠 …,綠燈維持 3 拍、黃燈 1 拍、紅燈 2 拍。 請用「狀態 + 計時」的方式設計,讓 tick() 每呼叫一次代表一拍,回傳當前燈號。跑 12 拍驗證循環正確。

看解答

關鍵在於:除了「哪個燈」這個狀態,還需要一個計數器記住「在這個燈待了幾拍」—— 這正是第 20 章的計數器在 FSM 裡的典型用途。時間到了才轉移到下一個燈,並把計數歸零。

class TrafficLight:
    # 每個狀態:(燈號, 要維持幾拍, 下一個狀態)
    PLAN = {
        'GREEN':  ('綠', 3, 'YELLOW'),
        'YELLOW': ('黃', 1, 'RED'),
        'RED':    ('紅', 2, 'GREEN'),
    }
    def __init__(self):
        self.state = 'GREEN'
        self.elapsed = 0                 # 在當前狀態已待的拍數

    def tick(self):
        light, hold, nxt = self.PLAN[self.state]
        self.elapsed += 1
        if self.elapsed >= hold:         # 時間到 → 轉移到下一個燈
            self.state = nxt
            self.elapsed = 0
        return light

tl = TrafficLight()
print([tl.tick() for _ in range(12)])
# ['綠','綠','綠','黃','紅','紅','綠','綠','綠','黃','紅','紅'] ← 3 綠 1 黃 2 紅,正確循環

進階:把「行人按鈕」當成一個輸入加進來(按下時,綠燈提早結束),這台機器就從 Moore 機(輸出只看狀態)悄悄多了一點 Mealy 的味道(轉移看輸入)。你也可以把它畫成狀態圖,體會「控制 FSM 指揮真實裝置」的感覺。

✏️ 練習 47.6 挑戰

回到 CMOS。圖 47-3 展示了用 4 顆電晶體做 NAND(下拉 NMOS 串聯、上拉 PMOS 並聯)。 請設計一個 2 輸入的 NOR 閘的 CMOS 接法:下拉網路與上拉網路各該怎麼接(串聯還是並聯)?為什麼?

看解答

NOR $= \neg(A + B)$:只有當 $A$、$B$ 都為 0 時才輸出 1,只要有一個為 1 就輸出 0。順著「下拉網路決定何時輸出 0」來想:

  • 下拉網路(NMOS):兩顆並聯。 只要 $A$ 或 $B$ 任一個為 1,對應的 NMOS 就導通,把輸出拉到 0——這正是 OR 的「至少一個」。
  • 上拉網路(PMOS):兩顆串聯。 它必須是下拉的互補:只有當 $A$、$B$ 都為 0 時,兩顆 PMOS(輸入 0 才導通)才一起導通,把輸出拉到 1。

對照一下就看出漂亮的對偶:NAND 是「下拉串聯 / 上拉並聯」, NOR 恰好相反,是「下拉並聯 / 上拉串聯」。 兩者都只要 4 顆電晶體,而且天生帶反相——這再次印證了:在 CMOS 裡,NAND 與 NOR 是最「便宜」的原生閘, 這跟它們在邏輯上「萬能」(第 19 章)互為表裡。串聯對應 AND/「都要」,並聯對應 OR/「至少一個」, 而上拉永遠是下拉的對偶——這條規律,骨子裡就是 De Morgan 定律。

把這一章串起來

我們從第 19、20 章那個乾淨的「理想閘」世界出發,掀開蓋子,看見了底下真實的電子工程, 最後又爬回抽象,抵達了通用的有限狀態機。回顧這趟旅程的路標:

  • 數位抽象:0/1 其實是兩段電壓區間。靠四個門檻撐出雜訊邊際、靠靜態規範讓每個閘都把訊號還原——雜訊因此不累積,數位電路才能堆疊到幾百億顆電晶體仍可靠。這是「為何用 0/1」最深刻的答案。
  • CMOS:電晶體是電壓控制的開關;NMOS 下拉、PMOS 上拉,互補配對。反相器只要 2 顆、NAND 只要 4 顆電晶體。因為互補,穩態幾乎不耗電——這是 CMOS 的省電祕密。而「下拉串聯/上拉並聯」的對偶,正是 De Morgan 長在矽上;NAND/NOR 天生最便宜,呼應了第 19 章的 NAND 萬能。
  • 卡諾圖:把布林化簡從「拼代數技巧」變成「在方格上圈相鄰的 1」。圈越大、消去越多變數,電路越省。它只是把合併律 $XY+X\neg Y=X$ 畫出來。
  • 時間維度:真實的閘有傳播延遲,會產生毛刺;最慢的關鍵路徑(加上 $t_{pcq}+t_{setup}+t_{skew}$)決定了時脈週期下限,也就是為什麼電路有速度上限。
  • 循序邏輯細節:latch 是準位觸發、flip-flop 是邊緣觸發;資料必須在邊緣前後的 setup/hold 視窗內穩定,否則會掉進亞穩態。setup 違反可靠放慢時脈救,hold 違反不行。時脈偏移還會偷走時序餘裕。
  • 有限狀態機:狀態 + 轉移 + 輸入/輸出。Moore(輸出看狀態)vs Mealy(輸出看狀態+輸入)。任何 FSM 都用「狀態暫存器 + 次態邏輯 + 輸出邏輯」這個通用架構實作——也就是第 20 章「算 + 記」的合體。FSM 就是 DFA 的硬體版(第 49 章)。
  • 資料路徑 + 控制:複雜系統 = 會算的資料路徑 + 指揮的控制 FSM。這正是 CPU 的組織原則(第 21 章控制單元),也是第 48 章的起點。

最值得帶走的一句直覺是:抽象與物理,是同一件事的兩面。「0 與 1」是電壓,「閘」是電晶體,「立刻」其實要花皮秒, 而紙上的「狀態機」在矽晶片裡是真實會滴答跳動的 flip-flop 與閘。當你能在這兩個世界之間自由切換—— 往上用抽象蓋大樓,往下用物理理解它為什麼站得住——你就真正讀懂了「數位」這兩個字。
