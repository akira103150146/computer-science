一個自動分類垃圾的工廠

想像你接了一個案子:某資源回收廠想裝一台相機,讓輸送帶上的東西一經過,就自動判斷它是塑膠、紙、金屬還是玻璃, 好把它撥到對的桶子。你讀過第 38 章,信心滿滿:這不就是個分類問題嗎?蒐集資料、訓練一個分類器就行。

問題是——機器學習的分類器(像 SVM、邏輯迴歸)只吃一串數字(特徵向量),它不吃「一張圖」。 一張 200×200 的彩色照片,攤平就是 200×200×3 = 120000 個像素值。你能直接把這十二萬個數字丟進 SVM 嗎? 可以,但效果會慘不忍睹:像素值對「亮一點、移一格、轉個角度」極度敏感,同一個寶特瓶換個位置,整串數字就面目全非。 分類器根本學不到「寶特瓶之所以是寶特瓶」的本質。

所以在深度學習之前,電腦視覺的核心功夫,是怎麼把一張圖,變成一串「聰明的數字」—— 一串能濃縮形狀、紋理、結構,又不被光照與位移干擾的特徵。這件事,過去由人來設計; 而這一章的主角,是讓機器自己學會這件事。我們先看人怎麼做(也是第 66 章傳統視覺的收尾),再看機器怎麼接手。

從手工特徵到自動學特徵

傳統範式:特徵 + 分類器

深度學習之前,影像辨識走的是一條清楚的兩段式流水線:

最經典的手工特徵之一是 HOG(方向梯度直方圖,Histogram of Oriented Gradients): 把影像切成許多小格,每格統計「邊緣朝哪些方向、有多強」。它對形狀敏感、又對光照變化穩定, 是行人偵測的傳奇特徵。在 OpenCV 裡,連訓練好的行人偵測器都內建了,幾行就能跑: 

import cv2

hog = cv2.HOGDescriptor()
# 載入 OpenCV 內建、預訓練好的「行人」SVM 偵測器
hog.setSVMDetector(cv2.HOGDescriptor_getDefaultPeopleDetector())
rects, weights = hog.detectMultiScale(img)   # 多尺度掃描整張圖找人形
# rects:框出來的人;weights:每個框的信心分數

這背後是一個SVM(支援向量機),正是第 38 章介紹過的那位「找最寬的街」的分類器。 如果要自己訓練,流程就是第 38 章那套監督式學習的標準骨架——切資料、fit 學、score 評: 

from sklearn.svm import SVC
from sklearn.model_selection import train_test_split

# X:每列是一張圖的「特徵向量」(如 HOG);y:每張圖的正確類別
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, random_state=0)
clf = SVC(kernel='rbf').fit(X_tr, y_tr)   # rbf 核能畫彎曲決策邊界
print(clf.score(X_te, y_te))              # 用「沒看過」的測試集評估
⚠️
第 38 章的鐵律,在影像上一樣成立

測試集只能在最後用來評估,絕不能拿去訓練或調參,否則就是資料洩漏(data leakage),分數會虛高、上線崩盤。 另外,SVM 對特徵尺度很敏感,記得先做特徵縮放(標準化)。這些第 38 章講透的觀念,在影像辨識裡一字不差地適用—— 因為影像辨識,骨子裡就是機器學習。

傳統範式的天花板

HOG+SVM 在「形狀單純、背景乾淨、類別不多」的場景(行人、手寫數字)表現很好。但它有兩道難以跨越的牆:

  • 特徵是人想的,人的想像力有限。HOG 抓得到形狀邊緣,但「貓的耳朵」「狗的鼻子」「金屬的反光質感」這些更高層、更抽象的概念,你要怎麼用一條公式設計出來?越複雜的辨識任務,越沒有人寫得出對應的特徵。
  • 每換一個任務,就要重想一套特徵。辨識行人用 HOG、辨識材質可能要換 LBP、辨識顏色又要色彩直方圖……特徵工程是一門又貴又不通用的手藝。

2012 年是個分水嶺。在影像辨識競賽 ImageNet 上,一個叫 AlexNet 的深度卷積網路, 把錯誤率從前一年的 26% 一舉砍到 16%,把所有手工特徵的方法遠遠甩在後頭。它做對了一件事: 不再由人設計特徵,而是讓網路從資料中,自己把「該抽什麼特徵」也一起學出來。這就是本章主角——CNN——的登場。

🔑
深度學習的核心翻轉:端到端

傳統做法是「人抽特徵 → 機器學分類」兩段分離;深度學習是「端到端(end-to-end)」: 從原始像素直接到最終答案,中間「抽什麼特徵」也是學出來的。換句話說,第 65、66 章那種「人手工設計的核」, 在 CNN 裡變成了「由訓練自動學出來的核」。這一個轉變,釋放了人類想像力的天花板。

CNN:為什麼卷積天生適合影像

終於來到重頭戲。卷積神經網路(convolutional neural network,CNN)是這整條影像辨識線的心臟。 它其實就是第 38 章講的神經網路,只是換上了一種特別適合影像的「零件」——卷積層。 要懂它,得先問:把一張圖丟進普通的全連接神經網路,有什麼不對?

全連接網路為什麼不適合影像

假設一張 200×200 的彩色圖,攤平成 12 萬個輸入,接到一層只有 1000 個神經元的全連接層。 光這一層就需要 $120000 \times 1000 = 1.2$ 億個權重——還沒算後面的層。這有三個致命問題:

  • 參數爆炸:上億個權重,記憶體吃不消、也極易過擬合。
  • 丟掉了空間結構:一攤平,「相鄰像素」這個關鍵資訊就沒了。可是影像的意義,正藏在「鄰近像素如何組成邊緣、邊緣如何組成形狀」裡。
  • 不耐位移:同一隻貓往右挪 10 個像素,對全連接層而言輸入完全變了,它得「重新學一次」貓在右邊長什麼樣。

CNN 用三個漂亮的點子,一次解決這三個問題:局部連接、參數共享、層層抽象。 而這三個點子,全都濃縮在一個你已經認識的操作裡——卷積。

卷積層:把第 65 章的「核」變成可以學的

在第 65 章,你已經學過卷積:拿一個小小的核(kernel,又叫濾鏡 filter), 在影像上一格一格滑動,每滑到一處,就把核與底下那塊像素「對應相乘再相加」,得到一個輸出值。 當時那些核是人設計的:Sobel 核抓水平/垂直邊緣、高斯核做模糊。

CNN 的卷積層做的完全是同一件事——只有一個革命性的差別:核裡的那 9 個(3×3)數字,不再由人填,而是當成「參數」,由訓練自動學出來。 一個卷積層通常有很多個核(例如 16 個),每個核掃過整張圖,各自產生一張特徵圖(feature map), 標示出「我這個核負責偵測的東西,在圖上哪些地方出現了」。

三個點子,各對應一個好處:

卷積帶來的是什麼解決了全連接的什麼問題
局部連接每個輸出只看一小塊(感受野),不是整張圖保住空間結構、大幅減少連線
參數共享同一個核走遍全圖,所有位置共用那 9 個權重參數從上億降到幾百;且天生耐位移
層層堆疊一層的特徵圖,當成下一層的輸入再卷積讓網路從簡單特徵,組合出複雜概念

層層抽象:邊緣 → 紋理 → 部件 → 物件

最迷人的是第三點。當卷積層一層疊一層,網路學到的特徵會越來越抽象, 這不是工程師硬編的,而是訓練自然「長」出來的層級——這也正是人們研究 CNN 內部時真實觀察到的現象:

池化:抓重點、縮尺寸、更耐位移

卷積層之間,常夾一個池化層(pooling)。最常用的是最大池化(max pooling): 把特徵圖切成 2×2 的小塊,每塊只留最大值。效果有三:把特徵圖長寬各砍一半(降維、省算力)、 保留「這附近最強的反應」、以及讓特徵更抗位移(物件晃一點點,最大值多半還在同一塊裡)。

把這些零件接起來,一個典型的 CNN 就是:卷積 → 激活(ReLU)→ 池化 反覆幾輪, 把空間尺寸越縮越小、特徵通道越疊越多(越深越抽象),最後攤平(flatten)成一條向量, 接上第 38 章那種全連接層,輸出「屬於各類別的分數」:

用 PyTorch 從零定義並訓練一個 CNN

理論到此,動手。我們用 PyTorch 把上圖那個小 CNN 寫出來。 PyTorch 的網路寫法有固定套路:繼承 nn.Module,在 __init__ 裡宣告會用到的層(像備好積木), 在 forward 裡定義資料怎麼一層層流過: 

import torch
import torch.nn as nn
import torch.nn.functional as F

class Net(nn.Module):
    """小型 CNN:兩層卷積 + 一層全連接。"""
    def __init__(self, n_cls):
        super().__init__()                          # 一定要先呼叫父類別建構子
        self.c1 = nn.Conv2d(3, 16, 3, padding=1)    # 輸入3通道→16張特徵圖,3×3核
        self.c2 = nn.Conv2d(16, 32, 3, padding=1)   # 16→32 張特徵圖
        self.fc = nn.Linear(32 * 8 * 8, n_cls)      # 攤平後接到「各類別分數」

    def forward(self, x):                           # 定義資料怎麼一層層流過
        x = F.max_pool2d(F.relu(self.c1(x)), 2)     # 卷積→ReLU→池化:32→16
        x = F.max_pool2d(F.relu(self.c2(x)), 2)     # 再一次:16→8
        return self.fc(x.flatten(1))                # 攤平成向量 → 全連接輸出

nn.Conv2d(3, 16, 3, padding=1) 就是圖 67-2 那個卷積層:吃 3 個輸入通道、產生 16 張特徵圖、用 3×3 的核; padding=1 是在邊緣補一圈零,讓輸出長寬和輸入一樣。F.relu 是激活函數(把負值砍成 0,注入非線性, 否則疊再多層都還是線性、白搭——這點第 38 章講過)。F.max_pool2d(..., 2) 就是 2×2 最大池化。

網路備好,接著是訓練迴圈。這是整個深度學習最核心的一段,而它的本質,你在第 38 章早就懂了: 前向算預測與損失 → 反向算梯度 → 梯度下降更新權重。在 PyTorch 裡是固定的「四步」: 

model = Net(n_cls=10)
opt   = torch.optim.Adam(model.parameters(), lr=1e-3)  # 最佳化器,1e-3 是學習率
lossf = nn.CrossEntropyLoss()                          # 分類用的交叉熵損失(第 38 章)

for epoch in range(10):                 # 把整份資料學 10 輪(10 個 epoch)
    for x, y in loader:                 # 每次拿一批(batch)影像 x 與標籤 y
        opt.zero_grad()                 # ① 清掉上一輪的梯度(忘了會累加、訓練爆掉!)
        pred = model(x)                 # ② 前向:算出預測
        loss = lossf(pred, y)           #    算「錯多少」
        loss.backward()                 # ③ 反向傳播:算出每個權重的梯度
        opt.step()                      # ④ 依梯度更新權重(往下坡走一步)
🔑
loss.backward() 背後,就是你學過的鏈鎖法則

第 38 章說過:反向傳播 = 鏈鎖法則 + 梯度下降。CNN 不過是把卷積也納入這條合成函數鏈。 你寫 loss.backward() 時,PyTorch 的自動微分(autograd)就沿著整個網路,由後往前把梯度逐層相乘、分配回每一個卷積核的每個權重—— 這正是第 12 章的鏈鎖法則,只是規模放大到百萬個參數、而且自動化了。你早就懂這個引擎,現在只是換了輛更大的車。

過擬合,還是老問題

CNN 參數雖比全連接少很多,但動輒幾百萬,過擬合(overfitting)依然是頭號敵人—— 症狀就是第 38 章那個:訓練準確率很高、測試/驗證準確率卻很差,模型在「背題庫」而非「學規律」。 對付它的武器,有些是 CNN 特有的,有些是第 38 章的老朋友:

  • 蒐集更多資料(最有效,但最貴)、以及下一節的資料增強(用變化「變」出更多資料)。
  • Dropout:訓練時隨機「關掉」一部分神經元,逼網路別過度依賴某幾條路,效果像在做集成。
  • 權重衰減(weight decay):其實就是第 38 章的 L2 正則化,懲罰過大的權重。
  • 提早停止(early stopping):盯著驗證誤差,一旦它開始回升,就停止訓練。

遷移學習:站在巨人的肩膀上

現在你會訓練 CNN 了。但真實世界有個殘酷現實:工廠只蒐得到 200 張瑕疵照、農場只有 5 種病蟲害各幾十張。 用這麼一點資料從零訓練一個幾百萬參數的 CNN,結果一定是慘烈的過擬合。難道沒有大數據就不能做了嗎?

有救,而且這一招是業界落地最關鍵的一手:遷移學習(transfer learning)。

為什麼有效:特徵是通用的

回想圖 67-3:CNN 的淺、中層學的是邊緣、紋理、形狀——這些東西,辨識貓、辨識瑕疵、辨識病蟲害都用得到, 是通用的視覺基本功。一個在 ImageNet(上百萬張、一千類)上訓練好的模型(如 ResNet), 早就把這套基本功練得爐火純青。我們何必從頭再練一次?

遷移學習的做法是:借用預訓練模型的「眼睛」(前面所有層),只把最後負責「下判斷」的分類層換成自己的。 因為前面的通用特徵直接重用,我們的任務只剩「重新組合這些特徵」,所以幾百張圖就夠了。 

import torch.nn as nn
from torchvision import models

m = models.resnet18(weights='IMAGENET1K_V1')      # 載入 ImageNet 預訓練的 ResNet18
for p in m.parameters():
    p.requires_grad = False                        # 凍結骨幹:這些權重訓練時不更新

# 把最後的分類層換成自己的(輸出改成你的類別數);新層預設 requires_grad=True
m.fc = nn.Linear(m.fc.in_features, num_classes)
# 於是只有 m.fc 會被訓練。之後可再解凍上層、用更小 lr 做第二階段微調
⚠️
兩個遷移學習翻車點

(1) 正規化要對齊:預訓練模型當初是用特定的平均/標準差(mean/std)正規化輸入的,你餵資料時要用同一組數值,否則特徵會錯位。 (2) 別一開始就解凍全網路 + 大學習率:那會在模型還沒站穩時,用你的小資料把辛苦學來的通用特徵「洗掉」,反而更差。先凍結、後微調,順序很重要。

資料增強:用變化「變」出更多資料

資料少還有一招:資料增強(data augmentation)。每次訓練時,隨機把圖裁切、翻轉、調色一下, 等於讓模型看到「同一張圖的許多變體」,逼它學「本質」而非死背某張照片的細節——泛化能力因此提升。 

from torchvision import transforms

train_aug = transforms.Compose([          # 把多個處理串成一條流水線
    transforms.RandomResizedCrop(224),    # 隨機裁一塊再縮到 224×224
    transforms.RandomHorizontalFlip(),    # 隨機左右翻
    transforms.ColorJitter(0.2, 0.2, 0.2),# 隨機微調亮度/對比/飽和度
    transforms.ToTensor(),                # 轉成 PyTorch 張量
])
⛔
資料增強只給訓練集!驗證/測試集不能增強

這是第 38 章「資料洩漏」鐵律在影像上的具體化身。只有訓練集做隨機增強;驗證/測試集只做 resize 與正規化。 否則你的評估會被隨機變動汙染、失真。更隱蔽的洩漏:切分資料時,同一個物件(或同一段影片的相鄰幀)別跨集—— 否則訓練集和測試集藏著幾乎一樣的圖,測試分數會虛高得離譜,上線立刻現形。把測試集鎖進保險箱,從頭到尾只用一次。

物件偵測:不只「是什麼」,還要「在哪裡」

到目前為止,我們做的都是分類:給整張圖一個標籤。但回收廠的輸送帶上,常常一次有好幾樣東西; 自駕車要同時知道畫面裡每一個人、車的位置。這需要更強的能力:物件偵測(object detection)—— 同時回答「在哪裡」與「是什麼」。先把三種任務的差別講清楚:

YOLO 的核心思想:You Only Look Once

偵測的早期做法(如 R-CNN)是「兩階段」:先用一個方法提議幾千個「可能有東西」的候選框,再逐個丟進分類器判斷——慢得無法即時。 YOLO(You Only Look Once)的革命性,在於把偵測變成「一次前向就搞定」的單階段問題,快到能即時:

它的點子很巧:把圖切成一個格網(grid),讓每一格同時負責預測「以我為中心,有沒有物件、框多大、是什麼類別」。 整張圖只看一次(一次卷積前向),所有格子的預測同時吐出來。這就是名字的由來——你只要看一次。

用 Ultralytics 的 YOLO,推論簡單到不可思議——先體驗成果,再回頭理解指標: 

from ultralytics import YOLO

model = YOLO('yolov8n.pt')      # 載入預訓練模型(COCO 80 類);n=最小最快版,首次自動下載
results = model('street.jpg')   # 丟圖片(也可丟資料夾/影片/網址/攝影機)
results[0].show()               # 顯示:框 + 類別 + 信心
for box in results[0].boxes:    # 取出每個框的細節
    print(box.cls, box.conf, box.xyxy)   # 類別、信心分數、座標(左上、右下)

偵測的三個關鍵指標:IoU、NMS、mAP

要評斷偵測好不好,有三個詞你會天天遇到。先講最基礎的 IoU(交並比,Intersection over Union): 衡量「預測框」和「正解框」貼得多準——就是兩框的交集面積 ÷ 聯集面積,值在 0 到 1,越高越貼。 慣例是 IoU ≥ 0.5 才算這次「命中」。

接著兩個:

  • NMS(非極大值抑制,Non-Maximum Suppression):同一個物件常被框出好幾個重疊的框。NMS 的工作是留下信心最高的那個框,把跟它重疊太多(IoU 大)的其餘框刪掉,讓一個物件最後只剩一個乾淨的框。
  • mAP(mean Average Precision,平均精度均值):偵測的「總成績單」。對每一類算 AP(精確率-召回率曲線下的面積,呼應第 38 章的 precision/recall),再對所有類別取平均。看 mAP 高低判斷模型好壞,比只數「框對幾個」客觀得多。

要訓練自己的偵測器(例如偵測安全帽、口罩),其實也是遷移學習:從預訓練的 yolov8n.pt 出發, 用你標註好的資料繼續學,幾百張框就能有不錯結果: 

model = YOLO('yolov8n.pt')                          # 從預訓練權重出發(遷移學習)
model.train(data='data.yaml', epochs=100, imgsz=640)  # data.yaml 指定資料路徑與類別名
metrics = model.val()                              # 驗證,看 mAP
print('mAP50-95:', metrics.box.map)
💡
即時偵測,就是第 65 章那個迴圈換個核心

把偵測接上攝影機就是即時應用——本質還是「讀一幀 → 處理 → 顯示」的迴圈,只是「處理」換成 YOLO:

import cv2
for r in model.predict(source=0, stream=True):  # source=0 攝影機;stream 逐幀吐結果
    cv2.imshow('det', r.plot())                 # r.plot() 把框畫上去
    if cv2.waitKey(1) & 0xFF == ord('q'):       # 按 q 離開
        break

「業務邏輯」就加在這迴圈裡:例如「偵測到未戴安全帽就畫紅框並計數」。資料品質比 epochs 更重要——標註鬆散、漏標,mAP 怎麼都上不去。

影像分割:精確到每一個像素

偵測給的是方框,但方框是粗的:一個框裡有一大半是背景。自駕車要知道「哪些像素是路面、哪些是行人」、 醫療影像要量腫瘤的面積、修圖要做無綠幕去背——這些都需要像素級的影像分割(image segmentation)。

語意分割 vs 實例分割

  • 語意分割(semantic segmentation):只管「每個像素屬於哪一類」,同類不分個體——畫面裡所有的車都標成同一種「車」色。
  • 實例分割(instance segmentation):更進一步,同類也要分出個體——車1、車2、車3 各有各的輪廓。

分割不能用準確率(accuracy)評估:因為背景像素通常占絕大多數,一個「全部猜背景」的爛模型, 準確率也能很高卻毫無用處。要改用 IoU 或 Dice 係數(都是衡量「預測遮罩與正解遮罩重疊多少」)—— 這又是第 38 章「類別不平衡時準確率會騙人」的翻版。

U-Net:先壓縮理解,再還原定位

語意分割的經典架構是 U-Net,因形狀像字母 U 而得名。它的直覺是個漂亮的兩難解法: 要「看懂整張圖在說什麼」,得把圖壓小、抽象化(才有大局觀);但輸出又要逐像素、和原圖一樣大(才能定位細節)。 U-Net 用兩條路解決:

現成的實例分割同樣一行就能跑(YOLO 的分割版,檔名多個 -seg): 

from ultralytics import YOLO

seg = YOLO('yolov8n-seg.pt')   # 分割版模型
r = seg('street.jpg')
r[0].show()                    # 框 + 半透明遮罩疊上去
masks = r[0].masks             # 取出每個實例的遮罩
ℹ️
SAM:點一下,就切出來

SAM(Segment Anything Model)是 Meta 推出的「萬物分割」基礎模型。它最神的地方是提示式、零訓練: 你只要給一個點或一個框當提示,它就切出那個物件,泛化能力極強,連沒見過的東西也切得乾淨。 這讓它成了「精準去背工具」與「加速標註」的利器——比起第 65、66 章靠顏色/邊緣去背,SAM 靠的是語意,複雜背景也難不倒它。

人臉與姿態:從整張圖,聚焦到關鍵點

前面都在處理「整張圖有什麼」。但很多最貼近生活的應用,聚焦在特定目標的關鍵點(landmark)上: 刷臉門禁、AR 美顏濾鏡、健身動作計數、駕駛疲勞偵測。這一節把人臉與人體一起講。

人臉:偵測 vs 辨識

這是兩件常被混為一談、其實截然不同的事:

  • 人臉偵測(detection):回答「有沒有臉、在哪」。傳統的 Haar 級聯、或現代 CNN 都能做。
  • 人臉辨識(recognition):回答「這是誰」。這才是難點,而它的解法非常優雅。

辨識的核心觀念是 embedding(嵌入向量):用一個 CNN,把每張臉編碼成一個向量(例如 128 個數字)。 模型訓練的目標是讓同一個人的向量彼此靠近、不同人的向量彼此遠離。於是「這兩張是不是同一人」, 就化簡成一個簡單的幾何問題:算兩個向量的距離,小於某個門檻就判定同一人。 

import face_recognition

known = face_recognition.face_encodings(known_rgb)[0]   # 已註冊者的 128 維向量
enc   = face_recognition.face_encodings(new_rgb)[0]      # 新來的臉,也編成 128 維
dist  = face_recognition.face_distance([known], enc)[0]  # 算距離,越小越像
print('同一人' if dist < 0.6 else '不同人')              # 門檻 0.6:太鬆會誤認,太嚴會拒認
🔑
辨識 = 比向量,不是比圖

人臉辨識的精髓:不是逐像素比對兩張照片像不像(那會被光照、角度、表情輕易騙過), 而是把臉濃縮成一個向量,再比向量的距離。這個「把複雜物件變成一個向量、用距離衡量相似度」的思想, 是現代 AI 的通用招式——它和第 54 章的向量空間、以及大型語言模型用的「文字 embedding」,是同一套數學。一個「考勤系統」,就是事先存好每人的 embedding,到場時偵測→編碼→找最近且小於門檻者。

姿態估計與「健身動作計數器」

有了人體 33 個關鍵點,就能做一個漂亮的旗艦專題:自動數深蹲/二頭彎舉次數的計數器。原理拆成兩步, 都不需要再訓練模型,純粹是幾何 + 邏輯:

第一步:用三點算夾角。例如二頭彎舉,看「肩-肘-腕」三點構成的手肘角度。這就是國中的餘弦定理變成程式 (用向量內積): 

import numpy as np

def angle(a, b, c):                       # b 是頂點(如手肘)
    ba, bc = a - b, c - b                 # 從頂點指向兩端的向量
    cos = np.dot(ba, bc) / (np.linalg.norm(ba) * np.linalg.norm(bc) + 1e-6)
    return np.degrees(np.arccos(np.clip(cos, -1, 1)))   # clip 防浮點誤差讓 arccos 變 NaN

第二步:用狀態機數次數。角度低於下限(手臂彎起)→ 進入「下」狀態;角度回到上限以上(手臂打直)→ 完成一次,計數 +1。關鍵是用不同的上下門檻(遲滯,hysteresis)避免角度在門檻附近抖動造成重複計數:

OCR:把影像裡的文字變成字串

還有一類辨識,目標既不是物件也不是人,而是文字:車牌、發票、名片、路標。 OCR(光學文字辨識,Optical Character Recognition)把影像中的文字變成可用的字串,通常分兩階段:

  • 文字偵測(detection):找出文字區域在哪裡(框出來)——這其實就是物件偵測。
  • 文字辨識(recognition):把框內的影像轉成字串。這是個序列問題(一個框裡有好幾個字、長度不定),常用 CTC 之類的序列模型解。

實務上不必自己造輪子,現成的 OCR 引擎很強。PaddleOCR 一行幫你做完兩階段、中英都認; Tesseract 則適合乾淨的印刷體: 

from paddleocr import PaddleOCR

ocr = PaddleOCR(use_angle_cls=True, lang='ch')   # use_angle_cls 自動校正方向;lang='ch' 中英都認
result = ocr.ocr('invoice.jpg')                  # 回傳每塊文字的 [框座標, (文字, 信心)]
# 注意:lang 沒設對(如忘了中文)會只認英數字!

# 另一選擇:Tesseract(適合乾淨印刷體,需另裝引擎與語言包)
# text = pytesseract.image_to_string(img, lang='chi_tra+eng')
💡
OCR 的勝負,常在「前處理」而非「換更強的模型」

歪斜、低對比、雜訊、字太小,都會讓辨識率崩盤。一套標準前處理往往比換模型更有效,而且全是第 65 章的舊招組合: 放大(小字放大筆畫才清楚)→ 去噪(保住邊)→ 自適應二值化(變成乾淨的白底黑字,OCR 最愛)→ 透視校正(把斜拍的車牌/文件拉正)。 最後再用格式規則做後處理(車牌樣式、發票統編 8 碼、日期格式)校正並過濾低信心結果。經典專題「車牌辨識(ANPR)」, 就是「YOLO 定位車牌 → 透視校正 → 二值化 → OCR 讀字 → 格式校正 → 去重紀錄」一條龍。

從 Notebook 到產品:即時部署與最佳化

最後一哩路,也是最被初學者低估的一哩:模型在 notebook 跑得動,不等於能用。 要變成「別人能用的產品」,你得面對延遲、掉幀、記憶體、跨機器執行、以及給非技術者操作的介面。這是工程化的功夫。

讓推論更快:量化、半精度、ONNX

即時應用要追求高 FPS(每秒幀數)。幾個常用手段:

  • 量化(quantization)與半精度(fp16):模型權重原本用 32 位元浮點數存。改用 16 位元(fp16)甚至 8 位元整數(int8),模型更小、跑更快,但準確率會略降——所以轉完一定要回頭驗證掉了多少精度,不能只看變快。
  • ONNX:一種通用模型格式。把訓練好的模型轉成 ONNX 後,可被各種更快的推論引擎(ONNX Runtime、TensorRT)載入,也方便跨框架、跨平台部署。
  • 架構層面:縮小輸入解析度、跳幀推論(不是每幀都偵測,中間用第 65 章的物件追蹤補)、用執行緒把「擷取」和「推論」分離,避免互相卡住。
from ultralytics import YOLO

YOLO('best.pt').export(format='onnx')   # 把訓練好的 .pt 轉成通用的 ONNX 格式
ℹ️
為什麼 GPU 讓深度學習快得驚人?

CNN 的前向與反向傳播,骨子裡全是大量的矩陣乘法。CPU 擅長「一件接一件做」,而 GPU 有數千個核心, 擅長「同一個運算對成千上萬筆資料同時做」——正中矩陣運算的下懷。這正是第 43 章講的資料平行(data parallelism), 也呼應第 48 章的硬體觀念。深度學習近年爆發,「算力」(GPU)正是三大功臣之一。 要部署到樹莓派、Jetson 這類邊緣裝置時,則反過來:選最小的模型(如 yolov8n)、降解析度與幀率,在有限算力下換取可用的即時性。

用 Streamlit / Gradio 做一個 demo

把模型變成「能上傳圖、按按鈕」的網頁,最快的路是 Streamlit 或 Gradio——用純 Python就生出網頁介面, 非技術者也能操作。這是「產品化」最有成就感的一步: 

import streamlit as st
from ultralytics import YOLO

@st.cache_resource          # 讓模型只載入一次,不要每次互動都重載
def load_model():
    return YOLO('best.pt')

model = load_model()
img = st.file_uploader('上傳圖片')   # 自動生出「上傳檔案」按鈕
if img:                             # 有上傳才處理
    res = model(read_image(img))
    st.image(res[0].plot(), channels='BGR')  # channels='BGR':OpenCV 是 BGR,網頁預期 RGB
⚠️
Streamlit 的兩個小陷阱

(1) 啟動方式不同:要用 streamlit run app.py,不是 python app.py。 (2) Streamlit 的執行模型是「使用者每次互動,就由上往下把整個檔重跑一遍」——所以「重的東西」(載模型)務必用 @st.cache_resource 快取起來,否則每點一下都重載模型,慢到不能用。 別忘了處理例外輸入(壞檔、沒偵測到物件),demo 才不會當場崩給人看。

當機器開始「看人」:電腦視覺的倫理

這一章教你打造了威力強大的工具。但和第 38 章結尾說的一樣—— 技術能力越強,責任越重。電腦視覺直接作用在「人」身上,它的倫理風險特別具體、特別貼身:

  • 隱私:人臉是最敏感的生物特徵之一。蒐集、儲存、比對人臉資料,必須取得當事人同意、妥善保管、明確用途。沒有同意的大規模臉部監控,是嚴重的隱私侵害。
  • 偏見與公平性:這呼應第 38 章。研究反覆發現,許多商用人臉辨識系統對深膚色族群、女性的錯誤率明顯偏高——因為訓練資料以特定族群為主。這種偏差一旦用於門禁、執法,會讓特定族群系統性地被誤認、被冤枉,後果由真實的人承擔。
  • 深偽(deepfake):同一套生成式技術,能合成以假亂真的人臉影片。它可以用於電影特效,也可以用於詐騙、假新聞、未經同意的偽造影像。這讓「眼見為憑」這個人類數千年的信任基礎開始動搖,是這個時代最棘手的資訊倫理難題之一。

這些議題的完整討論,留給第 41 章(科技倫理)。此刻請把一句話刻進心裡: 你做的每一個視覺系統,鏡頭對準的都是活生生的人。該不該做、資料怎麼來、誤判誰來承擔—— 這些問題,和「mAP 多少」一樣,是工程師職責的一部分。

✏️ 練習 67.1 實作

用 PyTorch 定義一個小 CNN,吃 1 通道的 28×28 灰階圖(如 MNIST 手寫數字),輸出 10 類。 要求:兩層卷積(各接 ReLU 與 2×2 池化),最後接一層全連接。請寫出 nn.Module,並算出全連接層的輸入維度。

看解答

關鍵是追蹤空間尺寸:28 →(池化)→ 14 →(池化)→ 7。第二層卷積輸出 32 通道,所以攤平後是 $32 \times 7 \times 7 = 1568$:

import torch.nn as nn
import torch.nn.functional as F

class SmallCNN(nn.Module):
    def __init__(self, n_cls=10):
        super().__init__()
        self.c1 = nn.Conv2d(1, 16, 3, padding=1)   # 1→16 通道,3×3 核
        self.c2 = nn.Conv2d(16, 32, 3, padding=1)  # 16→32 通道
        self.fc = nn.Linear(32 * 7 * 7, n_cls)     # 28→14→7,故 32*7*7=1568

    def forward(self, x):
        x = F.max_pool2d(F.relu(self.c1(x)), 2)    # 28×28 → 14×14
        x = F.max_pool2d(F.relu(self.c2(x)), 2)    # 14×14 → 7×7
        return self.fc(x.flatten(1))               # 攤平 → 10 類分數

檢查維度的好習慣:在 forward 裡暫時 print(x.shape),跑一筆假資料 model(torch.randn(1,1,28,28)) 看尺寸對不對。 算錯全連接的輸入維度,是新手最常見的報錯來源之一。

✏️ 練習 67.2 思考

用你自己的話解釋:卷積層是怎麼「學特徵」的?請涵蓋三點:(a) 卷積層的「核」和第 65 章影像處理的「核」有什麼相同、什麼不同? (b) 為什麼把卷積層疊深,網路能從「邊緣」一路學到「物件」?(c) 「參數共享」是什麼,它帶來哪兩個好處?

看解答

(a) 相同:兩者都是「拿一個小核在影像上滑動,逐元素相乘相加,產生特徵圖」這個運算,一模一樣。 不同:第 65 章的核(如 Sobel、高斯)是人手工設計、固定不變的;CNN 的核裡那些數字是參數,由訓練(梯度下降)自動學出來。 網路會學出「對我的任務最有用」的核,而不是人事先想好的。

(b) 因為一層的輸出(特徵圖)是下一層的輸入。淺層的核在原始像素上偵測邊緣;中層的核在「邊緣圖」上操作, 就能把邊緣組合成紋理、角點;再深一層把紋理組合成部件(眼睛、輪子);最深層把部件組合成完整物件。 「組合更簡單的特徵成更複雜的概念」這個能力,隨層數加深而自然湧現——這是單一手工特徵(只能停在邊緣那層)永遠做不到的。

(c) 參數共享 = 同一個核(同一組權重)走遍整張圖的每個位置,而不是每個位置各有一組權重。 兩個好處:(1) 參數量暴減(一個 3×3 核只有 9 個權重,卻能掃全圖),省記憶體又抗過擬合; (2) 位移不變性——「貓的特徵」不管出現在左上還右下,都用同一個核就能偵測到,網路不必為每個位置重學一次。

✏️ 練習 67.3 實作

你的農場只蒐到 5 種病蟲害、每類約 80 張照片。請(a) 說明為什麼不該從零訓練一個 CNN; (b) 用 PyTorch 寫出遷移學習的程式骨架:載入預訓練 ResNet18、凍結骨幹、換掉分類層; (c) 列出兩個你會採取的、對付小資料過擬合的具體措施。

看解答

(a) 400 張圖,對一個幾百萬參數的 CNN 來說太少,從零訓練幾乎必然嚴重過擬合(背下這 400 張、學不到能泛化的規律)。 而通用的視覺特徵(邊緣、紋理)在任何任務都共通,沒理由用這麼點資料從頭重學。

import torch.nn as nn
from torchvision import models

m = models.resnet18(weights='IMAGENET1K_V1')   # 載入預訓練權重
for p in m.parameters():
    p.requires_grad = False                     # 凍結骨幹,保住通用特徵
m.fc = nn.Linear(m.fc.in_features, 5)           # 換成 5 類的新分類層(只有它會被訓練)

# 之後只把「需要梯度」的參數丟給最佳化器
opt = torch.optim.Adam([p for p in m.parameters() if p.requires_grad], lr=1e-3)

(c) 任選兩個以上:① 資料增強(隨機裁切/翻轉/調色,只給訓練集);② 遷移學習本身(已用,大幅降低過擬合); ③ Dropout 或權重衰減(L2 正則化);④ 提早停止(盯驗證誤差);⑤ 先只訓尾端、再用很小的學習率解凍微調,別一開始就大 lr 全解凍把預訓練特徵洗掉。 另外記得:預訓練模型的正規化 mean/std 要對齊。

✏️ 練習 67.4 計算

有兩個邊界框(以左上、右下角座標表示):正解框 A = (10, 10, 50, 50),預測框 B = (30, 30, 70, 70)。 (a) 手算它們的 IoU。(b) 這次偵測算「命中」嗎(門檻 0.5)?(c) 寫一個 iou(boxA, boxB) 函式驗證你的答案。

看解答

(a) 手算:

  • 交集的左上角 = (max(10,30), max(10,30)) = (30, 30);右下角 = (min(50,70), min(50,70)) = (50, 50)。
  • 交集寬高 = (50−30)×(50−30) = 20×20 = 400。
  • 各框面積:A = 40×40 = 1600,B = 40×40 = 1600。
  • 聯集 = 1600 + 1600 − 400 = 2800。
  • IoU = 400 / 2800 ≈ 0.143。

(b) 0.143 < 0.5,所以不算命中——框雖然有重疊,但偏太多。

def iou(a, b):
    ax1, ay1, ax2, ay2 = a
    bx1, by1, bx2, by2 = b
    # 交集矩形
    ix1, iy1 = max(ax1, bx1), max(ay1, by1)
    ix2, iy2 = min(ax2, bx2), min(ay2, by2)
    iw, ih = max(0, ix2 - ix1), max(0, iy2 - iy1)   # max(0,..) 防無重疊變負
    inter = iw * ih
    area_a = (ax2 - ax1) * (ay2 - ay1)
    area_b = (bx2 - bx1) * (by2 - by1)
    union = area_a + area_b - inter
    return inter / union if union > 0 else 0.0

print(iou((10,10,50,50), (30,30,70,70)))   # 0.142857...

易錯點:(1) 兩框完全不重疊時,ix2 - ix1 會是負的,務必用 max(0, ...) 夾住,否則會算出假的正交集面積; (2) 聯集要減掉交集,別重複計算那塊。

✏️ 練習 67.5 實作

用 Ultralytics YOLO 寫一段推論程式骨架:載入預訓練模型、對一張圖偵測,然後只印出信心 ≥ 0.5 的框的類別名稱與信心分數。 (不需要真的執行,寫出正確結構即可。)

看解答 
from ultralytics import YOLO

model = YOLO('yolov8n.pt')          # 預訓練(COCO 80 類);首次自動下載
results = model('street.jpg')        # 也可丟資料夾/影片/網址/攝影機(source=0)

r = results[0]
names = r.names                      # 類別編號 → 名稱 的對照表
for box in r.boxes:
    conf = float(box.conf)
    if conf >= 0.5:                  # 只看夠有把握的框
        cls_id = int(box.cls)
        x1, y1, x2, y2 = box.xyxy[0].tolist()   # 框座標(左上、右下)
        print(f"{names[cls_id]}: {conf:.2f}  box=({x1:.0f},{y1:.0f},{x2:.0f},{y2:.0f})")

重點:r.boxes 可逐框走訪,每個 box 有 .conf(信心)、.cls(類別編號)、.xyxy(座標); r.names 把編號翻成可讀名稱。要接攝影機即時,把 model(...) 換成 model.predict(source=0, stream=True) 並用 for r in ... 逐幀處理。 提醒:YOLO 內部已經幫你做完 NMS,所以你拿到的框已經是去重後的乾淨結果。

✏️ 練習 67.6 挑戰

(呼應第 38 章)請設計一個不會資料洩漏的影像分類訓練流程,從一個「每個類別一個資料夾」的圖片集出發。 列出步驟,並特別指出影像任務中三個容易造成洩漏的陷阱,以及如何避免。

看解答

正確流程:

  1. 先切分:把資料切成 train / val / test 三份,測試集立刻鎖進保險箱,整個開發過程只在最後用一次。
  2. 只在訓練集上「學」前處理:資料增強只套用到訓練集;正規化的 mean/std 只用訓練集統計量,再套到 val/test。
  3. 用訓練集訓練、用驗證集挑超參數(學習率、epoch、是否解凍微調)與偵測過擬合。
  4. 定案後,才用測試集做唯一一次誠實評估。類別不平衡時,別只看準確率,看 F1 / 各類 recall。

影像任務特有的三個洩漏陷阱:

  • ① 同一物件/同一場景跨集。例如「同一個瑕疵品從不同角度拍的多張照」被分散到 train 和 test,或同一段影片的相鄰幀跨集——它們幾乎相同,等於讓模型「考過考古題」。避免:以「物件/場景/影片」為單位整組切分,不要以「單張圖」為單位。
  • ② 對全部資料做增強/正規化後才切分。這會把測試集的統計資訊洩漏進訓練。避免:永遠「先切分,再 fit 前處理」,而且增強只給訓練集。
  • ③ 反覆用測試集調參。每調一次參數就偷看一次測試集,等於慢慢對測試集過擬合,測試分數失去意義。避免:調參只准用驗證集;測試集從頭到尾碰一次。

核心心法和第 38 章完全一致:任何「向資料學習」的步驟,都只能用訓練集去 fit,再套用到 val/test。違反一次,做出來的高分都是自欺欺人。

✏️ 練習 67.7 挑戰

承上,你要把訓練好的健身計數器模型部署成一個能即時數二頭彎舉的 demo。 (a) 用「肩-肘-腕」三點 + 狀態機,寫出計數的核心邏輯(用前面的 angle 函式)。 (b) 說明你為什麼用「不同的上下門檻」,以及它防止了什麼問題。 (c) 若即時 FPS 太低、畫面卡頓,提出兩個最佳化方向。

看解答 
stage = 'up'      # 初始假設手臂打直
count = 0

# 每一幀:由 MediaPipe Pose 取得 肩 sh、肘 el、腕 wr 三點(已轉成像素座標的向量)
theta = angle(sh, el, wr)            # 手肘角度

if theta < 40 and stage == 'up':    # 從打直 → 彎到夠低
    stage = 'down'
elif theta > 160 and stage == 'down':  # 從彎起 → 完全打直,完成一次
    stage = 'up'
    count += 1
    print('完成第', count, '下')

(b) 用兩個不同門檻(40° 進「下」、160° 才回「上」)是遲滯(hysteresis)設計。 若只用單一門檻(如 90°),當角度在 90° 附近抖動(關鍵點本就有雜訊),狀態會反覆橫跳、瘋狂誤計數。 拉開上下門檻,角度必須「真的彎下去、又真的打直」才切換一次,計數才穩定、不漏不重。

(c) 任選兩個:① 降低輸入解析度或選更輕的模型;② 跳幀(不是每幀都跑姿態估計,中間用上一幀結果); ③ 用執行緒分離擷取與推論,避免互相等待;④ 過濾低 visibility 的關鍵點、對角度做平滑(移動平均)減少抖動,順帶減少誤計數。

把這一章串起來

我們從「相機只認得像素、認不得意義」出發,一路走到能偵測、分割、辨識人臉、讀文字、並把模型變成產品。回顧這趟旅程的路標:

  • 核心翻轉:從「人手工設計特徵(HOG)+ 分類器(SVM)」,翻轉成 CNN 的端到端學習——連「該抽什麼特徵」都讓機器自己學。第 65、66 章那種人設計的「核」,變成了訓練學出來的核。
  • CNN 靠卷積的三個點子適配影像:局部連接、參數共享、層層抽象(邊緣→紋理→部件→物件)。用 PyTorch 寫出 nn.Module 與訓練迴圈四步,而 loss.backward() 背後就是第 12、38 章的鏈鎖法則 + 梯度下降。
  • 遷移學習是小資料落地的關鍵:凍結預訓練骨幹、只換尾端;搭配資料增強。資料洩漏的鐵律(第 38 章)在影像上化身為「增強只給訓練集、同物件別跨集」。
  • 物件偵測:分類/偵測/分割的層級;YOLO「一次看全圖、格點預測」的單階段思想;以及 IoU、NMS、mAP 三個必懂指標。
  • 影像分割:語意 vs 實例;U-Net 用編碼-解碼 + skip connection 兼顧「理解」與「定位」;SAM 點一下就切。評估用 IoU/Dice 而非準確率。
  • 人臉與姿態:辨識 = 比 embedding 向量的距離(呼應第 54 章);MediaPipe 的關鍵點(注意 BGR/RGB 與歸一化座標兩坑);健身計數器 = 三點算角度 + 帶遲滯的狀態機。
  • OCR 的偵測+辨識兩階段,以及「前處理常比換模型更有效」。
  • 部署:量化、半精度、ONNX 加速;GPU 之所以快是因為矩陣運算的資料平行(第 43、48 章);用 Streamlit/Gradio 做 demo。
  • 倫理:人臉的隱私與偏見、深偽——鏡頭對準的是真實的人(第 41 章)。

如果你讀完第 38 章再讀完這一章,應該會有一個踏實的領悟: 影像辨識不是另一門新魔法,它就是機器學習,只是換上了適合影像的零件(卷積)。 假設函數、損失、梯度下降、泛化、過擬合、資料洩漏——這些第 38 章的血脈,在這裡一條都沒少,只是流過了一個更深、更專為影像而生的網路。 掌握了這層理解,你面對任何視覺模型,都不再是看魔術的觀眾,而是知道引擎怎麼轉、也知道何時該為它的決定負責的工程師。
